# ad-24-L3-中期层存储单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-24 |
| 模块名称 | L3 中期层存储单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 五层记忆层级存储 |
| 核心职责 | 存储已形成初步习惯的有效驾驶策略，占漏斗二总存储容量的 10%。接收来自 L2 晋升的条目，在留存满 30 日且满足重要度条件后进入 L4 晋升候选。本层经验被视为“已初步验证的策略”，享受遗忘保护。失败经验在晋升至 L4 前须过三道安全仲裁 |
| 依赖模块 | ad-22（L2 近期层存储单元，接收晋升条目）、ad-25（L3 中期层相似经验归并单元，周期性归并）、ad-38（晋升双条件判定单元，下发 L3 晋升候选清单）、ad-40（遗忘阈值判定单元，下发 L3 遗忘候选清单）、ad-43（失败经验安全仲裁三道校验单元，消费失败经验条目） |
| 被依赖模块 | ad-25（消费 L3 存储条目进行相似度归并）、ad-38（消费 L3 晋升候选条目）、ad-40（消费 L3 遗忘候选条目）、ad-43（消费失败经验条目进行安全仲裁） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收晋升条目，管理存储与流转 | 漏斗二激活，本层级初始化完成 |
| 存储将满 | `NEAR_FULL` | 存储占用率 > 85%，触发归并加速 | 存储占用率监测 |
| 存储已满 | `FULL` | 存储占用率 > 95%，强制归并并清除低价值条目 | 存储占用率监测 |
| 维护中 | `MAINTENANCE` | 正在执行相似经验归并或失败经验仲裁 | 定时维护或存储已满触发 |
| 冻结只读 | `FROZEN` | 漏斗二冻结或紧急接管，暂停写入仅保留查询 | 驾驶模式切换或紧急熔断 |

| 归并执行指令 | Struct（源条目 ID + 目标条目 ID + 归并策略） | ad-25 L3 中期层相似经验归并单元 | 归并周期触发时 | 普通 |
| 晋升失败回退条目 | Struct（条目 ID + 回退原因） | ad-38 晋升双条件判定单元 | L3 条目晋升 L4 失败时 | 普通 |

| 全局调度指令 | Enum（冻结/激活/维护） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 存储完成回执 | Struct（条目 ID + 存储层级=L3 + 存储时间戳 + 当前 L3 占用率） | ad-22、ad-38 | 每次晋升条目成功存储后 | 普通 |
| 晋升执行结果 | Struct（条目 ID + 原层级=L3 + 目标层级=L4 + 执行结果） | ad-38、ad-39 | 晋升完成或失败后 | **高** |

| 失败经验仲裁请求 | Struct（条目 ID + 经验内容 + 结果分类标签=策略失误 + 场景特征） | ad-43 | L3 中存在结果标签为“策略失误”的条目待晋升时 | **高** |
| L3 状态快照 | Struct（占用率 + 条目总数 + 各分槽条目分布 + 平均留存时长 + 平均 I 值 + 警示标签条目数） | ad-03、各对应分槽 | 每 30 秒周期性上报 | 普通 |
| 存储异常告警 | Struct（异常类型 + 当前占用率 + 详情） | ad-03、ad-01 | 存储写入失败或数据校验异常时 | **高** |


## 核心处理逻辑

```
FUNCTION l3_mid_term_storage_main_loop():
    SET internal_state = NORMAL
    初始化 L3 存储分区（占漏斗二总容量 10%，各分槽按策略价值动态分配）
    初始化条目索引表（条目 ID → 存储地址 + 晋升时间戳 + I 值 + 来源分槽号 + 结果分类标签）
    最大留存时间 = 30 日（720 小时）
    警示标签条目字典 = {}  // 条目 ID → 警示原因 + 连续安全通过次数

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = FROZEN
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 第 2 步：接收 L2 晋升条目
        IF 收到 L2 晋升条目 AND internal_state IN [NORMAL, NEAR_FULL]:
            // 2a. 检查存储空间
            IF internal_state == FULL:
                立即触发紧急归并（调用 ad-25 归并相似度 > 0.85 的条目）
                若归并后仍不足，清除 I 值最低的 3% 条目（L3 受保护，阈值极高）
                IF 释放空间仍不足:
                    向 ad-03 上报告警
                    返回存储失败（原因=L3 存储满）
                    CONTINUE
                SET internal_state = NEAR_FULL

            新条目 = 晋升条目

            // 2b. 结果分类标签检查
            IF 新条目.结果分类标签 == "策略失误":
                // 失败经验写入 L3，但标记为警示标签
                警示标签条目字典[新条目.ID] = {
                    警示原因: "策略失误",
                    连续安全通过次数: 0,
                    仲裁状态: "待仲裁"
                }
                // I 值不做提升（失败经验不奖励）
            ELSE IF 新条目.结果分类标签 == "成功优化":
                // 成功经验 I 值微调 +0.03
                新条目.I 值 = MIN(新条目.I 值 + 0.03, 1.0)
            ELSE IF 新条目.结果分类标签 == "不可抗力场景":
                // 不可抗力场景，I 值大幅提升，加速晋升
                新条目.I 值 = MIN(新条目.I 值 + 0.10, 1.0)

            // 2c. 分配空间并写入
            存储地址 = 分配空间(新条目.大小)
            IF 分配失败:
                触发碎片整理 → 重试
                IF 仍失败:
                    返回存储失败回执
                    CONTINUE

            写入存储介质
            更新条目索引表：
                条目 ID → {地址, NOW(), 新条目.I 值, 新条目.来源分槽号, 子类标记, 新条目.结果分类标签}
            向 ad-22、ad-38 返回存储完成回执

            // 2d. 更新占用率
            占用率 = 计算占用率()
            IF 占用率 > 0.95:
                SET internal_state = FULL
                上报告警
            ELSE IF 占用率 > 0.85 AND internal_state == NORMAL:
                SET internal_state = NEAR_FULL

        // 第 3 步：处理 L3 晋升候选
        IF 收到 ad-38 L3 晋升候选清单:
            FOR EACH 候选 IN 清单:
                条目 ID = 候选.条目 ID
                条目 = 索引表[条目 ID]

                // 3a. 失败经验晋升前安全仲裁
                IF 条目.结果分类标签 == "策略失误":
                    IF 警示标签条目字典[条目 ID].仲裁状态 == "待仲裁":
                        向 ad-43 发送失败经验仲裁请求
                        警示标签条目字典[条目 ID].仲裁状态 = "仲裁中"
                        向 ad-38 返回晋升暂缓（等待仲裁结果）
                        CONTINUE
                    ELSE IF 警示标签条目字典[条目 ID].仲裁状态 == "仲裁中":
                        向 ad-38 返回晋升暂缓
                        CONTINUE
                    ELSE IF 警示标签条目字典[条目 ID].仲裁状态 == "仲裁驳回":
                        // 仲裁未通过，仅保留 L3 警示标签，不晋升
                        向 ad-38 返回晋升驳回（原因=安全仲裁未通过）
                        CONTINUE
                    ELSE IF 警示标签条目字典[条目 ID].仲裁状态 == "仲裁通过":
                        // 仲裁通过，允许晋升
                        GOTO 执行晋升

                // 3b. 执行晋升
                LABEL 执行晋升:
                IF 候选.目标层级 == L4 AND L4 存储未满:
                    调用 ad-39 执行层级搬运（L3 → L4）
                    向 ad-38 返回晋升成功
                    从 L3 索引表中移除
                    IF 条目 ID IN 警示标签条目字典:
                        从警示标签条目字典中移除
                ELSE:
                    向 ad-38 返回晋升失败（原因=目标层级满或不存在）

        // 第 4 步：处理 L3 遗忘候选（L3 遗忘保护更强）
        IF 收到 ad-40 L3 遗忘候选清单:
            FOR EACH 候选 IN 清单:
                条目 ID = 候选.条目 ID
                条目 = 索引表[条目 ID]

                // L3 遗忘需满足额外条件
                IF 条目.I 值 < 该分槽 L3 最低遗忘阈值 AND 条目.结果分类标签 != "不可抗力场景":
                    // 优先冷归档而非直接删除
                    调用 ad-42 执行冷归档（ad-49）
                    向 ad-40 返回已归档
                    从索引表中移除
                ELSE:
                    向 ad-40 返回保留（不满足遗忘条件）

        // 第 5 步：处理失败经验仲裁结果
        IF 收到 ad-43 失败经验仲裁结果:
            条目 ID = 仲裁结果.条目 ID
            IF 仲裁结果.仲裁结论 == "放行晋升":
                警示标签条目字典[条目 ID].仲裁状态 = "仲裁通过"
                // 下次晋升周期自动处理
            ELSE IF 仲裁结果.仲裁结论 == "保留L3警示":
                警示标签条目字典[条目 ID].仲裁状态 = "仲裁驳回"
                // 永久保留在 L3 作为警示标签
            ELSE IF 仲裁结果.仲裁结论 == "永久锁定L5":
                // 特殊：直接锁定至 L5
                调用 ad-39 搬运至 L5
                从索引表和警示字典中移除

        // 第 6 步：处理晋升失败回退
        IF 收到晋升失败回退条目（从 L4 退回）:
            条目继续留在 L3
            标记回退次数 +1
            IF 回退次数 ≥ 2:
                // L3 回退两次即标记为“晋升困难”，不再尝试晋升
                标记为“长期保留L3”
                上报告警

        // 第 7 步：处理归并执行指令
        IF 收到 ad-25 归并执行指令:
            执行归并（合并相似条目的复用计数，保留更高 I 值条目）
            释放冗余空间

        // 第 8 步：定期维护——触发归并
        IF 距上次归并 >= 7 日 OR 占用率 > 80%:
            SET internal_state = MAINTENANCE
            向 ad-25 发送归并请求
            WAIT 归并完成
            SET internal_state = NORMAL

        // 第 9 步：警示标签自动降级检查
        IF 距上次降级检查 >= 30 日:
            FOR EACH 警示条目 IN 警示标签条目字典:
                IF 警示条目.仲裁状态 == "仲裁驳回":
                    // 检查同一场景是否连续 3 次无警示安全通过
                    IF 警示条目.连续安全通过次数 >= 3:
                        警示条目.警示原因 = "已降级为普通经验"
                        警示条目.仲裁状态 = "仲裁通过"  // 允许晋升
                        上报告警（降级事件）

        // 第 10 步：状态上报
        IF 距上次上报 >= 30 秒:
            向 ad-03 和各分槽上报 L3 状态快照

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| L3 存储满且归并无法释放足够空间 | 清除 I 值最低的 3% 条目（跳过不可抗力、警示标签、S≥0.7），上报告警 | 空间释放 |
| 失败经验仲裁超时（> 72 小时无结果） | 自动按“保留 L3 警示”处理，上报告警 | 仲裁完成 |
| L3 条目留存超 30 日但未满足晋升条件 | 由 ad-40 遗忘阈值判定单元评估，纳入遗忘候选（冷归档） | — |
| 晋升失败回退次数 ≥ 2 | 标记为“长期保留 L3”，不再尝试晋升，上报告警 | — |
| 警示标签条目连续 3 次无警示安全通过 | 自动降级为普通经验，允许正常晋升 | 降级完成 |
| 冻结状态写入 | 拒绝写入，返回原因 | 激活指令 |
| 条目索引表损坏 | 从存储分区重建，上报告警 | 重建完成 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 失败经验（结果标签=“策略失误”）在晋升 L4 前必须通过 ad-43 三道安全仲裁，任一未通过则永久保留在 L3 作为警示标签 |
| S-02 | L3 遗忘采用冷归档（ad-49）而非直接删除，确保中期经验可追溯恢复 |
| S-03 | 不可抗力场景经验（结果标签=“不可抗力”）在 L3 遗忘评估时豁免清除，直接保留 |
| S-04 | 警示标签条目在仲裁驳回后，同一场景连续 3 次无警示安全通过方可降级为普通经验 |
| S-05 | L3 条目晋升失败回退 ≥ 2 次须标记“长期保留 L3”，防止反复无效晋升尝试 |
| S-06 | 存储满时清除操作跳过不可抗力、警示标签、安全显著性 S≥0.7 的条目 |
| S-07 | 所有写入、晋升、遗忘、仲裁、归并操作全量写入 ad-51 变更日志 |
