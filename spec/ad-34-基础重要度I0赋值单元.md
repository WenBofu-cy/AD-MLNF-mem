# ad-34-基础重要度I₀赋值单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-34 |
| 模块名称 | 基础重要度 I₀ 赋值单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 三维重要度计算引擎 |
| 核心职责 | 基于新写入漏斗二的经验条目的结构化数据，为每条经验赋予基础重要度初始值 I₀（0–1）。I₀ 是三维重要度驱动公式 `I = I₀ + α·S + β·V + γ·C` 的初始基线，代表该经验在未经时间与复用检验前的固有价值 |
| 依赖模块 | ad-14（场景判定与分槽路由单元，提供经验写入请求）、各场景分槽（ad-15至ad-19，提供子类标签与路由标记） |
| 被依赖模块 | ad-36（综合重要度 I 值聚合计算单元，消费 I₀ 值）、ad-21（L1时序衰减单元，参考 I₀ 作为衰减基准） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收经验条目并赋予 I₀ | 漏斗二激活，本单元初始化完成 |
| 参数更新中 | `UPDATING` | I₀ 赋值规则库发生变更，重新加载 | 规则库 OTA 更新时 |
| 暂停服务 | `PAUSED` | 漏斗二冻结或紧急熔断 | 驾驶模式切换或紧急事件 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 经验写入请求 | Struct（条目 ID + 经验内容 + 场景标签 + 路由标记 + 来源分槽号 + 子类标记 + 触发模块） | ad-14 场景判定与分槽路由单元（经对应分槽转发） | 新经验生成时 | **高** |

| 全局调度指令 | Enum（冻结/激活） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| I₀ 赋值结果 | Struct（条目 ID + I₀ 值 + 赋值依据 + 赋值时间戳） | ad-36 综合重要度 I 值聚合计算单元 | 每次赋值完成后 | **高** |

## I₀ 赋值规则库

### 一、核心公式

```
I₀ = CLAMP( I_base × (1 + Σ 场景加成) + 事件加权 , 0.05 , 0.90 )
```

> I₀ 上限 0.90，为 α·S / β·V / γ·C 留出增量空间。I₀ 下限 0.05，新经验不设零值。

### 二、I_base 基础分值（按经验生成来源）

| 生成来源 | I_base | 说明 |
|----------|:---:|------|
| 人类示教（驾驶员演示） | 0.70 | 人类智慧直接输入，价值高 |
| 仿真回灌验证通过 | 0.60 | 经过离线验证的策略 |
| 系统自主探索成功 | 0.55 | 自主学习的成功经验 |
| 常规自动驾驶决策执行 | 0.50 | 日常决策，标准基线 |
| 低置信度降级路由 | 0.35 | 不确定场景下的经验，价值需验证 |
| 应急避险操作 | 0.80 | 涉及生命安全的操作，无论成败均高价值 |

### 三、场景加成系数（累加，上限 +0.30）

| 场景特征 | 加成系数 | 说明 |
|----------|:---:|------|
| 高风险目标（行人/非机动车） | +0.15 | 第四类目标为核心风险 |
| 极端天气（暴雪/浓雾/沙尘） | +0.10 | 特殊环境经验稀缺 |
| 夜间 + 无照明 | +0.05 | 感知受限场景 |
| 非铺装路面（乡村道路） | +0.05 | 特殊路况经验 |
| 高速场景（车速 > 80km/h） | +0.05 | 高速下的操作容错率低 |
| 首次遇到的新场景类型 | +0.10 | 探索价值 |
| 失败教训（策略失误） | +0.05 | 失败经验同样有价值（但不叠加） |

### 四、事件加权（ECS 语义标注）

| 事件类型 | 直接加成 | 说明 |
|----------|:---:|------|
| 成功避免碰撞 | +0.20 | 高价值安全经验 |
| 不可抗力成功应对 | +0.30（上限 0.90） | 极珍贵 |
| 法规合规示范 | +0.10 | 合规经验 |
| 通行效率优化 | +0.05 | 效率提升经验 |

### 五、特殊赋值规则

| 场景 | 规则 |
|------|------|
| 结果标签为“不可抗力场景” | I₀ = 0.90（直接上限），且优先级最高 |
| 安全显著性 S 已由 ad-31 计算为 ≥ 0.9 | I₀ = 0.90，直接触发 L5 直达 |
| 分槽为特殊环境槽（ad-18） | I₀ 额外 +0.05（特殊环境经验整体抬升） |


## 核心处理逻辑

```
FUNCTION i0_assignment_main_loop():
    SET internal_state = NORMAL
    加载 I₀ 赋值规则库（编译期固化 + OTA 增量）

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 第 2 步：处理经验写入请求
        IF 收到经验写入请求 AND internal_state == NORMAL:
            条目 ID = 写入请求.条目 ID
            结果标签 = 写入请求.经验内容.结果分类标签
            S 值 = 写入请求.经验内容.安全显著性（若已计算）

            // 2a. 不可抗力直通
            IF 结果标签 == "不可抗力场景":
                I₀ = 0.90
                赋值依据 = "不可抗力场景"
                GOTO 输出结果

            // 2b. S≥0.9 直通
            IF S 值 >= 0.9:
                I₀ = 0.90
                赋值依据 = "安全显著性 S≥0.9"
                GOTO 输出结果

            // 2c. 获取生成来源并查表
            生成来源 = 写入请求.经验内容.生成来源（默认=常规自动驾驶决策执行）
            I_base = 规则库.I_base[生成来源]
            IF I_base == NULL:
                I_base = 0.50  // 默认基线
                赋值依据 = "未知来源，使用默认基线"

            // 2d. 计算场景加成
            场景特征 = 写入请求.场景标签 + 写入请求.分槽号 + 写入请求.子类标记
            加成总和 = 0.0
            FOR EACH 加成条件 IN 规则库.场景加成系数:
                IF 场景特征 匹配 加成条件:
                    加成总和 += 加成条件.系数
            加成总和 = MIN(加成总和, 0.30)

            // 2e. 事件加权
            事件加权值 = 0.0
            IF 写入请求.事件语义标注 != NULL:
                FOR EACH 事件类型 IN 写入请求.事件语义标注:
                    IF 事件类型 IN 规则库.事件加权:
                        事件加权值 += 规则库.事件加权[事件类型]

            // 2f. 综合计算 I₀
            I₀ = I_base × (1 + 加成总和) + 事件加权值
            I₀ = CLAMP(I₀, 0.05, 0.90)

            // 2g. 特殊环境槽额外抬升
            IF 写入请求.来源分槽号 == ad-18:
                I₀ = MIN(I₀ + 0.05, 0.90)

            // 2h. 结果标签为“策略失误”不额外加分也不降级
            // I₀ 已客观反映经验固有价值，后续由三道仲裁处理

            赋值依据 = "生成来源: " + 生成来源 + ", 场景加成: " + 加成总和 + ", 事件加权: " + 事件加权值

        // 输出结果
        LABEL 输出结果:
            I₀ 结果 = {
                条目 ID: 条目 ID,
                I₀ 值: I₀,
                赋值依据: 赋值依据,
                赋值时间戳: NOW()
            }
            向 ad-36 发送 I₀ 赋值结果

        // 第 3 步：处理规则库更新
        IF 收到 I₀ 赋值规则库 OTA 更新:
            SET internal_state = UPDATING
            校验规则库签名与完整性
            加载新规则库
            向 ad-03 上报规则库更新日志
            SET internal_state = NORMAL

        // 第 4 步：周期性状态上报
        IF 距上次上报 >= 60 秒:
            向 ad-03、ad-01 上报赋值状态

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 经验写入请求缺少必要字段 | 使用默认 I₀ = 0.40，标记为“字段缺失-保守赋值”，上报告警 | 字段补全后正常赋值 |
| I₀ 计算溢出 > 0.90 | CLAMP 至 0.90 | — |
| 规则库加载失败 | 使用编译期固化的最小规则集，上报告警 | 规则库恢复 |
| 生成来源不在规则库中 | I_base 取 0.50，标记为未知来源 | 规则库扩展覆盖 |
| 场景加成系数累计超出上限 | CLAMP 至 0.30 | — |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | I₀ 上限硬编码为 0.90，为 I 值动态增量（α·S / β·V / γ·C）留出空间，避免初始值过高导致晋升失准 |
| S-02 | 不可抗力场景 I₀ 强制设为 0.90，确保顶级安全经验在记忆体系中获得最高初始权重 |
| S-03 | I₀ 赋值规则库须经过安全审计与签名校验，OTA 更新时校验完整性，失败则回退至编译期版本 |
| S-04 | I₀ 赋值不因经验的“成败”标签而产生极端偏差，策略失误经验保留客观固有价值，由后续三道安全仲裁把关 |
