# mem-63 驾驶场景分槽路由器

> 所属仓库: AD-mlnf-mem/spec/
> 模块编号: mem-63
> 所属分区: AD新增·双漏斗管理层


## 一、基本信息

| 项目 | 内容 |
|:---|:---|
| 模块编号 | mem-63 |
| 模块名称 | 驾驶场景分槽路由器 |
| 核心职责 | 根据世界模型输出的道路属性，自动判定当前驾驶场景类别，将漏斗二的经验条目路由到对应的场景分槽。检索时仅查询当前场景对应的槽，检索范围缩小80% |
| 输入数据 | 世界模型环境表征（来自mem-59）：道路等级、车道数、路面类型、交通标识密度、周边目标分布、天气与光照条件 |
| 输出数据 | 当前场景类别标签（高速巡航/城区路口/泊车低速/特殊环境/通用驾驶）、目标槽位ID、场景切换信号、检索范围约束参数 |


## 二、核心处理逻辑

**1. 场景类别判定**

系统根据世界模型输出的道路属性，按以下规则自动判定当前场景类别：

| 场景类别 | 判定条件 |
|:---|:---|
| 高速巡航 | 道路等级=高速公路/城市快速路，车道数≥2，路面类型=沥青/水泥，限速≥80km/h |
| 城区路口 | 道路等级=城市主干道/次干道/支路，500m内存在交通信号灯或人行横道 |
| 泊车低速 | 车速<10km/h持续30秒以上，或车辆处于R档，或检测到停车位标线 |
| 特殊环境 | 天气=雨/雪/雾/夜间，或路面类型=积水/积雪/施工区，或检测到隧道/收费站 |
| 通用驾驶 | 不满足上述任一条件的默认场景，含乡村非铺装道路（作为通用驾驶槽的重要子类） |

**2. 经验路由**

新的驾驶经验生成时，mem-63根据当前场景类别标签，将经验条目写入漏斗二对应槽位。路由规则：高速巡航场景→高速巡航槽；城区路口场景→城区路口槽；泊车低速场景→泊车低速槽；特殊环境场景→特殊环境槽；通用驾驶场景→通用驾驶槽。

**3. 检索范围约束**

当AD-ECC向记忆中枢发起经验检索请求时，mem-63根据当前场景类别标签，将检索范围限定在对应场景分槽内。检索范围缩小80%，大幅提升检索效率。仅在置信度不足或场景切换过渡期（场景切换后3秒内），同时检索上一场景槽和当前场景槽。

**4. 场景切换处理**

当车辆跨越场景边界时（如从高速匝道进入城区道路），mem-63检测场景类别变化，触发场景切换信号。切换过渡期3秒内，新旧两个场景槽同时参与检索。过渡期结束后，仅查询当前场景槽。场景切换期间的经验写入：写入新场景对应的槽位。

**5. 槽位维护**

持续监测各场景分槽的存储容量与活跃度。任一槽位存储容量超过上限时，触发该槽的遗忘策略（优先清理低重要度L1-L2条目）。任一槽位超过30天未激活，标记为“休眠”状态，不参与日常检索。


## 三、约束与异常处理

| 约束类型 | 内容 |
|:---|:---|
| 场景判定来源 | 仅接受mem-59（世界模型库）输出的道路属性作为判定依据，不直接读取传感器原始数据 |
| 判定确定性 | 场景类别一旦判定，在未检测到场景边界跨越前保持稳定，避免频繁切换 |
| 过渡期处理 | 场景切换过渡期3秒，新旧槽同时参与检索，确保过渡期经验不丢失 |
| 未知场景 | 若世界模型输出不完整（传感器失效等），默认判定为“通用驾驶”场景，不中断正常运行 |
| 检索权限 | 仅AD-ECC可发起经验检索请求，漏斗一不可查询漏斗二的场景分槽数据 |


## 四、接口校验用例

**用例1：高速巡航场景判定**

输入：世界模型输出道路等级=高速公路，车道数=3，路面类型=沥青，限速=120km/h，车速=100km/h。预期：判定为“高速巡航”场景，路由目标槽=高速巡航槽。

**用例2：城区路口场景判定**

输入：世界模型输出道路等级=城市主干道，400m内检测到交通信号灯，路面类型=沥青，车速=45km/h。预期：判定为“城区路口”场景，路由目标槽=城区路口槽。

**用例3：场景切换触发**

前置条件：当前场景=高速巡航。输入：世界模型检测到车辆驶入匝道，道路等级变为城市快速路辅路，限速降至40km/h，前方150m检测到红绿灯。预期：触发场景切换信号，切换过渡期3秒启动，新场景判定为“城区路口”。

**用例4：传感器失效时默认场景**

输入：世界模型因传感器失效无法输出完整的道路属性（仅知车速=60km/h）。预期：默认判定为“通用驾驶”场景，系统正常运行不受影响。

**用例5：检索范围约束验证**

前置条件：当前场景=高速巡航。操作：AD-ECC请求检索“变道策略”相关经验。预期：仅查询高速巡航槽内的经验条目，不查询城区路口槽、泊车低速槽等其他槽位。检索结果数量远小于全漏斗检索。
