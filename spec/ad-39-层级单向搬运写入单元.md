# ad-39-层级单向搬运写入单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-39 |
| 模块名称 | 层级单向搬运写入单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 执行经验条目从当前层级向上一层存储分区的物理搬运。接收晋升候选清单后，将条目的完整经验数据从源层级复制写入目标层级，校验写入完整性后，在源层级标记删除或正式清除。编译期硬约束禁止高层级向低层级回退搬运。确保搬运过程的原子性 |
| 依赖模块 | ad-38（晋升双条件判定单元，提供晋升候选清单）、ad-20/22/24/26/28（各层级存储单元，提供源数据与接收写入） |
| 被依赖模块 | ad-20/22/24/26（消费搬运指令，执行源层级删除）、ad-22/24/26/28（消费搬运指令，执行目标层级写入） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无搬运任务，等待晋升候选清单 | 系统初始化完成，或上一批次搬运结束 |
| 搬运执行中 | `TRANSFERRING` | 正在逐条目执行层级搬运 | 收到晋升候选清单 |
| 回滚中 | `ROLLING_BACK` | 搬运过程中发生异常，正在回滚已搬运条目 | 写入失败或校验异常 |
| 暂停服务 | `PAUSED` | 系统紧急熔断或漏斗二冻结 | 收到全局冻结或紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 晋升候选清单 | Dict（晋升路径 : List（条目 ID + 源层级 + 目标层级 + 晋升优先级 + I 值）） | ad-38 晋升双条件判定单元 | 每次晋升判定完成后 | **高** |
| 源条目完整数据 | Struct（条目 ID + 经验内容 + 元数据 + I 值 + 时间戳 + 来源分槽号 + 子类标记 + 结果分类标签 + 复用计数 + 不可抗力标记 + 安全仲裁状态） | 源层级存储单元（L1–L4） | 搬运开始时向源层级请求 | **高** |
| 目标层级写入确认 | Struct（条目 ID + 写入结果：成功/失败 + 失败原因） | 目标层级存储单元（L2–L5） | 每次写入操作后 | **高** |
| 源层级删除确认 | Struct（条目 ID + 删除结果：成功/失败） | 源层级存储单元（L1–L4） | 每次删除操作后 | **高** |
| 全局调度指令 | Enum（冻结/激活/紧急熔断） | ad-03 漏斗二专属调度单元、ad-01 | 模式切换或安全事件时 | **紧急/高** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 目标层级写入指令 | Struct（条目 ID + 完整经验数据 + 目标层级 + 写入类型：晋升写入） | 目标层级存储单元（L2–L5） | 搬运执行时 | **高** |
| 源层级删除指令 | Struct（条目 ID + 删除类型：晋升后删除 + 删除原因） | 源层级存储单元（L1–L4） | 目标层级写入确认成功后 | **高** |
| 搬运完成回执 | Struct（条目 ID + 源层级→目标层级 + 搬运结果 + 搬运时间戳） | ad-38、源层级、目标层级 | 单条目搬运完成（成功或失败） | **高** |
| 批次搬运汇总 | Struct（批次 ID + 总条目数 + 成功数 + 失败数 + 回滚数 + 耗时） | ad-38、ad-03、ad-01、ad-51 | 每批次搬运全部完成后 | **高** |
| 搬运异常告警 | Struct（条目 ID + 异常类型 + 当前步骤 + 处理方式） | ad-03、ad-01 | 写入失败、校验失败或回滚时 | **高** |


## 核心处理逻辑

```
FUNCTION layer_transfer_main_loop():
    SET internal_state = IDLE
    正在搬运标志 = False
    当前批次快照 = NULL  // 用于回滚

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断或冻结
        IF 收到全局冻结指令 OR 紧急熔断指令:
            IF internal_state == TRANSFERRING:
                // 中断搬运，不回滚（已完成的保留，未开始的取消）
                记录中断日志
                SET internal_state = PAUSED
            CONTINUE
        ELSE IF 收到激活指令 AND internal_state == PAUSED:
            SET internal_state = IDLE
            CONTINUE

        // 第 2 步：接收晋升候选清单
        IF 收到 ad-38 晋升候选清单 AND internal_state == IDLE:
            IF 晋升候选清单为空:
                CONTINUE

            SET internal_state = TRANSFERRING
            批次 ID = 生成唯一批次 ID()
            搬运统计 = {总条目数: 0, 成功数: 0, 失败数: 0, 回滚数: 0}
            当前批次快照 = 晋升候选清单

            // 合并所有晋升路径的候选条目，按优先级排序
            全部候选条目 = []
            FOR EACH 晋升路径, 候选列表 IN 晋升候选清单:
                FOR EACH 候选 IN 候选列表:
                    全部候选条目.append({条目 ID, 源层级, 目标层级, 晋升优先级, I 值})

            按晋升优先级降序排列 全部候选条目
            搬运统计.总条目数 = 全部候选条目.length

            // 第 3 步：逐条目搬运
            FOR EACH 候选 IN 全部候选条目:
                条目 ID = 候选.条目 ID
                源层级 = 候选.源层级
                目标层级 = 候选.目标层级

                // 3a. 单向约束校验（编译期硬编码）
                IF 源层级 >= 目标层级:
                    记录严重错误（层级编号异常）
                    搬运统计.失败数++
                    向 ad-01 发送搬运异常告警
                    CONTINUE

                IF 目标层级 - 源层级 != 1:
                    记录严重错误（跨层级跳跃）
                    搬运统计.失败数++
                    向 ad-01 发送搬运异常告警
                    CONTINUE

                // 3b. 从源层级读取完整经验数据
                向源层级存储单元发送数据读取请求(条目 ID)
                WAIT 源层级返回源条目完整数据

                IF 源条目数据 == NULL OR 条目已在源层级被删除:
                    // 可能在晋升判定后、搬运执行前被其他操作删除
                    搬运统计.失败数++
                    记录日志（条目不存在）
                    CONTINUE

                // 3c. 写入目标层级
                写入指令 = {
                    条目 ID: 条目 ID,
                    完整经验数据: 源条目数据,
                    目标层级: 目标层级,
                    写入类型: "晋升写入",
                    搬运时间戳: NOW()
                }
                向目标层级存储单元发送写入指令
                WAIT 目标层级返回写入确认

                IF 写入确认.结果 == 失败:
                    搬运统计.失败数++
                    向 ad-01 发送搬运异常告警(条目 ID, "目标层级写入失败", 写入确认.失败原因)
                    CONTINUE

                // 3d. 目标层级写入成功后，删除源层级条目
                删除指令 = {
                    条目 ID: 条目 ID,
                    删除类型: "晋升后删除",
                    删除原因: "已晋升至 L" + 目标层级
                }
                向源层级存储单元发送删除指令
                WAIT 源层级返回删除确认

                IF 删除确认.结果 == 失败:
                    // 目标层级已写入，源层级删除失败 → 数据重复但不丢失
                    搬运统计.失败数++
                    向 ad-01 发送搬运异常告警(条目 ID, "源层级删除失败", "数据可能重复")
                    // 记录供后续去重处理
                    CONTINUE

                // 3e. 搬运成功
                搬运统计.成功数++
                向 ad-38 发送搬运完成回执(条目 ID, 源层级→目标层级, 成功)

                // 3f. 搬运间短暂释放控制权
                SLEEP 5ms

            // 第 4 步：批次搬运完成
            批次汇总 = {
                批次 ID: 批次 ID,
                总条目数: 搬运统计.总条目数,
                成功数: 搬运统计.成功数,
                失败数: 搬运统计.失败数,
                回滚数: 搬运统计.回滚数,
                耗时: NOW() - 批次开始时间
            }
            向 ad-38、ad-03、ad-01、ad-51 发送批次搬运汇总
            当前批次快照 = NULL
            SET internal_state = IDLE

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 目标层级存储空间不足 | 该条目搬运失败，跳过，继续处理下一个条目。向 ad-38 返回失败原因 | 目标层级释放空间后，下一判定周期重新加入晋升候选 |
| 搬运过程中系统掉电 | 恢复后扫描各层级数据一致性：目标层级已写入但源层级未删除的条目保留在目标层，源层级重复条目由去重机制清理 | 系统重新初始化后自动修复 |
| 源条目在搬运开始前已被其他操作删除 | 该条目搬运失败，跳过 | — |
| 目标层级写入成功但源层级删除失败 | 目标层级保留有效数据，源层级条目标记为“待清理”，记录日志供后续去重 | 后续周期性去重任务清理 |
| 同一批次内搬运到一半收到紧急熔断 | 立即中断搬运，已成功搬运的条目保留，未开始的取消。记录中断日志 | 紧急解除后，下一判定周期重新评估 |
| 尝试从 L5 搬运至更高层级（不存在） | 编译期禁止，若运行时触发则记录严重错误并拒绝 | — |
| 尝试从高层级向低层级搬运 | 编译期硬编码拦截，不可执行 | — |

| MemoryBus | 读 | 源层级删除确认 | 只读 | 源层级返回 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 单向搬运为编译期硬约束。源层级编号必须严格小于目标层级编号，且差值必须为 1。任何违反此约束的搬运请求在编译期即被拒绝 |
| S-02 | 搬运操作不可跳过层级。L1 只能搬运至 L2，L2 只能搬运至 L3，以此类推。跨层级搬运请求记录为严重错误 |
| S-03 | 目标层级为 L5 的写入，须额外校验 ad-30 返回的令牌验证通过回执（由 ad-28 在写入前内部校验）。本单元在发送 L5 写入指令前，必须等待该令牌就绪 |
| S-04 | 搬运过程中目标层级写入成功后，源层级删除失败时，数据不得丢失。目标层级保留有效副本，源层级冗余条目由后续周期性去重任务安全清理 |
| S-05 | 紧急熔断时立即中断搬运，已完成的保留，未开始的取消。不得因搬运操作阻塞紧急安全指令 |
| S-06 | 所有搬运操作（含成功、失败、中断）全量写入 ad-51 不可变日志 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-39-01 | IDLE，L2→L3 候选 3 条，目标层级空间充足 | 晋升候选清单（3 条 L2→L3） | 逐条搬运：读源→写目标→删源，全部成功，返回批次汇总（成功 3） |
| TC-39-02 | IDLE，L1→L2 候选 1 条，目标层级空间不足 | 晋升候选清单（1 条 L1→L2） | 写入目标失败，条目保留 L1，搬运失败数 1 |
| TC-39-03 | IDLE，条目在搬运开始前已被删除 | 晋升候选清单（1 条，但源层级返回 NULL） | 搬运失败数 1，记录日志 |
| TC-39-04 | IDLE，写入 L5 的条目 | 晋升候选清单（1 条 L4→L5，令牌就绪） | 先校验令牌→写入 L5→删除 L4→成功 |
| TC-39-05 | TRANSFERRING，收到紧急熔断指令 | 紧急熔断信号 | 立即中断，已完成的 2 条保留，剩余 3 条取消，记录中断日志 |
| TC-39-06 | IDLE，收到跨层级跳跃请求（L1→L3） | 晋升候选清单（源 L1，目标 L3） | 校验拒绝，记录严重错误，搬运失败 |
| TC-39-07 | IDLE，收到反向搬运请求（L3→L2） | 晋升候选清单（源 L3，目标 L2） | 编译期硬编码拦截，不可执行 |
| TC-39-08 | IDLE，写入成功但删除源层级失败 | 目标写入确认成功，源删除确认失败 | 目标保留，源标记“待清理”，记录日志，搬运失败数 1 |
| TC-39-09 | 批次搬运完成 | 3 条全部成功 | 发送批次搬运汇总至 ad-38、ad-03、ad-01、ad-51 |
| TC-39-10 | PAUSED | 晋升候选清单 | 不处理，等待激活 |
