# ad-47-疑问缓存库 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-47 |
| 模块名称 | 疑问缓存库 |
| 所属分区 | 四、漏斗外挂扩展区（物理隔离） |
| 核心职责 | 暂存 ECC 认知大脑推理过程中产生的逻辑断点、未确认认知项、低置信度场景及待补全信息。为离线复盘、根因分析、策略修正、云端大模型异步识别提供结构化数据源。独立于双漏斗记忆系统运行，不参与记忆的沉淀、筛选、晋升与遗忘机制。任务周期结束后可选清理或归档 |
| 依赖模块 | ECC-01 情境解析模块（提交未知识别目标）、ECC-03 因果推理模块（提交推理断点）、ECC-04 心智模拟模块（提交低置信度方案）、ECC-08 元认知模块（提交认知偏差与能力缺口）、ECC-12 资源全域调度模块（查询疑问缓存用于云端异步处理） |
| 被依赖模块 | ECC-08（消费疑问缓存进行能力缺口分析）、ECC-12（向云端大模型提交未知识别请求）、离线复盘系统（消费疑问缓存进行根因分析与策略修正）、ad-27（L4 抽象提炼单元，参考疑问缓存进行规则归纳） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收疑问提交与查询 | 系统初始化完成 |
| 疑问记录中 | `RECORDING` | 正在接收并结构化存储疑问条目 | ECC 模块提交疑问时 |
| 批量查询中 | `QUERYING` | 正在处理疑问缓存查询请求 | ECC-12 或离线系统查询时 |
| 定期清理中 | `CLEANING` | 正在按生命周期策略清理过期疑问 | 周期性触发或容量告急 |
| 离线回放模式 | `REPLAY` | 向离线复盘系统批量导出疑问数据 | 车辆处于 P 档且充电/维护状态 |
| 暂停服务 | `PAUSED` | 系统紧急熔断 | 收到 ad-01 紧急熔断指令 |

| 紧急熔断指令 | Enum（全局熔断） | ad-01 总控漏斗 F₀ | 安全急停或碰撞事件时 | **紧急** |

## 疑问分类与生命周期

### 一、疑问类型定义

| 疑问类型 | 提交来源 | 内容描述 | 默认生命周期 | 处理方式 |
|----------|:---:|------|:---:|----------|
| 未知识别目标 | ECC-01 | 感知模块检测到实体但无法分类 | 72 小时 | 云端大模型异步识别 → 识别成功则更新世界模型建议 |
| 推理逻辑断点 | ECC-03 | 因果推理链路因信息缺失而中断 | 7 日 | 待相关信息补全后重新推理 |
| 低置信度场景 | ECC-04、ECC-08 | 决策置信度 C_total < 0.4 的异常场景 | 14 日 | 离线复盘 → 根因分析 → 策略修正 |
| 认知能力缺口 | ECC-08 | 元认知模块识别的系统能力短板 | 30 日 | 纳入自主学习目标，由内生动机模块驱动 |
| 法规模糊地带 | ECC-05 | 伦理仲裁遇到法规未明确覆盖的场景 | 永久（归档） | 定期汇总上报，用于法规库更新建议 |
| 传感器数据异常 | ECC-01 | 传感器数据质量异常但原因不明 | 48 小时 | 关联硬件诊断，超时未复现则清理 |

### 二、疑问生命周期管理

| 阶段 | 触发条件 | 操作 |
|------|----------|------|
| 创建 | ECC 模块提交疑问 | 分配唯一疑问 ID，记录时间戳，写入缓存 |
| 等待处理 | 疑问未被查询或回写 | 保留在活跃区，等待 ECC-12 查询或生命周期到期 |
| 已处理 | 云端回写识别结果、离线复盘完成、或关联信息补全 | 标记为“已处理”，保留 7 日后可清理 |
| 到期清理 | 超过生命周期且未标记为“已处理” | 普通疑问：安全删除；法规模糊地带：永久归档至 ad-49 |
| 容量告急清理 | 容量 > 80% 且无到期条目可清理 | 按优先级从低到高清理：传感器异常 > 未知识别 > 推理断点 > 低置信度 > 认知缺口（法规模糊不清理） |


## 核心处理逻辑

```
FUNCTION question_cache_main_loop():
    SET internal_state = NORMAL
    初始化疑问缓存存储分区
    疑问条目计数器 = 0
    疑问字典 = {}  // 疑问 ID → 疑问条目
    最大容量 = 50MB（可配置）

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF internal_state == PAUSED AND 紧急解除:
            SET internal_state = NORMAL

        // 第 2 步：接收疑问提交
        IF 收到疑问提交请求 AND internal_state == NORMAL:
            SET internal_state = RECORDING

            疑问类型 = 请求.疑问类型
            提交来源 = 请求.提交来源模块
            场景快照 = 请求.场景快照
            置信度 = 请求.置信度
            紧急程度 = 请求.紧急程度

            // 2a. 去重检查
            IF 疑问类型 == "未知识别目标":
                // 检查同一目标是否已有未处理的疑问
                FOR EACH 已有疑问 IN 疑问字典:
                    IF 已有疑问.类型 == "未知识别目标" AND 已有疑问.目标 ID == 请求.目标 ID AND 已有疑问.状态 == "等待处理":
                        // 更新已有疑问的出现频率与最新时间
                        已有疑问.出现频率++
                        已有疑问.最新时间戳 = NOW()
                        向提交方返回疑问条目确认回执(已有疑问.ID, "更新")
                        SET internal_state = NORMAL
                        CONTINUE

            // 2b. 创建新疑问条目
            疑问 ID = 生成唯一疑问 ID()
            生命周期 = 获取默认生命周期(疑问类型)
            疑问条目 = {
                疑问 ID: 疑问 ID,
                疑问类型: 疑问类型,
                提交来源: 提交来源,
                场景快照哈希: HASH(场景快照),
                场景摘要: 提取场景摘要(场景快照),
                置信度: 置信度,
                紧急程度: 紧急程度,
                状态: "等待处理",
                创建时间戳: NOW(),
                生命周期: 生命周期,
                过期时间: NOW() + 生命周期,
                出现频率: 1,
                关联疑问 ID 列表: 查找关联疑问(疑问类型, 场景快照),
                云端回写结果: NULL
            }

            // 2c. 存储疑问条目
            疑问字典[疑问 ID] = 疑问条目
            疑问条目计数器++

            // 2d. 返回确认回执
            向提交方返回疑问条目确认回执(疑问 ID, "创建", 生命周期)

            // 2e. 写入疑问日志
            向 ad-51 写入疑问创建日志(疑问 ID + 疑问类型 + 时间戳)

            SET internal_state = NORMAL

        // 第 3 步：处理云端大模型识别结果回写
        IF 收到 ECC-12 云端识别结果回写 AND internal_state == NORMAL:
            原疑问 ID = 回写.原疑问条目 ID
            识别结果 = 回写.识别结果
            识别置信度 = 回写.置信度

            IF 原疑问 ID IN 疑问字典:
                疑问条目 = 疑问字典[原疑问 ID]
                疑问条目.云端回写结果 = {
                    识别结果: 识别结果,
                    置信度: 识别置信度,
                    回写时间戳: NOW()
                }
                疑问条目.状态 = "已处理"

                // 若识别置信度 > 0.7，建议更新世界模型
                IF 识别置信度 > 0.7:
                    向 ECC-12 返回云端识别回写确认(原疑问 ID, "已处理", "建议更新世界模型")

                     // 生成世界模型更新建议
                    更新建议 = {
                        目标 ID: 疑问条目.目标 ID,
                        建议分类: 识别结果.建议分类,
                        置信度: 识别置信度,
                        来源: "云端大模型识别",
                        原始疑问 ID: 原疑问 ID
                    }
                    向 ECC-12 转发世界模型更新建议
                ELSE:
                    向 ECC-12 返回云端识别回写确认(原疑问 ID, "已处理", "置信度不足，保留待后续确认")
            ELSE:
                向 ECC-12 返回云端识别回写确认(原疑问 ID, "失败", "原疑问条目已清理")

        // 第 4 步：处理疑问查询请求
        IF 收到疑问查询请求 AND internal_state IN [NORMAL, CLEANING]:
            SET internal_state = QUERYING

            查询类型 = 请求.查询类型
            时间范围 = 请求.时间范围
            疑问类型筛选 = 请求.疑问类型筛选

            结果列表 = []
            FOR EACH 疑问条目 IN 疑问字典:
                IF 疑问条目.创建时间戳 IN 时间范围:
                    IF 疑问类型筛选 == NULL OR 疑问条目.疑问类型 IN 疑问类型筛选:
                        结果列表.append({
                            疑问 ID: 疑问条目.疑问 ID,
                            疑问类型: 疑问条目.疑问类型,
                            场景摘要: 疑问条目.场景摘要,
                            置信度: 疑问条目.置信度,
                            提交时间: 疑问条目.创建时间戳,
                            状态: 疑问条目.状态
                        })

            按创建时间戳降序排列 结果列表
            向请求方返回疑问查询结果(结果列表)
            SET internal_state = NORMAL

        // 第 5 步：定期清理过期疑问
        IF 收到定期清理触发信号 OR 容量使用率 > 80%:
            SET internal_state = CLEANING
            清理统计 = {清理条目数: 0, 归档条目数: 0, 释放空间: 0}
            当前时间 = NOW()

            FOR EACH 疑问条目 IN 疑问字典:
                应清理 = False

                // 5a. 已处理条目：保留 7 日后清理
                IF 疑问条目.状态 == "已处理" AND 当前时间 - 疑问条目.回写时间戳 > 7 日:
                    应清理 = True
                // 5b. 未处理条目：超过生命周期
                ELSE IF 疑问条目.状态 == "等待处理" AND 当前时间 > 疑问条目.过期时间:
                    应清理 = True
                // 5c. 容量告急时按优先级清理
                ELSE IF 容量使用率 > 80% AND 当前时间 - 疑问条目.创建时间戳 > 疑问条目.生命周期 × 0.5:
                    清理优先级 = [传感器数据异常, 未知识别目标, 推理逻辑断点, 低置信度场景, 认知能力缺口]
                    IF 疑问条目.疑问类型 IN 清理优先级[:3]:  // 取前 3 类优先清理
                        应清理 = True

                IF 应清理:
                    IF 疑问条目.疑问类型 == "法规模糊地带":
                        // 法规模糊不删除，永久归档
                        向 ad-49 发送归档请求(疑问条目)
                        清理统计.归档条目数++
                    ELSE:
                        REMOVE 疑问条目 FROM 疑问字典
                        清理统计.清理条目数++
                    清理统计.释放空间 += 估算条目大小(疑问条目)

            // 生成清理汇总报告
            向 ad-03、ad-01、ad-51 发送清理汇总报告

            // 容量仍高则上报告警
            IF 容量使用率 > 80%:
                向 ad-03、ad-01 发送容量告警

            SET internal_state = NORMAL

        // 第 6 步：离线回放模式（车辆 P 档 + 充电/维护）
        IF 收到离线回放请求 AND 车辆状态 IN [P档, 充电中, 维护中]:
            SET internal_state = REPLAY
            回放数据包 = 打包离线回放数据(疑问字典, 请求.时间范围)
            向离线复盘系统发送回放数据包
            SET internal_state = NORMAL

        // 第 7 步：定期向 ECC-12 推送未识别目标清单
        IF 距上次推送 >= 30 分钟:
            未识别清单 = 筛选(疑问字典, 疑问类型="未知识别目标", 状态="等待处理")
            按出现频率降序排列 未识别清单
            向 ECC-12 发送未识别目标清单(建议云端批量识别)
            // 同时更新出现频率为 0，下个周期重新统计

        // 第 8 步：状态上报
        IF 距上次上报 >= 60 秒:
            向 ad-03、ad-01 上报运行状态

        SLEEP 500ms  // 低频操作
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 同一未知识别目标重复提交 | 去重处理，更新出现频率与最新时间戳，不创建新条目 | — |
| 疑问条目数量超过 10,000 | 触发强制清理：清理所有已处理条目 + 超过生命周期 50% 的低优先级条目 | 条目数回落至 5,000 以下 |
| 云端大模型识别超时（> 72 小时无回写） | 疑问保持“等待处理”状态，过期后按生命周期正常清理 | 云端回写到达 |
| 疑问查询请求指定时间范围无匹配 | 返回空列表 | — |
| 容量使用率 > 90% | 触发紧急清理：跳过生命周期，优先清理传感器异常、未知识别、推理断点三类 | 容量回落至 < 70% |
| 离线回放时车辆状态变更（退出 P 档） | 中断回放，未传输数据保留 | 下次进入 P 档/充电时继续 |
| 紧急熔断期间收到疑问提交 | 暂存于提交方本地缓存（最长 30 秒），紧急解除后补交 | 紧急解除 |
| 疑问关联的原始场景数据过大（> 1MB） | 仅存储场景快照哈希与场景摘要，完整数据引用 ECC 决策日志中的存储地址 | — |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 疑问缓存独立于双漏斗记忆系统，不参与记忆的沉淀、筛选、晋升与遗忘机制。疑问数据不得自动转化为漏斗经验条目 |
| S-02 | 疑问条目中的场景快照数据须脱敏处理：剔除 GPS 精确坐标（仅保留道路类型+区域类型）、行人/车辆特征（仅保留类别标签）、时间戳精度降至小时级 |
| S-03 | 云端大模型识别请求须经 ECC-12 安全校验后发出，原始感知数据（图像/点云）不得直接上传云端，仅发送特征向量或脱敏描述 |
| S-04 | 疑问缓存数据在生命周期到期后必须安全清理（DoD 5220.22-M 标准覆写），法规模糊地带例外：永久归档至 ad-49 |
| S-05 | 疑问缓存数据仅供离线复盘、根因分析与系统优化使用，不作为实时驾驶决策的依据 |
| S-06 | 离线回放数据导出须在车辆安全停车状态（P 档）且经维护授权后方可执行 |
| S-07 | 所有疑问创建、处理、清理操作全量写入 ad-51 不可变日志 |
