# ad-22-L2-近期层存储单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-22 |
| 模块名称 | L2 近期层存储单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 五层记忆层级存储 |
| 核心职责 | 存储近 7 日内高频出现的驾驶场景经验，占漏斗二总存储容量的 25%。接收来自 L1 晋升的条目，在留存满 7 日且满足重要度条件后进入 L3 晋升候选。维护本层条目的热度统计，为晋升决策提供查询命中频率参考 |
| 依赖模块 | ad-20（L1 临时层存储单元，接收晋升条目）、ad-23（L2 近期层热度统计单元，提供热度数据）、ad-38（晋升双条件判定单元，下发 L2 晋升候选清单）、ad-40（遗忘阈值判定单元，下发 L2 遗忘候选清单） |
| 被依赖模块 | ad-23（消费 L2 存储条目进行热度统计）、ad-38（消费 L2 晋升候选条目）、ad-40（消费 L2 遗忘候选条目） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收晋升条目，管理存储与流转 | 漏斗二激活，本层级初始化完成 |
| 存储将满 | `NEAR_FULL` | 存储占用率 > 85%，触发温和清理 | 存储占用率监测 |
| 存储已满 | `FULL` | 存储占用率 > 95%，强制清除低重要度条目 | 存储占用率监测 |
| 维护中 | `MAINTENANCE` | 正在执行批量清除或碎片整理 | 定时维护或存储已满触发 |
| 冻结只读 | `FROZEN` | 漏斗二冻结或紧急接管，暂停写入仅保留查询 | 驾驶模式切换或紧急熔断 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| L1 晋升条目 | Struct（条目 ID + 经验内容 + I 值 + 来源分槽号 + 子类标记 + L1 停留时长） | ad-20 L1 临时层存储单元（经 ad-39 搬运） | L1 条目通过晋升判定后 | **高** |
| L2 晋升候选清单 | List（条目 ID + 晋升目标层级 + I 值 + 留存时长） | ad-38 晋升双条件判定单元 | 每个晋升判定周期 | **高** |
| L2 遗忘候选清单 | List（条目 ID + 当前层级 + I 值） | ad-40 遗忘阈值判定单元 | 每个遗忘判定周期 | **高** |
| 晋升失败回退条目 | Struct（条目 ID + 回退原因） | ad-38 晋升双条件判定单元 | L2 条目晋升 L3 失败时 | 普通 |
| 全局调度指令 | Enum（冻结/激活/维护） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |
| 批量清除指令 | Signal | ad-48 全局容量配额管控单元 | 全局容量逼近上限时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 存储完成回执 | Struct（条目 ID + 存储层级=L2 + 存储时间戳 + 当前 L2 占用率） | ad-20、ad-38 | 每次晋升条目成功存储后 | 普通 |
| 晋升执行结果 | Struct（条目 ID + 原层级=L2 + 目标层级 + 执行结果） | ad-38、ad-39 | 晋升完成或失败后 | **高** |
| 遗忘执行结果 | Struct（条目 ID + 删除/归档 + 结果） | ad-40、ad-42 | 遗忘执行后 | 普通 |
| L2 状态快照 | Struct（占用率 + 条目总数 + 各分槽条目分布 + 平均留存时长 + 平均 I 值） | ad-03、各对应分槽 | 每 10 秒周期性上报 | 普通 |
| 存储异常告警 | Struct（异常类型 + 当前占用率 + 详情） | ad-03、ad-01 | 存储写入失败或数据校验异常时 | **高** |


## 核心处理逻辑

```
FUNCTION l2_recent_storage_main_loop():
    SET internal_state = NORMAL
    初始化 L2 存储分区（占漏斗二总容量 25%，各分槽按活跃度动态分配）
    初始化条目索引表（条目 ID → 存储地址 + 晋升时间戳 + I 值 + 来源分槽号）
    最大留存时间 = 7 日（168 小时）

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = FROZEN
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 第 2 步：接收 L1 晋升条目
        IF 收到 L1 晋升条目 AND internal_state IN [NORMAL, NEAR_FULL]:
            // 2a. 检查存储空间
            IF internal_state == FULL:
                立即执行紧急清除：找出当前 L2 中 I 值最低的 5% 条目
                调用 ad-42 删除
                IF 释放空间仍不足:
                    向 ad-03 上报告警
                    返回存储失败（原因=L2 存储满）
                    CONTINUE
                ELSE:
                    SET internal_state = NEAR_FULL

            新条目 = 晋升条目
            // 2b. L2 存储 I 值微调（近期层经验价值略高于临时层）
            新条目.I 值 = MIN(新条目.I 值 + 0.02, 1.0)

            // 2c. 分配空间并写入
            存储地址 = 分配空间(新条目.大小)
            IF 分配失败:
                触发碎片整理 → 重试
                IF 仍失败:
                    返回存储失败回执
                    CONTINUE

            写入存储介质
            更新条目索引表：
                条目 ID → {地址, NOW(), 新条目.I 值, 新条目.来源分槽号, 子类标记}
            向 ad-20、ad-38 返回存储完成回执

            // 2d. 更新占用率
            占用率 = 计算占用率()
            IF 占用率 > 0.95:
                SET internal_state = FULL
                上报告警
            ELSE IF 占用率 > 0.85 AND internal_state == NORMAL:
                SET internal_state = NEAR_FULL

        // 第 3 步：处理 L2 晋升候选
        IF 收到 ad-38 L2 晋升候选清单:
            FOR EACH 候选 IN 清单:
                条目 ID = 候选.条目 ID
                IF 候选.目标层级 == L3 AND L3 存储未满:
                    调用 ad-39 执行层级搬运（L2 → L3）
                    向 ad-38 返回晋升成功
                    从 L2 索引表中移除
                ELSE:
                    向 ad-38 返回晋升失败（原因=目标层级满或不存在）

        // 第 4 步：处理 L2 遗忘候选
        IF 收到 ad-40 L2 遗忘候选清单:
            FOR EACH 候选 IN 清单:
                调用 ad-42 执行安全删除
                向 ad-40 返回遗忘执行结果
                从 L2 索引表中移除

        // 第 5 步：处理晋升失败回退
        IF 收到晋升失败回退条目（从 L3 退回）:
            条目继续留在 L2
            标记回退次数 +1
            IF 回退次数 ≥ 3:
                标记为“晋升困难条目”，降低其 I 值 0.05
            记录回退日志

        // 第 6 步：全局容量告急
        IF 收到 ad-48 批量清除指令:
            SET internal_state = MAINTENANCE
            按 I 值升序排列所有 L2 条目
            清除 I 值最低的 15% 条目（优先清除非安全条目）
            向 ad-42 批量发送清除指令
            向 ad-48 返回释放空间量
            SET internal_state = NORMAL

        // 第 7 步：定期碎片整理
        IF 距上次整理 >= 12 小时 AND 占用率 > 70%:
            SET internal_state = MAINTENANCE
            执行碎片整理
            SET internal_state = NORMAL

        // 第 8 步：状态上报
        IF 距上次上报 >= 10 秒:
            向 ad-03 和各分槽上报 L2 状态快照

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| L2 存储写入失败（硬件故障） | 重试 3 次，若均失败则标记坏块，使用备用存储块，上报告警 | 坏块修复或替换 |
| L1 晋升条目 I 值异常（> 1.0 或 < 0） | 钳制到 [0, 1]，上报告警 | — |
| L2 条目在 7 日内被高频查询但 I 值偏低 | 由 ad-23 热度统计单元上报，热度值作为额外晋升参考权重 | — |
| L2 条目留存超 7 日但未满足晋升条件 | 由 ad-40 遗忘阈值判定单元评估，纳入遗忘候选 | — |
| 条目索引表损坏 | 从存储分区重建，上报告警 | 重建完成 |
| 全局容量告急但 L2 安全条目占比高 | 仅清除非安全条目（S < 0.5），安全条目继续保留 | 容量释放 |
| 冻结状态写入 | 拒绝写入，返回原因 | 激活指令 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | L2 层所有条目最大留存时间硬编码为 7 日（168 小时），超时未晋升则进入遗忘评估 |
| S-02 | 安全显著性 S ≥ 0.7 的条目在批量清除时享有保护，不被强制清除 |
| S-03 | 晋升失败回退次数 ≥ 3 的条目须标记为“晋升困难”，降低 I 值并上报告警 |
| S-04 | 条目索引表每 12 小时自动备份至冗余分区 |
| S-05 | 存储写入须校验条目完整性 |
| S-06 | 冻结状态下禁止任何写入操作 |
| S-07 | 所有写入、晋升、遗忘、回退操作全量写入 ad-51 变更日志 |
