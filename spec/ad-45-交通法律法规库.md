# ad-45-交通法律法规库 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-45 |
| 模块名称 | 交通法律法规库 |
| 所属分区 | 四、漏斗外挂扩展区（物理隔离） |
| 核心职责 | 存储法定通行刚性规则与通行逻辑，以“场景条件-强制执行动作”形式固化，为 ECC 认知大脑提供法规合规底线查询。法规库独立于双漏斗记忆系统运行，不参与记忆的沉淀、筛选、晋升与遗忘机制。法规条目固化不可更改，仅接受经审批的追加写入。是自动驾驶系统决策的最终合规终审依据 |
| 依赖模块 | ECC-05 伦理仲裁模块（主要查询方）、ECC-03 因果推理模块（查询方）、ECC-04 心智模拟模块（查询方）、ad-09（行为判定标签单元，查询法规基准） |
| 被依赖模块 | ECC-05（消费法规条目进行伦理仲裁）、ECC-03/04（消费法规约束进行推理与模拟）、ad-09（消费法规基准判定驾驶行为） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常提供法规查询服务 | 系统初始化完成，法规库数据加载校验通过 |
| 法规查询中 | `QUERYING` | 正在根据场景条件匹配适用法规条目 | 收到法规查询请求 |
| 审批追加中 | `APPENDING` | 正在执行经审批的新法规条目追加 | 收到 ECC-12 转发的审批通过追加指令 |
| 数据校验中 | `VALIDATING` | 正在校验法规库数据完整性 | 系统初始化或定期校验周期 |
| 降级保守模式 | `DEGRADED` | 数据校验发现不一致，暂停追加仅保留查询，使用缓存基线 | 数据校验失败 |
| 暂停服务 | `PAUSED` | 系统紧急熔断 | 收到 ad-01 紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 法规查询请求 | Struct（场景条件 + 查询类型 + 请求来源模块 + 查询优先级） | ECC-05/03/04、ad-09 等模块（经 MemoryBus） | ECC 需要获取当前场景适用的法规约束时 | **高** |
| 法规基准查询请求 | Struct（行为类型 + 场景标签 + 道路等级 + 请求来源模块） | ad-09 行为判定标签单元 | 判定驾驶行为合规性时 | **高** |
| 审批追加指令 | Struct（新法规条目 + 追加原因 + 审批令牌 + 生效日期 + 审批时间戳） | ECC-12 资源全域调度模块（经 MemoryBus） | 交通法规更新或地方临时管制经审批后 | **高** |
| 数据校验指令 | Signal | ad-01 总控漏斗 F₀ 或系统初始化模块 | 系统上电自检或定期校验周期 | **高** |
| 紧急熔断指令 | Enum（全局熔断） | ad-01 总控漏斗 F₀ | 安全急停或碰撞事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 适用法规条目清单 | List（法规条目 ID + 规则内容 + 执行刚性等级 + 违规后果 + 匹配精度） | 请求来源模块（经 MemoryBus） | 法规查询请求处理完成后 | **高** |
| 法规基准判定参数 | Struct（行为类型 : 合规阈值 + 违规判定标准 + 场景调整系数） | ad-09 行为判定标签单元 | 法规基准查询请求处理完成后 | **高** |
| 临时伦理豁免凭证 | Struct（豁免条目 ID + 豁免原因 + 豁免有效期 + 审计标记） | ECC-05 伦理仲裁模块 | 极端安全困境且 ECC-05 发出豁免请求时 | **高** |
| 审批追加确认 | Struct（追加条目 ID + 追加结果 + 法规版本号 + 追加时间戳） | ECC-12、ad-51 | 审批追加执行完成后 | **高** |
| 数据校验报告 | Struct（校验时间 + 校验范围 + 法规条目总数 + 数据完整性 + 异常条目清单） | ad-01、ad-03 | 数据校验完成后 | **高** |
| 查询拒绝通知 | Struct（拒绝原因 + 建议操作） | 请求来源模块 | 查询条件不合法或系统降级时 | 普通 |
| 运行状态上报 | Struct（当前状态 + 近期查询次数 + 法规版本号 + 追加次数） | ad-03、ad-01 | 每 30 秒周期性上报 | 普通 |

## 核心处理逻辑

```
FUNCTION traffic_law_database_main_loop():
    SET internal_state = NORMAL
    加载交通法规库数据（编译期固化条目 + OTA 追加条目）
    校验法规库数据完整性（哈希校验 + 数字签名验证）
    法规版本号 = 初始版本号

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF internal_state == PAUSED AND 紧急解除:
            SET internal_state = NORMAL

        // 第 2 步：数据校验
        IF 收到数据校验指令 OR (距上次校验 >= 24 小时):
            SET internal_state = VALIDATING
            校验结果 = 执行全量哈希校验 + 数字签名验证 + 法规条目完整性检查
            IF 校验结果.存在异常:
                SET internal_state = DEGRADED
                向 ad-01 发送数据校验报告（含异常条目清单）
            ELSE:
                向 ad-01、ad-03 发送数据校验报告（正常）
                IF internal_state == DEGRADED:
                    SET internal_state = NORMAL
            CONTINUE

        // 第 3 步：处理法规查询请求
        IF 收到法规查询请求 AND internal_state IN [NORMAL, QUERYING, DEGRADED]:
            场景条件 = 请求.场景条件  // 含路口类型、信号灯状态、道路等级、交通标识等
            请求来源 = 请求.请求来源模块

            IF 请求来源 NOT IN 授权查询模块列表:
                向请求方返回查询拒绝通知（原因=未授权模块）
                CONTINUE

            SET internal_state = QUERYING

            // 3a. 精确匹配法规条目
            适用法规清单 = []
            FOR EACH 法规条目 IN 法规库:
                IF 法规条目.场景条件 匹配 场景条件:
                    适用法规清单.append({
                        法规 ID: 法规条目.ID,
                        规则内容: 法规条目.内容,
                        执行刚性等级: 法规条目.刚性等级,
                        违规后果: 法规条目.违规后果,
                        匹配精度: 计算匹配精度(场景条件, 法规条目.场景条件)
                    })

            // 3b. 按刚性等级排序：硬约束优先
            按执行刚性等级降序排列 适用法规清单

            // 3c. 返回查询结果
            向请求方返回适用法规条目清单
            SET internal_state = NORMAL

        // 第 4 步：处理法规基准查询请求（来自 ad-09）
        IF 收到法规基准查询请求 AND internal_state IN [NORMAL, DEGRADED]:
            行为类型 = 请求.行为类型
            场景标签 = 请求.场景标签
            道路等级 = 请求.道路等级

            // 根据行为类型匹配基准参数
            合规基准 = 获取合规基准(行为类型, 道路等级)
            场景调整系数 = 获取场景调整(场景标签)  // 如恶劣天气阈值收紧

            IF 合规基准 != NULL:
                法规基准参数 = {
                    行为类型: 行为类型,
                    合规阈值: 合规基准.阈值 × 场景调整系数,
                    违规判定标准: 合规基准.判定标准,
                    场景调整系数: 场景调整系数
                }
                向 ad-09 返回法规基准判定参数
            ELSE:
                向 ad-09 返回查询拒绝通知（原因=行为类型无对应法规基准）

        // 第 5 步：处理极端安全困境的临时伦理豁免
        IF 收到 ECC-05 临时伦理豁免请求 AND internal_state IN [NORMAL, DEGRADED]:
            豁免条目 ID = 请求.目标法规 ID
            豁免原因 = 请求.豁免原因（必须为“避免不可逆重度人身伤害”）
            场景快照 = 请求.场景快照

            // 5a. 验证豁免条件
            IF 豁免原因 != "避免不可逆重度人身伤害":
                向 ECC-05 返回查询拒绝通知（原因=豁免条件不满足）
                CONTINUE

            IF 法规条目[豁免条目 ID].刚性等级 != "硬约束":
                向 ECC-05 返回查询拒绝通知（原因=仅硬约束可申请豁免）
                CONTINUE

            // 5b. 生成临时伦理豁免凭证
            豁免凭证 = {
                豁免条目 ID: 豁免条目 ID,
                豁免原因: 豁免原因,
                豁免有效期: NOW() + 10 秒,  // 仅当次场景有效
                审计标记: "强制事后审计",
                场景快照哈希: HASH(场景快照),
                生成时间戳: NOW()
            }

            // 5c. 记录豁免事件
            向 ad-51 写入临时伦理豁免日志（含完整豁免凭证 + 场景快照）
            向 ECC-05 返回临时伦理豁免凭证
            CONTINUE

        // 第 6 步：处理审批追加（新法规条目）
        IF 收到 ECC-12 审批追加指令 AND internal_state IN [NORMAL, DEGRADED]:
            // 6a. 审批令牌与数字签名校验
            IF 审批令牌无效 OR 数字签名验证失败:
                向 ECC-12 返回审批追加确认(失败, 原因="令牌或签名无效")
                CONTINUE

            SET internal_state = APPENDING

            // 6b. 校验新条目格式与冲突
            新条目 = 指令.新法规条目
            IF 新条目.刚性等级 NOT IN [软约束, 软追加]:
                向 ECC-12 返回审批追加确认(失败, 原因="仅允许追加软约束/软追加条目")
                CONTINUE

            // 检查与已有硬约束是否冲突
            IF 检测冲突(新条目, 法规库.硬约束条目):
                向 ECC-12 返回审批追加确认(失败, 原因="与已有硬约束冲突")
                CONTINUE

            // 6c. 追加新条目
            法规库.追加(新条目)
            法规版本号++

            // 6d. 记录追加日志
            向 ad-51 写入法规库追加日志(新条目 + 版本号 + 时间戳)
            向 ECC-12 返回审批追加确认(成功, 版本号)
            SET internal_state = NORMAL

        // 第 7 步：定期状态上报
        IF 距上次上报 >= 30 秒:
            向 ad-03、ad-01 上报运行状态（含法规版本号 + 查询统计）

        SLEEP 20ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 查询场景条件无法匹配任何法规条目 | 返回空清单，附加通用安全规则（如“保持安全车速”） | — |
| 法规条目数字签名验证失败 | 该条目标记为“失效”，从查询结果中排除，上报告警 | 重新 OTA 下发正确条目 |
| 审批追加令牌无效 | 拒绝追加，记录安全日志 | — |
| 审批追加试图添加硬约束条目 | 拒绝追加（硬约束仅可通过固件升级修改），上报告警 | — |
| 新条目与已有硬约束冲突 | 拒绝追加，上报告警，提示冲突详情 | 冲突解决后重新审批 |
| 临时伦理豁免被滥用（频繁触发） | 上报告警至 ad-01，触发安全审计 | 审计确认无异常后解除 |
| 法规库数据文件损坏 | 状态=DEGRADED，从镜像分区加载备份，上报告警 | 备份加载成功且校验通过 |
| 紧急熔断期间收到查询 | 返回缓存基线法规数据 | 紧急解除 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 交通法规库独立于双漏斗记忆系统，不参与记忆的沉淀、筛选、晋升与遗忘机制。任何驾驶经验不得写入本模块 |
| S-02 | 硬约束法规条目（基础通行禁令、特种车辆优先、路权划分标准）为编译期固化，不可通过运行时 OTA 修改或删除 |
| S-03 | 软约束条目可经审批追加，但必须通过数字签名校验与冲突检测，且不可与已有硬约束冲突 |
| S-04 | 极端安全困境下的临时伦理豁免须 ECC-05 发起，豁免有效期仅 10 秒（当次场景），且强制事后审计 |
| S-05 | 法规库数据须每 24 小时校验一次数字签名与完整性，校验失败立即从镜像分区恢复 |
| S-06 | 法规库为自动驾驶系统决策的最终合规底线，任何自主学习的驾驶策略不得突破法规硬约束 |
| S-07 | 所有法规查询、豁免、追加操作全量写入 ad-51 不可变日志 |
