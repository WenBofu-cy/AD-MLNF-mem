# ad-18-特殊环境槽 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-18 |
| 模块名称 | 特殊环境槽 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 场景分槽管理 |
| 核心职责 | 承载雨、雪、雾、沙尘、积水、结冰、施工区、夜间无灯等特殊环境下的驾驶经验完整五层存储与晋升管理。执行专属遗忘策略：所有晋升阈值下调 30%，使特殊环境下的宝贵经验更快固化；遗忘阈值下调 40%，大幅延长保留时间。是本场景经验从 L1 到 L5 的完整生命周期管理者 |
| 依赖模块 | ad-14（场景判定与分槽路由单元，接收经验写入指令）、ad-20 至 ad-30（五层存储与晋升遗忘执行模块）、ad-36（综合重要度 I 值聚合计算单元） |
| 被依赖模块 | ad-14（上报存储占用率与活跃状态）、ad-03（漏斗二专属调度单元，接收状态汇总） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收经验条目并执行五层存储流转 | 漏斗二已激活，本槽已初始化 |
| 容量告急 | `CAPACITY_WARNING` | 存储占用率 > 90%，已触发保护性归档 | 本槽存储占用率监测 |
| 维护中 | `MAINTENANCE` | 正在执行相似经验归并或冷归档操作 | 定期维护周期或容量告急触发 |
| 冻结只读 | `FROZEN` | 漏斗二冻结或紧急接管，暂停写入但保留查询 | 驾驶模式切换或紧急熔断 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 经验写入指令 | Struct（含经验条目 + 写入优先级 + 路由标记 + 判定置信度） | ad-14 场景判定与分槽路由单元 | 每次场景路由至本槽时 | **高** |
| L1 至 L4 晋升候选清单 | List（经验条目 ID + 晋升目标层级 + 重要度 I） | ad-38 晋升双条件判定单元 | 每个晋升判定周期 | **高** |
| 遗忘候选清单 | List（经验条目 ID + 当前层级 + I 值） | ad-40 遗忘阈值判定单元 | 每个遗忘判定周期 | **高** |
| 归并建议 | Struct（源经验 ID + 目标经验 ID + 相似度） | ad-25 L3 中期层相似经验归并单元 | L3 归并操作时 | 普通 |
| 全局调度指令 | Enum（冻结/激活/维护） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 存储完成回执 | Struct（经验条目 ID + 存储层级 + 存储时间戳） | ad-14 | 每次经验成功存储后 | 普通 |
| 晋升执行结果 | Struct（经验条目 ID + 原层级 + 目标层级 + 执行结果） | ad-38、ad-39 | 晋升完成后 | **高** |
| 遗忘执行结果 | Struct（经验条目 ID + 删除/归档 + 结果） | ad-40、ad-42 | 遗忘执行后 | 普通 |
| 槽状态快照 | Struct（存储占用率 + 各层级条目数 + 活跃度） | ad-14、ad-03 | 每 1 秒周期性上报 | 普通 |
| 容量告警 | Struct（当前占用率 + 触发阈值 + 建议动作） | ad-03、ad-01 | 占用率 > 90% 时 | **高** |


## 专属遗忘策略

| 策略项 | 标准值 | 本槽调整 | 调整理由 |
|--------|:---:|:---:|----------|
| 所有晋升阈值 | L1→L2: 0.4, L2→L3: 0.6, L3→L4: 0.8 | **全部下调 30%**：L1→L2: 0.28, L2→L3: 0.42, L3→L4: 0.56 | 特殊环境经验极为宝贵（如冰雪路面制动距离、暴雨能见度判断），应大幅降低晋升门槛，使其更快脱离易失层级 |
| 晋升时间阈值 | L1:24h, L2:7日, L3:30日 | **全部下调 30%**：L1:17h, L2:5日, L3:21日 | 加速固化，因为特殊环境重复频率低，等太久可能永远无法晋升 |
| 最低重要度遗忘阈值 | 0.15 | **0.09（下调 40%）** | 极端天气、地质灾害等场景经验即使复用频次极低也须长期保留 |
| 安全显著性权重（α） | 0.5 | 0.55 | 特殊环境下安全风险放大，安全信号权重大 |
| 复用频次权重（γ） | 0.3 | 0.25 | 复用频次本就不高，权重适度降低 |
| 风格匹配度权重（β） | 0.2 | 0.20（不变） | — |
| 容量告警时加速遗忘 | 仅 L1/L2 | **完全禁止加速遗忘** | 宁可冷归档，不可因容量问题丢失特殊环境经验 |


## 核心处理逻辑

```
FUNCTION special_environment_slot_main_loop():
    SET internal_state = NORMAL
    初始化五层存储结构（配额向 L4/L5 倾斜，因为这些是救命经验）
    加载专属遗忘策略：
        α=0.55, β=0.20, γ=0.25
        晋升阈值：L1→L2: I>0.28, L2→L3: I>0.42, L3→L4: I>0.56
        晋升时间：L1:17h, L2:5日, L3:21日
        最低遗忘阈值=0.09
        禁止容量告急加速遗忘

    WHILE 系统运行中:
        // 1. 全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = FROZEN
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 2. 接收新经验
        IF 收到经验写入指令 AND internal_state == NORMAL:
            经验条目 = 写入指令.经验条目
            // 特殊环境经验价值极高，I₀ 自动 +0.10
            经验条目.I₀ = MIN(经验条目.I₀ + 0.10, 1.0)
            // 若涉及不可抗力事件标记，进一步保护
            IF 经验条目.结果分类标签 == "不可抗力场景":
                经验条目.I₀ = 1.0  // 最高重要度
                经验条目.晋升目标 = L5  // 直接申请 L5 锁定
            写入 L1
            向 ad-14 返回存储完成回执

        // 3. 晋升处理（阈值全面下调 30%）
        IF 收到晋升候选清单:
            FOR EACH 候选 IN 清单:
                IF 候选.目标层级存在且存储未满:
                    // 特殊环境槽使用专属的低阈值判定
                    调用 ad-39 搬运
                    向 ad-38 返回成功
                ELSE:
                    向 ad-38 返回失败

        // 4. 遗忘处理（阈值极低 0.09）
        IF 收到遗忘候选清单:
            FOR EACH 候选 IN 清单:
                IF 候选.层级 IN [L4, L5]:
                    SKIP  // L4/L5 永久保护
                ELSE IF 候选.层级 == L3 AND 候选.经验.结果分类 == "不可抗力":
                    SKIP  // 不可抗力相关 L3 也保护
                ELSE IF 候选.I < 0.09:
                    调用 ad-42 归档（优先冷归档，而非直接删除）
                    向 ad-40 返回已归档
                ELSE:
                    向 ad-40 返回保留

        // 5. L3 归并（谨慎归并，相似度阈值高）
        IF 距上次归并 >= 72h OR 占用率 > 85%:
            SET internal_state = MAINTENANCE
            向 ad-25 发送归并请求（相似度阈值=0.85，非常严格）
            执行归并
            SET internal_state = NORMAL

        // 6. 容量监控（不触发加速遗忘，而是冷归档）
        占用率 = 计算占用率()
        IF 占用率 > 0.90:
            SET internal_state = CAPACITY_WARNING
            上报告警
            启动 L1/L2 旧经验冷归档（ad-49），移动到冷存储而非删除
            向 ad-03 建议增加配额
        ELSE IF 占用率 < 0.70 AND internal_state == CAPACITY_WARNING:
            SET internal_state = NORMAL

        // 7. 状态上报
        IF 距上次上报 >= 1s:
            上报槽状态快照

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 不可抗力事件经验写入 | I₀ 自动提升至 1.0，直接申请 L5 锁定，永久保护 | — |
| 容量告急 | 禁止加速遗忘，启动 L1/L2 冷归档（ad-49），保留原始数据可恢复 | 容量释放 |
| 晋升候选条目 I 值不满足标准阈值但满足本槽专属阈值 | 仍允许晋升（本槽阈值更低） | — |
| 遗忘候选条目 I 值在 0.09–0.15 之间 | 本槽保留（标准槽会删除），因为特殊环境经验宝贵 | — |
| L3 归并相似度 < 0.85 | 取消归并，两条经验各自保留 | — |
| 晋升时间阈值被人为调低超过 30% | 校验下限（L1≥10h, L2≥3日, L3≥15日），拒绝突破 | 参数恢复 |
| 冻结状态写入 | 拒绝，返回原因 | 激活指令 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 不可抗力事件经验强制 I₀=1.0，终身锁定于 L5，任何运行时进程不可删除或降级 |
| S-02 | 晋升阈值下调 30% 为编译期默认值，运行时可调范围限于下调 20%–40% |
| S-03 | 最低遗忘阈值 0.09 为硬编码下限，不可通过运行时配置进一步下调 |
| S-04 | 容量告警时绝对禁止加速遗忘，仅允许冷归档，确保特殊环境经验零丢失 |
| S-05 | L3 归并相似度阈值硬编码 ≥ 0.85，防止不同特殊环境经验被错误合并 |
| S-06 | 所有不可抗力事件、晋升、遗忘、归档操作全量写入 ad-51 变更日志 |
