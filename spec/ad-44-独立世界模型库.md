# ad-44-独立世界模型库 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-44 |
| 模块名称 | 独立世界模型库 |
| 所属分区 | 四、漏斗外挂扩展区（物理隔离） |
| 核心职责 | 对接自动驾驶类脑全域客观世界模型，提供五大类目标分类、三维风险标签、动态重分类规则的查询接口。作为 ECC 认知大脑的客观认知底座，独立于双漏斗记忆系统运行，不参与记忆的沉淀、筛选、晋升与遗忘机制。只读查询为主，仅接受经审批的增量追加更新，禁止覆盖删除已有条目 |
| 依赖模块 | ECC-01 情境解析模块（主要查询方）、ECC-03 因果推理模块（查询方）、ECC-04 心智模拟模块（查询方） |
| 被依赖模块 | ECC-01/03/04/05（提供客观环境认知数据）、ad-08（上下文场景标记单元，提供场景特征查询）、ad-14（场景判定与分槽路由单元，提供道路属性查询） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常提供查询服务，接受动态更新 | 系统初始化完成，世界模型数据加载校验通过 |
| 动态更新中 | `UPDATING` | 正在接收感知模块的实时环境动态更新 | 感知模块推送新的环境状态 |
| 审批更新中 | `APPROVED_UPDATE` | 正在执行经审批的永久性结构变更 | 收到 ECC-12 转发的审批通过更新指令 |
| 数据校验中 | `VALIDATING` | 正在校验世界模型数据完整性 | 系统初始化或定期校验周期 |
| 降级只读 | `DEGRADED` | 数据校验发现不一致，暂停更新仅保留查询 | 数据校验失败 |
| 暂停服务 | `PAUSED` | 系统紧急熔断 | 收到 ad-01 紧急熔断指令 |

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 目标分类查询请求 | Struct（查询类型 + 目标 ID 列表 / 场景标签 + 请求来源模块 + 查询优先级） | ECC-01/03/04 等模块（经 MemoryBus） | ECC 需要获取目标分类与风险信息时 | **高** |
| 场景特征查询请求 | Struct（查询类型 + 道路属性需求 + GPS 坐标 + 请求来源模块） | ad-08、ad-14、ECC-01 | 需要判定当前场景类别与道路属性时 | **高** |
| 环境动态更新数据 | Struct（目标 ID + 更新类型 + 更新内容 + 感知置信度 + 时间戳） | ECC-01 情境解析模块 | 感知模块检测到环境变化时 | **高** |
| 审批更新指令 | Struct（更新类型 + 更新内容 + 审批令牌 + 更新原因 + 审批时间戳） | ECC-12 资源全域调度模块（经 MemoryBus） | 永久性环境变化经人工或系统审批后 | **高** |
| 数据校验指令 | Signal | ad-01 总控漏斗 F₀ 或系统初始化模块 | 系统上电自检或定期校验周期 | **高** |
| 紧急熔断指令 | Enum（全局熔断） | ad-01 总控漏斗 F₀ | 安全急停或碰撞事件时 | **紧急** |

|--------|----------|----------|----------|:---:|

| 场景特征查询结果 | Struct（道路等级 + 路面类型 + 车道数 + 交通标识密度 + 天气 + 光照 + 时段 + 特殊环境标记 + 场景分类建议） | 请求来源模块 | 场景查询处理完成后 | **高** |

| 审批更新确认 | Struct（更新条目 + 更新结果 + 版本号 + 更新时间戳） | ECC-12、ad-51 | 审批更新执行完成后 | **高** |
| 数据校验报告 | Struct（校验时间 + 校验范围 + 数据完整性 + 异常条目清单 + 处理建议） | ad-01、ad-03 | 数据校验完成后 | **高** |

## 核心处理逻辑

```
FUNCTION world_model_main_loop():
    SET internal_state = NORMAL
    加载世界模型数据（五大分类实体库、物理属性库、因果规则库、交通法规库索引）
    校验数据完整性（哈希校验）
    数据版本号 = 初始版本号

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF internal_state == PAUSED AND 紧急解除:
            SET internal_state = NORMAL

        // 第 2 步：数据校验
        IF 收到数据校验指令 OR (距上次校验 >= 24 小时):
            SET internal_state = VALIDATING
            校验结果 = 执行全量数据哈希校验 + 索引完整性检查
            IF 校验结果.存在异常:
                SET internal_state = DEGRADED
                向 ad-01 发送数据校验报告（含异常条目清单）
            ELSE:
                向 ad-01、ad-03 发送数据校验报告（正常）
                IF internal_state == DEGRADED:
                    SET internal_state = NORMAL
            CONTINUE

        // 第 3 步：处理目标分类查询
        IF 收到目标分类查询请求 AND internal_state IN [NORMAL, UPDATING, DEGRADED]:
            查询类型 = 请求.查询类型
            请求来源 = 请求.请求来源模块

            IF 请求来源 NOT IN 授权查询模块列表:
                向请求方返回查询拒绝通知（原因=未授权模块）
                CONTINUE

            IF 查询类型 == "目标查询":
                目标 ID 列表 = 请求.目标 ID 列表
                结果列表 = []
                FOR EACH 目标 ID IN 目标 ID 列表:
                    目标数据 = 查询实体库(目标 ID)
                    IF 目标数据 != NULL:
                        结果列表.append({
                            目标 ID, 
                            目标类别: 目标数据.类别,
                            三维风险向量: 目标数据.风险向量,
                            物理属性: 目标数据.物理属性,
                            动态重分类状态: 目标数据.重分类状态
                        })
                向请求方返回目标分类查询结果(结果列表)

            ELSE IF 查询类型 == "场景查询":
                GPS 坐标 = 请求.GPS 坐标
                道路属性 = 查询道路属性库(GPS 坐标)
                场景分类建议 = 执行场景分类判定(道路属性)
                向请求方返回场景特征查询结果(道路属性 + 场景分类建议)

        // 第 4 步：处理环境动态更新
        IF 收到 ECC-01 环境动态更新数据 AND internal_state IN [NORMAL, UPDATING]:
            SET internal_state = UPDATING
            目标 ID = 更新数据.目标 ID
            更新类型 = 更新数据.更新类型

            IF 更新类型 == "状态变更":
                // 更新目标的位置、速度、姿态等实时状态
                更新实体动态状态(目标 ID, 更新数据.更新内容)
            ELSE IF 更新类型 == "新目标注册":
                // 感知模块发现新目标，临时注册
                IF 目标 ID NOT IN 实体库:
                    临时注册目标(目标 ID, 更新数据.更新内容, 标记="待确认")
            ELSE IF 更新类型 == "目标消失":
                // 目标离开感知范围
                标记目标为"已离开"(目标 ID)

            向 ECC-01 返回动态更新确认(目标 ID, 成功)
            SET internal_state = NORMAL

        // 第 5 步：处理审批更新（永久性结构变更）
        IF 收到 ECC-12 审批更新指令 AND internal_state IN [NORMAL, DEGRADED]:
            // 5a. 审批令牌校验
            IF 审批令牌无效:
                向 ECC-12 返回审批更新确认(失败, 原因="令牌无效")
                CONTINUE

            // 5b. 更新类型校验
            更新类型 = 指令.更新类型
            IF 更新类型 NOT IN [新增永久设施, 路面结构变更, 长期障碍物注册]:
                向 ECC-12 返回审批更新确认(失败, 原因="不支持的更新类型")
                CONTINUE

            SET internal_state = APPROVED_UPDATE

            // 5c. 执行增量追加（禁止覆盖删除）
            IF 更新类型 == "新增永久设施":
                新设施 = 指令.更新内容
                实体库.追加(新设施)
                数据版本号++
            ELSE IF 更新类型 == "路面结构变更":
                路面变更 = 指令.更新内容
                道路属性库.追加变更记录(路面变更)  // 追加而非覆盖
                数据版本号++
            ELSE IF 更新类型 == "长期障碍物注册":
                障碍物 = 指令.更新内容
                实体库.追加(障碍物, 标记="长期固定")
                数据版本号++

            // 5d. 记录更新日志
            向 ad-51 写入世界模型更新日志(指令 + 版本号 + 时间戳)
            向 ECC-12 返回审批更新确认(成功, 版本号)
            SET internal_state = NORMAL

        // 第 6 步：定期数据版本同步
        IF 距上次版本上报 >= 30 秒:
            向 ad-03、ad-01 上报运行状态（含数据版本号 + 查询统计）

        SLEEP 20ms  // 快速响应查询
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 查询目标 ID 不存在 | 返回空结果，不报错 | — |
| 动态更新时目标 ID 冲突 | 以最新感知数据覆盖，保留变更日志 | — |
| 审批更新令牌无效 | 拒绝更新，记录安全日志 | — |
| 审批更新试图覆盖已有条目 | 拒绝覆盖，仅允许增量追加。若确需修改，须走“标记废弃+新增”流程 | — |
| 数据校验发现哈希不一致 | 状态=DEGRADED，暂停更新，保留查询，上报告警 | 从冗余备份恢复数据 |
| 世界模型数据文件损坏 | 状态=DEGRADED，从镜像分区加载备份，上报告警 | 备份加载成功且校验通过 |
| 查询请求频率过高（> 1000 次/秒） | 限流至 800 次/秒，上报告警 | 频率回落 |
| 紧急熔断期间收到查询 | 返回缓存数据（若有），否则返回“系统紧急熔断中” | 紧急解除 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 世界模型独立于双漏斗记忆系统，不参与记忆的沉淀、筛选、晋升与遗忘机制。任何经验数据不得写入本模块 |
| S-02 | 仅接受经 ECC-12 审批的永久性结构变更，审批令牌须经 ad-30 验证。所有审批更新全量写入 ad-51 不可变日志 |
| S-03 | 世界模型数据存储于只读分区，运行时仅允许增量追加，绝对禁止覆盖删除已有条目 |
| S-04 | 数据完整性校验每 24 小时执行一次，校验失败立即降级为 DEGRADED，从镜像分区恢复 |
| S-05 | 世界模型数据为客观认知底座，不受任何驾驶员偏好、驾驶风格或历史经验的影响 |
| S-06 | 查询服务对授权 ECC 模块开放，漏斗一模块无权查询世界模型数据 |
| S-07 | 世界模型数据版本号每次审批更新后递增，版本历史全量保留 |
