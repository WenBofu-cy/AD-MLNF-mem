# ad-40-遗忘阈值判定单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-40 |
| 模块名称 | 遗忘阈值判定单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 扫描漏斗二各层级中重要度 I 值低于遗忘阈值、且复用次数不足的经验条目，生成遗忘候选清单。L4/L5 层不参与遗忘扫描，L3 层遗忘优先冷归档而非直接删除。遗忘阈值受分槽专属策略与全局容量告急双重调节 |
| 依赖模块 | ad-36（综合重要度 I 值聚合计算单元，提供条目当前 I 值）、ad-20/22/24/26（各层级存储单元，提供条目元数据与复用计数）、ad-33（复用频次 C 值统计单元，提供复用次数）、ad-35（三维权重系数配置单元，获取各分槽最低遗忘阈值）、ad-48（全局容量配额管控单元，获取容量告急信号） |
| 被依赖模块 | ad-20/22/24/26（各层级存储单元，消费遗忘候选清单）、ad-42（冗余记忆删除与归档单元，接收遗忘执行指令） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 等待遗忘判定周期触发 | 系统初始化完成，或上次判定结束 |
| 判定执行中 | `JUDGING` | 正在逐层级、逐条目执行遗忘判定 | 周期触发或收到手动判定请求 |
| 暂停服务 | `PAUSED` | 漏斗二冻结或紧急熔断 | 驾驶模式切换或紧急事件 |

## 遗忘判定规则

### 一、通用遗忘条件

```
遗忘条件 = (I < θ_forget) AND (复用次数 < C_min)
```

两项条件必须同时满足，缺一不可。

| 参数 | 默认值 | 说明 |
|------|:---:|------|
| θ_forget（遗忘 I 阈值） | 0.15 | 各分槽可独立配置 |
| C_min（最小复用次数） | 2 | 复用次数 ≥ 2 的条目豁免遗忘 |

### 二、分层遗忘保护规则

| 层级 | 遗忘规则 |
|------|----------|
| L1 临时层 | 正常参与遗忘判定。遗忘方式 = 直接删除 |
| L2 近期层 | 正常参与遗忘判定。遗忘方式 = 直接删除 |
| L3 中期层 | 正常参与遗忘判定。遗忘方式 = **冷归档**（优先保留可追溯性），不可抗力场景豁免 |
| L4 长期层 | **不参与常规遗忘扫描**。仅在容量红色告急时，对 I < 0.10 且复用 < 1 的条目触发冷归档 |
| L5 核心层 | **永不参与遗忘扫描**。任何条件下均豁免 |

### 三、各分槽最低遗忘阈值（默认值）

| 分槽 | 最低遗忘 I 阈值 | 说明 |
|------|:---:|------|
| 高速巡航槽 (ad-15) | 0.12 | 低于默认，高速经验更安全 |
| 城区路口槽 (ad-16) | 0.10 | 低阈值保护安全经验 |
| 泊车低速槽 (ad-17) | 0.075 | 极低阈值，保护低频经验 |
| 特殊环境槽 (ad-18) | 0.09 | 低阈值，保护稀有经验 |
| 通用驾驶槽-常规 (ad-19) | 0.15 | 默认阈值 |
| 通用驾驶槽-乡村 (ad-19) | 0.075 | 低阈值，保护低频乡村经验 |

### 四、容量告急对遗忘阈值的动态调节

| 容量使用率 | 告急等级 | 遗忘阈值临时调整 |
|-----------|:---:|------|
| < 85% | 正常 | 使用各分槽默认遗忘阈值 |
| 85%–90% | 黄色 | 所有分槽遗忘阈值上调 20%（如 0.15→0.18） |
| 90%–95% | 橙色 | 所有分槽遗忘阈值上调 50%（如 0.15→0.225），L3 遗忘从冷归档升级为直接删除 |
| > 95% | 红色 | 遗忘阈值上调 100%（如 0.15→0.30），L4 纳入遗忘扫描（仅限 I<0.10） |

### 五、强制豁免规则（无条件跳过遗忘）

| 豁免条件 | 说明 |
|----------|------|
| 不可抗力标记 = True | 不可抗力经验永久保护 |
| 结果分类标签 = "不可抗力场景" | 同上 |
| 安全显著性 S ≥ 0.7（最近一次） | 高安全价值经验保护 |
| L5 核心层全部条目 | 终身锁定不可遗忘 |
| 人工锁定标记 = True | 管理员锁定经验 |
| 最近 7 日内被 ECC 成功采用 | 近期仍有实战价值的经验暂不遗忘 |


## 核心处理逻辑

```
FUNCTION forget_threshold_judge_main_loop():
    SET internal_state = IDLE
    判定间隔 = 120 秒
    last_judge_time = 0

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令 AND internal_state == PAUSED:
            SET internal_state = IDLE
            CONTINUE

        // 第 2 步：检查触发条件
        IF NOT (收到手动触发信号 OR NOW() - last_judge_time >= 判定间隔):
            SLEEP 1 秒
            CONTINUE

        SET internal_state = JUDGING
        start_time = NOW()
        last_judge_time = NOW()

        // 第 3 步：获取各层级条目数据与阈值
        各层级条目 = 各层级条目元数据快照
        分槽遗忘阈值库 = ad-35 各分槽最低遗忘阈值
        冷条目清单 = ad-33 冷条目清单
        容量状态 = ad-48 容量告急信号

        遗忘候选清单 = {L1: [], L2: [], L3: [], L4: []}  // L5 不参与
        豁免清单 = []

        // 第 4 步：根据容量状态调整阈值
        阈值调整系数 = 1.0
        L3 直接删除 = False
        L4 纳入扫描 = False
        IF 容量状态.使用率 >= 0.95:
            阈值调整系数 = 2.0; L3 直接删除 = True; L4 纳入扫描 = True
        ELSE IF 容量状态.使用率 >= 0.90:
            阈值调整系数 = 1.5; L3 直接删除 = True
        ELSE IF 容量状态.使用率 >= 0.85:
            阈值调整系数 = 1.2

        // 第 5 步：逐层级逐条目判定

        // 5a. L1 判定
        FOR EACH 条目 IN 各层级条目.L1:
            result = forget_judge(条目, "L1", 阈值调整系数, 默认遗忘方式="直接删除")
            IF result.结论 == "遗忘": 遗忘候选清单["L1"].append(result)
            ELSE IF result.结论 == "豁免": 豁免清单.append(result)

        // 5b. L2 判定
        FOR EACH 条目 IN 各层级条目.L2:
            result = forget_judge(条目, "L2", 阈值调整系数, 默认遗忘方式="直接删除")
            ... 同上 ...

        // 5c. L3 判定
        遗忘方式 = IF L3 直接删除 THEN "直接删除" ELSE "冷归档"
        FOR EACH 条目 IN 各层级条目.L3:
            result = forget_judge(条目, "L3", 阈值调整系数, 默认遗忘方式=遗忘方式)
            ... 同上 ...

        // 5d. L4 判定（仅容量红色告急时）
        IF L4 纳入扫描:
            FOR EACH 条目 IN 各层级条目.L4:
                result = forget_judge(条目, "L4", 阈值调整系数, 默认遗忘方式="冷归档")
                ... 同上 ...

        // 第 6 步：按遗忘优先级排序（I 值越低、复用越少、越久未访问 → 优先级越高）
        FOR EACH 层级 IN 遗忘候选清单:
            遗忘优先级 = (1.0 - I值) × 0.5 + (1.0 - C值) × 0.3 + (距上次访问天数/90) × 0.2
            按遗忘优先级降序排列 遗忘候选清单[层级]

        // 第 7 步：下发遗忘候选清单
        FOR EACH 层级, 候选列表 IN 遗忘候选清单:
            IF 候选列表不为空:
                向对应层级存储单元发送遗忘候选清单（经 ad-42 执行）

        // 第 8 步：容量压力评估
        预估释放空间 = 估算总大小(遗忘候选清单)
        IF 容量状态.使用率 > 0.90 AND 预估释放空间 < 总容量 × 0.02:
            向 ad-03、ad-01 发送容量压力告警（当前容量 + 本轮释放不足以显著缓解）

        // 第 9 步：汇总上报
        判定结果汇总 = {
            各层级扫描条目数,
            遗忘候选数: 各层级候选总数,
            豁免数: 豁免清单.length,
            冷归档数: 遗忘方式为冷归档的条目数,
            直接删除数: 遗忘方式为直接删除的条目数,
            预估释放空间: 预估释放空间,
            判定耗时: NOW() - start_time
        }
        向 ad-03、ad-01、ad-51 发送判定结果汇总

        SET internal_state = IDLE
        SLEEP 1 秒


FUNCTION forget_judge(条目, 层级, 阈值调整系数, 默认遗忘方式):
    // 强制豁免检查
    IF 条目.不可抗力标记 == True:
        RETURN {结论: "豁免", 原因: "不可抗力经验永久保护"}
    IF 条目.结果分类标签 == "不可抗力场景":
        RETURN {结论: "豁免", 原因: "不可抗力场景"}
    IF 条目.人工锁定标记 == True:
        RETURN {结论: "豁免", 原因: "人工锁定"}
    IF 层级 == "L5":
        RETURN {结论: "豁免", 原因: "L5 核心层永不遗忘"}
    IF 条目.最近访问时间 != NULL AND NOW() - 条目.最近访问时间 < 7 日:
        RETURN {结论: "豁免", 原因: "近 7 日内被成功采用"}
    IF 条目.S 值 >= 0.7:
        RETURN {结论: "豁免", 原因: "高安全显著性保护"}

    // 获取分槽专属遗忘阈值
    分槽号 = 条目.来源分槽号
    原始阈值 = 分槽遗忘阈值库[分槽号]
    IF 原始阈值 == NULL:
        原始阈值 = 0.15  // 默认

    // 动态阈值 = 原始阈值 × 容量调整系数（但不可超过上限 0.60）
    动态阈值 = MIN(原始阈值 × 阈值调整系数, 0.60)

    // 遗忘双条件判定
    IF 条目.I 值 < 动态阈值 AND 条目.复用计数 < 2:
        RETURN {
            结论: "遗忘",
            遗忘方式: 默认遗忘方式,
            I 值: 条目.I 值,
            遗忘原因: "I=" + 条目.I 值 + " < θ=" + 动态阈值 + ", 复用=" + 条目.复用计数,
            遗忘优先级: 待主循环计算
        }
    ELSE:
        RETURN {结论: "保留", 原因: "不满足遗忘双条件"}
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 条目 I 值缺失 | 使用条目 I₀ 替代，若仍缺失则保守保留（不遗忘），上报告警 | I 值补全后正常判定 |
| 分槽遗忘阈值未配置 | 使用全局默认 0.15，上报告警 | 阈值配置完成 |
| 容量告急但豁免条目占比 > 80% | 不强制清除豁免条目，上报告警建议手动干预扩容 | 容量扩展或手动清理 |
| 容量红色告警时 L4 纳入扫描 | 仅扫描 I<0.10 且复用<1 的 L4 条目，不可抗力仍豁免 | 容量恢复后自动退出 |
| 同一条目同时出现在遗忘候选和晋升候选 | 晋升优先，从遗忘候选中移除 | — |
| 手动遗忘触发信号但令牌无效 | 拒绝执行 | — |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | L5 核心层永不参与遗忘扫描，编译期硬编码豁免，任何运行时条件不可覆盖 |
| S-02 | 不可抗力场景经验（不可抗力标记=True）在所有层级均强制豁免遗忘，无一例外 |
| S-03 | L4 长期层仅在容量红色告警（>95%）时纳入有限遗忘扫描，且仅限 I<0.10 的非豁免条目 |
| S-04 | 遗忘阈值动态调节上限硬编码为 0.60，防止容量告急时过度清除 |
| S-05 | 遗忘判定仅生成候选清单，实际删除或归档由 ad-42 执行，本单元不直接操作存储 |
| S-06 | 所有遗忘判定（含遗忘候选、豁免、保留）全量写入 ad-51 不可变日志 |
