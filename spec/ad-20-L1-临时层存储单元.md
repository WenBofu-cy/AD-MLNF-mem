# ad-20-L1-临时层存储单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-20 |
| 模块名称 | L1 临时层存储单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 五层记忆层级存储 |
| 核心职责 | 存储本次行程的瞬时驾驶经验片段，作为漏斗二经验入口的第一站。占漏斗二总存储容量的 60%，是五层结构中流转速度最快、容量最大的层级。所有新经验首先写入本层，在留存满 24 小时后由 L1 时序衰减单元评估是否进入晋升候选或清除 |
| 依赖模块 | ad-14（场景判定与分槽路由单元，接收经验写入指令）、ad-21（L1 临时层时序衰减单元，提供衰减评估结果） |
| 被依赖模块 | ad-21（消费存储的 L1 条目进行衰减评估）、ad-38（晋升双条件判定单元，消费 L1 晋升候选条目） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收并存储经验条目，响应查询请求 | 漏斗二激活，本层级初始化完成 |
| 存储将满 | `NEAR_FULL` | 存储占用率 > 85%，触发温和清理 | 存储占用率监测 |
| 存储已满 | `FULL` | 存储占用率 > 95%，强制清除最低重要度条目 | 存储占用率监测 |
| 维护中 | `MAINTENANCE` | 正在执行批量清除或碎片整理 | 定时维护或存储已满触发 |
| 冻结只读 | `FROZEN` | 漏斗二冻结或紧急接管，暂停写入仅保留查询 | 驾驶模式切换或紧急熔断 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 经验写入指令 | Struct（含经验条目 ID + 经验内容 + 目标分槽号 + I₀ 值 + 时间戳 + 子类标记） | ad-14 场景判定与分槽路由单元（经对应分槽转发） | 每次新经验产生时 | **高** |
| 衰减评估结果 | Struct（条目 ID + 评估结论：晋升候选/继续保留/建议清除） | ad-21 L1 临时层时序衰减单元 | L1 条目留存满 24 小时后 | **高** |
| 晋升失败回退条目 | Struct（条目 ID + 回退原因） | ad-38 晋升双条件判定单元 | L1 条目晋升 L2 失败时 | 普通 |
| 全局调度指令 | Enum（冻结/激活/维护） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |
| 批量清除指令 | Signal | ad-48 全局容量配额管控单元 | 全局容量逼近上限时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 存储完成回执 | Struct（条目 ID + 存储位置 + 存储时间戳 + 当前 L1 占用率） | ad-14（经分槽转发） | 每次经验成功存储后 | 普通 |
| 晋升候选清单 | List（条目 ID + I 值 + 留存时长 + 来源分槽） | ad-38 晋升双条件判定单元 | 收到 ad-21 衰减评估为“晋升候选”后 | **高** |
| 清除执行回执 | Struct（条目 ID + 清除方式 + 时间戳） | ad-42 冗余记忆删除与归档单元 | 衰减评估为“建议清除”或强制清除时 | 普通 |
| L1 状态快照 | Struct（占用率 + 条目总数 + 各分槽条目分布 + 平均留存时长） | ad-03、对应分槽 | 每 5 秒周期性上报 | 普通 |
| 存储异常告警 | Struct（异常类型 + 当前占用率 + 详情） | ad-03、ad-01 | 存储写入失败或数据校验异常时 | **高** |


## 核心处理逻辑

```
FUNCTION l1_temporary_storage_main_loop():
    SET internal_state = NORMAL
    初始化 L1 存储分区（占漏斗二总容量 60%，平均分配给各活跃场景分槽）
    初始化条目索引表（条目 ID → 存储地址 + 写入时间戳 + I₀ + 来源分槽号）
    最大留存时间 = 24 小时

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = FROZEN
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 第 2 步：接收新经验并写入
        IF 收到经验写入指令 AND internal_state IN [NORMAL, NEAR_FULL]:
            // 2a. 检查存储空间
            IF internal_state == FULL:
                立即执行紧急清除：找出当前 L1 中 I 值最低的 5% 条目
                调用 ad-42 删除这些条目
                IF 释放空间仍不足:
                    向 ad-03 上报告警
                    返回存储失败回执
                    CONTINUE
                ELSE:
                    SET internal_state = NEAR_FULL

            新条目 = 写入指令.经验条目
            条目 ID = 新条目.ID

            // 2b. 校验条目完整性
            IF 新条目.经验内容 == NULL OR 新条目.I₀ == NULL:
                返回存储失败回执（原因=条目不完整）
                CONTINUE

            // 2c. 分配存储空间并写入
            存储地址 = 分配连续空间(新条目.大小)
            IF 分配失败:
                触发碎片整理
                重试分配
                IF 仍失败:
                    向 ad-03 上报告警
                    返回存储失败回执
                    CONTINUE

            写入存储介质
            更新条目索引表：
                条目 ID → {地址, NOW(), 新条目.I₀, 新条目.来源分槽号, 子类标记}
            向 ad-14 返回存储完成回执

            // 2d. 更新占用率并检查阈值
            占用率 = 计算占用率()
            IF 占用率 > 0.95:
                SET internal_state = FULL
                向 ad-03 上报告警
            ELSE IF 占用率 > 0.85 AND internal_state == NORMAL:
                SET internal_state = NEAR_FULL

        // 第 3 步：处理衰减评估结果（来自 ad-21）
        IF 收到衰减评估结果:
            FOR EACH 评估条目 IN 评估结果:
                IF 评估结论 == "晋升候选":
                    加入晋升候选清单
                ELSE IF 评估结论 == "建议清除":
                    调用 ad-42 执行安全删除
                    向 ad-42 返回清除执行回执
                    从条目索引表中移除
                ELSE:  // "继续保留"
                    不操作，条目继续留在 L1

            IF 晋升候选清单不为空:
                向 ad-38 发送晋升候选清单
                清空晋升候选清单

        // 第 4 步：处理晋升失败回退条目
        IF 收到晋升失败回退条目:
            条目 ID = 回退条目.条目 ID
            回退原因 = 回退条目.原因
            IF 回退原因 == "L2 存储满":
                // 条目继续留在 L1，等待下次衰减评估
                记录回退日志
            ELSE:
                // 其他原因（如条目损坏），标记为待清除
                调用 ad-42 删除
                从索引表中移除

        // 第 5 步：处理全局容量告急
        IF 收到 ad-48 批量清除指令:
            SET internal_state = MAINTENANCE
            按 I 值升序排列所有 L1 条目
            清除 I 值最低的 20% 条目（优先清除非安全相关条目）
            向 ad-42 批量发送清除指令
            向 ad-48 返回释放空间量
            SET internal_state = NORMAL

        // 第 6 步：定期碎片整理
        IF 距上次整理 >= 6 小时 AND 占用率 > 70%:
            SET internal_state = MAINTENANCE
            执行存储碎片整理，合并空闲空间
            SET internal_state = NORMAL

        // 第 7 步：状态上报
        IF 距上次上报 >= 5 秒:
            向 ad-03 和各分槽上报 L1 状态快照

        SLEEP 50ms  // 高频响应
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| L1 存储写入失败（硬件故障） | 重试 3 次，若均失败则标记该存储块为坏块，上报告警，使用备用存储块 | 坏块被修复或替换 |
| 条目索引表损坏 | 从存储分区重建索引表（遍历所有条目头），上报告警 | 重建完成 |
| L1 条目在 24 小时内被手动查询触发晋升 | 不提前晋升，仍等待 ad-21 的 24 小时衰减评估 | — |
| 新条目大小超过单块存储上限 | 分割存储（链式存储），上报告警 | — |
| 存储占用率长期 > 95% 且无法清除 | 向 ad-03 紧急请求配额扩容，暂停接收新经验 | 扩容完成或手动清理 |
| 冻结状态下收到写入 | 拒绝写入，返回原因 | 激活指令 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | L1 层仅存储临时经验数据，所有条目最大留存时间硬编码为 24 小时，超时未晋升则进入衰减评估 |
| S-02 | L1 层经验条目在 24 小时内不可被手动删除（除非全局容量告急触发批量清除） |
| S-03 | 安全显著性 S ≥ 0.9 的条目在 L1 批量清除时享有保护，不被清除 |
| S-04 | 存储写入须校验条目完整性（ID、内容、I₀、时间戳四要素齐全） |
| S-05 | 条目索引表每 24 小时自动备份至冗余分区，防止损坏后无法恢复 |
| S-06 | 冻结状态下禁止任何写入操作，保障模式切换数据一致性 |
| S-07 | 所有写入、清除、晋升候选操作全量写入 ad-51 变更日志 |
