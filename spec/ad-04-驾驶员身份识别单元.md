# ad-04-驾驶员身份识别单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-04 |
| 模块名称 | 驾驶员身份识别单元 |
| 所属分区 | 二、漏斗一：驾驶员画像漏斗 |
| 核心职责 | 融合中控屏手动选择、座椅记忆联动、人脸识别三种方式，确认当前驾驶员身份并匹配对应子画像槽。输出统一的驾驶员身份 ID 与槽位类型建议 |
| 依赖模块 | 无（独立传感器与 CAN 总线数据入口） |
| 被依赖模块 | ad-02（漏斗一专属调度单元，消费身份识别结果）、ad-05（子画像槽创建与初始化单元） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待车辆启动 | `WAITING_START` | 系统上电自检完成，等待车辆启动信号 | 冷启动自检通过 |
| 等待首次信号 | `WAITING_FIRST_SIGNAL` | 车辆已启动，等待任何身份识别信号输入 | 车辆启动（READY 状态） |
| 信号收集中 | `COLLECTING` | 已收到部分信号，正在等待补充信号 | 收到任一识别方式的部分数据 |
| 融合计算中 | `FUSING` | 正在对多源信号进行加权融合与身份判定 | 收集到足够信号量 |
| 身份已确认 | `CONFIRMED` | 驾驶员身份已确认，输出结果已下发 | 置信度 ≥ 阈值，或用户手动确认 |
| 等待用户确认 | `WAITING_USER` | 置信度低于阈值，等待中控屏用户手动确认 | 融合置信度 ∈ (0.4, 0.7) |
| 新驾驶员注册 | `NEW_DRIVER` | 识别到未注册的驾驶员面部特征 | 面部匹配分数低于注册阈值 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 中控屏手动选择信号 | Struct（用户选择的驾驶员 ID + 槽位类型 + 是否新注册） | 中控屏 CAN 总线 | 用户在中控屏手动操作时 | **最高** |
| 座椅记忆模块数据 | Struct（座椅位置参数向量 + 后视镜参数 + 方向盘位置） | 座椅记忆 CAN 总线 | 驾驶员调节座椅后按下记忆按钮 | **高** |
| 车内摄像头面部特征 | Struct（面部特征向量 + 活体检测结果） | 车内摄像头模块 | 驾驶员落座后自动采集，每 5 秒一次直至确认 | **高** |
| 人脸识别功能开关状态 | Bool | 中控屏用户设置 | 系统初始化、用户修改设置时 | 普通 |
| 已注册驾驶员面部特征库 | Dict（驾驶员 ID : 注册特征向量） | ad-05（子画像槽创建时回写） | 系统初始化、新驾驶员注册后更新 | 普通 |
| 车辆启动信号 | Signal | 车辆 CAN 总线 | 车辆从 OFF 切换至 READY | **高** |
| 车辆熄火信号 | Signal | 车辆 CAN 总线 | 车辆从 READY 切换至 OFF | 普通 |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 驾驶员身份识别结果 | Struct（身份 ID + 识别方式 + 置信度 + 建议槽位类型） | ad-02 漏斗一专属调度单元 | 身份确认或用户手动确认后 | **高** |
| 中控屏确认请求 | Struct（待确认驾驶员姓名或 ID + 置信度 + 匹配方式） | 中控屏 CAN 总线 | 置信度低于阈值时 | **高** |
| 新驾驶员注册通知 | Struct（面部特征向量 + 建议槽位类型 = 长期） | ad-05 子画像槽创建与初始化单元 | 识别到未注册驾驶员时 | **高** |
| 识别失败通知 | Struct（失败原因 + 建议操作） | 中控屏 CAN 总线 | 所有识别方式均无法确认身份时 | 普通 |
| 运行状态上报 | Struct（当前状态 + 最近一次识别结果摘要） | MemoryBus（记忆→ECC） | 每 5 秒周期性上报 | 普通 |


## 核心处理逻辑

```
FUNCTION driver_identification_main_loop():
    WAIT 车辆启动信号
    SET internal_state = WAITING_FIRST_SIGNAL

    WHILE 系统运行中:
        // 第 1 步：最高优先级——中控屏手动选择
        IF 收到中控屏手动选择信号:
            IF 用户选择已注册驾驶员:
                直接使用用户选择的身份 ID
                confidence = 1.0  // 用户主动确认为最高置信度
                SET internal_state = CONFIRMED
                输出身份识别结果 TO ad-02
                CONTINUE
            ELSE IF 用户选择“新驾驶员注册”:
                SET internal_state = NEW_DRIVER
                向 ad-05 发送新驾驶员注册通知
                采集当前面部特征作为新注册特征
                WAIT ad-05 返回新身份 ID
                输出身份识别结果 TO ad-02
                CONTINUE

        // 第 2 步：座椅记忆联动
        IF 收到座椅记忆模块数据:
            提取座椅位置参数向量
            检索已注册驾驶员的座椅记忆绑定记录
            IF 匹配到唯一驾驶员:
                seat_confidence = 0.85  // 座椅记忆置信度较高
                记录匹配结果
            ELSE IF 匹配到多个驾驶员:
                seat_confidence = 0.5
                记录候选列表
            ELSE:
                seat_confidence = 0.0
            SET internal_state = COLLECTING

        // 第 3 步：人脸识别
        IF 人脸识别功能开关状态 == True AND 收到车内摄像头面部特征:
            将面部特征向量与已注册驾驶员特征库进行相似度比对
            计算余弦相似度分数（0–1）
            IF 最高相似度 >= 0.85:
                face_confidence = 最高相似度
                记录匹配到的驾驶员 ID
            ELSE IF 最高相似度 >= 0.6:
                face_confidence = 最高相似度
                记录匹配到的驾驶员 ID + 标记为低置信度
            ELSE:
                face_confidence = 0.0
                标记为未注册驾驶员
            SET internal_state = COLLECTING
        ELSE IF 人脸识别功能开关状态 == False:
            face_confidence = 0.0  // 用户关闭人脸识别，不参与计算

        // 第 4 步：多源信号融合判定
        IF internal_state == COLLECTING AND 已收集到足够信号量:
            SET internal_state = FUSING

            // 4a. 中控屏手动选择优先
            IF 存在中控屏手动选择:
                直接使用手动选择结果，忽略其他信号
            ELSE:
                // 4b. 加权融合
                // 座椅记忆权重 = 0.4，人脸识别权重 = 0.6（人脸更可靠）
                IF 座椅匹配结果 == 人脸匹配结果:
                    fused_confidence = max(seat_confidence, face_confidence)
                    matched_id = 一致的身份 ID
                ELSE IF 座椅匹配结果 != 人脸匹配结果:
                    // 信号冲突，以人脸为准（权重更高）
                    matched_id = 人脸匹配的身份 ID
                    fused_confidence = face_confidence * 0.8  // 冲突惩罚
                ELSE IF 只有座椅匹配:
                    matched_id = 座椅匹配的身份 ID
                    fused_confidence = seat_confidence * 0.7  // 单源惩罚
                ELSE IF 只有人脸匹配:
                    matched_id = 人脸匹配的身份 ID
                    fused_confidence = face_confidence * 0.8  // 单源惩罚

            // 4c. 置信度判定
            IF fused_confidence >= 0.7:
                SET internal_state = CONFIRMED
                输出身份识别结果 TO ad-02
            ELSE IF fused_confidence >= 0.4:
                SET internal_state = WAITING_USER
                向中控屏发送确认请求（含匹配到的驾驶员姓名 + 置信度）
                WAIT 用户回复（超时 5 秒）
                IF 用户确认:
                    SET internal_state = CONFIRMED
                    输出身份识别结果（置信度 = 1.0）TO ad-02
                ELSE IF 用户选择其他身份:
                    按手动选择流程重新处理
                ELSE:
                    // 超时无回复，输出低置信度结果
                    输出身份识别结果（置信度 = fused_confidence）TO ad-02
                    SET internal_state = CONFIRMED
            ELSE:
                // 所有识别方式均无法确认
                输出识别失败通知到中控屏
                SET internal_state = WAITING_FIRST_SIGNAL  // 重置

        // 第 5 步：车辆熄火重置
        IF 收到车辆熄火信号:
            清除本次识别缓存
            SET internal_state = WAITING_START

        // 第 6 步：周期性状态上报
        IF 距上次上报 >= 5 秒:
            向 MemoryBus 上报运行状态与最近一次识别摘要

        SLEEP 200ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 人脸识别功能被用户关闭 | 仅使用座椅记忆匹配结果，融合计算时不考虑人脸权重 | 用户重新开启人脸识别功能 |
| 座椅记忆与人脸识别结果冲突 | 以人脸识别为准（权重 0.6 > 0.4），施加冲突惩罚（置信度 × 0.8），上报告警 | — |
| 人脸识别置信度在 0.6–0.7 之间且无座椅匹配 | 弹出中控屏确认对话框，显示最匹配的驾驶员姓名与相似度 | 用户手动确认或选择其他身份 |
| 中控屏确认请求 5 秒超时无回复 | 输出低置信度识别结果（置信度 = 融合置信度），激活临时画像槽 | — |
| 所有识别方式均无法确认身份（置信度 < 0.4） | 中控屏显示识别失败，提示手动选择或使用临时模式 | 用户手动选择身份 |
| 摄像头被遮挡或故障 | 人脸识别置信度自动置 0，仅使用座椅记忆，上报告警 | 摄像头恢复正常 |
| 活体检测失败（照片攻击） | 人脸识别结果作废，置信度置 0，记录安全日志 | — |
| 已注册驾驶员特征库为空 | 首次使用，任何面部特征均触发新驾驶员注册流程 | 至少注册一位驾驶员 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| CAN 总线 | 读 | 中控屏手动选择信号 | 只读 | 用户主动选择身份时接收 |
| CAN 总线 | 读 | 座椅记忆模块数据 | 只读 | 驾驶员调节座椅后触发 |
| CAN 总线 | 读 | 车内摄像头面部特征 | 只读 | 含面部特征向量与活体检测结果 |
| CAN 总线 | 读 | 车辆启动 / 熄火信号 | 只读 | 驾驶循环生命周期 |
| CAN 总线 | 写 | 中控屏确认请求 | 写入 | 置信度低时请求用户确认 |
| CAN 总线 | 写 | 识别失败通知 | 写入 | 所有方式均无法确认身份时 |
| MemoryBus | 写 | 驾驶员身份识别结果 | 专属写入 | 向 ad-02 下发最终结果 |
| MemoryBus | 写 | 新驾驶员注册通知 | 专属写入 | 向 ad-05 下发注册指令 |
| MemoryBus | 写 | 运行状态上报 | 周期性写入 | 每 5 秒向 ECC 上报 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 人脸识别原始图像不存储、不传输、不离开本地运算单元 |
| S-02 | 面部特征向量加密存储于本地安全分区，不可通过 OBD 或 OTA 导出 |
| S-03 | 人脸识别功能可通过中控屏一键关闭，关闭后特征向量仍保留但暂停采集 |
| S-04 | 活体检测失败三次以上，系统自动锁定人脸识别功能 10 分钟，防止欺骗攻击 |
| S-05 | 驾驶员身份识别结果仅用于漏斗一子画像槽路由，编译期禁止接入自动驾驶决策链路 |
| S-06 | 所有识别操作（含成功、失败、手动确认）全量写入 ad-51 号变更日志 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-04-01 | 已注册驾驶员“张三”落座，人脸识别开启 | 面部特征与“张三”注册特征匹配度 0.92 | 身份确认，输出 ID=张三，置信度 0.92 |
| TC-04-02 | 已注册驾驶员落座，人脸识别开启 | 中控屏手动选择“李四” | 直接输出 ID=李四，置信度 1.0，忽略人脸 |
| TC-04-03 | 驾驶员调节座椅后按下记忆按钮 1 | 座椅记忆数据与“张三”绑定记录匹配 | 匹配“张三”，座椅置信度 0.85，融合人脸（若一致） |
| TC-04-04 | 未注册驾驶员落座，人脸识别开启 | 面部特征与已注册库最高相似度 0.4 | 识别失败，中控屏提示手动选择或使用临时模式 |
| TC-04-05 | 人脸识别置信度 0.65，无座椅匹配 | 中控屏确认请求，用户 5 秒内确认 | 身份确认，置信度 1.0，输出结果 |
| TC-04-06 | 人脸识别置信度 0.65，中控屏确认请求 | 用户 5 秒内无回复 | 输出低置信度结果（0.65），激活临时画像槽 |
| TC-04-07 | 用户关闭人脸识别功能 | 仅座椅记忆数据匹配成功 | 输出座椅匹配结果，置信度 × 0.7（单源惩罚） |
| TC-04-08 | 座椅记忆匹配“张三”，人脸匹配“李四” | 信号冲突 | 以人脸为准输出“李四”，置信度 × 0.8（冲突惩罚） |
| TC-04-09 | 摄像头被遮挡 | 人脸识别无法获取有效特征 | 置信度置 0，仅使用座椅记忆，上报告警 |
