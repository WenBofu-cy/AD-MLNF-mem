# mem-58 疑问议题缓存库

> 所属仓库: AD-mlnf-mem/spec/
> 模块编号: mem-58
> 所属分区: 外置独立扩展区（AD场景适配）


## 一、基本信息

| 项目 | 内容 |
|:---|:---|
| 模块编号 | mem-58 |
| 模块名称 | 疑问议题缓存库 |
| 核心职责 | 存放AD-ECC认知层不确定推理、逻辑断点、决策置信度不足时产生的待确认认知项，以及感知失效时的环境疑问、自动返航前的条件评估疑问、极端伦理冲突场景标记。为离线复盘与持续学习提供待处理问题清单 |
| 输入数据 | AD-ECC各模块产生的疑问条目（来自adecc-01感知融合失效疑问、adecc-06意图预测置信度不足、adecc-09行为决策逻辑断点、adecc-15安全降级触发时的场景快照、adecc-16极端伦理冲突标记、adecc-22自动返航条件评估未通过项） |
| 输出数据 | 待解疑问清单（按优先级排序）、延后学习任务列表、认知补全参考数据、疑问统计报告（按类型/频率/场景分类） |


## 二、核心处理逻辑

**1. 疑问条目接收与分类**

接收来自AD-ECC各模块的疑问条目，按以下类别自动分类：

| 疑问类别 | 来源模块 | 典型内容 |
|:---|:---|:---|
| 感知失效疑问 | adecc-01/04 | “前方障碍物是什么？”——摄像头致盲，激光雷达点云密度不足，单传感器无法确认物体类别 |
| 决策置信度不足疑问 | adecc-09/15 | “这个路口该不该过？”——C_total<0.4，行为决策置信度不足，系统选择保守策略 |
| 逻辑断点疑问 | adecc-09/14 | “让行决策与变道决策矛盾”——因果推理链存在断裂，无法形成完整逻辑闭环 |
| 安全降级疑问 | adecc-15 | “当前场景触发二级降级”——记录触发降级的场景快照，供离线分析降级原因 |
| 极端伦理冲突标记 | adecc-16 | “刹车和转向都不可行”——记录极限场景的完整参数，标记为不可抗力场景 |
| 返航条件评估疑问 | adecc-22 | “剩余续航不足返航里程×1.3”——记录未通过的评估条件，供周期性重试参考 |

**2. 疑问优先级排序**

疑问条目按以下优先级排序：涉及人身安全的疑问（最高优先级）→ 导致安全降级的疑问 → 决策置信度不足的疑问 → 感知失效疑问 → 逻辑断点疑问（最低优先级）。同一优先级内按时间戳倒序排列（最新的优先处理）。

**3. 疑问生命周期管理**

每条疑问条目包含以下状态：待处理（新产生，尚未被离线复盘模块处理）、处理中（已被adecc-20离线复盘模块提取，正在分析）、已解决（离线复盘已完成，分析结论已沉淀）、已归档（超过30天的已解决疑问，转入冷归档存储）。

**4. 认知补全参考数据**

对于标记为“已解决”的疑问条目，系统自动提取其分析结论中的关键信息：根因分析结果、修正策略、验证状态。这些信息作为“认知补全参考数据”提供给adecc-20离线复盘模块，用于生成修正规则补丁。

**5. 存储容量管控**

疑问缓存库的存储容量上限为10000条。超过上限时，优先归档“已解决”状态中时间最早的非安全相关条目。涉及人身安全的疑问条目永不自动归档。30天未处理的低优先级疑问自动标记为“已过期”，转入冷归档。


## 三、约束与异常处理

| 约束类型 | 内容 |
|:---|:---|
| 存储上限 | 10000条，超限时按规则自动归档 |
| 安全优先 | 涉及人身安全的疑问永不自动归档 |
| 过期处理 | 30天未处理的低优先级疑问自动标记为“已过期” |
| 隐私保护 | 疑问条目中的传感器原始数据（摄像头帧/雷达点云）仅保留脱敏后的特征摘要 |
| 离线访问 | 疑问缓存库不参与实时决策流程，仅供离线复盘和认知补全使用 |


## 四、接口校验用例

**用例1：感知失效疑问接收与分类**

输入：adecc-01发送疑问条目“前方150m处检测到障碍物，摄像头致盲无法确认类别”。预期：自动分类为“感知失效疑问”，优先级=感知失效。返回疑问条目ID。

**用例2：极端伦理冲突标记存储**

输入：adecc-16发送极限场景标记“制动距离不足，旁侧有来车，本车道前方行人横穿，物理上不存在安全避让路径”。预期：自动分类为“极端伦理冲突标记”，优先级=涉及人身安全（最高）。该条目永不自动归档。

**用例3：疑问优先级排序验证**

前置条件：缓存库中有3条疑问：逻辑断点疑问（3小时前）、感知失效疑问（1小时前）、安全降级疑问（30分钟前）。操作：查询待解疑问清单。预期：排序结果为安全降级疑问（第1）、感知失效疑问（第2）、逻辑断点疑问（第3）。

**用例4：存储超限自动归档**

前置条件：缓存库已存储10000条疑问（达到上限），其中最早500条为“已解决”状态且不涉及人身安全。操作：新增一条感知失效疑问。预期：系统自动归档最早500条中的部分条目（腾出空间），新疑问成功写入。安全相关疑问不被归档。

**用例5：认知补全参考数据提取**

前置条件：一条“决策置信度不足疑问”已被adecc-20离线复盘模块处理完毕，分析结论为“该路口能见度不足触发保守策略，建议增加路口类型标记”。操作：adecc-20查询该疑问的认知补全参考数据。预期：返回根因分析结果、修正策略建议、验证状态。原始疑问状态更新为“已解决”。
