# ad-30-L5-核心层防篡改与只读管控单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-30 |
| 模块名称 | L5 核心层防篡改与只读管控单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 五层记忆层级存储 |
| 核心职责 | L5 核心层数据的唯一对外访问接口。管控所有读写权限：对外提供经过鉴权的只读查询服务，对内校验所有写入请求的安全令牌与操作合法性。任何对 L5 存储分区的访问（含读取）必须通过本单元，编译期禁止绕过。全量记录所有访问日志，实现软件层面的防篡改管控 |
| 依赖模块 | ad-28（L5 核心层存储单元，转发只读查询与写入确认）、ad-29（L5 核心层安全规则硬锁定单元，转发写入权限请求）、ad-51（记忆变更日志追溯单元，记录访问日志） |
| 被依赖模块 | ECC 大脑各模块（查询 L5 经验）、ad-01（总控漏斗 F₀，人工锁定指令）、ad-18（特殊环境槽，安全直达写入）、ad-26（L4 长期层存储单元，晋升写入） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常处理查询与写入鉴权 | 系统初始化完成，L5 已锁定 |
| 查询处理中 | `QUERYING` | 正在处理来自 ECC 的 L5 只读查询请求 | 收到查询请求 |
| 写入鉴权中 | `AUTHENTICATING` | 正在验证写入请求的安全令牌与操作合法性 | 收到写入请求 |
| 访问封禁 | `BLOCKED` | 检测到异常访问模式，临时封禁来源模块的访问权限 | 连续鉴权失败 3 次或检测到攻击特征 |
| 暂停服务 | `PAUSED` | 系统紧急熔断或 L5 硬件维护，暂停一切访问 | 紧急熔断指令或硬件维护通知 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 只读查询请求 | Struct（查询条件 + 请求来源模块 ID + 请求理由 + 查询优先级） | ECC 大脑各模块（经 MemoryBus） | ECC 需要查询 L5 核心经验时 | **高** |
| 写入鉴权请求 | Struct（请求 ID + 操作类型 + 安全令牌 + 请求来源模块 + 写入数据摘要） | ad-28 L5 核心层存储单元 | L5 需要执行写入前 | **高** |
| 安全令牌库 | Dict（令牌值 : 授权模块列表 + 权限级别 + 有效期） | 系统安全模块（编译期固化 + OTA 更新） | 系统初始化及令牌更新时 | **高** |
| 写入完成确认 | Struct（请求 ID + 写入结果 + 数据校验和） | ad-28 L5 核心层存储单元 | 写入操作完成 | 普通 |
| 访问封禁名单 | Dict（来源模块 ID : 封禁原因 + 封禁时间 + 解封条件） | 本单元维护 | 鉴权失败或异常检测触发 | 普通 |
| 系统紧急熔断指令 | Enum（全局熔断） | ad-01 总控漏斗 F₀ | 安全急停或碰撞事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 查询结果数据 | Struct（匹配条目列表 + 经验内容 + I 值 + 锁定状态 + 查询时间戳） | 请求来源模块（经 MemoryBus） | 查询鉴权通过且有匹配结果 | **高** |
| 查询拒绝通知 | Struct（拒绝原因 + 建议操作） | 请求来源模块 | 查询鉴权未通过 | **高** |
| 令牌验证结果 | Struct（令牌 + 验证结论：有效/无效 + 权限级别 + 剩余有效期） | ad-29 L5 核心层安全规则硬锁定单元 | 写入鉴权完成 | **高** |
| 鉴权拒绝通知 | Struct（请求 ID + 拒绝原因 + 安全日志） | ad-28、ad-29、ad-51 | 写入鉴权未通过 | **高** |
| 访问封禁通知 | Struct（被封禁模块 ID + 封禁原因 + 解封时间） | ad-01、ad-03 | 触发访问封禁时 | **高** |
| 访问日志记录 | Struct（访问时间戳 + 来源模块 + 操作类型 + 结果 + 详情） | ad-51 记忆变更日志追溯单元 | 每一次访问操作后 | **高** |
| 管控状态上报 | Struct（当前状态 + 近期查询次数 + 鉴权通过率 + 封禁列表） | ad-03、ad-01 | 每 5 分钟周期性上报 | 普通 |

### 一、只读查询权限矩阵

| 来源模块 | 权限级别 | 查询范围 | 限制条件 |
|----------|:---:|----------|----------|
| ECC-01 情境解析 | 高 | 全部 L5 经验 | 需附带查询理由 |
| ECC-03 因果推理 | 高 | 全部 L5 经验 | 需附带查询理由 |
| ECC-04 心智模拟 | 高 | 全部 L5 经验 | 需附带查询理由 |
| ECC-05 伦理仲裁 | 最高 | 全部 L5 经验 + 安全日志 | 安全审计用途，无限制 |
| ECC-08 元认知 | 高 | 全部 L5 经验 | 自我评估用途 |
| ECC-09 内生动机 | 中 | 仅限自主探索相关 L5 经验 | — |
| ECC-12 资源调度 | 中 | 仅限对外求助相关 L5 经验 | — |

| 漏斗一任意模块 | **禁止** | 无 | 编译期硬编码拒绝 |

### 二、写入操作鉴权规则

| 操作类型 | 合法来源模块 | 所需令牌级别 | 附加条件 |
|----------|:---:|:---:|----------|
| 晋升写入 | ad-26 | 系统内部令牌 | L4 晋升条目须 I ≥ 0.80，满足留存和复用条件 |
| 安全直达写入 | ad-18、ad-01 | 安全事件令牌 | 安全显著性 S ≥ 0.9，附事件快照 |
| 人工锁定写入 | ad-01 | 管理员令牌 | 令牌须在有效期内，附锁定理由 |
| 其他任何操作 | — | — | 一律拒绝 |

### 三、异常检测规则

| 异常模式 | 判定条件 | 处理 |
|----------|----------|------|
| 暴力鉴权 | 同一来源模块连续 3 次令牌验证失败 | 封禁该模块 10 分钟，上报告警 |
| 高频查询 | 同一来源模块每分钟查询 > 100 次 | 限流至 60 次/分钟，上报告警 |
| 跨模块令牌盗用 | 令牌对应模块与请求来源模块不一致 | 拒绝，记录安全日志，封禁来源模块 |
| 过期令牌 | 令牌有效期已过 | 拒绝，提示更新令牌 |


## 核心处理逻辑

```
FUNCTION l5_access_control_main_loop():
    SET internal_state = NORMAL
    加载安全令牌库（编译期固化 + OTA 可追加）
    初始化访问统计字典（来源模块 → {查询次数, 鉴权失败次数, 最近访问时间}）
    初始化封禁列表 = {}

    WHILE 系统运行中:
        // 第 1 步：紧急熔断优先
        IF 收到系统紧急熔断指令:
            SET internal_state = PAUSED
            拒绝一切新请求（返回“系统紧急熔断中”）
            CONTINUE
        ELSE IF internal_state == PAUSED AND 紧急解除信号:
            SET internal_state = NORMAL

        // 第 2 步：清理过期封禁
        FOR EACH 封禁条目 IN 封禁列表:
            IF NOW() >= 封禁条目.解封时间:
                移除封禁条目
                向 ad-01 发送解封通知

        // 第 3 步：处理只读查询请求
        IF 收到只读查询请求 AND internal_state == NORMAL:
            来源模块 = 请求.来源模块 ID

            // 3a. 封禁检查
            IF 来源模块 IN 封禁列表:
                向请求方返回查询拒绝通知（原因=模块已被封禁）
                CONTINUE

            // 3b. 漏斗一硬拦截
            IF 来源模块 IN [漏斗一所有模块]:
                向请求方返回查询拒绝通知（原因=编译期禁止漏斗一访问 L5）
                向 ad-51 写入拦截日志
                CONTINUE

            // 3c. 权限矩阵校验
            模块权限 = 权限矩阵[来源模块]
            IF 模块权限 == NULL:
                向请求方返回查询拒绝通知（原因=未授权模块）
                CONTINUE

            // 3d. 高频检测
            更新访问统计[来源模块]
            IF 访问统计[来源模块].近1分钟查询次数 > 100:
                向请求方返回查询拒绝通知（原因=查询频率超限，请稍后重试）
                向 ad-01 发送限流告警
                CONTINUE

            // 3e. 转发查询到 ad-28
            SET internal_state = QUERYING
            向 ad-28 转发只读查询请求（附加权限级别）
            WAIT ad-28 返回查询结果

            // 3f. 返回结果
            IF 查询结果不为空:
                向请求方返回查询结果数据
            ELSE:
                向请求方返回空结果
            向 ad-51 写入查询日志
            SET internal_state = NORMAL

        // 第 4 步：处理写入鉴权请求
        IF 收到 ad-28 写入鉴权请求 AND internal_state == NORMAL:
            请求 ID = 写入鉴权请求.请求 ID
            来源模块 = 写入鉴权请求.请求来源模块
            操作类型 = 写入鉴权请求.操作类型
            安全令牌 = 写入鉴权请求.安全令牌

            SET internal_state = AUTHENTICATING

            // 4a. 封禁检查
            IF 来源模块 IN 封禁列表:
                向 ad-29 返回令牌验证结果（无效）
                向 ad-28 返回鉴权拒绝通知（原因=模块已被封禁）
                SET internal_state = NORMAL
                CONTINUE

            // 4b. 操作类型合法性检查
            IF 操作类型 NOT IN [晋升写入, 安全直达写入, 人工锁定写入]:
                向 ad-29 返回令牌验证结果（无效）
                向 ad-28 返回鉴权拒绝通知（原因=非法操作类型）
                向 ad-51 写入鉴权失败日志
                SET internal_state = NORMAL
                CONTINUE

            // 4c. 操作类型与来源模块匹配检查
            IF NOT 操作类型-来源模块匹配(操作类型, 来源模块):
                向 ad-29 返回令牌验证结果（无效）
                向 ad-28 返回鉴权拒绝通知（原因=来源模块无权执行此操作）
                向 ad-51 写入鉴权失败日志
                SET internal_state = NORMAL
                CONTINUE

            // 4d. 安全令牌验证
            令牌信息 = 安全令牌库[安全令牌]
            IF 令牌信息 == NULL:
                鉴权失败计数[来源模块]++
                向 ad-29 返回令牌验证结果（无效）
                向 ad-28 返回鉴权拒绝通知（原因=无效令牌）
                向 ad-51 写入鉴权失败日志

                // 暴力鉴权检测
                IF 鉴权失败计数[来源模块] >= 3:
                    封禁列表[来源模块] = {封禁原因: "暴力鉴权", 封禁时间: NOW(), 解封时间: NOW()+10分钟}
                    向 ad-01 发送访问封禁通知
                SET internal_state = NORMAL
                CONTINUE

            // 4e. 令牌过期检查
            IF NOW() > 令牌信息.有效期:
                向 ad-29 返回令牌验证结果（无效）
                向 ad-28 返回鉴权拒绝通知（原因=令牌已过期）
                SET internal_state = NORMAL
                CONTINUE

            // 4f. 令牌与来源模块一致性检查
            IF 来源模块 NOT IN 令牌信息.授权模块列表:
                向 ad-29 返回令牌验证结果（无效）
                向 ad-28 返回鉴权拒绝通知（原因=令牌与来源模块不匹配，疑似盗用）
                封禁列表[来源模块] = {封禁原因: "令牌盗用嫌疑", 封禁时间: NOW(), 解封时间: NOW()+30分钟}
                向 ad-01 发送访问封禁通知
                向 ad-51 写入安全告警日志
                SET internal_state = NORMAL
                CONTINUE

            // 4g. 鉴权通过
            鉴权失败计数[来源模块] = 0
            向 ad-29 返回令牌验证结果（有效 + 权限级别）
            向 ad-51 写入鉴权通过日志
            SET internal_state = NORMAL

        // 第 5 步：处理写入完成确认
        IF 收到 ad-28 写入完成确认:
            向 ad-51 写入写入完成日志（含写入结果 + 数据校验和）

        // 第 6 步：周期性状态上报
        IF 距上次上报 >= 5 分钟:
            向 ad-03、ad-01 上报管控状态

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 漏斗一模块尝试查询 L5 | 编译期硬编码拒绝，记录安全日志 | — |
| 安全令牌库被篡改 | 校验令牌库哈希，不匹配则回退至编译期固化版本，上报告警 | OTA 重新下发正确令牌库 |
| 来源模块被封禁期间发起请求 | 返回封禁通知，不记录查询日志 | 封禁到期自动解封 |
| 同一模块连续 3 次鉴权失败 | 封禁该模块 10 分钟 | 10 分钟后自动解封 |
| ECC-05 伦理仲裁发起查询时被封禁 | 伦理仲裁不受封禁限制（最高权限豁免），正常放行 | — |
| 写入请求中操作类型与令牌权限级别不匹配 | 拒绝，如管理员令牌用于安全直达写入则放行 | — |
| 系统紧急熔断时收到查询请求 | 返回“系统紧急熔断中”，不记录查询日志 | 紧急解除 |
| 访问日志写入失败 | 缓存至本地临时队列（最长 5 分钟），重试写入 | 日志模块恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本单元是 L5 核心层数据的唯一对外访问接口，编译期保证所有对 L5 的访问必须通过本单元，不可绕过 |
| S-02 | 漏斗一所有模块访问 L5 的请求在编译期硬编码拒绝，无例外，不可通过运行时配置更改 |
| S-03 | ECC-05 伦理仲裁拥有最高查询权限，不受封禁列表和频率限制约束，保障安全审计畅通 |
| S-04 | 安全令牌库哈希值编译期固化，运行时每次加载前校验完整性，校验失败回退至默认安全版本 |
| S-05 | 每次访问操作（查询、鉴权、拒绝）全量写入 ad-51 不可变日志，包含时间戳、来源模块、操作类型、结果 |
| S-06 | 暴力鉴权检测：同一模块连续 3 次鉴权失败自动封禁 10 分钟，防暴力破解令牌 |
| S-07 | 令牌与来源模块绑定，跨模块使用令牌视为盗用行为，立即封禁来源模块 30 分钟 |
| S-08 | 系统紧急熔断时暂停一切访问，优先保障系统安全，紧急解除后恢复 |

## 质量自检清单
