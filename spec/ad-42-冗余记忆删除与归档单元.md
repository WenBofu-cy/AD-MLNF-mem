# ad-42-冗余记忆删除与归档单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-42 |
| 模块名称 | 冗余记忆删除与归档单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 执行遗忘流程的最终操作：对通过校验的遗忘条目执行安全删除（直接覆写擦除）或冷归档（压缩后转存至 ad-49）。是漏斗二记忆体系中唯一有权物理清除数据的模块。删除操作执行 DoD 5220.22-M 标准单次覆写，归档操作确保压缩数据可追溯恢复 |
| 依赖模块 | ad-41（最低复用次数校验单元，提供通过校验的遗忘执行清单）、ad-49（存储压缩与冷归档单元，接收归档数据）、ad-20/22/24/26（各层级存储单元，确认条目可删除）、ad-29（L5 核心层安全规则硬锁定单元，校验 L3 及以上条目是否受写保护） |
| 被依赖模块 | ad-20/22/24/26（接收删除完成确认，释放存储空间）、ad-49（接收归档数据）、ad-51（接收操作日志） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无删除或归档任务 | 系统初始化完成，或上一批次任务结束 |
| 执行中 | `EXECUTING` | 正在逐条目执行删除或归档 | 收到遗忘执行清单 |
| 回滚中 | `ROLLING_BACK` | 删除或归档过程中发生异常，正在回滚 | 写入失败或校验异常 |
| 暂停服务 | `PAUSED` | 系统紧急熔断或漏斗二冻结 | 收到全局冻结或紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 遗忘执行清单 | Dict（层级 : List（条目 ID + 遗忘方式 + I 值 + 复用次数 + 校验结论 + 来源分槽号）） | ad-41 最低复用次数校验单元 | 每次校验完成后 | **高** |
| 条目完整数据 | Struct（条目 ID + 经验内容 + 元数据 + 存储地址 + 大小） | 来源层级存储单元 | 执行删除或归档前请求 | **高** |
| 写保护校验结果 | Struct（条目 ID + 是否受写保护 + 保护类型） | ad-29 L5 核心层安全规则硬锁定单元 | L3 及以上条目执行前校验 | **高** |
| 归档写入确认 | Struct（条目 ID + 归档结果：成功/失败 + 归档位置） | ad-49 存储压缩与冷归档单元 | 每次归档操作后 | **高** |
| 全局调度指令 | Enum（冻结/激活/紧急熔断） | ad-03 漏斗二专属调度单元、ad-01 | 模式切换或安全事件时 | **紧急/高** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 源层级删除指令 | Struct（条目 ID + 删除类型 + 删除时间戳） | 来源层级存储单元 | 直接删除：覆写完成后；冷归档：归档确认后 | **高** |
| 归档数据包 | Struct（条目 ID + 压缩经验数据 + 元数据 + 归档原因 + 原层级 + 原分槽号） | ad-49 存储压缩与冷归档单元 | 冷归档条目执行时 | **高** |
| 单条目操作回执 | Struct（条目 ID + 操作类型 + 结果 + 时间戳） | 来源层级存储单元、ad-41 | 每处理完一个条目 | **高** |
| 批次操作汇总 | Struct（批次 ID + 总条目数 + 删除成功数 + 归档成功数 + 失败数 + 释放空间量 + 耗时） | ad-41、ad-40、ad-03、ad-01、ad-51 | 每批次全部完成后 | **高** |
| 异常告警 | Struct（条目 ID + 异常类型 + 处理方式 + 详情） | ad-03、ad-01 | 操作失败时 | **高** |


## 核心处理逻辑

```
FUNCTION redundant_prune_main_loop():
    SET internal_state = IDLE

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断或冻结
        IF 收到全局冻结指令 OR 紧急熔断指令:
            IF internal_state == EXECUTING:
                记录中断日志
                SET internal_state = PAUSED
            CONTINUE
        ELSE IF 收到激活指令 AND internal_state == PAUSED:
            SET internal_state = IDLE
            CONTINUE

        // 第 2 步：接收遗忘执行清单
        IF 收到 ad-41 遗忘执行清单 AND internal_state == IDLE:
            IF 遗忘执行清单所有层级均为空:
                CONTINUE

            SET internal_state = EXECUTING
            批次 ID = 生成唯一批次 ID()
            操作统计 = {总条目数: 0, 删除成功: 0, 归档成功: 0, 失败: 0, 释放空间: 0}

            // 合并所有层级的条目，按层级从低到高处理
            全部条目 = []
            FOR EACH 层级 IN [L1, L2, L3, L4]:
                IF 遗忘执行清单[层级] != NULL:
                    FOR EACH 条目 IN 遗忘执行清单[层级]:
                        全部条目.append({条目, 层级})

            操作统计.总条目数 = 全部条目.length

            // 第 3 步：逐条目执行删除或归档
            FOR EACH 待处理 IN 全部条目:
                条目 = 待处理.条目
                层级 = 待处理.层级
                条目 ID = 条目.条目 ID
                遗忘方式 = 条目.遗忘方式  // "直接删除" 或 "冷归档"

                // 3a. 从源层级获取条目完整数据
                向来源层级存储单元发送数据读取请求(条目 ID)
                WAIT 源层级返回条目完整数据

                IF 条目数据 == NULL:
                    操作统计.失败++
                    向 ad-01 发送异常告警(条目 ID, "条目数据不存在，可能已被其他操作删除")
                    CONTINUE

                // 3b. L3 及以上条目：校验写保护
                IF 层级 IN [L3, L4]:
                    向 ad-29 发送写保护校验请求(条目 ID)
                    WAIT ad-29 返回校验结果
                    IF 校验结果.受写保护:
                        操作统计.失败++
                        向 ad-01 发送异常告警(条目 ID, "条目受写保护，不可删除或归档")
                        CONTINUE

                // 3c. 根据遗忘方式执行操作
                IF 遗忘方式 == "冷归档":
                    // 压缩并发送至 ad-49
                    归档数据 = 压缩(条目数据)
                    归档数据包 = {
                        条目 ID: 条目 ID,
                        压缩经验数据: 归档数据,
                        元数据: 条目.元数据,
                        归档原因: 条目.遗忘原因,
                        原层级: 层级,
                        原分槽号: 条目.来源分槽号,
                        归档时间戳: NOW()
                    }
                    向 ad-49 发送归档数据包
                    WAIT ad-49 返回归档写入确认

                    IF 归档确认.结果 == 成功:
                        // 归档成功，删除源条目
                        向来源层级发送删除指令(条目 ID, "归档后删除")
                        WAIT 删除确认
                        操作统计.归档成功++
                        操作统计.释放空间 += 条目数据.大小
                        向 ad-41 发送单条目操作回执(条目 ID, "归档成功")
                    ELSE:
                        操作统计.失败++
                        向 ad-01 发送异常告警(条目 ID, "冷归档失败", 归档确认.失败原因)

                ELSE IF 遗忘方式 == "直接删除":
                    // 执行安全覆写擦除
                    覆写模式 = 0x00  // DoD 5220.22-M 单次全零覆写
                    存储地址 = 条目数据.存储地址
                    数据大小 = 条目数据.大小

                    // 覆写存储区域
                    覆写结果 = 安全覆写(存储地址, 数据大小, 覆写模式)
                    IF 覆写结果 != 成功:
                        操作统计.失败++
                        向 ad-01 发送异常告警(条目 ID, "安全覆写失败")
                        CONTINUE

                    // 刷新存储控制器缓存
                    FLUSH 存储控制器缓存

                    // 删除源条目索引
                    向来源层级发送删除指令(条目 ID, "覆写后删除")
                    WAIT 删除确认

                    操作统计.删除成功++
                    操作统计.释放空间 += 数据大小
                    向 ad-41 发送单条目操作回执(条目 ID, "删除成功")

                // 3d. 条目间短暂释放控制权
                SLEEP 5ms

            // 第 4 步：批次完成汇总
            批次汇总 = {
                批次 ID: 批次 ID,
                总条目数: 操作统计.总条目数,
                删除成功数: 操作统计.删除成功,
                归档成功数: 操作统计.归档成功,
                失败数: 操作统计.失败,
                释放空间量: 操作统计.释放空间,
                耗时: NOW() - 批次开始时间
            }
            向 ad-41、ad-40、ad-03、ad-01、ad-51 发送批次操作汇总
            SET internal_state = IDLE

        SLEEP 100ms


FUNCTION 安全覆写(存储地址, 数据大小, 覆写模式):
    // 执行 DoD 5220.22-M 标准单次覆写
    FOR offset = 0 TO 数据大小 STEP 覆写块大小（4KB）:
        写入 覆写模式 到 存储地址 + offset
        IF 写入失败:
            RETURN 失败
    RETURN 成功
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 冷归档时 ad-49 不可用 | 该条目保留在源层级，操作失败，上报告警。不降级为直接删除 | ad-49 恢复后重新执行 |
| 安全覆写时存储硬件故障 | 该条目操作失败，标记存储块为坏块，上报告警 | 硬件修复后重新执行 |
| 源条目在操作开始前已被其他操作删除 | 操作失败，统计失败数，记录日志 | — |
| 操作到一半收到紧急熔断 | 立即中断，已完成的保留，未开始的取消。记录中断日志 | 紧急解除后重新生成遗忘执行清单 |
| 条目受写保护但出现在遗忘清单中 | 操作失败，上报告警。说明上游 ad-40/ad-41 可能遗漏了豁免检查 | 上游模块修正 |
| 覆写后源层级删除失败 | 覆写已完成（数据已不可恢复），源层级条目标记为“待清理”，由后续去重任务处理 | 去重任务修复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 直接删除必须执行 DoD 5220.22-M 标准单次全零覆写，覆写完成后强制 FLUSH 存储控制器缓存，确保数据不可恢复 |
| S-02 | 冷归档条目在归档确认成功前不得删除源条目。归档失败时条目保留在源层级，数据零丢失 |
| S-03 | L3 及以上层级条目在执行任何操作前须向 ad-29 校验写保护状态，受写保护条目拒绝操作 |
| S-04 | 本单元是漏斗二中唯一有权物理清除数据的模块。其他任何模块不得绕过本单元直接删除经验条目 |
| S-05 | 紧急熔断时立即中断操作，已完成的保留，未开始的取消。不得因删除操作阻塞紧急安全指令 |
| S-06 | 所有删除、归档操作（含成功、失败、中断）全量写入 ad-51 不可变日志 |
