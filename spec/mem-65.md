# mem-65 双漏斗模式切换单元

> 所属仓库: AD-mlnf-mem/spec/
> 模块编号: mem-65
> 所属分区: AD新增·双漏斗管理层


## 一、基本信息

| 项目 | 内容 |
|:---|:---|
| 模块编号 | mem-65 |
| 模块名称 | 双漏斗模式切换单元 |
| 核心职责 | 管理人工驾驶↔自动驾驶切换时的漏斗冻结/激活、数据一致性校验、紧急应急接管时的双漏斗临时锁定只读。模式切换在0.5秒内完成 |
| 输入数据 | 驾驶模式切换信号（人工→自动/自动→人工）、紧急应急接管触发信号（来自AD-ECC安全仲裁adecc-14）、当前活跃漏斗状态（来自mem-61）、漏斗二运行状态 |
| 输出数据 | 切换完成确认信号、目标漏斗激活状态、数据一致性校验结果、紧急锁定状态通知 |


## 二、核心处理逻辑

**1. 人工驾驶→自动驾驶切换**

切换条件：车辆处于静止状态（车速=0，P档）或由人工持续操控中。

执行动作：
- 冻结漏斗一当前活跃子画像槽的所有写入操作
- 完成当前子画像槽的数据一致性校验（确认所有缓冲数据已落盘）
- 瞬间激活漏斗二（自动驾驶自成长漏斗）
- 向AD-ECC发送“漏斗二已激活”确认信号
- 切换全程在0.5秒内完成

切换后状态：漏斗一进入休眠冻结状态，仅保留只读权限供车内辅助提醒系统查询；漏斗二进入活跃运行状态，开始记录自动驾驶系统的驾驶行为经验。

**2. 自动驾驶→人工接管切换**

切换条件：驾驶员通过中控屏或物理按键发起接管请求，或AD-ECC安全降级触发人工接管请求。

执行动作：
- 暂停漏斗二的经验记录写入
- 完成漏斗二当前缓冲数据的一致性校验
- 即时调用mem-64识别当前驾驶员身份
- 激活漏斗一中对应驾驶员的子画像槽
- 向AD-ECC发送“漏斗一已激活”确认信号
- 接管过渡期（2秒）内双漏斗均设为只读，不写入新经验

切换后状态：漏斗二进入暂停休眠状态；漏斗一对应子画像槽进入活跃运行状态，开始记录当前驾驶员的驾驶行为。

**3. 紧急应急接管**

触发条件：AD-ECC安全仲裁（adecc-14）发出紧急接管信号，或驾驶员突然介入操控（大力转动方向盘/猛踩刹车）。

执行动作：
- 双漏斗临时锁定只读，立即停止所有经验写入
- 优先执行安全避险策略（由AD-MCC紧急避险模块admcc-09执行）
- 险情解除后，系统自检确认双漏斗数据一致性
- 数据一致性校验通过后恢复正常运行模式
- 数据一致性校验未通过时，标记异常并保留双漏斗的完整快照供离线分析

**4. 数据一致性校验**

每次模式切换前，系统自动执行以下校验：
- 确认漏斗一当前活跃槽的缓冲数据已全部落盘
- 确认漏斗二当前活跃槽的缓冲数据已全部落盘
- 校验双漏斗的存储分区完整性（无损坏、无丢失）
- 校验通过后执行切换；校验未通过时，保留当前模式并上报异常至mem-55日志系统


## 三、约束与异常处理

| 约束类型 | 内容 |
|:---|:---|
| 切换时间 | 人工→自动切换须在0.5秒内完成 |
| 接管过渡期 | 自动→人工接管过渡期2秒，过渡期内双漏斗只读 |
| 紧急接管 | 双漏斗临时锁定只读，先执行安全策略，险情解除后再校验数据 |
| 数据校验 | 每次切换前强制校验缓冲数据落盘状态，未通过则保留当前模式 |
| 切换失败 | 切换失败时保持当前模式不变，上报异常至日志系统，提示驾驶员手动接管 |
| 模式互斥 | 任何时刻漏斗一和漏斗二不能同时处于活跃写入状态 |


## 四、接口校验用例

**用例1：人工→自动驾驶正常切换**

前置条件：车辆静止（P档），漏斗一活跃槽“张三”正在写入。操作：驾驶员通过中控屏启动自动驾驶模式。预期：漏斗一“张三”槽冻结写入，漏斗二激活。切换在0.5秒内完成。AD-ECC收到“漏斗二已激活”确认信号。

**用例2：自动驾驶→人工接管切换**

前置条件：车辆处于自动驾驶模式，漏斗二正在写入经验。操作：驾驶员通过中控屏发起接管请求。预期：漏斗二暂停写入，mem-64识别驾驶员身份为“张三”，漏斗一“张三”槽激活。接管过渡期2秒内双漏斗只读。2秒后漏斗一正常写入。

**用例3：紧急应急接管**

前置条件：车辆处于自动驾驶模式，前方突然出现障碍物，AD-ECC安全仲裁发出紧急接管信号。操作：系统触发紧急应急接管。预期：双漏斗立即锁定只读，停止所有经验写入。AD-MCC紧急避险模块（admcc-09）执行紧急制动。险情解除后，系统自检双漏斗数据一致性，校验通过后恢复正常运行。

**用例4：数据一致性校验未通过**

前置条件：漏斗一活跃槽存在未落盘的缓冲数据（模拟存储故障）。操作：驾驶员尝试从人工驾驶切换至自动驾驶。预期：数据一致性校验未通过。系统保持人工驾驶模式不变，中控屏提示“系统自检中，请稍后再试”。异常上报至mem-55日志系统。

**用例5：模式互斥验证**

前置条件：漏斗一已激活。操作：尝试同时激活漏斗二（不经过正常切换流程）。预期：系统拒绝激活请求，返回“漏斗一当前活跃，无法同时激活漏斗二”。确保任何时刻双漏斗不会同时处于活跃写入状态。
