# ad-21-L1-临时层时序衰减单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-21 |
| 模块名称 | L1 临时层时序衰减单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 五层记忆层级存储 |
| 核心职责 | 管理 L1 临时层经验条目的时间衰减。L1 条目留存满 24 小时后，综合当前重要度 I 值与场景分槽专属晋升阈值，判定该条目应进入晋升候选、建议清除或继续保留。是五层记忆结构中连接“临时暂存”与“长期固化”的第一个判定关口 |
| 依赖模块 | ad-20（L1 临时层存储单元，提供 L1 条目索引与留存时长）、ad-36（综合重要度 I 值聚合计算单元，提供条目当前 I 值）、ad-35（三维权重系数配置单元，提供各分槽专属晋升阈值） |
| 被依赖模块 | ad-20（消费衰减评估结果，执行后续的晋升候选发送或清除） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 周期性扫描 L1 条目，执行衰减评估 | 漏斗二激活，L1 存在满 24 小时条目 |
| 批量评估中 | `BATCH_EVALUATING` | 正在处理大批量同时到期的条目 | 单次扫描发现 > 100 条满 24 小时条目 |
| 暂停评估 | `PAUSED` | 漏斗二冻结或 L1 处于维护状态，暂停评估 | 驾驶模式切换、L1 维护中 |
| 降级保守模式 | `CONSERVATIVE` | 全局容量告急，临时收紧清除阈值，加速淘汰 | 收到 ad-48 容量告急信号 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| L1 条目索引快照 | List（条目 ID + 写入时间戳 + 来源分槽号 + I₀ + 子类标记） | ad-20 L1 临时层存储单元 | 每 60 秒周期性推送 | 普通 |
| 条目当前 I 值 | Dict（条目 ID : 当前 I 值） | ad-36 综合重要度 I 值聚合计算单元 | 每 24 小时 I 值刷新后 | **高** |
| 分槽专属晋升阈值 | Dict（分槽号 : {L1→L2 晋升 I 阈值, L1→L2 晋升时间阈值}） | ad-35 三维权重系数配置单元 | 系统初始化及阈值变更时 | **高** |
| 容量告急信号 | Enum（全局容量 > 90%） | ad-48 全局容量配额管控单元 | 容量超过阈值时 | **高** |
| 全局调度指令 | Enum（冻结/激活） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 衰减评估结果 | List（条目 ID + 评估结论 + 当前 I 值 + 留存时长 + 建议操作） | ad-20 L1 临时层存储单元 | 每次衰减评估完成后 | **高** |
| 批量清除建议 | List（条目 ID + I 值 + 清除优先级） | ad-20、ad-42 | 容量告急时 | **高** |
| 评估统计上报 | Struct（评估总数 + 晋升候选数 + 建议清除数 + 继续保留数） | ad-03、ad-01 | 每次评估完成后 | 普通 |
| 阈值异常告警 | Struct（分槽号 + 异常阈值 + 详情） | ad-03、ad-01 | 检测到晋升阈值为 0 或异常时 | **高** |


## 衰减判定规则

### 一、通用判定规则

```
留存时长 < 24 小时 → 继续保留（不评估）
留存时长 ≥ 24 小时 → 进入衰减评估
```

### 二、衰减评估三维判定

| 评估结论 | 判定条件 | 后续操作 |
|----------|----------|----------|
| 晋升候选 | I ≥ 该分槽 L1→L2 晋升阈值 | 加入晋升候选清单，发送至 ad-38 |
| 建议清除 | I < 该分槽最低保留阈值（默认 0.10） | 发送至 ad-42 执行安全删除 |
| 继续保留 | I 在 [最低保留阈值, 晋升阈值) 之间 | 条目继续留在 L1，等待下次评估（下次评估周期 = 12 小时） |

### 三、各分槽专属晋升阈值

| 分槽 | L1→L2 晋升 I 阈值 | 最低保留 I 阈值 | 说明 |
|------|:---:|:---:|------|
| 高速巡航槽 (ad-15) | 0.40 | 0.10 | 标准 |
| 城区路口槽 (ad-16) | 0.40 | 0.08 | 最低保留阈值略低，安全经验优先保留 |
| 泊车低速槽 (ad-17) | 0.40 | 0.06 | 最低保留阈值极低，低频经验保护 |
| 特殊环境槽 (ad-18) | 0.28 | 0.06 | 晋升阈值下调 30%，加速固化 |
| 通用驾驶槽-常规子类 (ad-19) | 0.40 | 0.10 | 标准 |
| 通用驾驶槽-乡村道路子类 (ad-19) | 0.40 | 0.06 | 最低保留阈值下调，低频经验保护 |

### 四、容量告急时加速淘汰规则

| 条件 | 临时调整 |
|------|----------|
| 全局容量 > 90% | 所有分槽最低保留阈值临时上调 50%（如 0.10 → 0.15） |
| 全局容量 > 95% | 所有分槽最低保留阈值临时上调 100%（如 0.10 → 0.20） |
| L1 占用率 > 95% | 跳过“继续保留”，仅保留“晋升候选”与“建议清除”两档 |


## 核心处理逻辑

```
FUNCTION l1_decay_assessment_main_loop():
    SET internal_state = NORMAL
    评估间隔 = 60 秒
    继续保留条目下次评估间隔 = 12 小时
    已评估保留条目字典 = {}  // 条目 ID → 上次评估时间

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令 OR L1 处于维护状态:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 第 2 步：容量告急模式切换
        IF 收到 ad-48 容量告急信号（容量 > 90%）:
            SET internal_state = CONSERVATIVE
            所有分槽最低保留阈值临时上调
        ELSE IF 容量告急解除 AND internal_state == CONSERVATIVE:
            SET internal_state = NORMAL
            恢复各分槽标准最低保留阈值

        // 第 3 步：获取 L1 条目索引与当前 I 值
        IF 距上次评估 >= 60 秒:
            L1 条目列表 = ad-20 L1 条目索引快照
            条目 I 值字典 = ad-36 条目当前 I 值

            // 3a. 筛选到期条目
            到期条目列表 = []
            FOR EACH 条目 IN L1 条目列表:
                留存时长 = NOW() - 条目.写入时间戳
                IF 留存时长 >= 24 小时:
                    // 检查是否在“继续保留”冷却期内
                    IF 条目.ID IN 已评估保留条目字典:
                        IF NOW() - 已评估保留条目字典[条目.ID] < 12 小时:
                            CONTINUE  // 冷却期内，跳过
                    到期条目列表.append(条目)

            IF 到期条目列表为空:
                CONTINUE

            // 3b. 批量处理
            IF 到期条目列表.length > 100:
                SET internal_state = BATCH_EVALUATING

            // 3c. 逐条判定
            晋升候选清单 = []
            建议清除清单 = []
            继续保留清单 = []

            FOR EACH 条目 IN 到期条目列表:
                当前 I 值 = 条目 I 值字典[条目.ID]
                IF 当前 I 值 == NULL:
                    当前 I 值 = 条目.I₀  // 使用初始 I 值

                分槽号 = 条目.来源分槽号
                晋升阈值 = 分槽专属阈值[分槽号].晋升 I 阈值
                最低保留阈值 = 分槽专属阈值[分槽号].最低保留 I 阈值

                // 容量告急时调整阈值
                IF internal_state == CONSERVATIVE:
                    最低保留阈值 = 最低保留阈值 × 容量调整系数

                // L1 自身占用 > 95%：跳过继续保留
                IF L1 占用率 > 0.95:
                    IF 当前 I 值 >= 晋升阈值:
                        晋升候选清单.append({条目.ID, 当前 I 值, 留存时长})
                    ELSE:
                        建议清除清单.append({条目.ID, 当前 I 值, 留存时长})
                ELSE:
                    // 正常三维判定
                    IF 当前 I 值 >= 晋升阈值:
                        晋升候选清单.append({条目.ID, 当前 I 值, 留存时长})
                    ELSE IF 当前 I 值 < 最低保留阈值:
                        建议清除清单.append({条目.ID, 当前 I 值, 留存时长})
                    ELSE:
                        继续保留清单.append({条目.ID, 当前 I 值, 留存时长})
                        已评估保留条目字典[条目.ID] = NOW()

            // 3d. 发送评估结果
            评估结果 = {
                晋升候选: 晋升候选清单,
                建议清除: 建议清除清单,
                继续保留: 继续保留清单
            }
            向 ad-20 发送衰减评估结果

            // 3e. 统计上报
            向 ad-03、ad-01 上报评估统计

            IF internal_state == BATCH_EVALUATING:
                SET internal_state = NORMAL

        SLEEP 60 秒
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 条目当前 I 值缺失（ad-36 未返回） | 使用条目 I₀ 作为替代，标记为“I 值估算” | ad-36 恢复后使用精确值 |
| 分槽晋升阈值未配置 | 使用标准默认值（晋升=0.40，最低保留=0.10），上报告警 | 阈值配置完成 |
| 单个条目 I 值异常（> 1.0 或 < 0） | 钳制到 [0, 1] 范围，上报告警 | — |
| 条目时间戳异常（未来时间） | 标记为“时间戳异常”，跳过评估，等待下次扫描 | 时间戳恢复正常 |
| L1 条目数量巨大（> 10,000 条）导致评估超时 | 分批评估，每批 500 条，评估间隔缩短至 10 秒 | — |
| 冻结状态 | 暂停评估，状态 PAUSED | 激活指令 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 24 小时留存时间为硬编码判定门槛，任何运行时不满足此条件的条目不得进入衰减评估 |
| S-02 | 安全显著性 S ≥ 0.9 的条目，在容量告急时仍受最低保留阈值保护，不清除 |
| S-03 | 晋升阈值与最低保留阈值的修改须经权限校验，且不可低于编译期硬编码下限（晋升=0.20，最低保留=0.05） |
| S-04 | 继续保留条目 12 小时冷却期内不可重复评估，避免高频无效扫描 |
| S-05 | 所有衰减评估操作（晋升候选、建议清除、继续保留）全量写入 ad-51 变更日志 |
