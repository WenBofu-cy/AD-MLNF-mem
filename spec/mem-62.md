# mem-62 子画像槽隔离单元

> 所属仓库: AD-mlnf-mem/spec/
> 模块编号: mem-62
> 所属分区: AD新增·双漏斗管理层


## 一、基本信息

| 项目 | 内容 |
|:---|:---|
| 模块编号 | mem-62 |
| 模块名称 | 子画像槽隔离单元 |
| 核心职责 | 确保漏斗一内部不同驾驶员的子画像槽之间物理隔离，数据绝不混合。输出边界严格管控：所有统计数据仅用于车内辅助提醒，严禁以任何形式接入自动驾驶决策链路 |
| 输入数据 | 当前活跃子画像槽ID（来自mem-61）、各子画像槽的原始行为数据、AD-ECC的决策查询请求 |
| 输出数据 | 隔离状态确认信号、脱敏后的统计摘要（仅供车内辅助提醒）、越权访问告警 |


## 二、核心处理逻辑

**1. 物理存储隔离**

每个子画像槽在存储分区上分配独立的数据块。编译期通过内存分区与权限控制强制实施。任何子画像槽的数据指针不会被另一个子画像槽引用。张先生的驾驶数据永远不会出现在李女士的子画像槽中。

**2. 跨槽数据防泄漏**

系统不会因为张先生习惯跟车近，就在李女士开车时以张先生的数据做预警参考。所有跨槽数据引用请求在编译期即被拦截，不进入运行时判断流程。

**3. 输出边界严格管控**

所有子画像槽的原始行为数据仅对漏斗一内部开放。对外（AD-ECC、云端技能包、V2X通信模块）仅输出脱敏后的统计摘要。统计摘要仅包含：驾驶风格偏好（跟车时距/变道策略/加减速特性）、行为判定标签（优良习惯/常态陋习/应急特殊操作），不包含任何原始行为记录（具体时间/地点/车速/轨迹）。

**4. 自动驾驶决策链路隔离**

子画像槽数据严禁以任何形式接入自动驾驶决策链路。该隔离在编译期即通过内存分区与权限控制强制实施。AD-ECC的驾驶决策仅可调用漏斗二（自动驾驶自成长漏斗）的经验数据，不可调用漏斗一任何数据。

**5. 越权访问监控**

持续监测所有对子画像槽数据的访问请求。任何来自AD-ECC、云端技能包或V2X通信模块的原始数据读取请求均视为越权访问。越权访问事件立即上报mem-55（记忆变更日志追溯）并触发安全告警。


## 三、约束与异常处理

| 约束类型 | 内容 |
|:---|:---|
| 物理隔离 | 编译期强制实施，不依赖运行时配置 |
| 输出边界 | 仅向车内辅助提醒系统输出脱敏统计摘要，不输出原始数据 |
| 决策链路隔离 | 漏斗一数据禁止接入自动驾驶决策，编译期强制 |
| 越权告警 | 任何越权访问请求立即上报安全日志 |
| 数据擦除 | 删除子画像槽时执行DoD 5220.22-M标准覆写擦除 |


## 四、接口校验用例

**用例1：跨槽数据隔离验证**

前置条件：子画像槽A存储有张先生的驾驶数据（跟车时距1.8s），子画像槽B存储有李女士的驾驶数据（跟车时距2.5s）。操作：当前活跃槽为B（李女士），系统查询当前驾驶员的跟车偏好。预期：返回2.5s（李女士的数据），不返回张先生的1.8s。

**用例2：决策链路隔离验证**

前置条件：子画像槽A存储有张先生的驾驶数据。操作：AD-ECC尝试通过MemoryBus读取子画像槽A的原始行为数据，用于自动驾驶决策。预期：请求被mem-62拦截，返回“权限不足”。AD-ECC仅可通过漏斗二获取驾驶经验数据。

**用例3：对外输出脱敏验证**

前置条件：子画像槽A存储有张先生的完整驾驶行为记录（含GPS轨迹、具体车速、操作时间戳）。操作：车内辅助提醒系统请求张先生的驾驶偏好摘要。预期：仅返回脱敏后的统计摘要（“该驾驶员偏好跟车时距1.8s，变道策略积极”），不返回任何GPS坐标、具体车速数值或操作时间戳。

**用例4：越权访问告警验证**

前置条件：AD-ECC连续3次尝试读取子画像槽A的原始数据。预期：第1次越权请求即触发告警，上报至mem-55日志系统。3次累计后触发安全事件通知，标记AD-ECC为“异常访问模式”。
