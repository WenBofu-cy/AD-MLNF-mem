# mem-59 独立世界模型库

> 所属仓库: AD-mlnf-mem/spec/
> 模块编号: mem-59
> 所属分区: 外置独立扩展区（AD场景适配）


## 一、基本信息

| 项目 | 内容 |
|:---|:---|
| 模块编号 | mem-59 |
| 模块名称 | 独立世界模型库 |
| 核心职责 | 存储驾驶场景相关的物理常识与结构化世界模型。包含道路物理属性、交通参与者属性、非结构化道路参数、环境动态适配参数。以结构化属性向量与IF-THEN因果规则表形式存储，只读不改，永不遗忘，不参与任何漏斗晋升与遗忘机制 |
| 输入数据 | 世界模型更新请求（来自adecc-03局部场景建模）、环境查询请求（来自AD-ECC各认知模块及AD-MCC各执行模块） |
| 输出数据 | 物体属性基底（硬度/脆性/摩擦系数等量化等级）、物理约束规则、实时环境图谱、场景先验数据、路面类型与对应的摩擦系数和制动距离系数 |


## 二、核心处理逻辑

**1. 物理常识存储结构**

以结构化属性向量与IF-THEN因果规则表形式存储，存储占用<300KB。

道路实体物理参数：五大类目标的材质硬度、摩擦系数、质量等级、形变特性、可移动性。典型条目：干燥沥青μ=0.8，压实积雪μ=0.2，冰面μ=0.1。

环境物理影响阈值：积水滑水临界车速、冰雪附着系数衰减曲线、能见度-安全车距映射。典型条目：水膜厚度>3mm且车速>80km/h触发滑水风险。

车辆动力学硬边界：最大侧向加速度0.85g、极限制动减速度1.0g、侧翻临界阈值。

环境动态适配参数：随天气、光照、时段自动联动风险判定权重的修正系数。典型条目：夜间无灯路段，行人出现概率权重×1.5。

车辆能源物理参数：电池额定容量、燃油箱容积、额定续航、各工况能耗曲线。

**2. 非结构化道路物理属性**

| 属性 | 典型值 | 对驾驶的影响 |
|:---|:---|:---|
| 压实泥土路面摩擦系数 | μ=0.5-0.6 | 制动距离较沥青路面延长约30% |
| 松散碎石路面摩擦系数 | μ=0.3-0.4 | 急刹易侧滑，须降低车速上限 |
| 湿泥路面摩擦系数 | μ=0.2-0.3 | 极度危险，禁止急刹急转 |
| 路面可通过性判断 | 坑洼深度>15cm或凸起>20cm | 触发绕行规划，不可强行通过 |
| 道路宽度判断 | 单车道<2.5m | 不具备会车条件，须找避让点 |

**3. 交通参与者属性模板**

五大类交通参与者的物理属性预置模板：

- 行人（含儿童/老人/行动不便者）：可移动、脆性5（极高）、质量1（极轻）、避让优先级最高
- 非机动车（自行车/电动车）：可移动、速度范围0-25km/h、脆性4
- 普通机动车（乘用车/商用车）：可移动、质量3-5、速度范围0-120km/h、硬度4-5
- 静态障碍物（护栏/隔离带/路沿/坑洼）：不可移动、硬度4-5、脆性1-4
- 特种车辆（救护车/消防车/警车）：可移动、声光特征、优先通行权标记

**4. 查询与更新机制**

AD-ECC各认知模块通过MemoryBus向mem-59发起查询请求，获取当前场景相关的物理常识和世界模型数据。adecc-03（局部场景建模）负责将感知数据转化为结构化世界模型，增量更新至mem-59。更新内容须通过物理一致性校验（如“路面摩擦系数应在0.1-1.0范围内”），校验未通过则拒绝写入。世界模型数据与MLNF-Mem记忆漏斗物理隔离，互不干扰。

**5. 因果规则表**

以IF-THEN形式存储物理因果规则：

IF 水膜厚度>3mm AND 车速>80km/h THEN 滑水风险=极高
IF 路面类型=冰面 AND 制动减速度>2m/s² THEN 车轮抱死风险=高
IF 物体易碎(脆性≥4) AND 碰撞速度>10km/h THEN 物体损毁概率=极高
IF 路面宽度<车宽+0.5m THEN 不可通行
IF 泥泞深度>15cm THEN 不可通行（超过乘用车底盘离地间隙）
IF 涉水深度>30cm THEN 不可通行（超过安全涉水线）


## 三、约束与异常处理

| 约束类型 | 内容 |
|:---|:---|
| 只读保护 | 物理常识库数据不可被任何经验、习惯、学习行为修改或覆盖 |
| 存储占用 | 物理常识库总计<300KB，确保嵌入式环境下毫秒级查询延迟 |
| 更新校验 | 世界模型增量更新须通过物理一致性校验 |
| 查询失败 | 物理常识查询失败时，使用预设的保守值（如摩擦系数取最低值0.1） |
| 与记忆中枢隔离 | 世界模型与MLNF-Mem记忆漏斗物理分区存储，互不干扰 |
| 不可通行判定 | 触发不可通行判定时，立即上报AD-ECC行为决策模块，触发更换路线规划 |


## 四、接口校验用例

**用例1：正常路面摩擦系数查询**

输入：查询条件“路面类型=干燥沥青”。预期：返回摩擦系数μ=0.8，制动距离系数=1.0（基准）。

**用例2：积雪路面物理属性查询**

输入：查询条件“路面类型=压实积雪”。预期：返回摩擦系数μ=0.2，制动距离系数=1.5（制动距离延长50%）。联动环境动态适配参数：雪天行人出现概率权重×1.2。

**用例3：非铺装道路不可通行判定**

输入：世界模型检测到前方路面泥泞深度=20cm，查询mem-59是否可通行。预期：返回“不可通行”判定，原因=“泥泞深度20cm超过底盘离地间隙15cm”。触发绕行规划。

**用例4：物理常识查询失败时默认保守值**

前置条件：模拟mem-59存储故障，物理常识库查询失败。输入：查询“当前路面摩擦系数”。预期：返回保守默认值μ=0.1（冰面级），AD-ECC行为决策基于该保守值执行最高安全等级的速度限制。

**用例5：经验不可修改物理常识**

操作：漏斗二中存储了一条经验“该路段实际抓地力高于物理常识库标称值”。操作：尝试用该经验更新mem-59中的路面摩擦系数。预期：mem-59拒绝写入请求。物理常识库只接受adecc-03基于实际传感器数据的结构化更新，不接受经验漏斗的直接写入。
