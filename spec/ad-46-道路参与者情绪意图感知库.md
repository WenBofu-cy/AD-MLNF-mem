# ad-46-道路参与者情绪意图感知库 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-46 |
| 模块名称 | 道路参与者情绪意图感知库 |
| 所属分区 | 四、漏斗外挂扩展区（物理隔离） |
| 核心职责 | 对接外部情绪感知与意图推断模块，提供所有路上可见人员（四轮驾驶员、摩托车骑手、电单车骑手、自行车骑手、行人）的实时情绪状态与行为意图推断结果的查询接口。独立于双漏斗记忆系统运行，不参与记忆的沉淀、筛选、晋升与遗忘机制。仅提供决策辅助参考，不作为安全关键决策的唯一依据 |
| 依赖模块 | 外部情绪感知模块（摄像头面部表情/姿态分析、语音情绪特征提取）、外部意图推断引擎（行为轨迹预测）、ECC-10 社会心智模块（经 MemoryBus 查询） |
| 被依赖模块 | ECC-03 因果推理模块（查询情绪意图以辅助预判）、ECC-05 伦理仲裁模块（查询情绪意图以辅助风险评估）、ad-08 上下文场景标记单元（查询行人情绪以辅助应急标记）、ad-09 行为判定标签单元（查询驾驶员情绪以辅助行为判定） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常提供情绪意图查询服务 | 系统初始化完成，外部感知模块就绪 |
| 查询处理中 | `QUERYING` | 正在处理情绪意图查询请求 | 收到查询请求 |
| 数据更新中 | `UPDATING` | 正在接收外部感知模块的最新情绪数据 | 外部感知模块推送新数据 |
| 降级模式 | `DEGRADED` | 外部感知模块部分不可用，仅返回基于姿态的保守推断 | 外部感知模块故障或置信度过低 |
| 暂停服务 | `PAUSED` | 系统紧急熔断 | 收到 ad-01 紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 情绪意图查询请求 | Struct（目标 ID + 目标类别 + 查询类型 + 请求来源模块 + 查询优先级） | ECC-03/05、ad-08、ad-09 等模块（经 MemoryBus） | ECC 需要了解特定道路参与者的情绪状态时 | **高** |
| 外部情绪感知原始数据 | Struct（目标 ID + 面部表情特征向量 + 姿态特征向量 + 语音情绪特征 + 感知置信度 + 时间戳） | 外部情绪感知模块（摄像头/麦克风阵列） | 感知模块实时推送，每 200ms | **高** |
| 外部意图推断数据 | Struct（目标 ID + 行为意图标签 + 意图置信度 + 预测轨迹 + 时间戳） | 外部意图推断引擎 | 意图推断引擎周期性推送，每 500ms | **高** |
| 目标列表同步 | List（目标 ID + 目标类别 + 当前位置 + 运动状态） | ECC-01 情境解析模块（经 MemoryBus） | 感知融合更新后周期性推送 | 普通 |
| 全局调度指令 | Enum（冻结/激活） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |
| 紧急熔断指令 | Enum（全局熔断） | ad-01 总控漏斗 F₀ | 安全急停或碰撞事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 情绪状态查询结果 | Struct（目标 ID + 情绪状态标签 + 情绪强度 + 感知置信度 + 数据时效 + 查询时间戳） | 请求来源模块（经 MemoryBus） | 查询请求处理完成后 | **高** |
| 行为意图推断结果 | Struct（目标 ID + 意图标签 + 意图置信度 + 预测行为描述 + 预测时间窗口 + 查询时间戳） | 请求来源模块 | 查询请求处理完成后 | **高** |
| 综合情绪意图评估报告 | Struct（目标 ID + 情绪状态 + 意图推断 + 风险关联建议 + 综合置信度） | ECC-05 伦理仲裁模块、ECC-03 因果推理模块 | 高风险目标（TTC < 3s）情绪查询时 | **高** |
| 降级模式通知 | Struct（降级原因 + 可用能力范围 + 建议替代数据源） | 所有查询方模块 | 进入降级模式时 | **高** |
| 感知异常告警 | Struct（异常类型 + 影响范围 + 持续时长） | ad-03、ad-01 | 外部感知模块连续故障超 3 秒 | **高** |
| 运行状态上报 | Struct（当前状态 + 活跃目标数 + 近期查询次数 + 平均置信度 + 降级次数） | ad-03、ad-01 | 每 30 秒周期性上报 | 普通 |


## 情绪状态与意图标签体系

### 一、行人（第四类目标）

| 情绪/状态标签 | 识别特征 | 对行为意图的影响 |
|--------------|----------|------------------|
| 镇定从容 | 步伐平稳，视线正常，身体姿态放松 | 正常通行，行为可预测 |
| 慌张急躁 | 步伐急促，频繁看手机或看表，身体前倾 | 可能突然加速横穿，TTC 风险上调 |
| 犹豫徘徊 | 步伐不规律，左右张望，在路边来回踱步 | 可能在找时机横穿，需提前减速 |
| 注意力涣散 | 低头看手机，视线脱离道路，步伐缓慢 | 可能未注意来车，风险极高 |
| 暴怒冲动 | 大幅度手势，面部表情愤怒，快步冲向车道 | 可能主动拦车或冲入车道，触发紧急评估 |
| 老人/儿童特殊状态 | 行动迟缓（老人），行为不可预测（儿童） | 默认提高保护优先级，行为不可预测标记 |
| 注视车辆并点头/挥手 | 与驾驶员有眼神接触，做让行手势 | 高置信度让行意图，可安全通过 |
| 注视车辆但犹豫 | 看向本车，脚步减速但未停 | 可能让行也可能抢行，保守减速 |

### 二、非机动车骑行者（自行车/电单车/人力三轮）

| 情绪/状态标签 | 识别特征 | 对行为意图的影响 |
|--------------|----------|------------------|
| 正常骑行 | 速度稳定，车道位置居中或偏右，无异常摇摆 | 正常通行，行为可预测 |
| 疲劳/注意力下降 | 骑行速度波动，车把轻微摇摆，头部低垂 | 可能突然偏离车道，拉大横向距离 |
| 焦急赶路 | 高速骑行，频繁超越，观察后方频率高 | 可能突然变道或闯红灯 |
| 受惊/恐慌 | 突然加速或急刹，身体僵硬，回头看频率高 | 可能因本车鸣笛受惊，行为不可预测 |
| 违规倾向 | 逆行、闯红灯、在机动车道骑行 | 高风险，触发主动避让 |

### 三、机动车驾驶员（第二类目标）

| 情绪/状态标签 | 识别特征 | 对行为意图的影响 |
|--------------|----------|------------------|
| 正常驾驶 | 车道保持稳定，速度平稳，转向灯规范使用 | 正常跟车，可预测 |
| 路怒倾向 | 频繁鸣笛，急加速急刹，紧跟前车，大幅度手势 | 可能强行变道或故意别车，拉大安全距离 |
| 犹豫不决 | 车速偏慢，频繁刹车，转向灯反复开关 | 可能在找路或新手驾驶，保持耐心 |
| 疲劳驾驶 | 车辆轻微偏离车道，速度不规律，刹车反应延迟 | 高风险，远离该车辆并上报告警 |
| 友好让行 | 闪大灯示意，减速让出空间，打手势 | 可信赖的让行信号 |

### 四、摩托车/三轮机动车骑手（第二类子类）

| 情绪/状态标签 | 识别特征 | 对行为意图的影响 |
|--------------|----------|------------------|
| 正常骑行 | 速度适中，车道位置合理 | 正常跟车 |
| 激进驾驶 | 高速穿梭车流，压弯角度大，频繁变道 | 保持距离，避免并行 |
| 受惊/不稳 | 车身摇摆，脚点地，频繁回望 | 可能随时停车或摔倒，拉大距离 |


## 核心处理逻辑

```
FUNCTION emotion_intent_library_main_loop():
    SET internal_state = NORMAL
    初始化目标情绪缓存（目标 ID → 最新情绪状态 + 意图推断 + 时间戳）
    缓存有效期 = 2000ms（超过此时间的数据标记为“可能过时”）
    降级检查计数器 = 0

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = PAUSED
            清空查询等待队列
            CONTINUE
        ELSE IF internal_state == PAUSED AND 紧急解除:
            SET internal_state = NORMAL

        // 第 2 步：接收外部感知数据并更新缓存
        IF 收到外部情绪感知原始数据:
            IF internal_state == PAUSED:
                CONTINUE
            SET internal_state = UPDATING

            目标 ID = 数据.目标 ID
            感知置信度 = 数据.感知置信度

            IF 感知置信度 < 0.3:
                // 数据质量过低，不更新缓存
                CONTINUE

            // 更新情绪缓存
            情绪缓存[目标 ID] = {
                情绪状态标签: 解析情绪标签(数据.面部表情特征向量, 数据.姿态特征向量),
                情绪强度: 计算情绪强度(数据),
                感知置信度: 感知置信度,
                更新时间戳: NOW()
            }

            IF 感知置信度 < 0.5:
                降级检查计数器++
            ELSE:
                降级检查计数器 = MAX(0, 降级检查计数器 - 1)

            SET internal_state = NORMAL

        // 第 3 步：接收外部意图推断数据并更新缓存
        IF 收到外部意图推断数据:
            IF internal_state == PAUSED:
                CONTINUE

            目标 ID = 数据.目标 ID
            意图置信度 = 数据.意图置信度

            IF 意图置信度 < 0.4:
                CONTINUE

            IF 目标 ID IN 情绪缓存:
                情绪缓存[目标 ID].意图标签 = 数据.行为意图标签
                情绪缓存[目标 ID].意图置信度 = 意图置信度
                情绪缓存[目标 ID].预测轨迹 = 数据.预测轨迹
            ELSE:
                情绪缓存[目标 ID] = {
                    意图标签: 数据.行为意图标签,
                    意图置信度: 意图置信度,
                    更新时间戳: NOW()
                }

        // 第 4 步：降级检测
        IF 降级检查计数器 >= 15:  // 连续 15 次低置信度 ≈ 3 秒
            IF internal_state != DEGRADED:
                SET internal_state = DEGRADED
                向所有查询方模块发送降级模式通知（原因=感知质量过低，仅提供保守推断）
                向 ad-03、ad-01 发送感知异常告警
        ELSE IF internal_state == DEGRADED AND 降级检查计数器 < 5:
            SET internal_state = NORMAL
            向所有查询方模块发送降级解除通知

        // 第 5 步：处理情绪意图查询请求
        IF 收到情绪意图查询请求 AND internal_state IN [NORMAL, DEGRADED]:
            目标 ID = 请求.目标 ID
            目标类别 = 请求.目标类别
            请求来源 = 请求.请求来源模块

            IF 请求来源 NOT IN 授权查询模块列表:
                向请求方返回查询拒绝通知（原因=未授权模块）
                CONTINUE

            SET internal_state = QUERYING

            // 5a. 查询缓存
            缓存数据 = 情绪缓存[目标 ID]

            IF 缓存数据 == NULL:
                // 无缓存数据，返回基于目标类别的保守推断
                保守结果 = 生成保守情绪推断(目标类别)
                向请求方返回综合结果
                SET internal_state = NORMAL
                CONTINUE

            // 5b. 检查数据时效
            数据时效 = NOW() - 缓存数据.更新时间戳
            IF 数据时效 > 缓存有效期:
                标记数据为“可能过时”

            // 5c. 组装情绪状态结果
            情绪结果 = {
                目标 ID: 目标 ID,
                情绪状态标签: 缓存数据.情绪状态标签 ?? "未知",
                情绪强度: 缓存数据.情绪强度 ?? 0.0,
                感知置信度: 缓存数据.感知置信度 ?? 0.0,
                数据时效: 数据时效,
                查询时间戳: NOW()
            }

            // 5d. 组装行为意图结果
            意图结果 = {
                目标 ID: 目标 ID,
                意图标签: 缓存数据.意图标签 ?? "未知",
                意图置信度: 缓存数据.意图置信度 ?? 0.0,
                预测行为描述: 生成意图描述(缓存数据.意图标签, 目标类别),
                预测时间窗口: "未来 3 秒",
                查询时间戳: NOW()
            }

            // 5e. 高风险目标（TTC < 3s）生成综合评估报告
            IF 请求.查询优先级 == "高" OR 目标类别 IN [第三类, 第四类]:
                综合报告 = {
                    目标 ID: 目标 ID,
                    情绪状态: 缓存数据.情绪状态标签,
                    意图推断: 缓存数据.意图标签,
                    风险关联建议: 生成风险建议(缓存数据.情绪状态标签, 缓存数据.意图标签, 目标类别),
                    综合置信度: MIN(缓存数据.感知置信度 ?? 0.0, 缓存数据.意图置信度 ?? 0.0)
                }
                向 ECC-05、ECC-03 发送综合情绪意图评估报告

            // 5f. 返回查询结果
            综合结果 = {情绪状态: 情绪结果, 行为意图: 意图结果}
            向请求方返回综合结果

            SET internal_state = NORMAL

        // 第 6 步：定期清理过期缓存
        IF 距上次清理 >= 10 秒:
            清除情绪缓存中更新时间戳 > 10 秒的条目（目标已离开感知范围）

        // 第 7 步：状态上报
        IF 距上次上报 >= 30 秒:
            向 ad-03、ad-01 上报运行状态

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 外部感知模块完全不可用 | 状态=DEGRADED，仅返回基于目标类别的保守推断，上报告警 | 外部模块恢复且置信度稳定 > 0.5 |
| 查询目标不在缓存中 | 返回保守推断，不报错 | 外部感知模块推送该目标数据后 |
| 缓存数据超过有效期 | 标记“可能过时”，仍返回数据但附加时效警告 | 新数据到达 |
| 情绪与意图数据冲突（如表情友好但轨迹激进） | 以意图推断（行为轨迹）为准，情绪数据标记为参考 | — |
| 同一目标频繁查询（> 10 次/秒） | 返回缓存数据，不重新计算 | 查询频率恢复正常 |
| 紧急熔断期间收到查询 | 返回“系统紧急熔断中” | 紧急解除 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 情绪意图数据独立于双漏斗记忆系统，不参与记忆的沉淀、筛选、晋升与遗忘机制。任何情绪数据不得写入漏斗一或漏斗二的存储分区 |
| S-02 | 情绪感知原始数据（面部表情特征向量、语音特征、姿态特征）为只读参考，不本地落盘存储，查询缓存最长保留 10 秒后自动清除 |
| S-03 | 情绪意图数据仅作为决策辅助参考，不可作为安全关键决策（如紧急制动、紧急避让）的唯一依据。安全关键决策须以世界模型 TTC、目标类别、法规约束为核心判定条件 |
| S-04 | 行人/非机动车情绪查询结果不得包含任何可用于身份识别的生物特征数据，仅输出抽象情绪标签与意图标签 |
| S-05 | 本模块不得主动采集或存储任何车内驾驶员/乘客的情绪数据（该功能属于 HRI 人机交互模块范畴，不在 AD 世界模型范围内） |
| S-06 | 外部感知模块不可用时，本模块进入降级模式，仅提供基于目标类别的保守推断，并明确标记数据来源=保守推断 |
| S-07 | 所有高风险目标（TTC < 3s）的情绪意图评估报告全量写入 ad-51 不可变日志 |
