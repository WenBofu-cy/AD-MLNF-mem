# ad-26-L4-长期层存储单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-26 |
| 模块名称 | L4 长期层存储单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 五层记忆层级存储 |
| 核心职责 | 存储跨场景可泛化复用的高阶驾驶技能与已验证的关键经验，占漏斗二总存储容量的 4.5%。接收来自 L3 晋升的条目，在留存满 90 日且满足极高重要度条件后进入 L5 核心层晋升候选。本层经验被视为“成熟的驾驶智慧”，享受最高等级遗忘保护，采用冷归档替代直接删除 |
| 依赖模块 | ad-24（L3 中期层存储单元，接收晋升条目）、ad-27（L4 长期层经验抽象提炼单元，周期性提炼通用规则）、ad-38（晋升双条件判定单元，下发 L4 晋升候选清单）、ad-40（遗忘阈值判定单元，下发 L4 遗忘候选清单）、ad-43（失败经验安全仲裁三道校验单元，消费失败经验条目） |
| 被依赖模块 | ad-27（消费 L4 经验进行抽象提炼）、ad-38（消费 L4 晋升候选条目）、ad-40（消费 L4 遗忘候选条目）、ad-50（记忆导入导出与脱敏共享单元，消费 L4 泛化经验） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收晋升条目，管理存储与流转 | 漏斗二激活，本层级初始化完成 |
| 存储将满 | `NEAR_FULL` | 存储占用率 > 85%，触发归并与冷归档 | 存储占用率监测 |
| 存储已满 | `FULL` | 存储占用率 > 95%，强制归档最早条目 | 存储占用率监测 |
| 维护中 | `MAINTENANCE` | 正在执行经验抽象提炼或冷归档操作 | 定时维护或存储已满触发 |
| 冻结只读 | `FROZEN` | 漏斗二冻结或紧急接管，暂停写入仅保留查询 | 驾驶模式切换或紧急熔断 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| L3 晋升条目 | Struct（条目 ID + 经验内容 + I 值 + 来源分槽号 + 子类标记 + 在 L3 的留存时长 + 复用计数 + 结果分类标签） | ad-24 L3 中期层存储单元（经 ad-39 搬运） | L3 条目通过晋升判定且安全仲裁通过后 | **高** |

| L4 遗忘候选清单 | List（条目 ID + 当前层级=L4 + I 值 + 遗忘原因） | ad-40 遗忘阈值判定单元 | 每个遗忘判定周期 | **高** |
| 抽象提炼结果 | Struct（提炼出的通用规则 + 源条目 ID 列表 + 规则置信度） | ad-27 L4 长期层经验抽象提炼单元 | 周期性提炼完成后 | 普通 |

| 全局调度指令 | Enum（冻结/激活/维护） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 存储完成回执 | Struct（条目 ID + 存储层级=L4 + 存储时间戳 + 当前 L4 占用率） | ad-24、ad-38 | 每次晋升条目成功存储后 | 普通 |
| 晋升执行结果 | Struct（条目 ID + 原层级=L4 + 目标层级=L5 + 执行结果） | ad-38、ad-39 | 晋升完成或失败后 | **高** |

| L4 经验数据（脱敏后） | Struct（泛化经验规则 + 源条目数 + 平均 I 值） | ad-27 L4 长期层经验抽象提炼单元 | ad-27 请求时或周期性推送 | 普通 |
| 可导出经验包 | Struct（脱敏经验规则集 + 导出时间戳 + 导出范围） | ad-50 记忆导入导出与脱敏共享单元 | ad-50 请求导出时 | 普通 |
| L4 状态快照 | Struct（占用率 + 条目总数 + 各分槽条目分布 + 平均留存时长 + 平均 I 值 + 已提炼规则数） | ad-03、各对应分槽 | 每 60 秒周期性上报 | 普通 |
| 存储异常告警 | Struct（异常类型 + 当前占用率 + 详情） | ad-03、ad-01 | 存储写入失败或数据校验异常时 | **高** |


## 核心处理逻辑

```
FUNCTION l4_long_term_storage_main_loop():
    SET internal_state = NORMAL
    初始化 L4 存储分区（占漏斗二总容量 4.5%，各分槽按成熟度动态分配）
    初始化条目索引表：
        条目 ID → {存储地址, 晋升时间戳, I 值, 来源分槽号, 子类标记, 复用计数, 结果分类标签, 是否已提炼}
    最大留存时间 = 90 日（2160 小时）
    L4 最低遗忘 I 阈值 = 0.20（L4 遗忘门槛极高）

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = FROZEN
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 第 2 步：接收 L3 晋升条目
        IF 收到 L3 晋升条目 AND internal_state IN [NORMAL, NEAR_FULL]:
            // 2a. 检查存储空间
            IF internal_state == FULL:
                立即触发紧急冷归档：
                    找出 L4 中留存时间最长的 10% 条目（跳过 L5 候选条目）
                    调用 ad-42 批量冷归档至 ad-49
                若归档后仍不足，上报告警
                返回存储失败（原因=L4 存储满）
                SET internal_state = NEAR_FULL

            新条目 = 晋升条目

            // 2b. L4 晋升奖励：I 值固化 + 标记为可信
            新条目.I 值 = MIN(新条目.I 值 + 0.05, 1.0)
            新条目.是否已提炼 = False
            新条目.L4 晋升时间 = NOW()
            新条目.回退次数 = 0

            // 2c. 结果分类标签最终检查
            IF 新条目.结果分类标签 == "策略失误":
                // 能到达 L4 的策略失误经验必须已通过三道安全仲裁
                // 在此做最终确认
                IF 新条目.安全仲裁通过标记 != True:
                    向 ad-24 返回晋升驳回（原因=缺少安全仲裁通过标记）
                    CONTINUE
            ELSE IF 新条目.结果分类标签 == "不可抗力场景":
                // 不可抗力在 L4 中永久标记，加速向 L5 晋升
                新条目.不可抗力标记 = True

            // 2d. 分配空间并写入
            存储地址 = 分配空间(新条目.大小)
            IF 分配失败:
                触发碎片整理 → 重试
                IF 仍失败:
                    返回存储失败回执
                    CONTINUE

            写入存储介质
            更新条目索引表
            向 ad-24、ad-38 返回存储完成回执

            // 2e. 更新占用率
            占用率 = 计算占用率()
            IF 占用率 > 0.95:
                SET internal_state = FULL
                上报告警
            ELSE IF 占用率 > 0.85 AND internal_state == NORMAL:
                SET internal_state = NEAR_FULL

        // 第 3 步：处理 L4 晋升候选（→ L5）
        IF 收到 ad-38 L4 晋升候选清单:
            FOR EACH 候选 IN 清单:
                条目 ID = 候选.条目 ID
                条目 = 索引表[条目 ID]

                // 3a. L5 晋升条件极严苛
                晋升条件满足 = True
                IF 条目.I 值 < 0.80:
                    晋升条件满足 = False
                    原因 = "I 值不满足 L5 最低 0.80"
                ELSE IF 条目.留存时长 < 90 日 AND 条目.不可抗力标记 != True:
                    晋升条件满足 = False
                    原因 = "留存时长不足 90 日"
                ELSE IF 条目.复用计数 < 10 AND 条目.不可抗力标记 != True:
                    晋升条件满足 = False
                    原因 = "复用次数不足 10 次"

                IF 晋升条件满足 AND L5 存储未满:
                    调用 ad-39 执行层级搬运（L4 → L5）
                    向 ad-38 返回晋升成功
                    从 L4 索引表中移除
                ELSE IF 晋升条件满足 AND L5 存储已满:
                    向 ad-38 返回晋升失败（原因=L5 存储满，L5 条目受最高保护不可清除）
                ELSE:
                    向 ad-38 返回晋升暂缓（原因=不满足晋升条件）

        // 第 4 步：处理 L4 遗忘候选（L4 遗忘为冷归档，绝不直接删除）
        IF 收到 ad-40 L4 遗忘候选清单:
            FOR EACH 候选 IN 清单:
                条目 ID = 候选.条目 ID
                条目 = 索引表[条目 ID]

                // L4 遗忘保护机制
                IF 条目.I 值 < L4 最低遗忘 I 阈值 AND 条目.不可抗力标记 != True:
                    // 执行冷归档，而非删除
                    调用 ad-42 冷归档至 ad-49（压缩存储）
                    向 ad-40 返回已归档（归档位置=ad-49）
                    从索引表中移除（但保留归档索引）
                ELSE:
                    向 ad-40 返回保留（不满足遗忘条件或不可抗力保护）

        // 第 5 步：处理晋升失败回退
        IF 收到晋升失败回退条目（从 L5 退回）:
            条目.回退次数++
            IF 条目.回退次数 >= 1:
                // L4 条目晋升 L5 失败 1 次即标记
                标记为“L5 晋升困难”
                // 降低 I 值 0.03，避免反复尝试
                条目.I 值 = MAX(条目.I 值 - 0.03, 0.20)
            条目继续保留在 L4

        // 第 6 步：定期维护——触发抽象提炼
        IF 距上次抽象提炼 >= 30 日 AND L4 条目数 > 10:
            SET internal_state = MAINTENANCE
            向 ad-27 发送提炼请求（含 L4 当前所有条目数据）
            WAIT ad-27 返回抽象提炼结果
            IF 提炼出通用规则:
                向 ad-27 返回确认
                将提炼出的规则在本地备份（规则索引表）
            SET internal_state = NORMAL

        // 第 7 步：定期维护——冷归档检查
        IF 占用率 > 85%:
            SET internal_state = MAINTENANCE
            // 主动冷归档 L4 中最旧的 5% 条目
            最旧条目列表 = 按晋升时间升序取前 5%
            FOR EACH 条目 IN 最旧条目列表:
                IF 条目.不可抗力标记 != True AND 条目.复用计数 < 5:
                    调用 ad-42 冷归档至 ad-49
            SET internal_state = NORMAL

        // 第 8 步：处理导出请求
        IF 收到 ad-50 导出请求:
            按请求范围筛选 L4 经验条目
            执行脱敏处理：剔除 GPS 坐标、行人/车辆特征、具体时间戳
            打包为可导出经验包
            向 ad-50 返回可导出经验包

        // 第 9 步：状态上报
        IF 距上次上报 >= 60 秒:
            向 ad-03 和各分槽上报 L4 状态快照

        SLEEP 200ms  // L4 操作频率低
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| L4 存储满且无法冷归档释放 | 暂停接收新条目，上报告警，等待手动扩容 | 扩容或手动归档完成 |
| L4 条目留存超 90 日但未满足 L5 晋升条件 | 继续保留在 L4，参与冷归档但不删除 | — |
| 不可抗力条目数量占比 > 30% | 上报告警（可能为异常），但不影响存储 | 占比回落 |
| L4 条目被 ad-27 提炼后原条目 | 原条目继续保留在 L4，提炼出的规则存储在规则索引表中 | — |
| 晋升 L5 失败回退次数 ≥ 1 | 标记“L5 晋升困难”，I 值降低 0.03，避免反复尝试 | — |
| 冷归档时 ad-49 不可用 | 条目继续保留在 L4，上报告警，等待 ad-49 恢复 | ad-49 恢复后补归档 |
| 冻结状态写入 | 拒绝写入，返回原因 | 激活指令 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | L4 遗忘执行冷归档（ad-49）而非直接删除，所有归档经验可追溯恢复 |
| S-02 | 不可抗力场景经验（不可抗力标记=True）在 L4 遗忘评估时绝对豁免，不可归档或删除 |
| S-03 | L5 晋升条件硬编码：I ≥ 0.80，留存 ≥ 90 日，复用 ≥ 10 次（不可抗力豁免时间和复用限制） |
| S-04 | 晋升 L5 失败回退 ≥ 1 次即标记“L5 晋升困难”，避免反复无效晋升尝试 |
| S-05 | 导出至 ad-50 的经验包须执行脱敏处理：剔除 GPS 坐标、行人/车辆特征、具体时间戳 |
| S-06 | 策略失误经验进入 L4 须持有安全仲裁通过标记，否则退回 L3 |
| S-07 | 所有写入、晋升、遗忘、归档、导出操作全量写入 ad-51 变更日志 |
