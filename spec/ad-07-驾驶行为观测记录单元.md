# ad-07-驾驶行为观测记录单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-07 |
| 模块名称 | 驾驶行为观测记录单元 |
| 所属分区 | 二、漏斗一：驾驶员画像漏斗 |
| 核心职责 | 持续观测并结构化记录驾驶员当前操作行为。从车辆 CAN 总线实时采集方向盘转角、油门开度、制动压力、转向灯状态等操控数据，打包为标准化行为观测条目，经 ad-06 隔离校验后写入对应子画像槽 |
| 依赖模块 | ad-06（子画像槽数据隔离管控单元，校验写入权限）、ad-02（漏斗一专属调度单元，获取当前活跃槽号） |
| 被依赖模块 | ad-08（上下文场景标记单元，消费行为观测条目）、ad-10（行为累积统计单元，消费行为观测条目） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 系统初始化完成，等待人工驾驶模式激活 | 系统上电完成，驾驶模式未确认 |
| 观测采集中 | `OBSERVING` | 正在持续采集驾驶行为数据并结构化写入 | 驾驶模式 = 人工驾驶，当前活跃槽已激活 |
| 暂停观测 | `PAUSED` | 漏斗一冻结或槽位未就绪，暂时停止采集 | 驾驶模式切换、紧急接管、活跃槽未激活 |
| 故障降级 | `DEGRADED` | CAN 总线部分信号中断，仅采集可用信号 | 任一 CAN 信号源超时或异常 |
| 完全停采 | `STOPPED` | CAN 总线完全中断或系统故障，停止全部采集 | 全部 CAN 信号源中断 |

## 核心处理逻辑

```
FUNCTION behavior_observation_main_loop():
    SET internal_state = IDLE
    初始化 CAN 总线订阅：方向盘转角、转角速率、油门、制动压力、制动开关、转向灯、车速、档位

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = STOPPED
            停止全部 CAN 数据采集
            丢弃未写入的缓存数据
            CONTINUE

        // 第 2 步：驾驶模式与槽位就绪检查
        IF 驾驶模式 != 人工驾驶:
            IF internal_state == OBSERVING:
                SET internal_state = PAUSED
            CONTINUE  // 漏斗一仅人工驾驶模式运行

        IF 当前活跃槽号 == 未激活:
            IF internal_state == OBSERVING:
                SET internal_state = PAUSED
            CONTINUE

        // 第 3 步：CAN 信号健康检查
        CAN_healthy = 检测各 CAN 信号源是否正常
        IF 全部 CAN 信号源中断:
            SET internal_state = STOPPED
            上报告警 TO ad-01
            CONTINUE
        ELSE IF 部分 CAN 信号源中断:
            SET internal_state = DEGRADED
            IF 中断时长 > 500ms:
                上报告警 TO ad-01（含异常信号名称 + 持续时长）
        ELSE:
            IF internal_state IN [DEGRADED, STOPPED]:
                // CAN 信号恢复
                IF internal_state == STOPPED:
                    SET internal_state = PAUSED  // 信号恢复后重新等待槽位就绪
                ELSE:
                    SET internal_state = OBSERVING

        // 第 4 步：采集驾驶行为数据
        IF internal_state IN [IDLE, PAUSED] AND 驾驶模式 == 人工驾驶 AND 当前活跃槽号已激活:
            SET internal_state = OBSERVING

        IF internal_state == OBSERVING:
            // 4a. 读取当前帧 CAN 数据
            采集方向盘转角、转角速率、油门开度、制动压力、制动状态、转向灯、车速、档位

            // 4b. 数据质量评估
            IF 本帧全部信号正常:
                数据质量标记 = "完整"
            ELSE IF 部分信号缺失但可用插值估算:
                执行线性插值填充缺失值
                数据质量标记 = "插值估算"
            ELSE:
                数据质量标记 = "部分缺失"
                // 缺失严重的帧仍记录，但标记质量

            // 4c. 行为类型初步推断
            基于当前帧数据与上一帧对比：
                IF 制动压力 > 0.1 MPa:
                    行为类型 = "制动"
                ELSE IF 油门开度变化率 > 20%/秒:
                    行为类型 = "加速"
                ELSE IF 油门开度变化率 < -20%/秒:
                    行为类型 = "减速"
                ELSE IF abs(方向盘转角) > 10° AND 转向灯 == 开启:
                    行为类型 = "转弯"
                ELSE IF abs(方向盘转角) > 10° AND 转向灯 == 关闭:
                    行为类型 = "变道"（可能未打灯）
                ELSE IF 车速 < 1 km/h AND 制动踏板 == 踩下:
                    行为类型 = "停车"
                ELSE IF 车速 < 1 km/h AND 油门开度 > 0:
                    行为类型 = "起步"
                ELSE:
                    行为类型 = "匀速巡航"

            // 4d. 打包结构化观测条目
            observation = {
                观测时间戳 = NOW(),
                方向盘转角, 转角速率, 油门开度, 制动压力,
                制动状态, 转向灯状态, 当前车速, 当前档位,
                行为类型初步推断, 数据质量标记
            }

            // 4e. 写入目标槽（经 ad-06 隔离校验）
            向 ad-06 发送写入请求（目标槽号 = 当前活跃槽号，数据 = observation）
            WAIT ad-06 返回放行/拦截信号
            IF 放行:
                写入目标槽存储分区
                向 ad-08 发送 observation 副本
                向 ad-10 发送 observation 副本
            ELSE:
                记录拦截日志
                // 拦截事件已由 ad-06 上报

        // 第 5 步：周期性状态上报
        IF 距上次上报 >= 5 秒:
            向 ad-01 上报观测状态（含采集频率、最近写入时间戳）

        SLEEP 10ms  // 对应 100Hz 采集频率
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| CAN 总线全部信号中断 | 停止采集，状态 STOPPED，上报告警 | 全部信号恢复后，重新等待槽位就绪 |
| CAN 总线部分信号中断 | 状态 DEGRADED，对缺失信号执行插值估算或标记缺失，超 500ms 上报告警 | 信号恢复后自动回到 OBSERVING |
| 驾驶模式切换为非人工驾驶 | 暂停采集，状态 PAUSED，保留未写入缓存数据（等待恢复） | 驾驶模式恢复人工驾驶 |
| 活跃槽未激活 | 暂停采集，状态 PAUSED | ad-02 激活目标槽 |
| ad-06 拦截写入请求 | 记录拦截日志，不强制重试（隔离违规可能是系统级问题） | 拦截原因消除后（如槽状态恢复） |
| 存储空间不足（目标槽配额 > 95%） | 继续采集但上报告警，数据写入可能失败时缓存至临时队列（最长保留 60 秒） | 空间释放后从缓存队列补写 |
| 紧急熔断 | 停止采集，丢弃未写入缓存，状态 STOPPED | 紧急事件解除 + ad-01 恢复指令 |
| 采集频率下降（实际 < 50Hz） | 上报告警，标记数据质量，不停止采集 | 系统负载降低后自动恢复 100Hz |

| CAN 总线 | 读 | 当前档位 | 只读 | 状态变化时 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 漏斗一数据编译期禁止接入自动驾驶决策链路。本模块输出的所有观测条目仅写入漏斗一内部，不向 ECC 大脑传输 |
| S-02 | CAN 总线数据为只读订阅，本模块不得向 CAN 总线写入任何数据，防止干扰车辆操控系统 |
| S-03 | 行为观测条目中的方向盘转角、油门、制动等原始操控数据不得外传至云端或 OTA 导出 |
| S-04 | 所有观测条目写入前须经 ad-06 隔离校验，校验未通过则丢弃该条目并记录日志 |
| S-05 | 紧急熔断时立即停止采集并丢弃未写入缓存，确保安全事件优先 |
| S-06 | CAN 信号中断时对缺失数据的插值估算必须标记“插值估算”，不可伪造为“完整”数据 |
| S-07 | 观测条目保留 UTC 时间戳（毫秒精度），所有条目时序不可篡改 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-07-01 | OBSERVING 状态，全部 CAN 信号正常，活跃槽已激活 | 一帧完整 CAN 数据（方向盘 5°、油门 30%、制动 0、车速 50km/h、档位 D） | 生成观测条目（行为类型=匀速巡航，数据质量=完整），向 ad-06 发送写入请求 |
| TC-07-02 | OBSERVING 状态，制动踏板踩下 | 制动压力 2.0 MPa，车速 60→30km/h | 生成观测条目（行为类型=制动），写入目标槽 |
| TC-07-03 | OBSERVING 状态，转向灯开启 + 方向盘转角 > 10° | 转向灯=左转，方向盘转角=25° | 生成观测条目（行为类型=转弯） |
| TC-07-04 | OBSERVING 状态，方向盘转角 > 10° 但转向灯关闭 | 方向盘转角=15°，转向灯=关闭 | 生成观测条目（行为类型=变道，可能未打灯） |
| TC-07-05 | OBSERVING 状态，驾驶模式切换为自动驾驶 | 驾驶模式信号 = 自动驾驶 | 暂停采集，状态 PAUSED |
| TC-07-06 | OBSERVING 状态，收到紧急熔断指令 | 紧急熔断信号 | 停止采集，状态 STOPPED，丢弃未写入缓存 |
| TC-07-07 | DEGRADED 状态，部分 CAN 信号中断 600ms | 制动压力信号超时 | 插值估算制动压力，标记“插值估算”，上报告警 |
| TC-07-08 | OBSERVING 状态，ad-06 返回拦截 | 写入请求被拦截（如槽处于维护状态） | 记录拦截日志，不强制重试 |
| TC-07-09 | OBSERVING 状态，存储空间 > 95% | 写入请求 | 继续采集，上报告警，数据写入可能进入临时缓存队列 |
| TC-07-10 | STOPPED 状态，CAN 信号全部恢复 | CAN 信号恢复正常 | 状态转为 PAUSED，等待槽位就绪后恢复 OBSERVING |
