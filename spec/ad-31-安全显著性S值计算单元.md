# ad-31-安全显著性S值计算单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-31 |
| 模块名称 | 安全显著性 S 值计算单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 三维重要度计算引擎 |
| 核心职责 | 从每个驾驶经验条目对应的场景数据中，提取并量化安全相关物理信号，计算安全显著性分值 S（0–1）。S 值代表该场景中涉及人身安全与碰撞风险的紧迫与严重程度，直接驱动重要度 I 值计算与 L5 直达写入判定 |
| 依赖模块 | ad-44（独立世界模型库，提供风险特征）、ad-36（综合重要度 I 值聚合计算单元，接收 S 值）、ad-35（三维权重系数配置单元，获取 α 权重） |
| 被依赖模块 | ad-36（消费 S 值参与 I 值计算）、ad-28（S≥0.9 时触发安全事件直达写入 L5） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收场景数据并计算 S 值 | 漏斗二激活，本单元初始化完成 |
| 批量计算中 | `BATCH` | 正在批量处理历史经验条目的 S 值回溯计算 | 权重配置变更或系统初始化回溯 |
| 信号丢失降级 | `DEGRADED` | 部分安全信号源不可用，使用保守估算 | 感知模块或 CAN 信号中断 |
| 暂停服务 | `PAUSED` | 漏斗二冻结或紧急熔断 | 驾驶模式切换或紧急事件 |

## S 值计算模型

### 一、核心公式

```
S = CLAMP( S_base + S_event + S_EnvMod , 0.0 , 1.0 )
```

**S_base**：场景基础风险得分（0.0–0.5），由当前交通态势决定
**S_event**：离散危险事件触发增量（0.0–0.8），由突发危险信号驱动
**S_EnvMod**：环境风险调节因子（-0.1–+0.2），由天气、光照、路面状态修正

### 二、S_base 基础风险计算（0.0–0.5）

| 风险因子 | 计算方式 | 得分范围 |
|----------|----------|:---:|
| 碰撞时间 TTC | TTC > 4s: 0.0；2s < TTC ≤ 4s: 0.1；1s < TTC ≤ 2s: 0.2；TTC ≤ 1s: 0.3 | 0.0–0.3 |
| 跟车时距 THW | THW > 2.5s: 0.0；1.5s < THW ≤ 2.5s: 0.05；THW ≤ 1.5s: 0.1 | 0.0–0.1 |
| 目标风险类别 | 第四类（行人）: 0.15；第三类（动物）: 0.1；第二类（机动车）: 0.05；第一类（静态）: 0.0 | 0.0–0.15 |
| 车速风险 | 车速 / 路段限速 > 1.0: +0.1；> 1.2: +0.15；< 0.5: -0.05 | -0.05–+0.15 |

> S_base 取各因子累加值，上限 0.5。

### 三、S_event 离散危险事件触发增量（0.0–0.8）

| 事件 | 触发条件 | 增量 | 优先级 |
|------|----------|:---:|:---:|
| 紧急制动（AEB 触发） | 制动减速度 > 7m/s² 或 AEB 激活 | +0.6 | **最高** |
| 碰撞不可避免 | TTC < 0.5s 且无有效避让路径 | +0.8（直达 L5） | **最高** |
| ESC/ABS 介入 | 车身稳定或防抱死系统激活 | +0.4 | 高 |
| 行人/动物突然侵入 | 目标出现 TTC < 2s | +0.5 | 高 |
| 严重侧滑风险 | 侧向加速度 > 0.7g | +0.5 | 高 |
| 特种车辆临近 | 检测到应急车辆声光 | +0.3 | 中 |
| 交通违规风险 | 闯红灯/超速 > 20% | +0.2 | 中 |
| 盲区危险 | 变道时盲区有来车 | +0.3 | 中 |

> 多个事件并发时，取优先级最高的事件增量，次高事件增量 × 0.3 叠加，上限 0.8。

### 四、S_EnvMod 环境风险调节（-0.1–+0.2）

| 环境条件 | 调节量 | 说明 |
|----------|:---:|------|
| 路面湿滑/积雪/结冰 | +0.15 | 摩擦系数显著降低 |
| 能见度 < 100m（浓雾/暴雨） | +0.1 | 感知能力受限 |
| 夜间 + 无路灯 | +0.05 | 人类与动物风险增加 |
| 非铺装路面 | +0.1 | 车辆稳定性下降 |
| 晴天 + 干燥路面 + 日间 | -0.05 | 理想条件略降低敏感度 |
| 封闭园区/低速蠕行 | -0.1 | 极低风险场景适当降低 |

### 五、关键判定规则

| 规则 | 条件 | 动作 |
|------|------|------|
| L5 直达写入 | S ≥ 0.9 | 经验条目直接写入 L5 核心层，永久锁定，绕过常规晋升流程 |
| 高安全显著性标记 | S ≥ 0.7 | 经验条目在 I 值计算中 α 权重生效，加速晋升 |
| 安全信号丢失保守处理 | TTC/THW/目标列表任一缺失 | S_base 取历史最大值或 0.3（取高者），标记数据质量=降级 |


## 核心处理逻辑

```
FUNCTION s_value_calculation_main_loop():
    SET internal_state = NORMAL
    加载车辆动力学极限参数
    S 值缓存 = {}  // 条目 ID → S 值（用于回溯）

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 第 2 步：接收场景数据并计算 S 值
        IF 收到经验场景数据包 AND internal_state IN [NORMAL, BATCH]:
            条目 ID = 数据包.条目 ID
            场景时间窗口 = 数据包.场景时间窗口

            // 2a. 检查安全关键信号完整性
            信号状态 = 检查信号完整性(TTC, THW, 目标列表, 车速)
            IF 信号状态.任一关键信号缺失:
                IF 缺失时长 > 500ms:
                    SET internal_state = DEGRADED
                    向 ad-03、ad-01 发送信号丢失告警
                // 保守处理
                S_base = MAX(0.3, 历史 S_base 最大值)
                数据质量 = "降级"
            ELSE:
                IF internal_state == DEGRADED:
                    SET internal_state = NORMAL

                // 2b. 计算 S_base 基础风险
                S_base = 0.0
                S_base += 计算 TTC 得分(世界模型.TTC)
                S_base += 计算 THW 得分(世界模型.THW)
                S_base += 目标风险类别得分(世界模型.核心风险目标类别)
                S_base += 车速风险得分(车速, 路段限速)
                S_base = MIN(S_base, 0.5)
                数据质量 = "完整"

            // 2c. 计算 S_event 离散事件增量
            S_event = 0.0
            事件列表 = 检测危险事件(世界模型风险特征, CAN数据)
            IF 事件列表不为空:
                按优先级降序排列 事件列表
                主事件 = 事件列表[0]
                S_event = 主事件.增量
                IF 事件列表.length > 1:
                    次事件 = 事件列表[1]
                    S_event += 次事件.增量 × 0.3
                S_event = MIN(S_event, 0.8)

            // 2d. 计算 S_EnvMod 环境调节
            S_EnvMod = 0.0
            S_EnvMod += 路面状态调节(场景标签.路面状态)
            S_EnvMod += 能见度调节(场景标签.天气, 场景标签.光照)
            S_EnvMod += 道路类型调节(场景标签.路面类型)
            S_EnvMod = CLAMP(S_EnvMod, -0.1, 0.2)

            // 2e. 综合 S 值
            S = CLAMP(S_base + S_event + S_EnvMod, 0.0, 1.0)
            S 值缓存[条目 ID] = S

            // 2f. 输出计算结果
            S 值结果 = {
                条目 ID: 条目 ID,
                S 值: S,
                各因子得分: { S_base, S_event, S_EnvMod },
                计算时间戳: NOW(),
                数据质量标记: 数据质量
            }
            向 ad-36 发送 S 值计算结果

            // 2g. L5 直达写入判定
            IF S >= 0.9:
                L5 直达信号 = {
                    条目 ID: 条目 ID,
                    S 值: S,
                    事件快照: 主事件.描述,
                    直达原因: IF 主事件.类型 == "碰撞不可避免" THEN "S≥0.9-碰撞不可避免"
                               ELSE "S≥0.9-安全事件"
                }
                向 ad-28 发送 L5 直达写入触发信号
                向 ad-51 写入安全直达事件日志

        // 第 3 步：处理权重配置变更（触发回溯计算）
        IF 收到 ad-35 权重配置变更通知:
            SET internal_state = BATCH
            向 ad-14 请求近期（近 1 小时）所有经验条目 ID
            FOR EACH 条目 ID IN 近期条目列表:
                重新获取场景数据并计算 S 值
                更新 S 值缓存
                向 ad-36 推送更新后的 S 值
            SET internal_state = NORMAL

        // 第 4 步：周期性状态上报
        IF 距上次上报 >= 30 秒:
            向 ad-03、ad-01 上报计算状态

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| TTC 信号丢失 | S_base 取 0.3 或历史最大值，标记降级，上报告警 | 信号恢复 |
| 多个危险事件并发 | 主事件全量 + 次事件 × 0.3 叠加，S_event ≤ 0.8 | — |
| S 值计算溢出（>1.0） | CLAMP 至 1.0 | — |
| 权重变更触发批量回溯 | 状态切换 BATCH，处理完恢复 | 回溯完成 |
| 车辆动力学参数未加载 | 使用保守默认值，上报告警 | 参数加载完成 |
| 场景数据包缺失关键字段 | 跳过该条目计算，标记异常，返回空 S 值 | — |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | S ≥ 0.9 强制触发 L5 直达写入，绕过常规晋升与仲裁流程，直接物理锁定 |
| S-02 | 安全关键信号（TTC/THW/目标列表）丢失时采用保守估算（取高值），宁可高估不可低估 |
| S-03 | S 值计算基于实时世界模型数据，不受历史经验或驾驶员偏好影响 |
| S-04 | 碰撞不可避免（TTC < 0.5s 且无有效避让路径）S 值直接置为 0.8 起评 |
| S-05 | 所有 S ≥ 0.7 的计算结果全量写入 ad-51 变更日志 |
