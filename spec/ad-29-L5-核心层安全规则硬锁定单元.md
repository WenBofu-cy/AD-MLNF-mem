# ad-29-L5-核心层安全规则硬锁定单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-29 |
| 模块名称 | L5 核心层安全规则硬锁定单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 五层记忆层级存储 |
| 核心职责 | 编译期实施 L5 存储分区硬件写保护，管控 L5 写入权限的临时解除与恢复。任何对 L5 存储分区的写入、修改或删除操作，必须经过本单元的权限校验与临时解锁流程。写入完成后自动恢复写保护。物理层面防止非授权访问与数据篡改 |
| 依赖模块 | ad-28（L5 核心层存储单元，接收写保护状态报告与写入请求）、ad-30（L5 核心层防篡改与只读管控单元，提供安全令牌验证） |
| 被依赖模块 | ad-28（消费写入权限的临时解除与恢复）、ad-30（消费写保护状态信息用于查询管控） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 已锁定 | `LOCKED` | L5 分区写保护已激活，禁止一切写入/修改/删除 | 系统初始化完成，或写入完成后自动恢复 |
| 临时解锁 | `UNLOCKED_TEMP` | 已验证的安全写入请求，临时解除写保护，允许单次写入 | 收到合法写入请求且令牌验证通过 |
| 锁定异常 | `LOCK_FAILED` | 写保护机制可能出现故障，立即封锁所有写入请求并告警 | 硬件写保护控制器无响应或返回异常 |
| 硬件维护 | `HARDWARE_MAINT` | 存储硬件正在执行维护，暂停一切读写操作 | 系统检测到坏块或执行存储校验时 |

| 安全令牌验证结果 | Struct（令牌 + 验证结论：有效/无效 + 权限级别） | ad-30 L5 核心层防篡改与只读管控单元 | 每次权限校验后返回 | **高** |
| 硬件写保护控制器状态 | Struct（写保护开关状态 + 控制器健康度 + 硬件版本） | 硬件写保护控制器 | 周期性，每 1 秒 | 普通 |

## 核心处理逻辑

```
FUNCTION l5_hardware_lock_main_loop():
    SET internal_state = LOCKED
    初始化硬件写保护控制器接口（GPIO / SPI / 专用安全芯片）
    激活写保护：向控制器发送 LOCK 指令
    临时解锁超时 = 2000ms（默认，写操作必须在 2 秒内完成）
    写入请求队列 = []
    last_hw_check_time = 0

    WHILE 系统运行中:
        // 第 1 步：最高优先级——系统紧急熔断
        IF 收到 ad-01 紧急熔断指令:
            IF internal_state == UNLOCKED_TEMP:
                立即向硬件控制器发送 FORCE_LOCK 指令
                中止当前写入操作（不等待写入完成）
                LOG 紧急中断事件 TO ad-51
            SET internal_state = LOCKED
            CONTINUE

        // 第 2 步：定期检查硬件控制器健康状态
        IF NOW() - last_hw_check_time >= 1 秒:
            last_hw_check_time = NOW()
            硬件状态 = 读取硬件写保护控制器状态()

            IF 硬件状态 == 无响应 OR 硬件状态 == 异常:
                IF internal_state != LOCK_FAILED:
                    SET internal_state = LOCK_FAILED
                    向 ad-28、ad-03、ad-01 发送异常锁定告警
                    强制封锁所有写入请求队列
                CONTINUE
            ELSE IF internal_state == LOCK_FAILED AND 硬件状态 == 正常:
                SET internal_state = LOCKED
                向 ad-01 发送恢复通知

            // 上报写保护状态
            向 ad-28、ad-03、ad-01 发送写保护状态报告

        // 第 3 步：处理存储介质健康报告
        IF 收到存储介质健康报告:
            IF 存储介质.坏块数 > 安全阈值 OR 存储介质.读写错误率 > 安全阈值:
                向 ad-28、ad-03 发送硬件维护通知
                IF internal_state == UNLOCKED_TEMP:
                    等待当前写入完成，然后进入 HARDWARE_MAINT
                ELSE:
                    SET internal_state = HARDWARE_MAINT

        // 第 4 步：处理写入权限临时解除请求
        IF 收到 ad-28 写入权限临时解除请求:
            IF internal_state == LOCK_FAILED:
                向 ad-28 返回权限拒绝通知（原因=写保护控制器异常）
                CONTINUE
            IF internal_state == HARDWARE_MAINT:
                向 ad-28 返回权限拒绝通知（原因=硬件维护中）
                CONTINUE
            IF internal_state == UNLOCKED_TEMP:
                // 当前已有写入操作在进行中，加入等待队列
                写入请求队列.append(请求)
                CONTINUE

            // 4a. 验证安全令牌
            向 ad-30 发送安全令牌验证请求（含请求中的令牌）
            WAIT ad-30 返回验证结果
            IF 验证结果 == 无效 OR 权限级别不足:
                向 ad-28 返回权限拒绝通知
                向 ad-51 写入权限拒绝日志
                CONTINUE

            // 4b. 校验请求合法性
            IF 请求.操作类型 NOT IN [晋升写入, 安全直达写入, 人工锁定写入]:
                向 ad-28 返回权限拒绝通知（原因=非法操作类型）
                CONTINUE
            IF 请求.写入数据大小 > L5 剩余容量:
                向 ad-28 返回权限拒绝通知（原因=写入数据超限）
                CONTINUE

            // 4c. 解除写保护
            向硬件写保护控制器发送 UNLOCK 指令（带超时参数）
            WAIT 控制器返回 UNLOCK 确认
            IF 控制器返回失败 OR 超时:
                向 ad-28 返回权限拒绝通知（原因=写保护解除失败）
                SET internal_state = LOCK_FAILED（若连续失败 3 次）
                CONTINUE

            // 4d. 解除成功，通知 ad-28 可以写入
            SET internal_state = UNLOCKED_TEMP
            启动临时解锁计时器（超时 = 临时解锁超时）
            向 ad-28 发送写保护解除确认（含超时倒计时）

        // 第 5 步：处理写入完成确认信号
        IF 收到 ad-28 写入完成确认信号 AND internal_state == UNLOCKED_TEMP:
            // 5a. 立即向硬件控制器发送 LOCK 指令
            向硬件写保护控制器发送 LOCK 指令
            WAIT 控制器返回 LOCK 确认
            IF 控制器返回失败:
                重试 3 次
                IF 仍失败:
                    SET internal_state = LOCK_FAILED
                    向 ad-28、ad-03、ad-01 发送异常锁定告警
                    CONTINUE

            // 5b. 恢复锁定状态
            SET internal_state = LOCKED
            停止临时解锁计时器
            向 ad-28 发送写保护恢复确认

            // 5c. 处理等待队列中的下一个写入请求
            IF 写入请求队列不为空:
                下一个请求 = 写入请求队列.出队()
                重新触发该请求的处理（回到第 4 步）

        // 第 6 步：临时解锁超时检查
        IF internal_state == UNLOCKED_TEMP AND 临时解锁计时器已超时:
            // 写入操作超时未完成，强制锁定
            向硬件写保护控制器发送 FORCE_LOCK 指令
            SET internal_state = LOCKED
            向 ad-28 发送写入超时中止通知
            向 ad-51 写入超时事件日志
            上报告警 TO ad-01

        // 第 7 步：硬件维护状态检查
        IF internal_state == HARDWARE_MAINT:
            // 等待硬件维护完成
            IF 收到存储介质健康报告 AND 健康报告.状态 == 正常:
                SET internal_state = LOCKED
                向 ad-28、ad-03 发送硬件维护完成通知

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 硬件写保护控制器无响应 | 状态=LOCK_FAILED，拒绝一切写入请求，上报告警 | 控制器恢复响应，连续 3 次正常 |
| 写保护解除指令连续失败 3 次 | 状态=LOCK_FAILED，封锁所有写入，上报告警 | 硬件维修或更换 |
| 写入操作超过 2 秒超时未完成 | 强制锁定，中止写入，通知 ad-28，上报告警 | 重新发起写入请求 |
| 系统紧急熔断时正好处于临时解锁 | 立即强制锁定，不等写入完成，L5 数据安全优先 | 紧急解除后重新评估 L5 一致性 |
| 安全令牌验证超时（ad-30 无响应） | 拒绝本次写入，不清除队列，上报告警 | ad-30 恢复 |
| 存储介质出现坏块 | 进入 HARDWARE_MAINT，暂停写入，保留只读，启用镜像 | 坏块被替换或隔离 |
| 多个写入请求并发到达 | 请求排队，先到先服务，队首处理完毕自动处理下一个 | — |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | L5 存储分区写保护由硬件写保护控制器实现，编译期固化，不可通过运行时软件指令直接修改写保护状态 |
| S-02 | 临时解锁窗口最大 2 秒，超时后硬件自动强制锁定，确保写入异常时数据仍受保护 |
| S-03 | 任何绕过本单元的 L5 存储分区直接写入尝试，在物理层面被硬件写保护控制器拦截并告警 |
| S-04 | 硬件写保护控制器须选用独立于主 CPU 的安全芯片或专用存储控制器，不可与通用 GPIO 共用 |
| S-05 | 写入权限临时解除须同时验证安全令牌、操作类型合法性、剩余容量，三项缺一不可 |
| S-06 | 系统紧急熔断时，强制锁定优先于一切写入操作，已写入一半的数据作废 |
| S-07 | 所有写保护解除、恢复、拒绝、强制锁定事件全量写入 ad-51 不可变日志 |
| S-08 | 存储介质出现坏块时，优先保护已有数据，暂停新写入，利用镜像分区保障数据完整性 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-29-01 | LOCKED，控制器正常，令牌有效 | ad-28 写入请求（晋升写入，令牌有效，数据 2KB） | 令牌验证通过 → UNLOCK → 返回解除确认 → 等待写入完成 → 恢复 LOCK |
| TC-29-02 | LOCKED，令牌无效 | ad-28 写入请求（令牌无效） | 令牌验证失败 → 返回权限拒绝通知 → 记录日志 → 保持 LOCKED |
| TC-29-03 | UNLOCKED_TEMP，写入完成 | ad-28 写入完成确认（数据校验和正确） | 向控制器发送 LOCK → 恢复 LOCKED → 返回恢复确认 |
| TC-29-04 | UNLOCKED_TEMP，超过 2 秒未收到写入完成 | 计时器超时 | 强制锁定 → 通知 ad-28 写入超时中止 → 上报告警 |
| TC-29-05 | LOCKED，系统紧急熔断指令 | 紧急熔断信号 | 若当前为 UNLOCKED_TEMP → 立即强制锁定，不等写入完成 → 状态 LOCKED |
| TC-29-06 | 硬件控制器连续 3 次 UNLOCK 失败 | 第 4 次写入请求 | 状态=LOCK_FAILED → 拒绝一切写入 → 上报告警 |
| TC-29-07 | LOCK_FAILED，硬件控制器恢复 | 硬件控制器状态=正常 | 状态恢复 LOCKED → 向 ad-01 发送恢复通知 |
| TC-29-08 | LOCKED，存储介质报告坏块超阈值 | 存储介质健康报告 | 进入 HARDWARE_MAINT → 暂停写入 → 保留只读 |
| TC-29-09 | HARDWARE_MAINT，存储介质恢复 | 存储介质健康报告=正常 | 恢复 LOCKED → 通知 ad-28、ad-03 |
| TC-29-10 | 多个写入请求并发到达（3 个） | 请求 1、请求 2、请求 3 几乎同时到达 | 请求 1 处理中，请求 2、3 排队；请求 1 完成后自动处理请求 2 |
| TC-29-11 | LOCKED，写入请求操作类型为“删除” | ad-28 写入请求（操作类型=删除） | 非法操作类型 → 返回权限拒绝通知 |
