# ad-06-子画像槽数据隔离管控单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-06 |
| 模块名称 | 子画像槽数据隔离管控单元 |
| 所属分区 | 二、漏斗一：驾驶员画像漏斗 |
| 核心职责 | 编译期强制实施子画像槽间物理存储隔离，管控所有跨槽数据访问请求的放行与拦截，确保漏斗一数据绝对不泄露至自动驾驶决策链路 |
| 依赖模块 | ad-02（漏斗一专属调度单元，接收槽位激活/冻结指令）、ad-05（子画像槽创建与初始化单元，接收新槽注册） |
| 被依赖模块 | ad-07（驾驶行为观测记录单元，须经本模块校验后方可写入目标槽）、ad-11（驾驶辅助提醒生成单元，须经本模块校验后方可读取目标槽数据）、ad-12（临时画像槽自动清除单元，须经本模块校验后方可执行擦除） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 所有槽位访问校验规则正常执行 | 系统初始化完成 |
| 紧急只读 | `EMERGENCY_RO` | 安全事件触发，全部子画像槽强制只读，拦截一切写入请求 | 收到 ad-01 紧急熔断指令 |
| 全部冻结 | `ALL_FROZEN` | 漏斗一全部子画像槽暂停写入，仅允许只读查询 | 驾驶模式切换为非人工驾驶 |
| 单槽维护 | `SLOT_MAINTENANCE` | 指定子画像槽正在被擦除或修复，该槽暂停读写 | 收到 ad-12 擦除指令或槽数据校验失败 |
| 隔离规则更新中 | `RULE_UPDATING` | 新的子画像槽注册，正在更新隔离规则表 | 收到 ad-05 新槽注册通知 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 数据写入请求 | Struct（请求来源模块 + 目标槽号 + 数据类型 + 数据内容） | ad-07 驾驶行为观测记录单元 | 每次驾驶行为观测完成后 | **高** |
| 数据读取请求 | Struct（请求来源模块 + 源槽号 + 查询条件） | ad-11 驾驶辅助提醒生成单元 | 每次生成辅助提醒前 | **高** |
| 数据擦除请求 | Struct（请求来源模块 + 目标槽号 + 擦除范围） | ad-12 临时画像槽自动清除单元 | 临时槽到期或一次性行程结束时 | **高** |
| 跨槽访问请求 | Struct（请求来源模块 + 源槽号 + 目标槽号 + 操作类型） | 任意模块 | 任何尝试跨槽访问时 | **高** |
| 紧急熔断指令 | Enum（全局熔断） | ad-01 总控漏斗 F₀ | 安全急停或碰撞事件时 | **紧急** |
| 漏斗冻结指令 | Enum（全部冻结） | ad-01 总控漏斗 F₀ | 驾驶模式切换时 | **高** |
| 新槽注册通知 | Struct（新槽号 + 存储分区指针 + 访问权限标记） | ad-05 子画像槽创建与初始化单元 | 新子画像槽创建完成时 | **高** |
| 槽位激活/冻结指令 | Struct（目标槽号 + 操作类型） | ad-02 漏斗一专属调度单元 | 子画像槽生命周期变更时 | **高** |

## 核心处理逻辑

```
FUNCTION isolation_gate_main_loop():
    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断
        IF 收到 ad-01 紧急熔断指令:
            SET internal_state = EMERGENCY_RO
            拦截一切写入与擦除请求（只读查询仍可放行）
            LOG 熔断事件 TO ad-51
            CONTINUE

        // 第 2 步：漏斗冻结处理
        IF 收到 ad-01 漏斗冻结指令:
            SET internal_state = ALL_FROZEN
            拦截全部写入请求，只读查询放行
            CONTINUE

        // 第 3 步：新槽注册——更新隔离规则表
        IF 收到 ad-05 新槽注册通知:
            SET internal_state = RULE_UPDATING
            在隔离规则表中新增条目：
                - 槽号 = 新槽号
                - 存储分区基址 = 通知中的指针
                - 存储分区大小 = 通知中的大小
                - 访问白名单模块 = [ad-07, ad-10, ad-11]（仅漏斗一内部模块）
                - 跨槽访问 = 禁止
                - 自动驾驶模块访问 = 编译期禁止
            SET internal_state = NORMAL
            CONTINUE

        // 第 4 步：处理数据读写擦除请求
        IF 收到数据写入请求:
            result = validate_request(请求, "写入")
            IF result == PASS:
                放行，返回写入放行信号 TO ad-07
            ELSE:
                拦截，返回写入拦截信号 TO ad-07
                上报告警 TO ad-01 + 写入安全日志 TO ad-51

        ELSE IF 收到数据读取请求:
            result = validate_request(请求, "读取")
            IF result == PASS:
                放行，返回读取放行信号 TO ad-11
            ELSE:
                拦截，返回读取拦截信号 TO ad-11
                上报告警 TO ad-01 + 写入安全日志 TO ad-51

        ELSE IF 收到数据擦除请求:
            result = validate_request(请求, "擦除")
            IF result == PASS:
                IF 当前状态 == SLOT_MAINTENANCE AND 目标槽号 == 维护中的槽号:
                    放行
                ELSE IF 当前状态 == NORMAL:
                    放行
                ELSE:
                    拦截（紧急/冻结状态禁止擦除）
            ELSE:
                拦截

        // 第 5 步：周期性状态上报
        IF 距上次上报 >= 5 秒:
            向 ad-01 上报运行状态与近期拦截统计

        SLEEP 50ms  // 隔离校验须快速响应


FUNCTION validate_request(request, operation_type):
    // 校验规则按优先级从高到低排列

    // 规则 0：紧急熔断状态仅放行只读查询
    IF internal_state == EMERGENCY_RO:
        IF operation_type != "读取":
            RETURN FAIL, "紧急熔断状态，禁止非只读操作"

    // 规则 0b：全部冻结状态禁止写入
    IF internal_state == ALL_FROZEN:
        IF operation_type == "写入" OR operation_type == "擦除":
            RETURN FAIL, "漏斗一全部冻结，禁止写入和擦除操作"

    // 规则 1：编译期禁止自动驾驶决策模块访问漏斗一
    IF request.source_module IN [自动驾驶决策相关模块列表]:
        RETURN FAIL, "编译期禁止：漏斗一数据不可接入自动驾驶决策链路"

    // 规则 2：校验目标槽号是否存在
    IF request.target_slot NOT IN 隔离规则表:
        RETURN FAIL, "目标槽号不存在"

    // 规则 3：跨槽访问检测
    IF operation_type == "读取" AND request.source_slot != request.target_slot:
        // 任何跨槽读取均拦截
        RETURN FAIL, "跨槽访问禁止：子画像槽间物理隔离"

    // 规则 4：槽位维护状态检查
    IF 隔离规则表[request.target_slot].status == "MAINTENANCE":
        RETURN FAIL, "目标槽处于维护状态，暂停访问"

    // 规则 5：来源模块白名单校验
    IF request.source_module NOT IN 隔离规则表[request.target_slot].allowed_modules:
        RETURN FAIL, "来源模块不在目标槽访问白名单中"

    // 全部校验通过
    RETURN PASS
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 自动驾驶决策模块尝试访问漏斗一数据 | 编译期强制拦截，返回拦截原因码，上报告警，写入安全日志。此拦截不可被任何运行时配置绕过 | — |
| 跨槽读取请求（槽 A 读槽 B 的数据） | 拦截，原因码=“子画像槽物理隔离”，上报告警 | — |
| 来源模块不在目标槽白名单中 | 拦截，原因码=“未授权访问”，上报告警 | 仅当该模块经安全审计后被添加到白名单（须 OTA 更新） |
| 目标槽号不存在 | 拦截，原因码=“无效槽号” | — |
| 紧急熔断状态下收到写入请求 | 拦截，仅放行只读查询 | 紧急事件解除且 ad-01 下发恢复指令 |
| 全部冻结状态下收到写入请求 | 拦截，仅放行只读查询 | 驾驶模式恢复人工驾驶 |
| 目标槽处于维护状态 | 拦截该槽所有请求，其他槽不受影响 | 维护完成且槽状态恢复 ACTIVE |
| 隔离规则表损坏或丢失 | 标记全部槽为 UNKNOWN，全部请求拦截，触发全局告警 | 从冗余备份恢复规则表 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| MemoryBus | 读 | 数据写入请求（来自 ad-07） | 只读 | 每次驾驶行为观测后 |
| MemoryBus | 读 | 数据读取请求（来自 ad-11） | 只读 | 每次生成辅助提醒前 |
| MemoryBus | 读 | 数据擦除请求（来自 ad-12） | 只读 | 临时槽到期或一次性行程结束 |
| MemoryBus | 读 | 紧急熔断指令（来自 ad-01） | 只读 | 最高优先级 |
| MemoryBus | 读 | 漏斗冻结指令（来自 ad-01） | 只读 | 驾驶模式切换时 |
| MemoryBus | 读 | 新槽注册通知（来自 ad-05） | 只读 | 新槽创建完成时 |
| MemoryBus | 读 | 槽位激活/冻结指令（来自 ad-02） | 只读 | 槽生命周期变更时 |
| MemoryBus | 写 | 写入放行/拦截信号 | 专属写入 | 向 ad-07 返回 |
| MemoryBus | 写 | 读取放行/拦截信号 | 专属写入 | 向 ad-11 返回 |
| MemoryBus | 写 | 擦除放行信号 | 专属写入 | 向 ad-12 返回 |
| MemoryBus | 写 | 隔离违规告警 | 专属写入 | 向 ad-01、ad-51 上报 |
| MemoryBus | 写 | 运行状态上报 | 周期性写入 | 每 5 秒向 ad-01 上报 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 漏斗一数据编译期禁止接入自动驾驶决策链路。此规则以编译期常量形式硬编码，不可通过运行时配置、OTA 或用户设置绕过 |
| S-02 | 子画像槽间绝对物理存储隔离。任何跨槽读取请求（即使同属漏斗一）均拦截，无一例外 |
| S-03 | 数据访问白名单为编译期常量，仅包含漏斗一内部模块（ad-07、ad-10、ad-11），其他模块一律禁止 |
| S-04 | 紧急熔断状态下仅放行只读查询，任何写入或擦除操作均拦截 |
| S-05 | 全部冻结状态下仅放行只读查询，写入和擦除均拦截 |
| S-06 | 每一次拦截事件均写入 ad-51 不可变安全日志，含违规类型、来源模块、目标槽号、时间戳 |
| S-07 | 隔离规则表存储于本地安全分区，定期（每 60 秒）自动备份至冗余分区，防止损坏 |

| TC-06-02 | NORMAL 状态，ad-11 请求读取槽 A | 读取请求（来源=ad-11，源槽=槽 A） | 放行，返回读取放行信号 |

| TC-06-04 | NORMAL 状态，自动驾驶决策模块尝试读取槽 A | 读取请求（来源=自动驾驶决策模块） | 编译期拦截，原因码=“禁止接入自动驾驶决策链路” |
| TC-06-05 | EMERGENCY_RO 状态 | 写入请求（来源=ad-07，目标槽=槽 A） | 拦截，仅放行只读查询 |
| TC-06-06 | ALL_FROZEN 状态 | 写入请求（来源=ad-07） | 拦截，原因码=“漏斗一全部冻结” |
| TC-06-07 | NORMAL 状态，未注册模块请求访问 | 读取请求（来源=未授权模块） | 拦截，原因码=“未授权访问” |
| TC-06-08 | NORMAL 状态，目标槽号不存在 | 写入请求（目标槽=不存在的槽号） | 拦截，原因码=“无效槽号” |
| TC-06-09 | ad-05 注册新槽 C | 新槽注册通知（槽号=C，存储分区指针） | 更新隔离规则表，返回确认，状态恢复 NORMAL |
| TC-06-10 | 目标槽处于 MAINTENANCE 状态 | 写入请求（目标槽=维护中的槽） | 拦截，原因码=“目标槽处于维护状态” |
