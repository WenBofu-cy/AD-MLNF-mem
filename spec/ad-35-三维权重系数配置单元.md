# ad-35-三维权重系数配置单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-35 |
| 模块名称 | 三维权重系数配置单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 三维重要度计算引擎 |
| 核心职责 | 存储、管理与分发三维重要度驱动公式 `I = I₀ + α·S + β·V + γ·C` 中的权重系数 α、β、γ，以及各场景分槽的专属晋升阈值与遗忘策略参数。提供参数读取、动态调整（受权限控制）与一致性校验功能。是记忆系统可配置性的核心枢纽 |
| 依赖模块 | ad-36（综合重要度 I 值聚合计算单元，消费权重系数）、ad-31/32/33（消费权重）、各分槽（消费分槽专属阈值）、ad-01（总控漏斗 F₀，接收参数变更指令） |
| 被依赖模块 | ad-36、ad-31、ad-32、ad-33、各场景分槽、ad-21（L1 时序衰减单元，消费晋升阈值）等 |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 提供参数读取服务，参数已验证通过 | 系统初始化完成 |
| 参数更新中 | `UPDATING` | 正在执行权重或阈值的动态调整 | 收到合法的参数变更请求 |
| 参数回退 | `ROLLBACK` | 检测到参数异常，正在回退至上一个安全版本 | 参数校验失败或配置冲突时 |
| 暂停服务 | `PAUSED` | 紧急熔断或系统维护，暂停参数读取与更新 | 收到 ad-01 紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 权重系数读取请求 | Struct（请求模块 ID + 需要参数列表 + 请求时间戳） | ad-36、ad-31、ad-32、ad-33、各分槽 | 系统初始化或模块运行时 | **高** |
| 参数变更指令 | Struct（目标参数 + 新值 + 变更理由 + 操作者权限令牌） | ad-01 总控漏斗 F₀（经人工或云端） | 用户改变驾驶风格、分槽策略调优 | **高** |

## 参数库与管理规则

### 一、全局默认权重（编译期固化）

| 参数 | 默认值 | 硬约束范围 | 说明 |
|------|:---:|:---:|------|
| α（安全显著性权重） | 0.50 | [0.30, 0.70] | 安全维度核心 |
| β（风格匹配度权重） | 0.20 | [0.10, 0.40] | 用户体验维度 |
| γ（复用频次权重） | 0.30 | [0.10, 0.50] | 经验验证维度 |

> 约束：α + β + γ = 1.0 强制，任何变更必须保持总和为 1.0。若变更导致总和 ≠ 1.0，系统自动按比例缩放。

### 二、各场景分槽专属参数（默认值，编译期固化，可单独调整）

| 分槽 | α | β | γ | 最低遗忘阈值 | L1→L2 晋升 I 阈值 | L2→L3 晋升 I 阈值 | L3→L4 晋升 I 阈值 |
|------|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| 高速巡航槽 (ad-15) | 0.50 | 0.14 | 0.36 | 0.12 | 0.40 | 0.60 | 0.80 |
| 城区路口槽 (ad-16) | 0.60 | 0.16 | 0.24 | 0.10 | 0.40 | 0.60 | 0.80 |
| 泊车低速槽 (ad-17) | 0.40 | 0.25 | 0.35 | 0.075 | 0.40 | 0.60 | 0.80 |
| 特殊环境槽 (ad-18) | 0.55 | 0.20 | 0.25 | 0.09 | 0.28 | 0.42 | 0.56 |
| 通用驾驶槽-常规 (ad-19) | 0.50 | 0.20 | 0.30 | 0.15 | 0.40 | 0.60 | 0.80 |
| 通用驾驶槽-乡村 (ad-19) | 0.55 | 0.25 | 0.20 | 0.075 | 0.40 | 0.60 | 0.80 |

> 晋升时间阈值（L1→L2, L2→L3, L3→L4）默认分别为 24h/7日/30日，各分槽可单独调整，但受硬约束保护（见安全边界）。

### 三、参数变更权限

| 变更级别 | 所需令牌 | 允许调整范围 | 示例 |
|----------|:---:|------|------|
| 用户级 | 用户令牌 | β 权重 ±0.10，风格基准参数 | 切换“平顺舒适”风格 |
| 工程师级 | 管理员令牌 | 分槽阈值在硬约束内调整，α/γ 权重微调 | 优化特殊环境槽晋升速度 |
| 系统级 | 系统内部令牌 | 因容量告急临时调整遗忘阈值 | 全局容量 > 90% 时的自动调整 |
| 禁止级 | — | 任何超出硬约束范围的调整 | α 设为 0.80（超出上限） |


## 核心处理逻辑

```
FUNCTION weight_config_main_loop():
    SET internal_state = NORMAL
    加载编译期默认参数库（硬约束固件版本）
    当前参数库 = 编译期默认参数库
    参数变更日志 = []  // 用于回滚
    参数版本号 = 1

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF internal_state == PAUSED AND 紧急解除:
            SET internal_state = NORMAL

        // 第 2 步：处理参数读取请求
        IF 收到参数读取请求 AND internal_state == NORMAL:
            请求模块 = 请求.请求模块 ID
            需求参数列表 = 请求.需要参数列表
            IF 请求模块在授权列表内:
                参数数据包 = 从当前参数库提取需求参数
                向请求模块返回参数数据包
            ELSE:
                返回拒绝（无权限）

        // 第 3 步：处理参数变更指令
        IF 收到参数变更指令 AND internal_state == NORMAL:
            SET internal_state = UPDATING

            // 3a. 权限校验
            令牌 = 变更指令.操作者权限令牌
            向 ad-30 请求验证令牌
            IF 令牌无效 OR 权限级别不足:
                向 ad-01 返回参数变更拒绝通知（原因=权限不足）
                SET internal_state = NORMAL
                CONTINUE

            // 3b. 硬约束检查
            目标参数 = 变更指令.目标参数
            新值 = 变更指令.新值
            硬约束 = 编译期默认参数库[目标参数].硬约束
            IF 新值超出硬约束范围:
                向 ad-01 返回参数变更拒绝通知（原因=超出硬约束范围）
                SET internal_state = NORMAL
                CONTINUE

            // 3c. 总和校验（如涉及 α/β/γ）
            IF 目标参数 IN [α, β, γ]:
                暂存当前 α,β,γ 副本
                修改指定参数为 新值
                IF α+β+γ != 1.0:
                    等比例缩放其他两个参数使总和=1.0
                    记录缩放日志
                IF 缩放后 α,β,γ 有任何超出硬约束:
                    向 ad-01 返回参数变更拒绝通知（原因=总和约束无法满足）
                    回滚 α,β,γ 到暂存副本
                    SET internal_state = NORMAL
                    CONTINUE

            // 3d. 执行变更
            旧值 = 当前参数库[目标参数]
            当前参数库[目标参数] = 新值
            参数版本号++
            参数变更日志.append({目标参数, 旧值, 新值, 时间戳, 参数版本号})

            // 3e. 通知相关模块
            向 ad-01 发送参数变更确认（含旧值→新值，版本号）
            向所有订阅了该参数的模块（如 ad-36, 各分槽）广播参数变更确认

            SET internal_state = NORMAL

        // 第 4 步：参数异常检测（定期）
        IF 距上次检测 >= 300 秒:
            FOR EACH 参数 IN 当前参数库:
                IF 参数值超出硬约束 OR (参数 IN [α,β,γ] AND α+β+γ != 1.0):
                    SET internal_state = ROLLBACK
                    执行回滚至上一个安全版本
                    向 ad-01 及所有模块发送参数回滚通知
                    SET internal_state = NORMAL
                    BREAK

        // 第 5 步：周期性状态上报
        IF 距上次上报 >= 60 秒:
            向 ad-03、ad-01 上报配置状态

        SLEEP 200ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| α+β+γ 总和 ≠ 1.0 的变更请求 | 自动等比例缩放，确保总和=1.0，若缩放后仍违规则拒绝变更 | — |
| 请求修改的参数不在参数库中 | 返回拒绝（参数不存在） | — |
| 分槽专属阈值被设为 0（即跳过晋升） | 不允许，最低阈值有硬约束下限 | 恢复至 ≥ 下限 |
| 参数变更日志存储空间满 | 覆盖最旧日志，上报告警 | — |
| 参数回滚时无安全版本可用 | 回退至编译期默认参数库，上报告警 | — |
| 紧急熔断期间收到变更请求 | 拒绝，返回“系统紧急熔断中” | 熔断解除后重新发起 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | α, β, γ 的硬约束范围编译期固化，任何运行时操作不可突破上限或下限 |
| S-02 | α + β + γ = 1.0 为强制约束，系统自动缩放确保一致性，但缩放后 α 权重不得低于 0.30 |
| S-03 | 晋升时间阈值存在编译期硬下限：L1→L2 ≥ 10h, L2→L3 ≥ 3日, L3→L4 ≥ 15日，防止过度加速导致经验泡沫 |
| S-04 | 所有参数变更（含自动缩放）全量写入 ad-51 不可变日志，并包含操作者与变更前后值 |
| S-05 | 参数回滚机制确保任何异常可在 300 秒内检测并自动恢复至上一个安全版本 |
| S-06 | 用户级令牌不可调整 α 和 γ，仅可调整 β 及相关风格参数 |
