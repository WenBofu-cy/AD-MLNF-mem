# ad-50-记忆导入导出与脱敏共享单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-50 |
| 模块名称 | 记忆导入导出与脱敏共享单元 |
| 所属分区 | 五、存储与系统运维 |
| 核心职责 | 管理漏斗二中 L4/L5 层泛化驾驶经验的导出、脱敏处理与外部经验包的合规导入。是记忆系统与云端技能库、其他车辆或外部存储设备之间的唯一数据交换接口。确保导出的经验绝对不包含 GPS 坐标、行人/车辆特征、时间戳等敏感隐私数据 |
| 依赖模块 | ad-26（L4 长期层存储单元，提供可导出的泛化经验）、ad-28（L5 核心层存储单元，提供经审批可导出的核心经验）、ad-30（L5 核心层防篡改与只读管控单元，校验导出权限）、ad-51（记忆变更日志追溯单元，记录导入导出日志） |
| 被依赖模块 | 云端技能库（外部系统，接收经验包）、其他车辆记忆系统（外部系统，接收经验包）、本地维护工具（导入经验包） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无导入导出任务，等待指令 | 系统初始化完成，或上一批次任务结束 |
| 导出执行中 | `EXPORTING` | 正在筛选、脱敏并打包经验数据 | 收到合法的导出请求 |
| 导入执行中 | `IMPORTING` | 正在接收外部经验包并执行合规校验 | 收到合法的导入请求 |
| 暂停服务 | `PAUSED` | 系统紧急熔断或维护 | 收到全局冻结或紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 导出请求 | Struct（导出范围 + 导出目标 + 操作者权限令牌 + 请求时间戳） | ad-01 总控漏斗 F₀（经管理员或云端调度） | 管理员主动触发或云端技能库同步请求 | **高** |
| L4 泛化经验数据 | List（条目 ID + 经验内容 + 适用场景 + 重要度 I 值 + 贡献条目数） | ad-26 L4 长期层存储单元 | 导出时请求 | **高** |
| L5 经审批可导出经验 | List（条目 ID + 经验内容 + 锁定类型 + 审批令牌） | ad-28 L5 核心层存储单元（经 ad-30 校验） | 导出时请求 | **高** |
| 导入经验包 | Struct（经验包文件 + 来源标识 + 数字签名 + 导入令牌） | 云端技能库 / 其他车辆 / 本地维护工具 | 管理员触发或云端下发 | **高** |
| 权限令牌验证结果 | Struct（令牌 + 有效/无效 + 权限级别） | ad-30 L5 核心层防篡改与只读管控单元 | 每次导出或导入前校验 | **高** |
| 全局调度指令 | Enum（冻结/激活） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 导出经验包 | Struct（经验包版本 + 导出时间戳 + 脱敏经验规则列表 + 数字签名 + 导出范围说明） | 云端技能库 / 其他车辆 / 本地存储 | 导出完成后 | **高** |
| 导出完成回执 | Struct（导出条目总数 + 脱敏条目数 + 导出包大小 + 数字签名 + 导出时间戳） | ad-01、ad-51 | 导出完成后 | **高** |
| 导入确认回执 | Struct（导入条目总数 + 成功导入数 + 拒绝数 + 拒绝原因清单 + 导入时间戳） | ad-01、ad-51 | 导入完成后 | **高** |
| 导入经验写入指令 | Struct（条目 ID + 经验内容 + I₀ 建议值 + 来源标记） | ad-14 场景判定与分槽路由单元 | 导入校验通过后 | **高** |
| 请求拒绝通知 | Struct（拒绝原因 + 建议操作） | 请求来源 | 权限不足、令牌无效或数据校验失败时 | **高** |


## 脱敏处理规则

### 一、导出脱敏规则

| 数据字段 | 脱敏处理 | 说明 |
|----------|----------|------|
| GPS 精确坐标 | 剔除，替换为道路类型 + 区域类型（如“城市主干道”、“乡村非铺装”） | 保留场景特征，去除位置隐私 |
| 精确时间戳 | 剔除，替换为时间段（如“日间”、“夜间”、“高峰时段”） | 保留时间特征，去除精确时刻 |
| 行人/车辆特征 | 剔除，替换为目标类别标签（如“第四类行人”、“第二类轿车”） | 仅保留世界模型标准分类 |
| 车牌号/人脸特征 | 完全剔除，不留任何替代 | 绝对隐私保护 |
| 车辆自身 VIN/ID | 完全剔除 | 匿名化共享 |
| 决策动作参数 | 保留（如“跟车时距 2.5s”、“制动减速度 3.0m/s²”） | 核心驾驶技能数据 |
| 场景特征向量 | 保留泛化后的特征 | 用于相似场景匹配 |
| I 值 / S 值 / V 值 / C 值 | 保留 | 用于经验可信度评估 |

### 二、导出范围限制

| 限制项 | 规则 |
|--------|------|
| L4 经验 | 仅导出已提炼为泛化规则的条目（由 ad-27 标记），原始具体经验不导出 |
| L5 经验 | 仅导出标记为“可导出”的条目，且须经 ad-30 审批令牌验证 |
| 漏斗一数据 | 绝对禁止导出，编译期硬编码拦截 |
| 警示标签经验 | 不可导出（失败经验不共享） |
| 单次导出上限 | 500 条经验规则 |

### 三、导入合规校验

| 校验项 | 通过标准 |
|--------|----------|
| 数字签名验证 | 经验包数字签名与来源标识匹配，且签名在有效期内 |
| 法规合规校验 | 经验包中每条规则通过 ad-45 交通法律法规库的合规检查 |
| 物理可行校验 | 经验包中决策动作参数在 ad-44 世界模型物理极限内 |
| 冲突检测 | 导入规则不与 L5 核心层已有安全规则冲突 |
| 来源可信度 | 经验包来源标识在可信来源列表中 |


## 核心处理逻辑

```
FUNCTION memory_import_export_main_loop():
    SET internal_state = IDLE

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令 OR 紧急熔断指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令 AND internal_state == PAUSED:
            SET internal_state = IDLE
            CONTINUE

        // 第 2 步：处理导出请求
        IF 收到导出请求 AND internal_state == IDLE:
            SET internal_state = EXPORTING

            // 2a. 权限校验
            令牌 = 导出请求.操作者权限令牌
            向 ad-30 发送权限令牌验证请求(令牌, 操作类型="导出")
            WAIT 验证结果

            IF 验证结果.无效 OR 权限级别 < 管理员级:
                向请求方返回请求拒绝通知(原因="权限不足")
                SET internal_state = IDLE
                CONTINUE

            // 2b. 检查导出范围
            导出范围 = 导出请求.导出范围
            IF 导出范围包含漏斗一数据:
                向请求方返回请求拒绝通知(原因="漏斗一数据禁止导出")
                SET internal_state = IDLE
                CONTINUE

            // 2c. 从 L4 获取泛化经验
            L4 经验列表 = []
            IF 导出范围包含 L4:
                向 ad-26 发送数据请求(类型="泛化经验", 筛选条件="已提炼标记=True")
                WAIT ad-26 返回 L4 泛化经验数据
                L4 经验列表 = 返回数据

            // 2d. 从 L5 获取可导出经验
            L5 经验列表 = []
            IF 导出范围包含 L5:
                向 ad-28 发送数据请求(类型="可导出经验", 审批令牌=令牌)
                WAIT ad-28 返回 L5 经审批可导出经验
                L5 经验列表 = 返回数据

            全部经验列表 = L4 经验列表 + L5 经验列表

            IF 全部经验列表.length > 500:
                向请求方返回请求拒绝通知(原因="导出条目超过单次上限 500")
                SET internal_state = IDLE
                CONTINUE

            // 2e. 执行脱敏处理
            脱敏后规则列表 = []
            FOR EACH 经验条目 IN 全部经验列表:
                IF 经验条目.结果分类标签 == "警示标签":
                    CONTINUE  // 失败经验不导出

                脱敏规则 = {
                    规则内容: 经验条目.规则内容,  // 已由 ad-27 提炼为 IF-THEN
                    适用场景标签: 脱敏场景(经验条目.适用场景),
                    I 值: 经验条目.I 值,
                    S 值: 经验条目.S 值,
                    V 值: 经验条目.V 值,
                    C 值: 经验条目.C 值,
                    贡献条目数: 经验条目.贡献条目数,
                    来源分槽: 经验条目.来源分槽号
                }
                // 剔除所有隐私字段（GPS/时间戳/车牌/人脸/车辆VIN）
                脱敏后规则列表.append(脱敏规则)

            // 2f. 打包并签名
            经验包 = {
                经验包版本: 生成版本号(),
                导出时间戳: NOW(),
                脱敏经验规则列表: 脱敏后规则列表,
                导出范围说明: 导出范围,
                数字签名: 生成数字签名(脱敏后规则列表)
            }

            // 2g. 输出导出结果
            向导出目标（云端/其他车辆/本地）发送导出经验包
            向 ad-01、ad-51 发送导出完成回执(脱敏规则列表.length, SIZE(经验包), 数字签名)

            SET internal_state = IDLE
            CONTINUE

        // 第 3 步：处理导入请求
        IF 收到导入经验包 AND internal_state == IDLE:
            SET internal_state = IMPORTING

            // 3a. 权限与数字签名校验
            令牌 = 导入经验包.导入令牌
            向 ad-30 发送权限令牌验证请求(令牌, "导入")
            WAIT 验证结果

            IF 验证结果.无效 OR 权限级别 < 管理员级:
                向请求方返回请求拒绝通知(原因="权限不足")
                SET internal_state = IDLE
                CONTINUE

            // 3b. 数字签名验证
            数字签名验证 = 验证数字签名(导入经验包)
            IF 数字签名验证 != 通过:
                向请求方返回请求拒绝通知(原因="数字签名无效")
                SET internal_state = IDLE
                CONTINUE

            // 3c. 来源可信度检查
            IF 导入经验包.来源标识 NOT IN 可信来源列表:
                向请求方返回请求拒绝通知(原因="来源不在可信列表中")
                SET internal_state = IDLE
                CONTINUE

            // 3d. 逐条规则合规校验
            成功导入数 = 0
            拒绝导入清单 = []

            FOR EACH 规则 IN 导入经验包.脱敏经验规则列表:
                // 法规合规校验
                法规校验 = 向 ad-45 查询(规则.规则内容)
                IF 法规校验.结论 == "违反":
                    拒绝导入清单.append({规则 ID, 原因="法规合规校验未通过: " + 法规校验.详情})
                    CONTINUE

                // 物理可行校验
                物理校验 = 向 ad-44 查询(规则.规则内容中的决策动作参数)
                IF 物理校验.结论 == "超出极限":
                    拒绝导入清单.append({规则 ID, 原因="物理可行校验未通过"})
                    CONTINUE

                // 与 L5 安全规则冲突检测
                L5 冲突 = 向 ad-28 查询(规则.规则内容, "冲突检测")
                IF L5 冲突.存在冲突:
                    拒绝导入清单.append({规则 ID, 原因="与 L5 核心安全规则冲突"})
                    CONTINUE

                // 校验通过，写入漏斗二
                I₀ 建议值 = 规则.I 值 × 0.7  // 导入经验初始重要度略降低
                向 ad-14 发送导入经验写入指令(规则, I₀ 建议值, 来源标记="外部导入")
                成功导入数++

            // 3e. 输出导入结果
            向 ad-01、ad-51 发送导入确认回执(成功导入数, 拒绝导入清单)
            IF 拒绝导入清单不为空:
                向请求方返回导入确认回执(成功导入数, 拒绝导入清单)

            SET internal_state = IDLE
            CONTINUE

        // 第 4 步：周期性状态上报
        IF 距上次上报 >= 300 秒:
            向 ad-03、ad-01 上报运行状态

        SLEEP 500ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 导出请求尝试导出漏斗一数据 | 编译期硬编码拦截，直接拒绝 | — |
| 导出范围包含警示标签经验 | 自动跳过，不导出也不报错 | — |
| 导出条目数超过单次上限 500 | 拒绝导出，提示缩小范围或分批导出 | 调整导出范围后重新请求 |
| 导入经验包数字签名失效 | 拒绝导入，记录安全日志 | — |
| 导入规则与 L5 安全规则冲突 | 该规则拒绝导入，其他规则正常处理 | — |
| 导出过程中系统紧急熔断 | 中断导出，已打包数据作废 | 紧急解除后重新导出 |
| 导入过程中目标分槽存储满 | 该规则导入失败，加入拒绝清单 | 存储释放后重新导入 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 漏斗一任何数据（子画像槽、行为统计、驾驶辅助提示）禁止以任何形式导出，编译期硬编码拦截 |
| S-02 | 导出经验包必须执行完整脱敏处理，绝对不可包含 GPS 坐标、精确时间戳、行人/车辆特征、车牌号、人脸特征、车辆 VIN |
| S-03 | 导入经验包必须通过数字签名验证、法规合规校验、物理可行校验、L5 安全规则冲突检测四道关卡 |
| S-04 | L5 核心层经验仅允许经 ad-30 审批令牌验证的条目导出，且须标记为“可导出” |
| S-05 | 警示标签经验（失败经验）禁止导出，防止不安全策略跨车传播 |
| S-06 | 导入经验的初始 I₀ 值须降至原始 I 值的 70%，以体现外部经验在本车未验证的谨慎态度 |
| S-07 | 所有导入导出操作全量写入 ad-51 不可变日志 |
