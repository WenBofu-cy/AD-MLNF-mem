## 四、AD新增：双漏斗管理层（mem-61 ~ mem-66）

AD方向采用双漏斗架构：**漏斗一（驾驶员画像漏斗）**仅在人工驾驶模式下激活，记录每位驾驶员的驾驶习惯；**漏斗二（自动驾驶自成长漏斗）**仅在自动驾驶模式下激活，沉淀系统自身的驾驶经验。以下六个模块支撑这套双漏斗体系的并行运作。

| 编号 | 模块名称 | 核心职责 |
|:---:|------|------|
| mem-61 | 漏斗一管理单元 | 管理驾驶员画像漏斗。多人子画像槽的创建、切换、删除、隐私保护。默认6个长期槽+1个临时槽+1个一次性槽。所有子画像槽数据仅本地加密存储，物理隔绝云端同步通道。删除时执行DoD 5220.22-M标准覆写擦除 |
| mem-62 | 子画像槽隔离单元 | 确保不同驾驶员的子画像槽之间物理隔离，编译期强制实施。不同驾驶员的驾驶数据永不混合。输出边界严格管控：所有统计数据仅用于车内辅助提醒，严禁以任何形式接入自动驾驶决策链路 |
| mem-63 | 驾驶场景分槽路由器 | 根据世界模型输出的道路属性（道路等级、车道数、路面类型、交通标识密度），自动判定当前场景类别（高速巡航/城区路口/泊车低速/特殊环境/通用驾驶），将经验条目路由到漏斗二对应槽位。检索时仅查询当前场景对应槽，检索范围缩小80% |
| mem-64 | 驾驶员身份识别单元 | 统一管理中控屏手动选择、座椅记忆联动、人脸识别（可选关闭）三种身份识别方式。车辆启动时提示选择驾驶员身份，5秒内无选择且无识别结果默认启用临时画像槽。行程中可随时通过中控屏切换身份 |
| mem-65 | 双漏斗模式切换单元 | 管理人工驾驶↔自动驾驶切换时的漏斗冻结/激活/数据一致性校验。模式切换在0.5秒内完成。紧急应急接管时双漏斗临时锁定只读，险情解除后系统自检确认数据一致性再恢复正常 |
| mem-66 | 乡村道路经验保护单元 | 乡村非铺装道路经验遗忘阈值下调50%，标记“乡村道路”特殊标签。遗忘策略的最低重要度阈值较常规场景下调，确保关键经验不被过早清除 |


## 五、独立外置：交通法规库（mem-67）

| 编号 | 模块名称 | 核心职责 |
|:---:|------|------|
| mem-67 | 交通法规库 | 存储法定通行刚性规则，以“场景条件-强制执行动作”形式固化，优先级高于一切驾驶经验。只读不改，永不遗忘，不参与任何漏斗晋升与遗忘机制 |

**交通法规库与物理常识库（mem-59）并列，构成固定知识底座的两大支柱：**

- **物理常识库（mem-59）**：管“路面摩擦系数多少”、“滑水临界车速多少”——这些是物理规律
- **交通法规库（mem-67）**：管“红灯必须停车”、“人行横道行人绝对优先”——这些是法律规则

任何自主学习经验、驾驶员习惯、时效优先策略均不得突破法规约束。标记为“硬约束”的规则，物理不可违抗，不经过推理大脑的推演评估，直接作为不可修改的前提条件注入决策流程。


## 六、三驱动重要度映射（AD场景配置）

核心公式 `I = I₀ + α·S + β·V + γ·C` 不变，但S/V/C的映射关系配置如下：

| 驱动信号 | HR场景 | AD场景 | 触发条件示例 |
|:---|:---|:---|:---|
| S（情绪等价） | 碰撞/摔落/危险操作 | 急刹/碰撞预警/违规罚款/事故 | TTC<2s时S=0.9；ABS触发时S=0.7；ESC介入时S=0.8 |
| V（意义标签） | 身份绑定/目标关联 | 安全规则标签/偏好路线标签 | 平顺变道且纵向冲击度<2m/s³时V=0.8 |
| C（重复复用） | 技能练习/习惯强化 | 同一路段多次通行/同一操作反复执行 | 成功复用100次以上时C=1.0 |


## 七、模块编号锁定规则

1. mem-01 ~ mem-57：漏斗内核集群，永久锁定，版本内禁止新增、删减、重排
2. mem-58 ~ mem-60：外置扩展区，保留编号，仅做场景适配，不改变核心职能
3. mem-61 ~ mem-66：AD方向新增模块，编号从61开始向后追加，不得修改已有编号
4. mem-67：交通法规库，独立外置，与物理常识库并列
5. 本地不再新增记忆类独立模块，高阶拓展能力统一由云端技能包迭代补充
