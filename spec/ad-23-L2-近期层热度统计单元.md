# ad-23-L2-近期层热度统计单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-23 |
| 模块名称 | L2 近期层热度统计单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 五层记忆层级存储 |
| 核心职责 | 统计 L2 近期层各经验条目的查询命中频率与最近访问时间，计算热度权重。热度值作为条目向 L3 晋升的参考权重之一，与重要度 I 值共同决定晋升优先级。识别高频复用的经验，加速其向中期层固化；标记长期未被查询的“冷”条目，为其遗忘判定提供辅助依据 |
| 依赖模块 | ad-22（L2 近期层存储单元，提供 L2 条目索引与查询日志）、ad-38（晋升双条件判定单元，接收热度加权后的晋升建议） |
| 被依赖模块 | ad-22（接收热度统计结果，影响条目排序）、ad-40（遗忘阈值判定单元，参考热度数据辅助遗忘判定） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 周期性统计 L2 条目的查询热度 | 漏斗二激活，L2 存在条目 |
| 批量统计中 | `BATCH_PROCESSING` | 正在处理大量条目的热度计算 | 单次统计条目数 > 500 |
| 暂停统计 | `PAUSED` | L2 处于维护或冻结状态，暂停统计 | 驾驶模式切换、L2 维护中 |

## 热度计算模型

### 一、热度值公式

```
热度值 H = w1 × 近期查询频率 + w2 × 累计查询次数衰减值 + w3 × 最近查询时间衰减因子

其中：
- w1 = 0.5（近期频率权重最高）
- w2 = 0.3（累计次数反映长期价值）
- w3 = 0.2（最近访问时间反映当前相关性）
```

### 二、各因子计算方法

**近期查询频率（归一化 0–1）**
```
近期（近 72 小时）查询次数 / 72
超过 72 小时内查询 3 次以上 → 1.0
```

**累计查询次数衰减值（归一化 0–1）**
```
累计查询次数 / (累计查询次数 + 5)   // 5 次为半衰点
0 次 → 0.0
5 次 → 0.5
20 次以上 → 0.8+
```

**最近查询时间衰减因子（归一化 0–1）**
```
IF 从未被查询:
    R = 0.0
ELSE IF 最近查询距今 < 6 小时:
    R = 1.0
ELSE IF 最近查询距今 < 24 小时:
    R = 0.8
ELSE IF 最近查询距今 < 72 小时:
    R = 0.4
ELSE:
    R = 0.1
```

### 三、热度等级划分

| 热度值范围 | 热度等级 | 含义 | 对晋升的影响 |
|-----------|:---:|------|-------------|
| H ≥ 0.7 | 热 | 高频复用经验，驾驶策略价值极高 | 晋升 I 值加权 +0.05 |
| 0.3 ≤ H < 0.7 | 温 | 正常复用频率 | 无调整 |
| 0.1 ≤ H < 0.3 | 凉 | 复用频率偏低 | 无调整，但标记关注 |
| H < 0.1 | 冷 | 长期未被查询或从未使用 | 纳入冷条目清单，辅助遗忘判定 |


## 核心处理逻辑

```
FUNCTION l2_heat_statistics_main_loop():
    SET internal_state = NORMAL
    查询日志缓冲区 = []  // 累积查询日志
    上次统计时间 = NOW()
    统计间隔 = 60 秒

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令 OR L2 处于维护状态:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 第 2 步：接收查询日志（持续累积）
        IF 收到 L2 条目查询日志:
            将查询日志追加到缓冲区

        // 第 3 步：周期性热度统计
        IF NOW() - 上次统计时间 >= 统计间隔:
            上次统计时间 = NOW()
            L2 条目列表 = ad-22 L2 条目索引快照

            IF L2 条目列表为空:
                CONTINUE

            IF L2 条目列表.length > 500:
                SET internal_state = BATCH_PROCESSING

            热度结果列表 = []
            冷条目列表 = []
            当前时间 = NOW()

            FOR EACH 条目 IN L2 条目列表:
                条目 ID = 条目.ID

                // 3a. 计算近期查询频率（近 72 小时）
                近期查询次数 = 统计缓冲区中 条目 ID 在 [NOW()-72h, NOW()] 内的出现次数
                近期频率 = MIN(近期查询次数 / 72, 1.0)

                // 3b. 计算累计查询次数衰减值
                累计查询次数 = 统计缓冲区中 条目 ID 的总出现次数
                累计衰减值 = 累计查询次数 / (累计查询次数 + 5)

                // 3c. 计算最近查询时间衰减因子
                最近查询时间 = 缓冲区中 条目 ID 的最新查询时间戳
                IF 最近查询时间 == NULL:
                    R = 0.0
                ELSE IF 当前时间 - 最近查询时间 < 6 小时:
                    R = 1.0
                ELSE IF 当前时间 - 最近查询时间 < 24 小时:
                    R = 0.8
                ELSE IF 当前时间 - 最近查询时间 < 72 小时:
                    R = 0.4
                ELSE:
                    R = 0.1

                // 3d. 综合热度值
                H = 0.5 × 近期频率 + 0.3 × 累计衰减值 + 0.2 × R

                // 3e. 热度等级判定
                IF H >= 0.7:
                    热度等级 = "热"
                ELSE IF H >= 0.3:
                    热度等级 = "温"
                ELSE IF H >= 0.1:
                    热度等级 = "凉"
                ELSE:
                    热度等级 = "冷"
                    冷条目列表.append({条目 ID, 上次查询距今: 当前时间 - 最近查询时间})

                热度结果列表.append({
                    条目 ID: 条目 ID,
                    热度值: H,
                    最近查询时间: 最近查询时间,
                    累计查询次数: 累计查询次数,
                    热度等级: 热度等级
                })

            // 3f. 输出热度排行（按热度值降序）
            按热度值降序排列 热度结果列表
            向 ad-22 发送热度排行
            向 ad-38 发送热度排行（含热条目 I 值加权建议）
            向 ad-40 发送冷条目清单

            // 3g. 定期热度统计报告
            IF 距上次统计报告 >= 30 分钟:
                生成热度统计报告
                向 ad-03、ad-01 上报

            // 3h. 清理过期查询日志（保留 7 天）
            清除缓冲区中 7 天前的查询日志

            IF internal_state == BATCH_PROCESSING:
                SET internal_state = NORMAL

        SLEEP 10 秒
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 查询日志缓冲区溢出（> 100,000 条） | 自动清除最旧的 20% 日志，上报告警 | — |
| L2 条目被删除但查询日志仍残留 | 统计时忽略已删除条目，残留日志由定期清理机制清除（7 天） | — |
| 某条目查询频率异常高（单分钟 > 10 次） | 可能为系统异常重复查询，标记但不上报，正常统计 | — |
| 统计过程中 L2 条目数量变化 | 使用快照数据，本次统计不受实时变更影响 | — |
| 冻结状态 | 暂停统计，不清空缓冲区，状态 PAUSED | 激活指令 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| MemoryBus | 读 | L2 条目查询日志 | 只读 | ad-22 实时推送 |
| MemoryBus | 读 | L2 条目索引快照 | 只读 | ad-22 每 60 秒推送 |
| MemoryBus | 读 | 全局调度指令 | 只读 | ad-03 下发 |
| MemoryBus | 写 | 条目热度排行 | 专属写入 | ad-22、ad-38、ad-40 |
| MemoryBus | 写 | 冷条目清单 | 专属写入 | ad-40 |
| MemoryBus | 写 | 热度统计报告 | 周期性写入 | ad-03、ad-01 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 热度统计数据仅用于漏斗二内部晋升参考与遗忘辅助判定，不向漏斗一传输 |
| S-02 | 查询日志缓冲区仅保留 7 天，超期自动清除，不持久化存储 |
| S-03 | 热度加权仅影响晋升优先级，不可替代重要度 I 值作为晋升的主判定依据 |
| S-04 | 冷条目清单仅作为遗忘判定的辅助参考，最终遗忘决策仍由 ad-40 综合重要度与复用次数做出 |
| S-05 | 热度统计过程中使用快照数据隔离实时变更，防止并发冲突 |
