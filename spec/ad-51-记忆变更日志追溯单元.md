# ad-51-记忆变更日志追溯单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-51 |
| 模块名称 | 记忆变更日志追溯单元 |
| 所属分区 | 五、存储与系统运维 |
| 核心职责 | 全链路记录双漏斗记忆系统中所有记忆操作事件的不可变审计日志。覆盖写入、晋升、遗忘、归档、导入导出、配置变更、安全事件等全部操作类型。日志存储于独立安全分区，支持按时间范围、操作类型、来源模块等多维条件检索。是系统合规审计、故障回溯与数据一致性校验的唯一日志依据 |
| 依赖模块 | 全部漏斗内核模块（ad-01 至 ad-43）、外挂模块（ad-44 至 ad-47）、运维模块（ad-48 至 ad-50）——作为日志生产者 |
| 被依赖模块 | ad-01 总控漏斗 F₀（查询审计日志）、ad-30 L5 核心层防篡改与只读管控单元（查询安全事件日志）、离线审计系统（批量导出日志） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收并持久化日志事件 | 系统初始化完成，日志分区挂载校验通过 |
| 日志写入中 | `WRITING` | 正在将日志条目写入安全分区 | 收到日志记录请求 |
| 日志查询中 | `QUERYING` | 正在处理日志检索请求 | 收到查询请求 |
| 日志归档中 | `ARCHIVING` | 正在将较旧日志压缩归档以释放空间 | 日志分区使用率 > 80% 或定期触发 |
| 存储降级 | `DEGRADED` | 日志分区可用容量不足或硬件异常 | 日志分区使用率 > 95% 或硬件告警 |
| 暂停服务 | `PAUSED` | 系统紧急熔断，暂停日志接收 | 收到 ad-01 紧急熔断指令 |

## 核心处理逻辑

```
FUNCTION change_log_main_loop():
    SET internal_state = NORMAL
    日志分区路径 = 独立安全存储分区（与漏斗存储物理隔离）
    日志保留期限 = 6 个月（默认），安全事件日志保留 3 年
    最大日志容量 = 20MB（默认，可配置）
    上一日志哈希 = NULL  // 链式日志结构
    日志计数器 = 0
    日志缓冲区 = []  // 批量写入缓冲区
    缓冲区最大大小 = 50 条
    缓冲区刷新间隔 = 500ms

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断
        IF 收到紧急熔断指令:
            // 紧急刷新缓冲区，确保安全事件日志不丢失
            刷新日志缓冲区()
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF internal_state == PAUSED AND 紧急解除:
            SET internal_state = NORMAL

        // 第 2 步：接收日志分区健康报告
        IF 收到日志分区健康报告:
            IF 健康报告.坏块数 > 安全阈值 OR 健康报告.读写错误率 > 安全阈值:
                SET internal_state = DEGRADED
                向 ad-03、ad-01 发送容量告警(原因=硬件异常)
            ELSE IF 健康报告.已用容量 / 健康报告.总容量 > 0.95:
                SET internal_state = DEGRADED
                向 ad-03、ad-01 发送容量告警(原因=使用率>95%)
            ELSE IF 健康报告.已用容量 / 健康报告.总容量 > 0.80:
                向 ad-03、ad-01 发送容量告警(原因=使用率>80%)
                IF internal_state == NORMAL:
                    触发日志归档()

        // 第 3 步：接收日志记录请求
        IF 收到日志记录请求 AND internal_state IN [NORMAL, WRITING]:
            SET internal_state = WRITING

            // 3a. 生成日志条目
            日志 ID = 生成唯一日志 ID()
            当前时间 = NOW()
            日志条目 = {
                日志 ID: 日志 ID,
                事件时间戳: 请求.操作时间戳,
                记录时间戳: 当前时间,
                事件类型: 请求.事件类型,
                来源模块编号: 请求.来源模块编号,
                来源模块名称: 请求.来源模块名称,
                操作描述: 请求.操作描述,
                操作详情: 请求.操作详情,
                操作结果: 请求.操作结果,
                关联条目 ID: 请求.关联条目 ID,
                关联分槽号: 请求.关联分槽号,
                事件等级: 判定事件等级(请求.事件类型, 请求.操作结果),
                数据校验和: NULL,  // 待计算
                上一日志哈希: 上一日志哈希
            }

            // 3b. 计算本条日志校验和
            日志条目.数据校验和 = SHA256(序列化(日志条目))

            // 3c. 加入写入缓冲区
            日志缓冲区.append(日志条目)
            日志计数器++
            上一日志哈希 = 日志条目.数据校验和

            // 3d. 缓冲区满或超时则批量刷新
            IF 日志缓冲区.length >= 缓冲区最大大小:
                刷新日志缓冲区()

            // 3e. 安全事件日志立即刷新
            IF 请求.事件类型 == SAFETY_EVENT OR 请求.事件等级 == CRITICAL:
                刷新日志缓冲区()

            // 3f. 返回写入确认
            向日志来源模块返回日志写入确认(日志 ID, 成功)

            SET internal_state = NORMAL

        // 第 4 步：定时刷新日志缓冲区
        IF 距上次刷新 >= 缓冲区刷新间隔 AND 日志缓冲区不为空:
            刷新日志缓冲区()

        // 第 5 步：处理日志查询请求
        IF 收到日志查询请求 AND internal_state IN [NORMAL, DEGRADED]:
            SET internal_state = QUERYING

            时间范围 = 请求.时间范围
            事件类型筛选 = 请求.事件类型筛选
            来源模块筛选 = 请求.来源模块筛选
            关联条目 ID = 请求.关联条目 ID
            分页参数 = 请求.分页参数

            // 从日志分区检索匹配条目
            匹配结果 = 检索日志分区(时间范围, 事件类型筛选, 来源模块筛选, 关联条目 ID)
            总匹配数 = 匹配结果.总数
            分页结果 = 应用分页(匹配结果, 分页参数)

            向请求方返回日志查询结果(分页结果, 总匹配数, 分页参数)

            SET internal_state = NORMAL

        // 第 6 步：处理日志导出请求
        IF 收到日志导出请求 AND internal_state IN [NORMAL, DEGRADED]:
            // 权限校验
            令牌 = 导出请求.操作者权限令牌
            向 ad-30 发送权限令牌验证请求(令牌, "审计日志导出")
            WAIT 验证结果

            IF 验证结果.无效 OR 权限级别 < 管理员级:
                向请求方返回请求拒绝通知(原因="权限不足")
                CONTINUE

            导出时间范围 = 导出请求.时间范围
            导出格式 = 导出请求.导出格式（JSON/CSV）
            日志列表 = 检索日志分区(导出时间范围)

            导出数据包 = {
                导出时间范围: 导出时间范围,
                日志条目列表: 日志列表,
                导出格式: 导出格式,
                数字签名: 生成数字签名(日志列表),
                导出时间戳: NOW()
            }

            向请求方返回日志导出数据包(导出数据包)

        // 第 7 步：日志归档
        IF 日志分区使用率 > 80% OR 距上次归档 >= 30 日:
            SET internal_state = ARCHIVING

            归档截止时间 = NOW() - 6 个月  // 6 个月前的日志可归档
            安全事件截止时间 = NOW() - 3 年  // 3 年前的安全日志可归档

            归档日志列表 = 检索日志分区(时间范围=[最旧日志时间, 归档截止时间])
            // 排除安全事件日志（保留 3 年）
            归档日志列表 = 筛选(归档日志列表, 事件类型 != SAFETY_EVENT OR 事件时间戳 < 安全事件截止时间)

            IF 归档日志列表不为空:
                压缩归档数据 = 压缩(归档日志列表)
                归档位置 = 写入冷归档分区(压缩归档数据)
                从活跃日志分区删除已归档日志
                向 ad-03、ad-01 发送日志归档汇总报告

            SET internal_state = NORMAL

        // 第 8 步：周期性状态上报
        IF 距上次上报 >= 60 秒:
            统计 今日写入量、平均写入延迟、各事件类型分布
            向 ad-03、ad-01 上报运行状态

        SLEEP 50ms


FUNCTION 刷新日志缓冲区():
    IF 日志缓冲区为空:
        RETURN

    // 批量写入日志分区
    FOR EACH 日志条目 IN 日志缓冲区:
        写入结果 = 追加写入日志分区(日志条目)
        IF 写入结果 != 成功:
            重试 3 次
            IF 仍失败:
                向 ad-03、ad-01 发送容量告警(原因="日志写入失败")
                // 保留在缓冲区，下次重试
                RETURN

    清空日志缓冲区


FUNCTION 判定事件等级(事件类型, 操作结果):
    IF 事件类型 IN [SAFETY_EVENT, ACCESS_DENIED, L5_UNLOCK_TEMP]:
        RETURN CRITICAL
    ELSE IF 事件类型 IN [CONFIG_CHANGE, ARBITRATION, FORGET_EXECUTE]:
        RETURN HIGH
    ELSE IF 操作结果 == FAILURE OR 操作结果 == REJECTED:
        RETURN WARNING
    ELSE:
        RETURN NORMAL
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 日志写入失败（存储硬件故障） | 重试 3 次，若均失败则保留在缓冲区，上报告警 | 硬件恢复后自动补写 |
| 日志分区使用率 > 95% | 状态=DEGRADED，暂停新日志接收（安全事件日志除外），强制触发紧急归档 | 使用率回落至 < 80% |
| 日志缓冲区溢出（> 500 条） | 强制刷新到日志分区，若刷新失败则覆盖最旧缓冲区条目（安全事件日志除外） | — |
| 日志链断裂（上一日志哈希不匹配） | 标记断裂点，后续日志继续记录但附加断裂标记，上报告警 | 手动审计确认完整性 |
| 日志查询时间范围过大（> 1 年） | 分批返回，每批最多 1000 条 | — |
| 紧急熔断期间收到日志请求 | 安全事件日志立即刷新写入，其他日志缓存于缓冲区 | 紧急解除后批量写入 |
| 日志分区损坏 | 从镜像分区恢复，缺失日志由缓冲区补写 | 恢复完成 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 日志存储于独立安全分区，与漏斗存储物理隔离，运行时仅本模块拥有写入权限，其他模块只读 |
| S-02 | 日志采用链式结构（每条日志含上一日志哈希），任何日志条目的篡改或删除将导致后续所有哈希断裂，可被即时检测 |
| S-03 | 安全事件日志（SAFETY_EVENT）保留期限 ≥ 3 年，普通操作日志保留期限 ≥ 6 个月，到期归档不可直接删除 |
| S-04 | 日志条目一旦写入即不可修改。本模块不提供日志修改或删除接口，任何对日志数据的修改须在物理层面离线操作 |
| S-05 | 日志导出须经管理员令牌验证，导出数据包须附带数字签名，以确保离线审计时可验证数据来源与完整性 |
| S-06 | 紧急熔断时安全事件日志享有最高写入优先级，强制刷新缓冲区，确保安全事件不因系统故障而丢失 |
| S-07 | 日志分区须每日自动备份至冗余镜像分区，备份文件同样受链式哈希保护 |
