# ad-10-行为累积统计单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-10 |
| 模块名称 | 行为累积统计单元 |
| 所属分区 | 二、漏斗一：驾驶员画像漏斗 |
| 核心职责 | 按行为维度（跟车/变道/路口通行/加减速/让行/停车/起步）累计同类行为频次，按判定标签（优良习惯/常态陋习/应急特殊操作）分类统计，生成多时间粒度统计周期报表（近 7 日/近 30 日/总累计），供驾驶辅助提醒生成单元调用 |
| 依赖模块 | ad-09（行为判定标签单元，提供带判定标签的行为条目）、ad-05（子画像槽创建与初始化单元，提供统计基线）、ad-02（漏斗一专属调度单元，提供当前活跃槽号） |
| 被依赖模块 | ad-11（驾驶辅助提醒生成单元，消费统计周期报表）、ad-13（子画像槽长期未活跃提醒单元，消费最后活跃时间戳） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收判定条目并更新统计 | 系统初始化完成，活跃槽已激活 |
| 统计周期切换中 | `ROLLING` | 正在执行日/周/月统计周期的滚动更新 | 跨自然日/周/月边界时 |
| 基线初始化 | `INIT` | 新槽创建，等待统计基线清零确认 | 收到 ad-05 统计基线初始化数据 |
| 暂停统计 | `PAUSED` | 漏斗一冻结或驾驶模式切换，暂停接收新条目 | 驾驶模式切换为非人工驾驶 |
| 紧急只读 | `EMERGENCY_RO` | 紧急事件触发，暂停统计更新 | 收到 ad-01 紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 带判定标签的行为条目 | Struct（含原始观测 + 场景标签 + 判定标签 + 判定依据 + 判定置信度） | ad-09 行为判定标签单元 | 每次判定完成后 | **高** |
| 统计基线初始化数据 | Struct（槽号 + 全零基线 + 初始时间戳） | ad-05 子画像槽创建与初始化单元 | 新子画像槽创建完成时 | **高** |
| 驾驶模式信号 | Enum（人工驾驶/自动驾驶/紧急接管） | ad-02 漏斗一专属调度单元 | 驾驶模式变更时 | **高** |
| 当前活跃槽号 | Int | ad-02 漏斗一专属调度单元 | 槽位激活/切换时 | **高** |
| 紧急熔断指令 | Enum（全局熔断） | ad-01 总控漏斗 F₀ | 安全急停或碰撞事件时 | **紧急** |
| 系统时钟信号 | UTC 时间戳 | 系统时钟 | 周期性，每 60 秒 | 普通 |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 统计周期报表 | Struct（见下方详表） | ad-11 驾驶辅助提醒生成单元 | 每次统计更新后 / ad-11 主动查询时 | **高** |
| 最后活跃时间戳更新 | Struct（槽号 + 最新活跃时间） | ad-13 子画像槽长期未活跃提醒单元 | 每次统计更新后 | 普通 |

## 核心处理逻辑

```
FUNCTION behavior_statistics_main_loop():
    SET internal_state = INIT
    初始化统计数据结构（按行为维度 × 3 时间粒度 × 4 标签类型）
    last_rollover_check = NOW()

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = EMERGENCY_RO
            暂停统计更新
            CONTINUE

        // 第 2 步：驾驶模式检查
        IF 驾驶模式 != 人工驾驶:
            SET internal_state = PAUSED
            CONTINUE

        // 第 3 步：统计基线初始化
        IF 收到 ad-05 统计基线初始化数据:
            为该槽号初始化全零统计结构
            记录初始时间戳
            SET internal_state = NORMAL
            CONTINUE

        // 第 4 步：统计周期滚动检查（每 60 秒检查一次）
        IF NOW() - last_rollover_check >= 60 秒:
            last_rollover_check = NOW()
            IF 跨自然日:
                SET internal_state = ROLLING
                将“近 7 日”窗口向前滑动 1 天（删除最早 1 天的数据）
                重新计算近 7 日统计
                IF 跨自然周:
                    将“近 30 日”窗口向前滑动 7 天
                    重新计算近 30 日统计
                    归档上周期报表至本地日志
                SET internal_state = NORMAL

        // 第 5 步：接收判定条目并更新统计
        IF 收到 ad-09 带判定标签的行为条目:
            IF internal_state IN [EMERGENCY_RO, PAUSED, INIT]:
                暂存至临时队列（最长保留 30 秒）
                CONTINUE

            // 5a. 提取关键字段
            行为维度 = 判定条目.原始条目.行为类型
            判定标签 = 判定条目.判定标签
            判定置信度 = 判定条目.判定置信度
            目标槽号 = 当前活跃槽号

            // 5b. 置信度加权（陋习判定置信度低时加权处理）
            统计权重 = 1.0
            IF 判定标签 == "常态陋习" AND 判定置信度 < 0.5:
                统计权重 = 1.2  // 保守加权，低置信度陋习权重大
            ELSE IF 判定标签 == "优良习惯" AND 判定置信度 < 0.5:
                统计权重 = 0.8  // 低置信度优良权重小

            // 5c. 更新统计计数器
            统计结构[目标槽号][行为维度][总累计].总次数 += 1 × 统计权重
            统计结构[目标槽号][行为维度][总累计][判定标签] += 1 × 统计权重
            统计结构[目标槽号][行为维度][近7日].总次数 += 1 × 统计权重
            统计结构[目标槽号][行为维度][近7日][判定标签] += 1 × 统计权重
            统计结构[目标槽号][行为维度][近30日].总次数 += 1 × 统计权重
            统计结构[目标槽号][行为维度][近30日][判定标签] += 1 × 统计权重

            // 同步更新标签汇总
            统计结构[目标槽号][按判定标签汇总][*][判定标签] += 1 × 统计权重

            // 5d. 更新陋习排行榜（近 30 日）
            重新排序陋习排行榜

            // 5e. 更新改善趋势
            重新计算 近7日综合优良率 vs 近30日综合优良率 差值

            // 5f. 生成统计周期报表
            report = 打包统计报表(目标槽号)

            // 5g. 输出到 ad-11
            向 ad-11 发送 report

            // 5h. 更新最后活跃时间戳
            向 ad-13 发送 { 槽号 = 目标槽号, 最后活跃时间 = NOW() }

        // 第 6 步：统计数据完整性校验（每 30 分钟）
        IF 距上次校验 >= 30 分钟:
            FOR EACH 活跃槽号:
                // 校验：各行为维度汇总 = 标签汇总 = 条目总数
                IF 行为维度总次数合计 != 标签汇总总次数:
                    上报告警 TO ad-01（含槽号 + 不一致详情）
                    标记该槽统计为 NEED_REPAIR
                // 校验：近 7 日 ≤ 近 30 日 ≤ 总累计
                IF 近7日总次数 > 近30日总次数 OR 近30日总次数 > 总累计总次数:
                    上报告警

        // 第 7 步：周期性状态上报
        IF 距上次上报 >= 60 秒:
            向 ad-01 上报统计状态（含总条目数、各标签占比、综合优良率）

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 驾驶模式切换为非人工驾驶 | 暂停统计，状态 PAUSED，未处理条目暂存临时队列（最长 30 秒） | 恢复人工驾驶后从队列补处理 |
| 紧急熔断 | 暂停统计，状态 EMERGENCY_RO，清空临时队列 | 紧急事件解除 + 恢复指令 |
| 统计数据校验不一致（行为维度合计 ≠ 标签合计） | 上报告警，标记 NEED_REPAIR，继续使用当前数据（不阻塞） | 下次校验通过后自动清除标记 |
| 统计计数器溢出 | 使用 64 位整型，上限 9.2×10¹⁸，若溢出则触发全局告警 | — |
| 临时队列积压超过 30 秒 | 丢弃最旧条目，上报告警 | 系统恢复 NORMAL 后清空队列 |
| 新槽初始化时仍有旧槽统计在内存中 | 旧槽统计归档至冷存储后从内存卸载 | — |
| 用户手动重置某维度统计 | 仅重置该维度近 7 日/近 30 日数据，总累计保留 | — |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 漏斗一数据编译期禁止接入自动驾驶决策链路。本模块输出的统计报表仅流向 ad-11，不向 ECC 大脑或任何自动驾驶决策模块传输 |
| S-02 | 子画像槽间统计数据绝对物理隔离。各槽统计结构在内存中独立分配，禁止跨槽聚合或比较 |
| S-03 | 统计报表中的陋习排行榜仅用于车内驾驶辅助提示，不向云端同步，不用于任何形式的驾驶员评分或保险费率计算 |
| S-04 | 统计基线初始化数据由 ad-05 下发，本模块不得主动修改基线值 |
| S-05 | 紧急熔断时立即暂停统计更新，清空临时队列，确保安全事件优先 |
| S-06 | 统计周期报表在输出到 ad-11 前须脱敏处理，不包含驾驶员身份 ID 明文，仅含槽号索引 |
| S-07 | 所有统计结构变更（周期滚动、基线初始化、异常修复）全量写入 ad-51 变更日志 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-10-01 | NORMAL 状态，活跃槽 A | 判定条目（行为=变道，标签=常态陋习，置信度=0.9） | 槽 A 变道统计：近7日/近30日/总累计 陋习次数各 +1，报表更新输出到 ad-11 |
| TC-10-02 | NORMAL 状态，判定置信度=0.4 | 判定条目（行为=跟车，标签=常态陋习，置信度=0.4） | 统计权重=1.2（保守加权），陋习次数 +1.2 |
| TC-10-03 | NORMAL 状态，判定置信度=0.4 | 判定条目（行为=跟车，标签=优良习惯，置信度=0.4） | 统计权重=0.8（保守降权），优良次数 +0.8 |
| TC-10-04 | NORMAL 状态，收到多个条目后 | 连续 10 条判定条目（8 优良 + 2 陋习） | 综合优良率=80%，陋习排行榜更新，改善趋势计算 |
| TC-10-05 | 跨自然日边界 | 系统时钟触发新的一天 | 近 7 日窗口滑动 1 天，重新计算统计，上周期归档 |
| TC-10-06 | 统计数据校验不一致 | 校验周期触发 | 上报告警，标记 NEED_REPAIR |
| TC-10-07 | 驾驶模式切换为自动驾驶 | 判定条目 | 暂停统计，状态 PAUSED，条目暂存临时队列 |
| TC-10-08 | EMERGENCY_RO 状态 | 判定条目 | 丢弃，不处理 |
| TC-10-09 | 收到 ad-05 统计基线初始化数据 | 新槽 B 初始化 | 为槽 B 初始化全零统计结构，状态 NORMAL |
| TC-10-10 | 用户手动重置变道维度统计 | 重置指令 | 变道近 7 日/近 30 日清零，总累计保留 |
