# ad-38-晋升双条件判定单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-38 |
| 模块名称 | 晋升双条件判定单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 严格按照“留存时长达标 + 重要度 I 值达标”双重条件，对漏斗二中每一层级的经验条目进行晋升资格判定。将符合条件的条目按优先级排序后生成晋升候选清单，下发至对应层级的存储单元执行晋升搬运。L4→L5 晋升额外校验安全仲裁状态与不可抗力标记 |
| 依赖模块 | ad-36（综合重要度 I 值聚合计算单元，提供条目当前 I 值）、ad-20/22/24/26（各层级存储单元，提供条目留存时长与元数据）、ad-35（三维权重系数配置单元，获取各分槽专属晋升阈值）、ad-43（失败经验安全仲裁三道校验单元，校验 L4→L5 失败经验） |
| 被依赖模块 | ad-20/22/24/26（各层级存储单元，消费晋升候选清单）、ad-39（层级单向搬运写入单元，接收搬运指令） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 等待晋升判定周期触发 | 系统初始化完成，或上次判定结束 |
| 判定执行中 | `JUDGING` | 正在逐层级、逐条目执行双条件判定 | 周期触发或收到手动判定请求 |
| 仲裁等待中 | `WAITING_ARBITRATION` | L4→L5 晋升中有失败经验待安全仲裁 | 向 ad-43 发送仲裁请求后等待 |
| 暂停服务 | `PAUSED` | 漏斗二冻结或紧急熔断 | 驾驶模式切换或紧急事件 |

## 晋升双条件判定规则

### 一、通用双条件公式

```
晋升条件 = (留存时长 ≥ T_threshold) AND (I ≥ I_threshold)
```

两项条件必须同时满足，缺一不可。

### 二、各层级晋升阈值（默认值，各分槽可独立配置）

| 晋升路径 | 默认留存时间阈值 | 默认 I 值阈值 | 特殊说明 |
|----------|:---:|:---:|------|
| L1 → L2 | 24 小时 | 0.40 | 特殊环境槽降至 0.28 |
| L2 → L3 | 7 日 | 0.60 | — |
| L3 → L4 | 30 日 | 0.80 | 失败经验须通过三道安全仲裁 |
| L4 → L5 | 90 日 | 0.80 | 失败经验须通过安全仲裁，且复用≥10 次（不可抗力豁免时间与复用限制） |

### 三、晋升优先级计算

当同一层级有多个条目满足晋升条件，但目标层级存储容量有限时，按以下优先级排序：

```
晋升优先级 = I 值 × 0.6 + (留存时长 / 晋升时间阈值) × 0.2 + 热度值 × 0.2
```

> 热度值由 ad-23 L2 热度统计单元提供（L2→L3 时），其他层级热度值取 0.5 中性值。

### 四、失败经验特殊判定规则（L3→L4 及 L4→L5）

| 条件 | 处理 |
|------|------|
| 结果分类标签 = "策略失误"，安全仲裁状态 = 未仲裁 | 向 ad-43 发送仲裁请求，状态=WAITING_ARBITRATION，条目保留原层级等待 |
| 结果分类标签 = "策略失误"，安全仲裁状态 = 仲裁中 | 暂缓晋升，保留原层级等待 |
| 结果分类标签 = "策略失误"，安全仲裁状态 = 仲裁驳回 | 驳回晋升，条目保留原层级为警示标签 |
| 结果分类标签 = "策略失误"，安全仲裁状态 = 仲裁通过 | 允许正常参与晋升判定 |
| 结果分类标签 = "策略失误"，晋升路径 = L4→L5 | 除安全仲裁通过外，还需复用计数 ≥ 10 |
| 不可抗力标记 = True | 豁免留存时间与复用次数限制，仅需 I ≥ 0.80 |


## 核心处理逻辑

```
FUNCTION promotion_dual_condition_judge_main_loop():
    SET internal_state = IDLE
    判定间隔 = 60 秒
    last_judge_time = 0
    仲裁等待字典 = {}  // 条目 ID → {请求时间, 超时时间}

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令 AND internal_state == PAUSED:
            SET internal_state = IDLE
            CONTINUE

        // 第 2 步：检查判定触发条件
        触发判定 = False
        IF 收到手动判定触发信号:
            触发判定 = True
            判定类型 = "手动"
        ELSE IF NOW() - last_judge_time >= 判定间隔:
            触发判定 = True
            判定类型 = "周期"
        ELSE IF 仲裁等待字典不为空 AND 有仲裁结果返回:
            触发判定 = True
            判定类型 = "仲裁返回"

        IF NOT 触发判定:
            SLEEP 1 秒
            CONTINUE

        SET internal_state = JUDGING
        start_time = NOW()
        last_judge_time = NOW()

        // 第 3 步：获取各层级候选条目与 I 值
        各层级候选条目 = ad-20/22/24/26 候选条目快照
        I 值字典 = ad-36 条目当前 I 值字典
        晋升阈值库 = ad-35 分槽专属晋升阈值

        晋升候选清单 = {L1→L2: [], L2→L3: [], L3→L4: [], L4→L5: []}
        暂缓清单 = []
        驳回清单 = []
        仲裁请求清单 = []

        // 第 4 步：逐层级逐条目判定

        // 4a. L1 → L2 判定
        FOR EACH 条目 IN 各层级候选条目.L1:
            条目 ID = 条目.ID
            留存时长 = 条目.留存时长
            分槽号 = 条目.来源分槽号
            阈值 = 晋升阈值库[分槽号].L1→L2
            I 值 = I 值字典[条目 ID] ?? 条目.I₀

            IF 留存时长 >= 阈值.时间阈值 AND I 值 >= 阈值.I 阈值:
                晋升候选清单["L1→L2"].append({条目 ID, I 值, 留存时长, 晋升优先级=计算优先级(I值,留存时长,热度=0.5)})
            ELSE IF 留存时长 < 阈值.时间阈值:
                暂缓清单.append({条目 ID, 暂缓原因="留存时长不足", 还需等待=阈值.时间阈值-留存时长})
            ELSE:
                驳回清单.append({条目 ID, 驳回原因="I 值不满足", 当前 I=I 值, 要求 I=阈值.I 阈值})

        // 4b. L2 → L3 判定（增加热度参考）
        FOR EACH 条目 IN 各层级候选条目.L2:
            条目 ID = 条目.ID
            热度值 = 查询 ad-23 热度值(条目 ID) ?? 0.5
            ... 同 4a 逻辑，晋升优先级计算时加入热度值 ...
            IF 满足条件:
                晋升候选清单["L2→L3"].append({..., 晋升优先级=计算优先级(I值,留存时长,热度值)})
            ELSE: ...

        // 4c. L3 → L4 判定（增加失败经验仲裁检查）
        FOR EACH 条目 IN 各层级候选条目.L3:
            条目 ID = 条目.ID
            结果标签 = 条目.结果分类标签

            IF 结果标签 == "策略失误" AND 条目.安全仲裁状态 NOT IN ["仲裁通过", "仲裁驳回"]:
                仲裁请求清单.append(条目)
                CONTINUE
            ELSE IF 结果标签 == "策略失误" AND 条目.安全仲裁状态 == "仲裁驳回":
                驳回清单.append({条目 ID, 驳回原因="安全仲裁未通过"})
                CONTINUE

            ... 同 4a 双条件判定 ...
            IF 满足条件:
                晋升候选清单["L3→L4"].append({...})

        // 4d. L4 → L5 判定（最严苛）
        FOR EACH 条目 IN 各层级候选条目.L4:
            条目 ID = 条目.ID
            不可抗力标记 = 条目.不可抗力标记

            IF 不可抗力标记 == True:
                // 豁免留存时间与复用限制，仅需 I ≥ 0.80
                IF I 值 >= 0.80:
                    晋升候选清单["L4→L5"].append({..., 备注="不可抗力豁免"})
                CONTINUE

            复用计数 = 条目.复用计数
            IF 复用计数 < 10:
                暂缓清单.append({条目 ID, 暂缓原因="复用次数不足", 当前=复用计数, 要求=10})
                CONTINUE

            ... 同 4c 失败经验检查 + 双条件判定 ...
            IF 满足条件:
                晋升候选清单["L4→L5"].append({...})

        // 第 5 步：发送安全仲裁请求
        IF 仲裁请求清单不为空:
            SET internal_state = WAITING_ARBITRATION
            FOR EACH 条目 IN 仲裁请求清单:
                向 ad-43 发送安全仲裁请求(条目 ID, 条目.经验内容, 条目.结果分类标签, 条目.场景特征)
                仲裁等待字典[条目 ID] = {请求时间: NOW(), 超时时间: NOW()+72h}
            // 本次判定中需仲裁的条目暂不晋升，等待下次判定

        // 第 6 步：处理返回的仲裁结果
        IF 仲裁等待字典不为空:
            FOR EACH 条目 ID IN 仲裁等待字典:
                IF 条目 ID IN 本次返回的仲裁结果:
                    仲裁结果 = 本次返回的仲裁结果[条目 ID]
                    IF 仲裁结果.仲裁结论 == "放行晋升" OR 仲裁结果.仲裁结论 == "仲裁通过":
                        移除仲裁等待
                    ELSE IF 仲裁结果.仲裁结论 == "保留L3警示" OR 仲裁结果.仲裁结论 == "仲裁驳回":
                        驳回清单.append({条目 ID, 驳回原因="安全仲裁驳回"})
                        移除仲裁等待
                    ELSE IF 仲裁结果.仲裁结论 == "永久锁定L5":
                        晋升候选清单["L4→L5"].append({条目 ID, 备注="仲裁直达L5"})
                        移除仲裁等待

        // 第 7 步：目标层级容量检查与优先级截断
        FOR EACH 晋升路径, 候选列表 IN 晋升候选清单:
            按晋升优先级降序排列 候选列表
            目标层级 = 晋升路径.目标层级
            目标层级剩余容量 = 查询目标层级剩余容量()
            IF 候选列表.length > 目标层级剩余容量:
                截断 候选列表 至 目标层级剩余容量
                被截断条目 → 暂缓清单（原因=目标层级容量不足）

        // 第 8 步：下发晋升候选清单
        FOR EACH 晋升路径, 候选列表 IN 晋升候选清单:
            IF 候选列表不为空:
                向 晋升路径.来源层级存储单元 发送晋升候选清单（经 ad-39）

        // 第 9 步：下发暂缓与驳回通知
        FOR EACH 条目 IN 暂缓清单:
            向条目.来源层级存储单元 发送晋升暂缓通知
        FOR EACH 条目 IN 驳回清单:
            向条目.来源层级存储单元 发送晋升驳回通知

        // 第 10 步：汇总上报
        判定结果汇总 = {
            各层级判定条目数, 晋升候选数: 各晋升路径候选总数,
            暂缓数: 暂缓清单.length, 驳回数: 驳回清单.length,
            仲裁请求数: 仲裁请求清单.length, 判定耗时: NOW()-start_time
        }
        向 ad-03、ad-01、ad-51 发送判定结果汇总

        IF internal_state == WAITING_ARBITRATION AND 仲裁等待字典为空:
            SET internal_state = IDLE
        ELSE IF internal_state == JUDGING:
            SET internal_state = IDLE

        SLEEP 1 秒
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 条目 I 值缺失 | 使用条目 I₀ 替代，标记为“I 值估算”，上报告警 | I 值补全后自动修正 |
| 分槽晋升阈值未配置 | 使用全局默认阈值，上报告警 | 阈值配置完成 |
| 目标层级容量为 0（已满） | 全部候选条目暂缓，上报告警 | 目标层级释放空间 |
| 安全仲裁超时（> 72 小时） | 自动按“保留 L3 警示”处理，上报告警 | 仲裁完成 |
| 同一条目同时出现在晋升候选和暂缓清单 | 晋升候选优先，从暂缓中移除 | — |
| 晋升候选清单下发后条目已被删除 | ad-39 执行前二次校验，不存在则跳过 | — |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 晋升双条件（留存时长 + I 值）必须同时满足，单条件不可放行。编译期固化判定逻辑 |
| S-02 | 失败经验（结果标签="策略失误"）晋升 L3→L4 及 L4→L5 必须通过 ad-43 三道安全仲裁 |
| S-03 | 不可抗力经验晋升 L5 豁免留存时间与复用次数限制，但仍需 I ≥ 0.80 |
| S-04 | 晋升优先级计算仅决定同一批次内的晋升顺序，不得因优先级低而永久阻止晋升 |
| S-05 | 所有晋升判定（含晋升、暂缓、驳回、仲裁）全量写入 ad-51 不可变日志 |
| S-06 | 目标层级容量为 0 时禁止强行晋升，防止数据溢出覆盖已有经验 |
