# ad-32-风格匹配度V值计算单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-32 |
| 模块名称 | 风格匹配度 V 值计算单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 三维重要度计算引擎 |
| 核心职责 | 将系统实际执行的驾驶动作与用户设定的驾驶风格偏好（平顺舒适/标准通勤/高效通行）进行契合度量化比对，计算风格匹配度分值 V（0–1）。V 值反映当前驾驶行为是否符合用户期望的驾乘体验，参与综合重要度 I 值计算 |
| 依赖模块 | ad-35（三维权重系数配置单元，获取 β 权重及各风格基准参数）、ad-36（综合重要度 I 值聚合计算单元，接收 V 值） |
| 被依赖模块 | ad-36（消费 V 值参与 I 值计算） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收驾驶数据并计算 V 值 | 漏斗二激活，本单元初始化完成 |
| 参数更新中 | `UPDATING` | 用户切换风格或调整风格参数，正在刷新基准值 | 收到 ad-35 风格参数变更通知 |
| 暂停服务 | `PAUSED` | 漏斗二冻结或紧急熔断 | 驾驶模式切换或紧急事件 |


## 输入数据

|--------|----------|----------|----------|:---:|
| 驾驶行为数据包 | Struct（条目 ID + 行为类型 + 实际执行参数 + 场景特征 + 分槽号） | 各场景分槽（经 ad-14 转发） | 新经验生成后 | **高** |

| 车辆物理约束 | Struct（最大纵向冲击度、最大横向冲击度、最小跟车时距法规下限） | 编译期常量 | 系统初始化 | 普通 |
| 全局调度指令 | Enum（冻结/激活） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| V 值计算结果 | Struct（条目 ID + V 值 + 各维度得分 + 计算时间戳） | ad-36 综合重要度 I 值聚合计算单元 | 每次计算完成后 | **高** |
| 风格偏差告警 | Struct（条目 ID + 偏差维度 + 偏差幅度 + 建议） | ad-03、ad-01 | 连续多次 V < 0.3 时 | 普通 |
| 计算状态上报 | Struct（当前状态 + 近期平均 V 值 + 各风格使用占比） | ad-03、ad-01 | 每 60 秒周期性上报 | 普通 |


## V 值计算模型

### 一、核心公式

```
V = CLAMP( 1.0 - Σ( w_i × 偏差_i ) , 0.0 , 1.0 )
```

其中 `偏差_i` 是实际执行参数与风格基准的归一化偏差，`w_i` 是各行为维度的权重。

### 二、驾驶风格基准参数库

| 行为维度 | 平顺舒适 | 标准通勤 | 高效通行 | 物理/法规硬约束 |
|----------|:---:|:---:|:---:|------|
| 跟车时距 (s) | 2.5 | 2.0 | 1.8 | ≥ 1.8s（法规下限） |
| 纵向冲击度 (m/s³) | ≤ 2.0 | ≤ 3.0 | ≤ 3.5 | ≤ 5.0 |
| 横向冲击度 (m/s³) | ≤ 1.5 | ≤ 2.0 | ≤ 2.5 | ≤ 3.0 |
| 制动减速度 (m/s²) | ≤ 2.5 | ≤ 3.5 | ≤ 4.0 | ≤ 7.0（非紧急） |
| 起步加速度 (m/s²) | ≤ 1.5 | ≤ 2.0 | ≤ 2.5 | — |
| 转弯车速/限速比 | ≤ 0.5 | ≤ 0.7 | ≤ 0.8 | ≤ 0.9 |
| 变道间隙 (s) | ≥ 3.0 | ≥ 2.5 | ≥ 2.0 | ≥ 1.5 |
| 刹停点头控制 | 平顺，纵向冲击度 < 1.0 | 轻微，< 1.5 | 可接受，< 2.0 | — |

### 三、偏差计算方法

**归一化偏差公式（以跟车时距为例）：**
```
IF 实际值 ≥ 风格基准值:
    偏差 = 0.0  // 超出基准为正向（更安全/更舒适）
ELSE:
    偏差 = (风格基准值 - 实际值) / (风格基准值 - 法规下限值)
    偏差 = CLAMP(偏差, 0.0, 1.0)
```

**纵向冲击度偏差示例：**
```
IF 实际值 ≤ 风格基准值:
    偏差 = 0.0  // 低于基准为正向
ELSE:
    偏差 = (实际值 - 风格基准值) / (物理上限 - 风格基准值)
    偏差 = CLAMP(偏差, 0.0, 1.0)
```

### 四、行为维度权重分配

| 行为维度 | 权重 | 说明 |
|----------|:---:|------|
| 跟车时距 | 0.20 | 纵向安全核心 |
| 纵向冲击度 | 0.15 | 驾乘舒适度核心 |
| 横向冲击度 | 0.15 | 变道/转弯舒适度 |
| 制动减速度 | 0.15 | 制动舒适与安全 |
| 起步加速度 | 0.10 | 起步体验 |
| 转弯车速比 | 0.10 | 弯道体感 |
| 变道间隙 | 0.10 | 变道安全与平顺 |
| 刹停点头 | 0.05 | 停车最后一刻体验 |

### 五、各分槽 V 值特殊处理

| 分槽 | 特殊处理 | 理由 |
|------|----------|------|
| 高速巡航槽 (ad-15) | 跟车时距权重提升至 0.30，刹停点头权重降至 0.0 | 高速跟车与刹停频率差异 |
| 城区路口槽 (ad-16) | 制动减速度权重提升至 0.25，起步加速度提升至 0.15 | 城区频繁启停 |
| 泊车低速槽 (ad-17) | 使用独立风格基准（所有阈值 × 0.6 收紧） | 低速场景对精度要求更高 |
| 特殊环境槽 (ad-18) | V 值强制置为 1.0（安全优先，不做风格评判） | 特殊环境下不纠结风格 |
| 通用驾驶槽-乡村子类 (ad-19) | 转弯车速比权重提升至 0.20，纵向冲击度权重降至 0.10 | 乡村弯道多、路面不平 |


## 核心处理逻辑

```
FUNCTION v_value_calculation_main_loop():
    SET internal_state = NORMAL
    加载当前用户风格设定（默认=标准通勤）
    加载风格基准参数库
    加载分槽 V 值特殊处理规则
    近期 V 值历史 = []  // 用于偏差告警

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 第 2 步：用户风格变更处理
        IF 收到用户风格设定变更:
            SET internal_state = UPDATING
            更新当前风格 = 变更后的风格
            从 ad-35 获取对应风格的基准参数库
            向 ad-03 上报风格切换日志
            SET internal_state = NORMAL
            CONTINUE

        // 第 3 步：接收驾驶行为数据并计算 V 值
        IF 收到驾驶行为数据包 AND internal_state == NORMAL:
            条目 ID = 数据包.条目 ID
            行为类型 = 数据包.行为类型
            实际参数 = 数据包.实际执行参数
            分槽号 = 数据包.分槽号

            // 3a. 特殊环境槽直接返回 V=1.0
            IF 分槽号 == ad-18:
                V = 1.0
                向 ad-36 发送 V 值计算结果(条目 ID, V=1.0)
                CONTINUE

            // 3b. 获取当前风格基准
            风格基准 = 风格基准参数库[当前风格]
            分槽权重 = 获取分槽权重(分槽号, 行为类型)

            // 3c. 逐维度计算偏差
            总加权偏差 = 0.0
            各维度得分 = {}

            // 跟车时距
            IF 实际参数.跟车时距 != NULL AND 行为类型 IN [跟车, 巡航]:
                偏差 = 计算正偏差(实际值=实际参数.跟车时距, 基准值=风格基准.跟车时距, 下限=法规下限)
                总加权偏差 += 分槽权重.跟车时距 × 偏差
                各维度得分["跟车时距"] = {实际: 实际参数.跟车时距, 基准: 风格基准.跟车时距, 偏差: 偏差}

            // 纵向冲击度
            IF 实际参数.纵向冲击度 != NULL:
                偏差 = 计算负偏差(实际值=实际参数.纵向冲击度, 基准值=风格基准.纵向冲击度, 上限=物理上限)
                总加权偏差 += 分槽权重.纵向冲击度 × 偏差
                各维度得分["纵向冲击度"] = {实际: 实际参数.纵向冲击度, 基准: 风格基准.纵向冲击度, 偏差: 偏差}

            // 横向冲击度
            IF 实际参数.横向冲击度 != NULL AND 行为类型 IN [变道, 转弯]:
                偏差 = 计算负偏差(实际值=实际参数.横向冲击度, 基准值=风格基准.横向冲击度, 上限=物理上限)
                总加权偏差 += 分槽权重.横向冲击度 × 偏差
                各维度得分["横向冲击度"] = {实际: 实际参数.横向冲击度, 基准: 风格基准.横向冲击度, 偏差: 偏差}

            // 制动减速度
            IF 实际参数.制动减速度 != NULL AND 行为类型 IN [制动, 刹停]:
                // 区分紧急制动（紧急制动不参与 V 值评判，由 S 值主导）
                IF 实际参数.制动减速度 > 7.0:
                    各维度得分["制动减速度"] = {实际: 实际参数.制动减速度, 备注: "紧急制动，不纳入V值"}
                ELSE:
                    偏差 = 计算负偏差(实际值=实际参数.制动减速度, 基准值=风格基准.制动减速度, 上限=7.0)
                    总加权偏差 += 分槽权重.制动减速度 × 偏差
                    各维度得分["制动减速度"] = {实际: 实际参数.制动减速度, 基准: 风格基准.制动减速度, 偏差: 偏差}

            // 起步加速度
            IF 实际参数.起步加速度 != NULL AND 行为类型 == 起步:
                偏差 = 计算负偏差(实际值=实际参数.起步加速度, 基准值=风格基准.起步加速度, 上限=5.0)
                总加权偏差 += 分槽权重.起步加速度 × 偏差
                各维度得分["起步加速度"] = {实际: 实际参数.起步加速度, 基准: 风格基准.起步加速度, 偏差: 偏差}

            // 转弯车速比
            IF 实际参数.转弯车速比 != NULL AND 行为类型 == 转弯:
                偏差 = 计算负偏差(实际值=实际参数.转弯车速比, 基准值=风格基准.转弯车速比, 上限=0.9)
                总加权偏差 += 分槽权重.转弯车速比 × 偏差
                各维度得分["转弯车速比"] = {实际: 实际参数.转弯车速比, 基准: 风格基准.转弯车速比, 偏差: 偏差}

            // 变道间隙
            IF 实际参数.变道间隙 != NULL AND 行为类型 == 变道:
                偏差 = 计算正偏差(实际值=实际参数.变道间隙, 基准值=风格基准.变道间隙, 下限=法规下限)
                总加权偏差 += 分槽权重.变道间隙 × 偏差
                各维度得分["变道间隙"] = {实际: 实际参数.变道间隙, 基准: 风格基准.变道间隙, 偏差: 偏差}

            // 刹停点头
            IF 实际参数.刹停冲击度 != NULL AND 行为类型 == 刹停:
                偏差 = 计算负偏差(实际值=实际参数.刹停冲击度, 基准值=风格基准.刹停点头, 上限=2.0)
                总加权偏差 += 分槽权重.刹停点头 × 偏差
                各维度得分["刹停点头"] = {实际: 实际参数.刹停冲击度, 基准: 风格基准.刹停点头, 偏差: 偏差}

            // 3d. 综合 V 值
            V = CLAMP(1.0 - 总加权偏差, 0.0, 1.0)

            // 3e. 输出结果
            V 值结果 = {
                条目 ID: 条目 ID,
                V 值: V,
                各维度得分: 各维度得分,
                计算时间戳: NOW()
            }
            向 ad-36 发送 V 值计算结果

            // 3f. 连续低 V 值告警
            近期 V 值历史.append({条目 ID, V})
            IF 近期 V 值历史.length > 10:
                移除最旧记录
            IF 近期 V 值历史.length >= 5 AND 平均 V 值 < 0.3:
                向 ad-03、ad-01 发送风格偏差告警
                近期 V 值历史.clear()

        // 第 4 步：周期性状态上报
        IF 距上次上报 >= 60 秒:
            向 ad-03、ad-01 上报计算状态

        SLEEP 50ms


FUNCTION 计算正偏差(实际值, 基准值, 下限):
    // 实际值 ≥ 基准值 → 无偏差
    IF 实际值 >= 基准值:
        RETURN 0.0
    ELSE:
        RETURN MIN((基准值 - 实际值) / (基准值 - 下限), 1.0)

FUNCTION 计算负偏差(实际值, 基准值, 上限):
    // 实际值 ≤ 基准值 → 无偏差
    IF 实际值 <= 基准值:
        RETURN 0.0
    ELSE:
        RETURN MIN((实际值 - 基准值) / (上限 - 基准值), 1.0)
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 用户未选择风格（风格设定=NULL） | 默认使用“标准通勤”风格基准 | — |
| 某行为维度数据缺失 | 该维度不参与本次 V 值计算，权重归零，其他维度权重等比放大 | — |
| 紧急制动（减速度 > 7m/s²） | 制动维度不参与 V 值计算，由 S 值主导 | — |
| 用户切换风格 | 状态切换 UPDATING，加载新基准后恢复 NORMAL | 参数加载完成 |
| 车辆物理约束参数未加载 | 使用编译期默认上限值，上报告警 | 参数加载完成 |
| 分槽权重配置缺失 | 使用标准默认权重，上报告警 | 配置补全 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 紧急制动事件（减速度 > 7m/s²）不参与 V 值计算，避免因安全操作而误判为“风格不匹配” |
| S-02 | 特殊环境槽（ad-18）经验 V 值强制置 1.0，安全优先于风格评判 |
| S-03 | 任何风格参数的物理/法规下限硬编码（如跟车时距 ≥ 1.8s），基准值不可低于此下限 |
| S-04 | V 值计算仅用于经验评估与晋升参考，不可作为实时驾驶干预的依据 |
| S-05 | 用户风格偏好数据属于个人隐私，不参与云端同步或导出 |
