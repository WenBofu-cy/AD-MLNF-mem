# ad-13-子画像槽长期未活跃提醒单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-13 |
| 模块名称 | 子画像槽长期未活跃提醒单元 |
| 所属分区 | 二、漏斗一：驾驶员画像漏斗 |
| 核心职责 | 检测超过 90 天未活跃的长期槽，向用户发出提醒，仅执行用户手动确认的删除。绝对禁止自动删除长期槽。定期扫描各长期槽的最后活跃时间戳，生成未活跃槽清单并推送至中控屏 |
| 依赖模块 | ad-10（行为累积统计单元，提供最后活跃时间戳更新）、ad-02（漏斗一专属调度单元，获取长期槽列表与用户反馈） |
| 被依赖模块 | 中控屏交互模块（硬件，接收提醒显示）、ad-02（接收用户确认删除指令） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 定期扫描未活跃槽，正常推送提醒 | 系统初始化完成，漏斗一处于人工驾驶模式 |
| 提醒冷却中 | `REMINDER_COOLDOWN` | 已推送过提醒，冷却期内不再重复推送 | 同一次驾驶循环中已推送过同类提醒 |
| 用户已确认待删除 | `AWAIT_DELETION` | 用户已在中控屏确认删除某个长期槽，等待 ad-02 下发最终删除指令 | 用户手动确认删除 |
| 暂停服务 | `PAUSED` | 漏斗一冻结或非人工驾驶模式，暂停提醒 | 驾驶模式切换或紧急接管 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 最后活跃时间戳更新 | Struct（槽号 + 最后活跃时间 + 驾驶员身份 ID 掩码） | ad-10 行为累积统计单元 | 每次行为统计更新后（约每 5 秒） | 普通 |
| 长期槽列表 | List（槽号 + 创建日期 + 驾驶员掩码名称） | ad-02 漏斗一专属调度单元 | 系统初始化及槽位变更时 | **高** |
| 用户手动确认删除信号 | Struct（目标槽号 + 确认删除） | 中控屏（通过 ad-02 转发） | 用户在中控屏操作 | **高** |
| 驾驶模式信号 | Enum（人工驾驶/自动驾驶/紧急接管） | ad-02 漏斗一专属调度单元 | 驾驶模式变更 | **高** |
| 系统时钟信号 | UTC 时间戳 | 系统时钟 | 周期性，每 60 秒 | 普通 |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 未活跃槽提醒请求 | Struct（未活跃槽列表 + 建议操作 + 提醒等级） | 中控屏（通过车内显示总线） | 检测到超过 90 天未活跃的长期槽 | 普通 |
| 删除触发建议 | Struct（目标槽号 + 用户已确认 + 建议立即删除） | ad-02 漏斗一专属调度单元 | 用户确认删除后，转发正式删除请求 | **高** |
| 提醒事件日志 | Struct（槽号 + 未活跃天数 + 提醒时间 + 用户响应） | ad-51 记忆变更日志追溯单元 | 每次提醒及用户响应后 | 普通 |
| 状态上报 | Struct（当前状态 + 未活跃槽数量 + 已提醒次数） | ad-01 总控漏斗 F₀ | 每 60 秒周期性上报 | 普通 |


## 核心处理逻辑

```
FUNCTION inactivity_reminder_main_loop():
    SET internal_state = NORMAL
    初始化 已提醒槽记录 字典（槽号 -> 最后提醒时间）
    last_scan_time = 0
    扫描间隔 = 24 小时（或每次车辆启动时立即扫描一次）

    WHILE 系统运行中:
        // 第 1 步：驾驶模式与状态检查
        IF 驾驶模式 != 人工驾驶 OR 收到紧急熔断指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF internal_state == PAUSED AND 驾驶模式 == 人工驾驶:
            SET internal_state = NORMAL

        // 第 2 步：更新活跃时间戳（持续接收 ad-10 更新）
        IF 收到 ad-10 最后活跃时间戳更新:
            更新内存中对应槽号的 last_active_time

        // 第 3 步：定期扫描未活跃槽（每 24 小时或车辆启动后首次）
        IF (NOW() - last_scan_time >= 24 小时) OR (车辆刚启动 AND 首次扫描未执行):
            last_scan_time = NOW()
            unactive_slots = []

            FOR EACH 长期槽 IN 长期槽列表:
                最后活跃 = 内存中槽号的 last_active_time
                IF 最后活跃 == NULL:
                    最后活跃 = 槽创建时间  // 从未有过活跃记录

                未活跃天数 = DAYS(NOW() - 最后活跃)
                IF 未活跃天数 > 90:
                    // 检查冷却期：如果在上次提醒后的 30 天内且用户未响应，暂不重复提醒
                    IF 槽号 IN 已提醒槽记录:
                        IF (NOW() - 已提醒槽记录[槽号]) < 30 天:
                            CONTINUE  // 还在冷却期，跳过
                    
                    unactive_slots.append({
                        槽号: 槽号,
                        未活跃天数: 未活跃天数,
                        建议操作: "长期未使用，建议删除以释放空间"
                    })

            IF unactive_slots 非空:
                // 将提醒推送至中控屏（每次驾驶循环仅推送一次）
                向中控屏发送未活跃槽提醒请求( unactive_slots )
                FOR EACH slot IN unactive_slots:
                    已提醒槽记录[slot.槽号] = NOW()
                向 ad-51 写入提醒事件日志

        // 第 4 步：处理用户手动确认删除
        IF 收到用户手动确认删除信号:
            目标槽号 = 信号.目标槽号
            // 再次确认该槽确实超过 90 天未活跃（防止用户误操作或界面延迟）
            IF 槽号对应的未活跃天数 > 90:
                SET internal_state = AWAIT_DELETION
                向 ad-02 发送删除触发建议（目标槽号 + 用户已确认 + 类型=长期槽手动删除）
                向 ad-51 写入用户确认删除日志
                // 从已提醒槽记录中移除以防止后续误提醒
                REMOVE 已提醒槽记录[目标槽号]
                SET internal_state = NORMAL
            ELSE:
                // 活跃状态已改变（可能在提醒后用户使用了该槽），忽略删除
                向中控屏返回提示“该账户近期已活跃，无需删除”

        // 第 5 步：周期性状态上报
        IF 距上次上报 >= 60 秒:
            向 ad-01 上报状态

        SLEEP 10 秒  // 低频轮询即可
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 长期槽从未有过活跃记录（新创建但未使用） | 使用槽创建时间作为最后活跃时间计算 | — |
| 用户连续忽略提醒超过 90 天（未活跃持续 > 180 天） | 每 30 天提醒一次，不执行自动删除。提醒等级可逐步提升（如从“建议”变为“强烈建议”） | 用户手动确认删除或账户重新活跃 |
| 用户在确认删除后立即取消 | 删除操作尚未执行（ad-02 收到请求但用户可在执行前通过中控屏撤销），本模块记录取消日志 | — |
| 驾驶模式切换为非人工驾驶 | 暂停扫描与提醒，状态 PAUSED | 恢复人工驾驶 |
| 紧急熔断 | 暂停一切提醒，状态 PAUSED | 紧急解除后恢复 |
| 中控屏硬件故障无法显示提醒 | 上报告警，缓存未活跃槽清单，待硬件恢复后补发 | 硬件恢复 |
| ad-02 未返回长期槽列表 | 本次扫描跳过，使用上一次缓存的列表，上报告警 | ad-02 恢复通信 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| MemoryBus | 读 | 最后活跃时间戳更新 | 只读 | 由 ad-10 周期性发送 |
| MemoryBus | 读 | 长期槽列表 | 只读 | 由 ad-02 在初始化及变更时下发 |
| MemoryBus | 读 | 用户手动确认删除信号 | 只读 | 由 ad-02 转发（来自中控屏） |
| MemoryBus | 读 | 驾驶模式信号 | 只读 | 由 ad-02 下发 |
| MemoryBus | 写 | 删除触发建议 | 专属写入 | 向 ad-02 发送（用户已确认删除） |
| MemoryBus | 写 | 提醒事件日志 | 专属写入 | 向 ad-51 写入 |
| 车内显示总线 | 写 | 未活跃槽提醒请求 | 专属写入 | 向中控屏发送 |
| MemoryBus | 写 | 状态上报 | 周期性写入 | 每 60 秒向 ad-01 上报 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 绝对禁止自动删除任何长期槽。即使用户连续忽略提醒，系统也不得自行执行删除操作。删除的唯一触发条件是用户手动在中控屏确认删除 |
| S-02 | 用户确认删除后，本模块仅转发“建议删除”请求至 ad-02，最终执行仍由 ad-02 调度 ad-12 完成，且需经过 ad-06 权限校验 |
| S-03 | 未活跃槽提醒内容不得包含驾驶员的真实姓名或身份 ID 明文，仅以掩码名称（如“用户1”）或槽编号显示 |
| S-04 | 提醒频率最低为 30 天一次，避免频繁骚扰用户。用户可在中控屏关闭长期未活跃提醒（但不影响 90 天计时） |
| S-05 | 所有提醒及用户确认删除操作均全量写入 ad-51 不可变日志 |

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-13-01 | NORMAL 状态，有长期槽 A 最后活跃在 95 天前，距上次提醒超过 30 天 | 定期扫描触发 | 生成未活跃清单，推送至中控屏（建议删除），记录日志，更新提醒时间 |
| TC-13-02 | 用户在中控屏对槽 A 点击“确认删除” | 用户手动确认删除信号（槽 A） | 校验未活跃天数>90 → 向 ad-02 发送删除触发建议 → 记录日志 |
| TC-13-03 | 槽 B 最后活跃在 60 天前 | 定期扫描触发 | 不在未活跃清单中，不提醒 |
| TC-13-04 | 用户已确认删除槽 A，但尚未执行 | 再次收到扫描触发 | 槽 A 已从提醒记录移除，不再提醒 |
| TC-13-05 | 上次提醒在 15 天前，槽 A 仍未活跃（未活跃 110 天） | 定期扫描触发 | 冷却期未过（<30天），本次扫描跳过，不重复提醒 |
| TC-13-06 | 用户忽略提醒，槽 A 未活跃 210 天 | 定期扫描触发（距上次提醒 30 天） | 再次提醒，提醒等级提升为“强烈建议” |
| TC-13-07 | 驾驶模式切换为自动驾驶 | 扫描触发信号 | 暂停服务，状态 PAUSED |
| TC-13-08 | 用户在中控屏关闭“长期未活跃提醒” | 扫描触发 | 不推送中控屏显示，但仍记录日志 |
| TC-13-09 | ad-10 更新槽 A 的最后活跃时间为今天 | 接收活跃时间戳更新 | 槽 A 从未活跃清单中自动移除，若已推送提醒则下次不再提醒 |
| TC-13-10 | 系统启动时检测到长期槽 C 从未有过活跃记录，创建于 100 天前 | 启动扫描 | 计算未活跃 100 天，触发提醒 |
