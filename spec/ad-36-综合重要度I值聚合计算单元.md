# ad-36-综合重要度I值聚合计算单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-36 |
| 模块名称 | 综合重要度 I 值聚合计算单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 三维重要度计算引擎 |
| 核心职责 | 执行三维重要度驱动公式 `I = I₀ + α·S + β·V + γ·C`，将基础重要度 I₀、安全显著性 S、风格匹配度 V、复用频次 C 四项因子按权重系数聚合为综合重要度 I 值。I 值是漏斗二中所有晋升、遗忘、排序决策的唯一量化依据 |
| 依赖模块 | ad-31（S 值输入）、ad-32（V 值输入）、ad-33（C 值输入）、ad-34（I₀ 输入）、ad-35（权重系数 α/β/γ 输入） |
| 被依赖模块 | ad-38（晋升判定，消费 I 值）、ad-40（遗忘判定，消费 I 值）、ad-21（L1 时序衰减，参考 I 值）、ad-37（定时刷新，触发 I 值更新）、ad-28（L5 存储，S≥0.9 时 I 值直达写入） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 接收各因子输入，完成计算后输出 I 值 | 系统初始化完成，权重系数已加载 |
| 等待数据补全 | `WAITING_DATA` | 某条目因子尚未全部到达，暂存已到因子 | 新条目首次因子到达，但缺少 ≥1 项因子 |
| 批量重算中 | `BATCH_RECALC` | 权重系数变更或定时刷新触发，正在批量更新全量条目的 I 值 | ad-37 触发或 ad-35 权重变更 |
| 暂停服务 | `PAUSED` | 系统紧急熔断，暂停计算 | 收到 ad-01 紧急熔断指令 |

|--------|----------|----------|----------|:---:|
| S 值计算结果 | Struct（条目 ID + S 值 + 计算时间戳） | ad-31 安全显著性 S 值计算单元 | S 值计算完成后 | **高** |
| V 值计算结果 | Struct（条目 ID + V 值 + 计算时间戳） | ad-32 风格匹配度 V 值计算单元 | V 值计算完成后 | **高** |

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| I 值计算结果 | Struct（条目 ID + I 值 + 各因子明细 + 计算时间戳 + 数据完整性标记） | ad-38 晋升双条件判定单元、ad-40 遗忘阈值判定单元 | 每次 I 值计算或更新完成后 | **高** |

| 计算状态上报 | Struct（当前状态 + 近期平均 I 值 + I 值分布 + 待补全条目数） | ad-03、ad-01 | 每 60 秒周期性上报 | 普通 |


## I 值计算公式与规则

### 一、核心公式

```
I = CLAMP( I₀ + α·S + β·V + γ·C , 0.05 , 1.0 )
```

| 因子 | 含义 | 取值范围 | 来源模块 | 说明 |
|------|------|:---:|------|------|
| I₀ | 基础重要度 | [0.05, 0.90] | ad-34 | 经验固有价值基线 |
| S | 安全显著性 | [0.0, 1.0] | ad-31 | 场景安全风险 |
| V | 风格匹配度 | [0.0, 1.0] | ad-32 | 用户风格契合度 |
| C | 复用频次 | [0.0, 1.0) | ad-33 | 实战验证程度 |
| α | S 权重 | [0.30, 0.70] | ad-35 | 默认 0.50 |
| β | V 权重 | [0.10, 0.40] | ad-35 | 默认 0.20 |
| γ | C 权重 | [0.10, 0.50] | ad-35 | 默认 0.30 |

> 硬约束：α + β + γ = 1.0。I 值下限 0.05（新经验最低保底），上限 1.0。

### 二、数据到达策略

| 场景 | 策略 |
|------|------|
| 新条目四条因子全部到达 | 立即计算 I 值，输出结果，数据完整性=完整 |
| 新条目部分因子到达 | 进入等待数据补全，最长等待 30 秒。超时后用当前已有因子计算（缺失因子按 0 处理），数据完整性=部分 |
| 已有条目的 C 值更新（复用触发） | I₀/S/V 使用最近缓存值，结合新 C 值立即重算 I |
| 权重系数变更 | 触发批量重算，所有已缓存条目按新权重重新计算 I |
| 已有条目的 S/V/I₀ 更新 | 立即重算该条目的 I 值 |

### 三、各分槽权重微调规则

> 当条目来自特定分槽时，实际计算使用的权重 = 该分槽在 ad-35 中配置的专属权重（见 ad-35 参数库），而非全局默认权重。本模块从 ad-35 获取时自动带出对应分槽的专属权重。


## 核心处理逻辑

```
FUNCTION i_value_aggregation_main_loop():
    SET internal_state = NORMAL
    I 值缓存 = {}  // 条目 ID → {I₀, S, V, C, I值, 各因子时间戳, 数据完整性}
    权重系数 = {α: 0.50, β: 0.20, γ: 0.30}
    权重版本号 = 1
    待补全队列 = {}  // 条目 ID → {已到因子, 缺失因子列表, 首次到达时间}

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 第 2 步：接收权重系数
        IF 收到 ad-35 权重系数数据包:
            权重系数 = 数据包.{α, β, γ}
            权重版本号 = 数据包.版本号
            // 权重变更，触发全量重算
            SET internal_state = BATCH_RECALC
            FOR EACH 条目 ID, 缓存数据 IN I 值缓存:
                IF 缓存数据.数据完整性 == "完整":
                    新 I 值 = CLAMP(缓存数据.I₀ + 权重系数.α × 缓存数据.S + 权重系数.β × 缓存数据.V + 权重系数.γ × 缓存数据.C, 0.05, 1.0)
                    缓存数据.I值 = 新 I 值
                    向 ad-38、ad-40 发送 I 值计算结果(条目 ID, 新 I 值, 各因子明细, 数据完整性)
            SET internal_state = NORMAL
            CONTINUE

        // 第 3 步：接收各因子输入
        IF 收到因子输入（S / V / C / I₀）AND internal_state == NORMAL:
            条目 ID = 因子输入.条目 ID
            因子类型 = 判断输入类型(S/V/C/I₀)

            // 3a. 初始化或获取缓存
            IF 条目 ID NOT IN I 值缓存:
                I 值缓存[条目 ID] = {I₀: NULL, S: NULL, V: NULL, C: 0.0, I值: 0.05}
                待补全队列[条目 ID] = {已到因子: [因子类型], 缺失因子: [S,V,C,I₀] - 因子类型, 首次到达时间: NOW()}
                SET internal_state = WAITING_DATA

            // 3b. 更新缓存中的对应因子
            缓存 = I 值缓存[条目 ID]
            IF 因子类型 == "S": 缓存.S = 因子输入.S值; 缓存.S时间戳 = NOW()
            ELSE IF 因子类型 == "V": 缓存.V = 因子输入.V值; 缓存.V时间戳 = NOW()
            ELSE IF 因子类型 == "C": 缓存.C = 因子输入.C值; 缓存.C时间戳 = NOW()
            ELSE IF 因子类型 == "I₀": 缓存.I₀ = 因子输入.I₀值; 缓存.I₀时间戳 = NOW()

            // 3c. 更新待补全队列
            IF 条目 ID IN 待补全队列:
                待补全队列[条目 ID].缺失因子.REMOVE(因子类型)

            // 3d. 判断是否可计算
            IF 缓存.I₀ != NULL AND 缓存.S != NULL AND 缓存.V != NULL:
                // 四条因子齐备，立即计算
                缓存.数据完整性 = "完整"
                新 I 值 = CLAMP(缓存.I₀ + 权重系数.α × 缓存.S + 权重系数.β × 缓存.V + 权重系数.γ × 缓存.C, 0.05, 1.0)
                缓存.I值 = 新 I 值

                向 ad-38、ad-40 发送 I 值计算结果(条目 ID, 新 I 值, 各因子明细, "完整")

                // L5 直达判定
                IF 新 I 值 >= 0.9 OR 缓存.S >= 0.9:
                    向 ad-28 发送 L5 直达写入触发确认(条目 ID, 新 I 值, 缓存.S)

                从待补全队列中移除 条目 ID
                IF internal_state == WAITING_DATA AND 待补全队列为空:
                    SET internal_state = NORMAL

            ELSE IF 条目 ID IN 待补全队列 AND NOW() - 待补全队列[条目 ID].首次到达时间 > 30 秒:
                // 超时，使用已有因子计算（缺失因子按 0）
                缓存.数据完整性 = "部分"
                缺失S = IF 缓存.S == NULL THEN 0.0 ELSE 缓存.S
                缺失V = IF 缓存.V == NULL THEN 0.0 ELSE 缓存.V
                缺失I₀ = IF 缓存.I₀ == NULL THEN 0.20 ELSE 缓存.I₀  // I₀ 缺失保守取 0.20
                新 I 值 = CLAMP(缺失I₀ + 权重系数.α × 缺失S + 权重系数.β × 缺失V + 权重系数.γ × 缓存.C, 0.05, 1.0)
                缓存.I值 = 新 I 值

                向 ad-03、ad-01 发送因子缺失告警(条目 ID, 待补全队列[条目 ID].缺失因子, 30秒)
                向 ad-38、ad-40 发送 I 值计算结果(条目 ID, 新 I 值, 各因子明细, "部分")

                从待补全队列中移除 条目 ID

        // 第 4 步：定时批量重算（权重未变但定时刷新）
        IF 收到 ad-37 批量重算触发信号 AND internal_state == NORMAL:
            SET internal_state = BATCH_RECALC
            FOR EACH 条目 ID, 缓存数据 IN I 值缓存:
                IF 缓存数据.数据完整性 == "完整":
                    新 I 值 = CLAMP(缓存数据.I₀ + 权重系数.α × 缓存数据.S + 权重系数.β × 缓存数据.V + 权重系数.γ × 缓存数据.C, 0.05, 1.0)
                    IF 新 I 值 != 缓存数据.I值:  // 仅更新发生变化的
                        缓存数据.I值 = 新 I 值
                        向 ad-38、ad-40 发送 I 值计算结果(条目 ID, 新 I 值, 各因子明细, "完整")
            SET internal_state = NORMAL

        // 第 5 步：周期性状态上报
        IF 距上次上报 >= 60 秒:
            统计 平均 I 值、I 值分布（0-0.2/0.2-0.4/0.4-0.6/0.6-0.8/0.8-1.0）、待补全条目数
            向 ad-03、ad-01 上报计算状态

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 新条目 I₀ 和 S 值同时缺失 | I₀ 缺失取保守值 0.20，S 缺失取 0.0。若超 30 秒仍缺失，以此值计算 | 因子到达后更新 |
| I 值计算溢出 > 1.0 | CLAMP 至 1.0，上报告警 | — |
| I 值计算 < 0.0 | CLAMP 至 0.05（下限），上报告警 | — |
| 权重系数未加载（初始化阶段） | 使用编译期默认值 α=0.50, β=0.20, γ=0.30，上报告警 | 权重系数加载完成 |
| 权重总和 ≠ 1.0（异常） | 自动按比例缩放至 1.0，上报告警 | ad-35 修正配置 |
| 某条目长期处于待补全（> 5 分钟） | 清除待补全队列中的该条目，缓存保留已有因子，等待下次因子到达时重新触发 | — |
| 批量重算期间新因子到达 | 新因子更新缓存，待批量重算结束后以最新值计算 | 批量重算完成 |
| 缓存条目数超过内存上限（> 100,000） | 清除最久未更新的 20% 缓存条目（L5 条目豁免），上报告警 | — |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | I 值计算公式编译期固化，运行时不接受非授权的公式修改 |
| S-02 | α + β + γ = 1.0 为强制约束，任何情况下实际计算使用的权重总和必须为 1.0 |
| S-03 | I 值 ≥ 0.9 或 S ≥ 0.9 时，必须触发 L5 直达写入确认，不得遗漏 |
| S-04 | 缺失因子超时后保守处理：I₀ 取 0.20（偏低），S 取 0.0（假设无风险，但后续由 S 更新修正） |
| S-05 | I 值缓存中的各因子须保留时间戳，在计算时检查各因子时效性。S/V/I₀ 超过 7 日未更新的条目触发重新计算请求 |
| S-06 | 所有 I 值计算结果全量写入 ad-51 变更日志 |
