# ad-17-泊车低速槽 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-17 |
| 模块名称 | 泊车低速槽 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 场景分槽管理 |
| 核心职责 | 承载泊车入库、狭窄空间穿行、人车混行区域蠕行等低速场景驾驶经验的完整五层存储与晋升管理。执行专属遗忘策略：最低重要度阈值下调 50%，大幅延长泊车与低速复杂场景经验的保留时间，避免低频但关键的操作经验被过早遗忘 |
| 依赖模块 | ad-14（场景判定与分槽路由单元，接收经验写入指令）、ad-20 至 ad-30（五层存储与晋升遗忘执行模块）、ad-36（综合重要度 I 值聚合计算单元） |
| 被依赖模块 | ad-14（上报存储占用率与活跃状态）、ad-03（漏斗二专属调度单元，接收状态汇总） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收经验条目并执行五层存储流转 | 漏斗二已激活，本槽已初始化 |
| 容量告急 | `CAPACITY_WARNING` | 存储占用率 > 90%，已触发加速遗忘机制 | 本槽存储占用率监测 |
| 维护中 | `MAINTENANCE` | 正在执行相似经验归并或冷归档操作 | 定期维护周期或容量告急触发 |
| 冻结只读 | `FROZEN` | 漏斗二冻结或紧急接管，暂停写入但保留查询 | 驾驶模式切换或紧急熔断 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 经验写入指令 | Struct（含经验条目 + 写入优先级 + 路由标记 + 判定置信度） | ad-14 场景判定与分槽路由单元 | 每次场景路由至本槽时 | **高** |
| L1 至 L4 晋升候选清单 | List（经验条目 ID + 晋升目标层级 + 重要度 I） | ad-38 晋升双条件判定单元 | 每个晋升判定周期 | **高** |
| 遗忘候选清单 | List（经验条目 ID + 当前层级 + I 值） | ad-40 遗忘阈值判定单元 | 每个遗忘判定周期 | **高** |
| 归并建议 | Struct（源经验 ID + 目标经验 ID + 相似度） | ad-25 L3 中期层相似经验归并单元 | L3 归并操作时 | 普通 |
| 全局调度指令 | Enum（冻结/激活/维护） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 存储完成回执 | Struct（经验条目 ID + 存储层级 + 存储时间戳） | ad-14 | 每次经验成功存储后 | 普通 |
| 晋升执行结果 | Struct（经验条目 ID + 原层级 + 目标层级 + 执行结果） | ad-38、ad-39 | 晋升完成后 | **高** |
| 遗忘执行结果 | Struct（经验条目 ID + 删除/归档 + 结果） | ad-40、ad-42 | 遗忘执行后 | 普通 |
| 槽状态快照 | Struct（存储占用率 + 各层级条目数 + 活跃度） | ad-14、ad-03 | 每 1 秒周期性上报 | 普通 |
| 容量告警 | Struct（当前占用率 + 触发阈值 + 建议动作） | ad-03、ad-01 | 占用率 > 90% 时 | **高** |


## 专属遗忘策略

| 策略项 | 标准值 | 本槽调整 | 调整理由 |
|--------|:---:|:---:|----------|
| 最低重要度遗忘阈值 | 0.15 | **0.075（下调 50%）** | 泊车与低速复杂场景操作频次低但操作精度要求高，遗忘阈值大幅降低，确保宝贵经验不被轻易丢弃 |
| 安全显著性权重（α） | 0.5 | 0.40 | 低速场景碰撞风险相对较低，权重适度让渡给复用和风格 |
| 复用频次权重（γ） | 0.3 | 0.35 | 停车的重复路径、特定车位的操作值得强化 |
| 风格匹配度权重（β） | 0.2 | 0.25 | 泊车舒适度与平顺性是重要评价维度 |
| L1→L2 晋升时间阈值 | 24h | 48h（延长） | 泊车场景重复周期可能跨天，延长临时层保护时间 |
| L2→L3 晋升时间阈值 | 7日 | 10日（延长） | 避免周末一次的成功泊车经验在下一周被过早晋升 |
| L3→L4 晋升时间阈值 | 30日 | 45日（延长） | 泊车技能需更长时间检验其泛化能力 |


## 核心处理逻辑

```
FUNCTION parking_low_speed_slot_main_loop():
    SET internal_state = NORMAL
    初始化五层存储结构（配额向 L3/L4 倾斜，因为经验少而精）
    加载专属遗忘策略：
        α=0.40, β=0.25, γ=0.35
        最低遗忘阈值=0.075
        L1→L2 时间=48h, L2→L3 时间=10日, L3→L4 时间=45日

    WHILE 系统运行中:
        // 1. 全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = FROZEN
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 2. 接收新经验
        IF 收到经验写入指令 AND internal_state == NORMAL:
            经验条目 = 写入指令.经验条目
            // 泊车低速场景：初始重要度 I₀ 默认 +0.05（鼓励保留）
            经验条目.I₀ = MIN(经验条目.I₀ + 0.05, 1.0)
            写入 L1
            向 ad-14 返回存储完成回执

        // 3. 晋升处理
        IF 收到晋升候选清单:
            FOR EACH 候选 IN 清单:
                IF 候选目标层级存在且存储未满:
                    调用 ad-39 搬运
                    返回成功
                ELSE:
                    返回失败

        // 4. 遗忘处理（阈值极低，大量保留）
        IF 收到遗忘候选清单:
            FOR EACH 候选 IN 清单:
                IF 候选.层级 IN [L4, L5]:
                    SKIP
                ELSE IF 候选.I < 0.075:  // 极低阈值
                    调用 ad-42 删除
                    返回已删除
                ELSE:
                    返回保留

        // 5. L3 归并（泊车场景相似度高，归并激进一些以节省空间）
        IF 距上次归并 >= 48h OR 占用率 > 85%:
            SET internal_state = MAINTENANCE
            向 ad-25 发送归并请求
            执行归并（相似度阈值放宽至 0.75）
            SET internal_state = NORMAL

        // 6. 容量监控
        占用率 = 计算占用率()
        IF 占用率 > 0.90:
            SET internal_state = CAPACITY_WARNING
            上报告警
            // 泊车槽不建议激进遗忘，优先触发归并和归档
            向 ad-03 建议增加配额或冷归档
        ELSE IF 占用率 < 0.70 AND internal_state == CAPACITY_WARNING:
            SET internal_state = NORMAL

        // 7. 状态上报
        IF 距上次上报 >= 1s:
            上报槽状态快照

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 泊车经验频繁触发遗忘但 I 值低 | 最低遗忘阈值 0.075 提供强力保护，只要不是完全无价值的经验都会保留 | — |
| 特定车位经验长期未复用（> 90天） | 不会自动删除，仅当容量告急且 I < 0.075 时才考虑清除 | — |
| 容量告急但无可归并经验 | 上报告警，启动 L3/L4 冷归档（ad-49）释放空间 | 手动扩容或归并完成 |
| 晋升时间阈值被人为缩短 | 校验下限（L1→L2 ≥ 24h, L2→L3 ≥ 7日, L3→L4 ≥ 30日），拒绝突破底线 | 参数恢复 |
| 冻结状态写入 | 拒绝，返回原因 | 激活指令 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 最低遗忘阈值下限为 0.075，编译期硬编码，不可被运行时配置下调 |
| S-02 | 晋升时间阈值下限（L1→L2: 24h, L2→L3: 7日, L3→L4: 30日）为编译期硬约束，不可突破 |
| S-03 | 泊车经验因涉及低速近距离人车交互，涉及行人的经验 I₀ 仍自动 +0.10 |
| S-04 | 容量告急时优先触发归并和冷归档，而非加速遗忘，以保护低频经验 |
| S-05 | 所有操作日志写入 ad-51 |
