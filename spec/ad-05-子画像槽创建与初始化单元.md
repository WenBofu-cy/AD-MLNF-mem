# ad-05-子画像槽创建与初始化单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-05 |
| 模块名称 | 子画像槽创建与初始化单元 |
| 所属分区 | 二、漏斗一：驾驶员画像漏斗 |
| 核心职责 | 为新驾驶员创建独立子画像槽，分配物理存储分区，初始化统计基线与元数据。将新驾驶员面部特征向量回写至 ad-04 的特征库。槽位上限、存储配额、初始化参数均受编译期硬约束 |
| 依赖模块 | ad-02（漏斗一专属调度单元，下发创建指令）、ad-04（驾驶员身份识别单元，接收新驾驶员注册通知）、ad-48（全局容量配额管控单元，校验剩余容量） |
| 被依赖模块 | ad-06（子画像槽数据隔离管控单元，接收新槽注册）、ad-07（驾驶行为观测记录单元，写入目标槽）、ad-10（行为累积统计单元，初始化统计基线） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无创建任务，等待指令 | 系统初始化完成、上一次创建任务结束 |
| 槽位校验中 | `VALIDATING` | 校验槽位类型、槽位上限、剩余容量是否满足创建条件 | 收到 ad-02 创建指令 |
| 分区分配中 | `ALLOCATING` | 正在物理存储分区中划分子画像槽专属空间 | 校验通过 |
| 基线初始化中 | `INITIALIZING` | 写入初始元数据、统计基线清零、关联驾驶员身份信息 | 分区分配完成 |
| 特征回写中 | `REGISTERING` | 将新驾驶员面部特征向量写入 ad-04 特征库 | 基线初始化完成 |
| 创建完成 | `DONE` | 子画像槽已就绪，通知下游模块 | 全部步骤完成 |
| 创建失败 | `FAILED` | 校验未通过或分配异常，返回失败原因 | 任一环节失败 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 子画像槽创建指令 | Struct（驾驶员身份 ID + 槽位类型 + 面部特征向量 + 请求来源） | ad-02 漏斗一专属调度单元 | ad-02 判定需要创建新槽时 | **高** |
| 槽位上限配置 | Int | 编译期常量 | 系统初始化 | — |
| 单槽存储配额 | Int（MB） | 编译期常量 | 系统初始化 | — |
| 全局剩余存储容量 | Int（MB） | ad-48 全局容量配额管控单元 | 创建前校验时查询 | 普通 |
| 已有槽位数量 | Int | ad-02 漏斗一专属调度单元 | 创建前校验时查询 | 普通 |
| 创建回滚指令 | Signal | ad-01 总控漏斗 F₀ | 全局熔断或紧急事件时 | **紧急** |

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|

| 面部特征回写 | Struct（驾驶员身份 ID + 特征向量） | ad-04 驾驶员身份识别单元 | 新驾驶员注册时 | **高** |

## 核心处理逻辑

```
FUNCTION slot_creation_main_loop():
    WHILE 系统运行中:
        // 第 1 步：紧急事件优先
        IF 收到 ad-01 创建回滚指令:
            终止当前创建任务
            回滚已分配分区（若已分配）
            SET internal_state = FAILED
            输出创建失败通知 TO ad-02
            CONTINUE

        // 第 2 步：等待创建指令
        IF 收到 ad-02 子画像槽创建指令:
            SET internal_state = VALIDATING

            // 2a. 槽位类型校验
            IF 槽位类型 NOT IN [长期, 临时, 一次性]:
                失败原因 = "非法槽位类型"
                GOTO 失败处理

            // 2b. 槽位上限校验
            IF 槽位类型 == 长期:
                查询当前长期槽数量
                IF 当前长期槽数量 >= 6:
                    失败原因 = "长期槽位已满（6/6）"
                    建议操作 = "提示用户释放旧槽或使用临时模式"
                    GOTO 失败处理
            ELSE IF 槽位类型 == 临时:
                查询当前临时槽数量
                IF 当前临时槽数量 >= 1:
                    失败原因 = "临时槽位已满（1/1）"
                    建议操作 = "降级为一次性记录槽"
                    GOTO 失败处理
            ELSE IF 槽位类型 == 一次性:
                查询当前一次性槽数量
                IF 当前一次性槽数量 >= 1:
                    // 覆盖当前一次性槽（数据在行程结束时即清除）
                    触发 ad-12 快速擦除当前一次性槽

            // 2c. 容量校验
            向 ad-48 查询全局剩余存储容量
            单槽配额 = 根据槽位类型确定：
                长期槽：预分配配额 × 1.0
                临时槽：预分配配额 × 0.3
                一次性槽：预分配配额 × 0.1
            IF 全局剩余容量 < 单槽配额:
                失败原因 = "存储容量不足"
                建议操作 = "提示用户清理旧数据或扩展存储"
                GOTO 失败处理

            // 第 3 步：分配物理存储分区
            SET internal_state = ALLOCATING
            在漏斗一专用存储分区内申请单槽配额大小的连续空间
            IF 分配失败:
                失败原因 = "存储分区分配失败"
                GOTO 失败处理
            生成新槽号 = 递增序列号（全局唯一，永不重用）
            记录存储分区指针（起始地址 + 大小）

            // 第 4 步：初始化统计基线
            SET internal_state = INITIALIZING
            写入元数据：
                - 驾驶员身份 ID
                - 槽位类型
                - 创建时间戳
                - 存储分区指针
                - 槽状态 = ACTIVE
            写入全零统计基线：
                - 跟车行为计数 = 0
                - 变道行为计数 = 0
                - 路口通行计数 = 0
                - 加减速计数 = 0
                - 让行行为计数 = 0
            通知 ad-10 初始化该槽的统计基线

            // 第 5 步：面部特征回写
            IF 请求来源 == 新驾驶员注册 AND 面部特征向量不为空:
                SET internal_state = REGISTERING
                向 ad-04 回写面部特征向量与身份 ID 绑定
                WAIT ad-04 回写确认

            // 第 6 步：下发创建完成通知
            SET internal_state = DONE
            向 ad-02 返回创建完成回执
            向 ad-06 发送新槽注册通知
            向 ad-48 更新存储分配记录
            向 ad-51 写入创建事件日志
            SET internal_state = IDLE
            CONTINUE

        // 失败处理
        LABEL 失败处理:
            SET internal_state = FAILED
            向 ad-02 返回创建失败通知（含失败原因码 + 建议操作）
            向 ad-51 写入失败事件日志
            回滚已分配分区（若已分配）
            SET internal_state = IDLE

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 长期槽位已满（已达 6 个上限） | 返回失败，建议用户手动释放旧槽或使用临时模式。编译期硬编码上限 6，不可绕过 | 用户手动删除或转存旧槽后释放槽位 |
| 临时槽位已满（已达 1 个） | 返回失败，建议降级为一次性记录槽。不自动覆盖已有临时槽（可能仍有活跃行程） | 当前临时槽行程结束自动清除后释放 |
| 一次性槽位已满（已达 1 个） | 自动触发 ad-12 快速擦除当前一次性槽（其数据本身行程结束时即清除），覆盖创建 | — |
| 全局剩余存储容量不足 | 返回失败，建议用户清理旧数据或扩展存储。上报告警 | 容量释放后 |
| 存储分区分配失败（内存碎片、硬件故障） | 返回失败，上报告警至 ad-01，标记存储模块为 DEGRADED | 存储硬件恢复、碎片整理完成 |
| 面部特征向量为空（新驾驶员注册但人脸未采集成功） | 跳过特征回写步骤，子画像槽正常创建。后续 ad-04 可在人脸采集成功后补充回写 | 人脸采集成功 |
| 创建过程中收到紧急熔断指令 | 立即终止当前操作，回滚已分配分区，返回失败通知 | 紧急事件解除后重新发起创建 |
| 新槽号生成溢出 | 编译期保留足够大的槽号空间，若溢出则触发全局告警 | — |

| 总线 | 操作 | 数据内容 | 权限 | 说明 |

| MemoryBus | 读 | 子画像槽创建指令 | 只读 | 由 ad-02 下发 |
| MemoryBus | 读 | 全局剩余存储容量 | 只读 | 向 ad-48 查询 |

| MemoryBus | 写 | 新槽注册通知 | 专属写入 | 向 ad-06 下发 |
| MemoryBus | 写 | 统计基线初始化数据 | 专属写入 | 向 ad-10 下发 |
| MemoryBus | 写 | 面部特征回写 | 专属写入 | 向 ad-04 回写 |
| MemoryBus | 写 | 创建失败通知 | 专属写入 | 向 ad-02 返回 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 长期槽位上限 6 为编译期硬编码常量，不可通过运行时配置、OTA 或用户设置绕过 |
| S-02 | 每个子画像槽分配独立物理存储分区，分区基址与大小在创建时锁定，不可动态扩缩 |
| S-03 | 新槽号使用全局递增序列号，永不重用，即使旧槽已删除也不可回收其槽号 |
| S-04 | 子画像槽创建记录（含驾驶员身份 ID、槽号、存储分区、时间戳）全量写入 ad-51 不可变日志 |
| S-05 | 一次性槽覆写前必须执行安全擦除（DoD 5220.22-M 单次覆写），旧数据不可残留 |
| S-06 | 面部特征向量写入 ad-04 前须确认用户已授权人脸识别，未授权则跳过回写步骤 |
| S-07 | 紧急熔断时立即终止创建操作并回滚，不残留未完成的存储分区 |
| S-08 | 子画像槽创建后默认编译期禁止接入自动驾驶决策链路，该隔离由 ad-06 强制执行 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-05-01 | 系统空闲，长期槽位 3/6，剩余容量充足 | 创建指令（新驾驶员 ID=王五，槽位类型=长期，含面部特征） | 创建成功，返回新槽号与存储分区指针，通知 ad-06、ad-10、ad-04、ad-48、ad-51 |
| TC-05-02 | 长期槽位已满 6/6 | 创建指令（新驾驶员，槽位类型=长期） | 返回失败，原因码=“长期槽位已满”，建议操作=“提示用户释放旧槽或使用临时模式” |
| TC-05-03 | 临时槽位已满 1/1，当前仍有活跃行程 | 创建指令（新驾驶员，槽位类型=临时） | 返回失败，原因码=“临时槽位已满”，建议操作=“降级为一次性记录槽” |
| TC-05-04 | 一次性槽位已满 1/1 | 创建指令（新驾驶员，槽位类型=一次性） | 触发 ad-12 快速擦除旧一次性槽，覆盖创建成功 |
| TC-05-05 | 剩余容量不足以分配长期槽配额 | 创建指令（槽位类型=长期） | 返回失败，原因码=“存储容量不足” |
| TC-05-06 | 非法槽位类型 = “永久” | 创建指令（槽位类型=非法值） | 返回失败，原因码=“非法槽位类型” |
| TC-05-07 | 创建过程中收到紧急熔断指令 | 创建指令（正常）→ 中途收到 ad-01 回滚指令 | 立即终止，回滚已分配分区，返回失败通知 |
| TC-05-08 | 新驾驶员注册，但面部特征向量为空 | 创建指令（含空特征向量） | 跳过特征回写步骤，其余正常创建成功 |
| TC-05-09 | 创建成功 | 创建完成回执 | ad-02 收到回执，ad-06 收到注册通知，ad-10 初始化基线，ad-51 记录日志 |
