# ad-33-复用频次C值统计单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-33 |
| 模块名称 | 复用频次 C 值统计单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 三维重要度计算引擎 |
| 核心职责 | 统计每条驾驶经验条目在同类场景下被成功查询调用的标准化次数，计算复用频次分值 C（0–1）。C 值代表该经验的实战验证程度，复用次数越高，经验越成熟可靠，重要度权重越应得到强化 |
| 依赖模块 | ad-36（综合重要度 I 值聚合计算单元，接收 C 值）、各场景分槽及 ECC 模块（提供经验查询命中日志） |
| 被依赖模块 | ad-36（消费 C 值参与 I 值计算）、ad-37（重要度增量定时刷新单元，触发周期更新） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收查询日志并更新 C 值 | 漏斗二激活，本单元初始化完成 |
| 批量刷新中 | `BATCH_REFRESH` | 正在周期性批量更新全量条目的 C 值 | 每 24 小时触发或配置变更时 |
| 暂停服务 | `PAUSED` | 漏斗二冻结或紧急熔断 | 驾驶模式切换或紧急事件 |

|--------|----------|----------|----------|:---:|

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| C 值更新结果 | Struct（条目 ID + 新 C 值 + 累计复用次数 + 最近复用时间） | ad-36 综合重要度 I 值聚合计算单元 | 每次 C 值更新后 | **高** |
| 冷条目清单 | List（条目 ID + 上次复用距今时长 + 建议关注） | ad-40 遗忘阈值判定单元 | 批量刷新时，识别长期未被复用的条目 | 普通 |
| 统计状态上报 | Struct（当前状态 + 近期平均 C 值 + 高复用条目数） | ad-03、ad-01 | 每 60 秒周期性上报 | 普通 |


## C 值计算模型

### 一、核心公式

```
C = 累计复用次数 / (累计复用次数 + K)

其中 K 为半衰常数，默认 K = 5，表示复用 5 次时 C 值达到 0.5。
```

**衰减变体（考虑时间衰减的 C 值，用于遗忘辅助）：**
```
C_decay = C × 时间衰减因子

时间衰减因子 = 1.0  (最近复用 < 30 日)
             = 0.8  (最近复用 30–90 日)
             = 0.5  (最近复用 90–180 日)
             = 0.2  (最近复用 > 180 日)
```

> 标准 C 值用于 I 值计算（不衰减）。C_decay 仅供遗忘判定参考。

### 二、各层级 C 值初始值与特殊规则

| 层级 | 初始 C 值 | 特殊规则 |
|------|:---:|------|
| L1 临时层 | 0.0 | 新条目无复用，首次被查询后更新 |
| L2 近期层 | 继承 L1 的复用计数 | — |
| L3 中期层 | 继承 L2 的复用计数 | 归并时复用计数相加 |
| L4 长期层 | 继承 L3 的复用计数 | 提炼后原条目保留，复用计数不清零 |
| L5 核心层 | 继承 L4 的复用计数 | 不可遗忘，C 值仅用于统计 |

### 三、复用次数的有效计数条件

**有效复用**：经验被 ECC 大脑查询后，该经验提供的策略被 ECC 实际采用并执行成功。仅查询但未采用不计数。

**成功采用判定**：经验查询命中 → ECC 基于该经验生成决策 → 决策执行后结果标签为“成功优化”或“不可抗力场景”。

> 由 ECC-04 心智模拟模块在执行后反馈采用结果。


## 核心处理逻辑

```
FUNCTION c_value_statistics_main_loop():
    SET internal_state = NORMAL
    C 值缓存 = {}  // 条目 ID → {累计复用次数, 最近复用时间, C 值}
    last_refresh_time = NOW()

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 第 2 步：接收查询命中日志，实时更新 C 值
        IF 收到经验查询命中日志 AND internal_state == NORMAL:
            条目 ID = 命中日志.条目 ID
            采用结果 = 命中日志.采用结果（来自 ECC-04 反馈）

            // 仅当实际采用并成功时计数
            IF 采用结果 NOT IN ["成功优化", "不可抗力场景"]:
                CONTINUE

            // 更新缓存
            IF 条目 ID NOT IN C 值缓存:
                C 值缓存[条目 ID] = {累计复用次数: 0, 最近复用时间: NULL, C 值: 0.0}

            C 值缓存[条目 ID].累计复用次数 += 1
            C 值缓存[条目 ID].最近复用时间 = NOW()
            新 C 值 = C 值缓存[条目 ID].累计复用次数 / (C 值缓存[条目 ID].累计复用次数 + 5)
            C 值缓存[条目 ID].C 值 = 新 C 值

            // 推送更新
            向 ad-36 发送 C 值更新结果(条目 ID, 新 C 值, 累计复用次数, 最近复用时间)

        // 第 3 步：周期性批量刷新（24 小时）
        IF 收到 ad-37 周期刷新触发信号 OR (NOW() - last_refresh_time >= 24 小时):
            SET internal_state = BATCH_REFRESH
            last_refresh_time = NOW()

            冷条目清单 = []
            FOR EACH 条目 ID, 缓存数据 IN C 值缓存:
                // 重新计算 C 值（确保一致性）
                新 C 值 = 缓存数据.累计复用次数 / (缓存数据.累计复用次数 + 5)
                缓存数据.C 值 = 新 C 值

                // 检查是否为冷条目（超过 90 天未复用）
                距上次复用 = IF 缓存数据.最近复用时间 != NULL THEN NOW() - 缓存数据.最近复用时间 ELSE 999 日
                IF 距上次复用 > 90 日:
                    冷条目清单.append({条目 ID, 上次复用距今: 距上次复用})

                // 推送更新
                向 ad-36 发送 C 值更新结果(条目 ID, 新 C 值, 缓存数据.累计复用次数, 缓存数据.最近复用时间)

            // 向遗忘模块提供冷条目参考
            IF 冷条目清单不为空:
                向 ad-40 发送冷条目清单

            SET internal_state = NORMAL

        // 第 4 步：周期性状态上报
        IF 距上次上报 >= 60 秒:
            统计 平均 C 值、高复用条目数(C>0.8)
            向 ad-03、ad-01 上报统计状态

        SLEEP 200ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 条目尚未被任何查询命中 | C 值保持 0.0，由 I₀ 独立支撑初始重要度 | — |
| 条目被删除后仍收到查询日志 | 忽略该条目的更新，清理缓存中对应记录 | — |
| 批量刷新时条目数量极大（> 10,000） | 分批处理，每批 1000 条，避免阻塞 | 刷新完成 |
| 查询日志中采用结果字段缺失 | 默认不计数（保守处理），上报告警 | 字段恢复 |
| 系统时间异常导致最近复用时间错乱 | 使用系统启动时间戳校正，上报告警 | 系统时间恢复正常 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 复用频次计数必须基于“采用成功”，仅查询未采用的经验不得计数，防止无效经验被错误强化 |
| S-02 | C 值仅作为经验成熟度的辅助指标，不可替代安全显著性 S 值在 I 值计算中的核心地位 |
| S-03 | 冷条目清单仅供遗忘判定的辅助参考，最终遗忘决策仍由 ad-40 综合 I 值、S 值、留存时长等多维因素做出 |
| S-04 | C 值衰减变体 C_decay 仅用于遗忘辅助，不参与正式 I 值计算 |
| S-05 | 查询日志中的来源模块与场景信息不可包含驾驶员身份数据 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-33-01 | 条目 A 首次被成功采用 | 查询命中日志（条目 A，采用结果=成功优化） | 累计复用次数=1，C=1/(1+5)≈0.167，推送更新 |
| TC-33-02 | 条目 B 累计已成功采用 5 次 | 第 5 次成功采用 | 累计复用次数=5，C=5/(5+5)=0.5，推送更新 |
| TC-33-03 | 条目 C 被查询但未采用 | 查询命中日志（条目 C，采用结果=策略失误） | 不计数，C 值不变 |
| TC-33-04 | 条目 D 累计 20 次成功采用 | 第 20 次成功采用 | C=20/(20+5)≈0.8，高复用标记 |
| TC-33-05 | 批量刷新触发 | 条目 E 累计 3 次，最近复用 100 天前 | C=3/8=0.375，C_decay=0.375×0.5=0.1875，纳入冷条目清单 |
| TC-33-06 | 条目 F 最近复用 10 天前 | 批量刷新 | C_decay=0.375×1.0=0.375，不纳入冷条目清单 |
| TC-33-07 | 条目 G 从未被复用 | 批量刷新 | C=0.0，距上次复用=999，纳入冷条目清单 |
| TC-33-08 | 条目 H 已被删除 | 收到其查询日志 | 忽略更新，清理缓存 |
| TC-33-09 | PAUSED | 查询命中日志 | 不处理 |
| TC-33-10 | 条目 I 缓存不存在但收到查询日志 | 查询命中日志（条目 I，首次出现） | 新建缓存，累计=1，C≈0.167 |
