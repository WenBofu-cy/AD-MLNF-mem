# ad-16-城区路口槽 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-16 |
| 模块名称 | 城区路口槽 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 场景分槽管理 |
| 核心职责 | 承载城市道路场景下红绿灯通行、人行横道礼让、无保护左转、拥堵跟车等驾驶经验的完整五层存储与晋升管理。执行专属遗忘策略：安全显著性权重上调 20%，强化涉及行人及高风险场景的经验保留优先级。是本场景经验从 L1 到 L5 的完整生命周期管理者 |
| 依赖模块 | ad-14（场景判定与分槽路由单元，接收经验写入指令）、ad-20 至 ad-30（五层存储与晋升遗忘执行模块）、ad-36（综合重要度 I 值聚合计算单元） |
| 被依赖模块 | ad-14（上报存储占用率与活跃状态）、ad-03（漏斗二专属调度单元，接收状态汇总） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收经验条目并执行五层存储流转 | 漏斗二已激活，本槽已初始化 |
| 容量告急 | `CAPACITY_WARNING` | 存储占用率 > 90%，已触发加速遗忘机制 | 本槽存储占用率监测 |
| 维护中 | `MAINTENANCE` | 正在执行相似经验归并或冷归档操作 | 定期维护周期或容量告急触发 |
| 冻结只读 | `FROZEN` | 漏斗二冻结或紧急接管，暂停写入但保留查询 | 驾驶模式切换或紧急熔断 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 经验写入指令 | Struct（含经验条目 + 写入优先级 + 路由标记 + 判定置信度） | ad-14 场景判定与分槽路由单元 | 每次场景路由至本槽时 | **高** |
| L1 至 L4 晋升候选清单 | List（经验条目 ID + 晋升目标层级 + 重要度 I） | ad-38 晋升双条件判定单元 | 每个晋升判定周期 | **高** |
| 遗忘候选清单 | List（经验条目 ID + 当前层级 + I 值） | ad-40 遗忘阈值判定单元 | 每个遗忘判定周期 | **高** |
| 归并建议 | Struct（源经验 ID + 目标经验 ID + 相似度） | ad-25 L3 中期层相似经验归并单元 | L3 归并操作时 | 普通 |
| 全局调度指令 | Enum（冻结/激活/维护） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 存储完成回执 | Struct（经验条目 ID + 存储层级 + 存储时间戳） | ad-14 | 每次经验成功存储后 | 普通 |
| 晋升执行结果 | Struct（经验条目 ID + 原层级 + 目标层级 + 执行结果） | ad-38、ad-39 | 晋升完成后 | **高** |
| 遗忘执行结果 | Struct（经验条目 ID + 删除/归档 + 结果） | ad-40、ad-42 | 遗忘执行后 | 普通 |
| 槽状态快照 | Struct（存储占用率 + 各层级条目数 + 活跃度） | ad-14、ad-03 | 每 1 秒周期性上报 | 普通 |
| 容量告警 | Struct（当前占用率 + 触发阈值 + 建议动作） | ad-03、ad-01 | 占用率 > 90% 时 | **高** |


## 专属遗忘策略

| 策略项 | 标准值 | 本槽调整 | 调整理由 |
|--------|:---:|:---:|----------|
| 安全显著性权重（α） | 0.5 | **0.60（上调 20%）** | 城区路口场景行人、非机动车密集，安全风险极高，涉及安全事件的经验应优先保留 |
| 复用频次权重（γ） | 0.3 | 0.24（相应下调） | 保持权重总和 1.0 |
| 风格匹配度权重（β） | 0.2 | 0.16（相应下调） | 同上 |
| 最低重要度遗忘阈值 | 0.15 | 0.10（下调 33%） | 安全场景经验即使重要度较低也优先保留 |
| L1→L2 晋升时间阈值 | 24h | 12h（缩短） | 城区路口场景变化快，有用的经验需要更快脱离 L1 易失区 |
| L2→L3 晋升时间阈值 | 7日 | 5日（缩短） | 城区驾驶策略需更快固化，以应对高频类似场景 |
| L4 安全底线锁定 | 人工锁定 | 自动锁定（S>0.85 自动申请 L5 锁定） | 安全相关经验应更快进入不可遗忘区 |


## 核心处理逻辑

```
FUNCTION urban_intersection_slot_main_loop():
    SET internal_state = NORMAL
    初始化五层存储结构，加载专属遗忘参数：
        α=0.60, β=0.16, γ=0.24
        最低遗忘阈值=0.10
        L1→L2 晋升时间阈值=12h
        L2→L3 晋升时间阈值=5日
        L4 自动锁定条件：S > 0.85

    WHILE 系统运行中:
        // 1. 全局调度
        IF 收到冻结指令:
            SET internal_state = FROZEN
            CONTINUE
        ELSE IF 收到激活指令:
            SET internal_state = NORMAL
            CONTINUE

        // 2. 新经验存储
        IF 收到经验写入指令 AND internal_state == NORMAL:
            经验条目 = 写入指令.经验条目
            // 如果涉及行人或非机动车，自动提升初始 I₀（基础重要度）10%
            IF 经验条目.核心风险目标.类别 IN [第四类, 第三类]:
                经验条目.I₀ = MIN(经验条目.I₀ * 1.10, 1.0)
            将经验条目写入 L1
            向 ad-14 返回存储完成回执

        // 3. 晋升处理
        IF 收到晋升候选清单:
            FOR EACH 候选 IN 清单:
                IF 候选.目标层级存在 AND 存储未满:
                    调用 ad-39 搬运
                    向 ad-38 返回成功
                ELSE:
                    向 ad-38 返回失败

        // 4. 遗忘处理（本槽阈值更低）
        IF 收到遗忘候选清单:
            FOR EACH 候选 IN 清单:
                IF 候选.层级 IN [L4, L5]:
                    SKIP
                ELSE:
                    // 使用专属遗忘阈值 0.10 判定
                    IF 候选.I < 0.10:
                        调用 ad-42 删除
                        向 ad-40 返回已删除
                    ELSE:
                        向 ad-40 返回保留

        // 5. L3 归并
        IF 距上次归并 >= 24h OR 占用率 > 90%:
            SET internal_state = MAINTENANCE
            向 ad-25 发送归并请求
            执行归并
            SET internal_state = NORMAL

        // 6. 容量监控
        占用率 = 计算占用率()
        IF 占用率 > 0.90:
            SET internal_state = CAPACITY_WARNING
            上报告警
            临时上调 L1/L2 遗忘阈值 20%
        ELSE IF 占用率 < 0.70 AND internal_state == CAPACITY_WARNING:
            SET internal_state = NORMAL
            恢复遗忘阈值

        // 7. 状态上报
        IF 距上次上报 >= 1s:
            上报槽状态快照

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 行人/非机动车相关经验 I₀ 提升超限 | 限制 I₀ ≤ 1.0，上报告警 | — |
| 安全显著性权重被人为调低 | 拒绝修改，回退默认值，上报告警 | 参数校验通过 |
| 容量告急时 L1/L2 加速遗忘 | 仅针对 L1/L2，L3 及以上保留 | 占用率 < 70% |
| 无保护左转经验被频繁遗忘 | 低 I 值时仍受最低阈值 0.10 保护 | — |
| 冻结状态写入 | 拒绝，返回原因 | 收到激活指令 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 安全显著性权重上调 20% 为编译期默认值，运行时可调范围 0.50–0.65 |
| S-02 | 涉及行人/非机动车的经验 I₀ 自动提升，确保安全经验在晋升中占优 |
| S-03 | 行人礼让相关的 L5 经验永久锁定，不可遗忘 |
| S-04 | 容量告警时加速遗忘仅限 L1/L2，保障中期以上经验安全 |
| S-05 | 所有操作日志写入 ad-51 |
