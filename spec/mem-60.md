## 二、核心处理逻辑

### 2.1 路权优先级与车车协商规则

**路权优先级排序（固定不可变更）：**

| 优先级 | 交通参与者 | 说明 |
|:---:|:---|:---|
| 1（最高） | 行人（尤其是儿童、老人、行动不便者） | 任何情况下行人绝对优先 |
| 2 | 非机动车（自行车、电动车、人力三轮车） | 次于行人，优于机动车 |
| 3 | 特种应急车辆（仅当本车非应急状态） | 救护车、消防车、警车优先通行 |
| 4 | 普通机动车 | 直行优先、转弯让直行 |
| 5（最低） | 静态障碍物 | 最后考虑 |

**车车协商规则：**

| 场景 | 规则 |
|:---|:---|
| 交替通行 | 左右车道车辆交替依次通行，遵循“拉链式”合流原则 |
| 窄路会车 | 有避让条件方让行；坡度>5%时下坡车让上坡车 |
| 拥堵蠕行 | 保持安全跟车距离，不频繁变道加塞 |
| 路口无信号灯 | 让右方来车先行；转弯让直行 |
| 夜间会车 | 提前切换近光灯，降低车速至15km/h以下 |

### 2.2 行人/骑行者神态行为分析

**分析对象：** 所有被adecc-05标记为“行人”、“骑行者”（含自行车、电动车、摩托车、人力三轮车）的交通参与者。

**分析维度一：面部表情与情绪状态识别**

| 情绪状态 | 视觉特征 | 风险权重 |
|:---|:---|:---|
| 平静 | 面部肌肉放松，视线稳定 | ×1.0（基准） |
| 焦躁/急切 | 频繁左右张望、皱眉、嘴唇紧抿 | ×1.5 |
| 愤怒 | 眉毛下压、怒视、面部紧绷 | ×1.4 |
| 紧张/恐惧 | 眼睛睁大、瞳孔扩张、身体僵硬 | ×1.3 |
| 分心 | 视线向下（看手机）、目光呆滞 | ×1.6（横穿风险最高） |
| 无法识别 | 面部被遮挡、光线不足、距离过远 | ×1.2（保守处理） |

**分析维度二：身体姿态与横穿意图预测**

| 横穿意图等级 | 判定条件 | 横穿概率 | 建议动作 |
|:---|:---|:---:|:---|
| 极低 | 背对车道、静止不动、与他人交谈 | <10% | 正常行驶，保持观察 |
| 低 | 面向车道但静止、在路边等待 | 10%-30% | 轻微减速，做好制动准备 |
| 中 | 身体微倾朝向车道、脚步有启动趋势 | 30%-60% | 明显减速，扩大横向距离 |
| 高 | 一只脚已踏出或前轮已进入车道 | 60%-80% | 主动让行，准备紧急制动 |
| 极高 | 正在横穿中或身体已完全进入车道 | >80% | 立即让行或紧急制动 |

**分析维度三：骑行者专属神态分析**

| 骑行者特征 | 风险含义 | 联动动作 |
|:---|:---|:---|
| 频繁回头看车 | 可能准备变道或横穿 | 主动让出空间，不抢行 |
| 单手扶把+另一手打手势 | 即将转向 | 按手势方向预判轨迹，减速让行 |
| 身体大幅摆动、站立踩踏 | 正在加速或急加速 | 保持更大安全距离，不并行 |
| 外卖骑手特征（统一制服+改装电动车） | 行业性高风险群体 | 风险权重额外×1.3，跟车距离×1.5 |
| 老人骑人力三轮车 | 反应慢、车速低、可能突然偏移 | 超车时留足1.5m以上横向距离 |

**分析维度四：场景上下文联动**

- 公交站台附近行人，横穿概率自动+15%
- 学校区域，儿童横穿概率默认+20%
- 夜市/集市区域，行人横穿概率默认+25%
- 雨天无遮挡的行人，焦躁情绪概率+30%
- 夜间照明不足区域，所有行人风险权重×1.3

### 2.3 分析结果的记录与学习闭环

**核心原则：无论预测准确还是错误，所有判定结果均需记录。**

**记录内容（每次判定以纯文字形式写入）：**

| 字段 | 说明 |
|:---|:---|
| 判定时间戳 | UTC时间，精确到毫秒 |
| 目标ID | 来自adecc-05的唯一ID |
| 目标类型 | 行人/自行车/电动车/摩托车/人力三轮车 |
| 预测情绪状态 | 平静/焦躁/愤怒/紧张/分心/无法识别 |
| 预测横穿概率 | 数值，0-100% |
| 实际行为 | 横穿/未横穿/抢道/让行/其他 |
| 预测准确度 | 正确/误判-假阳性/误判-假阴性 |
| 场景特征 | 文字化描述（场景类型+光照+天气） |
| 目标位置 | 相对本车距离与方位 |

**记录存储策略：**

- 预测准确：标记为“成功案例”，存入漏斗二对应场景分槽，作为正面经验
- 预测失误：标记为“异常案例”，存入疑问缓存（经mem-58中转），供离线复盘
- 重大失误（假阴性导致险些碰撞）：直接提升至L3中期层，触发专项复盘

**群体共享机制：**

- 文字化判定结果脱敏后可上报云端技能包体系
- 其他车辆在相似场景下可查询群体经验数据
- 高频误判模式标记为“群体学习课题”，触发云端专项优化


## 三、约束与异常处理

| 约束类型 | 内容 |
|:---|:---|
| 只读保护 | 社会规则库的路权优先级和协商规则不可被任何经验修改 |
| 隐私保护·日常正确判断 | 人脸图像在完成推理后立即从内存中丢弃，不做持久化存储。仅保留纯文字描述的分析记录。不上传云端 |
| 隐私保护·日常错误判断 | 与人脸图像相关的判定错误记录，保留完整的文字描述和场景特征，标记为异常案例供离线复盘。人脸图像用后即焚，不上传云端 |
| 隐私保护·事故相关判断 | 若某次神态分析直接关联交通事故（如假阴性预测导致碰撞），该次判断的完整原始数据须本地加密保留：含文字记录、人脸图像、人体骨架序列、场景视频片段。存储周期≥3年。仅当监管部门依法调取或云端群体安全分析需要时，经车主书面确认后方可提取。不上传云端 |
| 群体共享脱敏 | 上报云端的数据剔除GPS坐标、时间戳精确度降至小时、目标特征仅为脱敏后的文字标签。事故相关数据不参与群体共享，仅在本地保留 |
| 判定失败 | 无法识别目标情绪时，采用保守风险权重×1.2，不中断正常行驶 |
| 误判记录 | 所有误判均需以文字描述形式写入疑问缓存，不得丢弃。事故相关误判须保留完整原始数据 |
| 应急优先 | 本车紧急避险场景下暂停神态分析，优先执行避险动作 |


## 四、接口校验用例

**用例1：正常行人情绪识别**

输入：摄像头捕捉到前方人行道旁一名行人，面部表情显示频繁左右张望、皱眉。目标ID=“PED-001”，目标类型=行人。预期：情绪状态识别为“焦躁/急切”。横穿概率评估为中（30%-60%）。风险权重×1.5。判定结果以文字描述形式写入漏斗二。原始图像数据已丢弃，不可检索。

**用例2：分心行人高横穿风险**

输入：摄像头捕捉到一名行人低头看手机，视线从未看向车道。目标ID=“PED-002”。预期：情绪状态识别为“分心”。横穿概率评估为高（60%-80%）。风险权重×1.6。建议主动让行，准备紧急制动。

**用例3：外卖骑手专属风险加权**

输入：摄像头捕捉到一名统一制服的外卖电动车骑手，频繁回头看车，单手扶把。目标ID=“BIKE-001”。预期：识别为“外卖骑手”类别，风险权重额外×1.3。骑手频繁回头看车判定为可能准备变道。建议主动让出空间，不抢行。

**用例4：预测失误的记录**

输入：系统预测行人横穿概率为高（75%），但行人实际在路边停下等待，未横穿。预测结果=假阳性。预期：系统将此次判定以文字描述形式完整记录为“异常案例-假阳性”，写入疑问缓存。原始图像数据已丢弃。文字记录供离线复盘分析。

**用例5：事故相关判断的完整保留**

输入：系统预测一名行人横穿概率为低（20%），但行人实际突然横穿，导致碰撞事故。预测结果=假阴性（事故关联）。预期：该次判断的完整原始数据——文字记录、人脸图像、人体骨架序列、场景视频片段——全部本地加密保留。存储周期≥3年。不上传云端。仅当监管部门依法调取或经车主书面确认后方可提取。

**用例6：群体共享数据的脱敏验证**

输入：本车记录的100条判定结果准备上报云端。操作：执行数据脱敏——剔除GPS坐标、时间戳精确度降至小时、目标特征仅为脱敏后的文字标签。预期：脱敏后数据仅含目标类型、情绪类别、横穿概率、预测准确度、场景类型。任何第三方无法从共享数据中还原出具体个人。事故相关数据未参与共享。

**用例7：逆光时无法识别的保守处理**

输入：逆光条件下，摄像头无法清晰捕捉行人面部表情。目标ID=“PED-003”。预期：情绪状态标记为“无法识别”，采用保守风险权重×1.2。横穿概率按中等处理。继续行驶但保持观察。无法识别原因（“逆光”）写入记录。

**用例8：隐私保护——图像用后即焚验证**

输入：摄像头捕捉到行人面部图像，执行情绪推理。操作：推理完成后，尝试从本地存储中检索原始面部图像。预期：原始面部图像已丢弃，不可检索。仅保留脱敏后的文字描述记录。人脸特征向量、关键点坐标等任何生物特征数据均不可检索。
