# ad-02-漏斗一专属调度单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-02 |
| 模块名称 | 漏斗一专属调度单元 - 驾驶员画像漏斗管家 |
| 所属分区 | 一、顶层总控中枢 |
| 核心职责 | 漏斗一内部子画像槽的全生命周期管理。负责驾驶员身份识别结果的接收与槽位匹配，管控子画像槽的创建、激活、冻结与删除 |
| 依赖模块 | ad-04（驾驶员身份识别单元）、ad-05（子画像槽创建与初始化单元）、ad-06（子画像槽数据隔离管控单元）、ad-12（临时画像槽自动清除单元）、ad-13（子画像槽长期未活跃提醒单元） |
| 被依赖模块 | ad-01（总控漏斗 F₀）、ad-07（驾驶行为观测记录单元）、ad-11（驾驶辅助提醒生成单元） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待身份确认 | `WAITING_ID` | 车辆启动后等待驾驶员身份识别结果 | 系统上电、驾驶模式为人工驾驶 |
| 槽位匹配中 | `MATCHING` | 正在将识别到的身份与已有子画像槽进行匹配 | 收到驾驶员身份识别结果 |
| 长期槽就绪 | `LONG_TERM_READY` | 已匹配到已有长期槽，激活就绪 | 身份与已有长期槽匹配成功 |
| 新槽创建中 | `CREATING` | 未匹配到已有槽，正在创建新子画像槽 | 身份未匹配到任何已有槽 |
| 临时槽激活 | `TEMP_ACTIVE` | 临时画像槽已激活，7 天倒计时开始 | 用户未确认身份或选择临时使用 |
| 一次性槽激活 | `ONESHOT_ACTIVE` | 一次性记录槽已激活，行程结束后自动清除 | 代客泊车等一次性场景 |
| 槽位冻结 | `FROZEN` | 子画像槽暂停写入，保留数据只读 | 自动驾驶模式激活、紧急接管 |
| 槽位清理中 | `CLEANING` | 正在执行临时槽或一次性槽的安全擦除 | 临时槽满 7 天或一次性行程结束 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 驾驶员身份识别结果 | Struct（身份 ID + 识别方式 + 置信度） | ad-04 驾驶员身份识别单元 | 身份识别完成时 | **高** |
| 驾驶模式信号 | Enum（人工驾驶 / 自动驾驶 / 紧急接管） | ad-01 总控漏斗 F₀ | 驾驶模式变更时 | **高** |
| 子画像槽状态快照 | Dict（各槽状态 + 占用情况） | 各子画像槽 | 周期性，每 1 秒 | 普通 |
| 中控屏手动选择信号 | Struct（用户选择的驾驶员 ID + 槽位类型） | 中控屏 CAN 总线 | 用户手动操作时 | **高** |
| 临时槽计时器触发 | Signal（7 天到期） | 系统时钟 | 临时槽创建后满 7 天 | 普通 |
| 行程结束信号 | Signal | 车辆 CAN 总线 | 车辆熄火、P 档 + 手刹 | 普通 |
| 全局调度指令 | Enum（激活 / 冻结 / 锁定只读） | ad-01 总控漏斗 F₀ | 模式切换、安全事件时 | **紧急 / 高** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 目标子画像槽激活指令 | Struct（槽号 + 激活模式） | ad-06 子画像槽数据隔离管控单元 | 身份匹配或新槽创建完成时 | **高** |
| 新槽创建指令 | Struct（驾驶员 ID + 槽位类型 + 存储分区指针） | ad-05 子画像槽创建与初始化单元 | 未匹配到已有槽时 | **高** |
| 临时槽激活指令 | Struct（槽号 + 7 天计时器启动） | ad-06、ad-12 | 用户未确认身份时 | **高** |
| 一次性槽激活指令 | Struct（槽号 + 行程绑定） | ad-06、ad-12 | 代客泊车等一次性场景 | **高** |
| 槽位冻结指令 | Enum（冻结 / 解冻） | 目标子画像槽 | 驾驶模式切换时 | **高** |
| 临时槽清除触发信号 | Signal | ad-12 临时画像槽自动清除单元 | 临时槽满 7 天或一次性行程结束 | 普通 |
| 未活跃槽提醒请求 | Struct（槽号列表） | ad-13 子画像槽长期未活跃提醒单元 | 周期性检测，每 24 小时 | 普通 |
| 调度状态上报 | Struct（当前活跃槽 + 槽位占用统计） | ad-01 总控漏斗 F₀ | 周期性，每 1 秒 | 普通 |


## 核心处理逻辑

```
FUNCTION F1_dispatcher_main_loop():
    WHILE 系统运行中:
        // 第 1 步：最高优先级——全局调度指令
        IF 收到 ad-01 下发的全局调度指令:
            IF 指令 == 冻结 OR 锁定只读:
                SET internal_state = FROZEN
                冻结当前活跃子画像槽写入
                通知 ad-06 切换为只读模式
            ELSE IF 指令 == 激活:
                IF internal_state == FROZEN:
                    解除冻结，恢复写入
            CONTINUE

        // 第 2 步：驾驶模式监听
        IF 驾驶模式 != 人工驾驶:
            IF internal_state NOT IN [FROZEN, WAITING_ID]:
                冻结当前活跃子画像槽写入
                SET internal_state = FROZEN
            CONTINUE  // 漏斗一仅人工驾驶模式运行

        // 第 3 步：身份识别结果处理
        IF 收到 ad-04 驾驶员身份识别结果 OR 中控屏手动选择信号:
            SET internal_state = MATCHING
            检索已有长期槽中的匹配身份

            IF 匹配成功:
                激活对应长期槽
                SET internal_state = LONG_TERM_READY
            ELSE IF 用户选择临时使用 OR 5 秒内无确认:
                IF 临时槽有空位:
                    激活临时画像槽
                    SET internal_state = TEMP_ACTIVE
                    启动 7 天倒计时
                ELSE:
                    激活一次性记录槽
                    SET internal_state = ONESHOT_ACTIVE
            ELSE:
                创建新长期槽
                SET internal_state = CREATING
                向 ad-05 发送创建指令
                WAIT 创建完成
                激活新槽
                SET internal_state = LONG_TERM_READY

        // 第 4 步：临时槽与一次性槽生命周期管理
        IF internal_state == TEMP_ACTIVE AND 7 天计时器到期:
            触发 ad-12 执行安全擦除
            SET internal_state = CLEANING
        IF internal_state == ONESHOT_ACTIVE AND 行程结束信号:
            触发 ad-12 执行安全擦除
            SET internal_state = CLEANING

        // 第 5 步：定期未活跃检测
        IF 距上次未活跃检测 >= 24 小时:
            扫描所有长期槽的最后活跃时间戳
            IF 存在超过 90 天未活跃的槽:
                向 ad-13 发送提醒请求（仅提示，不自动删除）
            IF 存在超过 90 天未活跃且用户已手动确认删除的槽:
                触发 ad-12 执行安全擦除

        // 第 6 步：周期性状态上报
        IF 距上次上报 >= 1 秒:
            汇总当前活跃槽信息与槽位占用统计
            上报 TO ad-01

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 长期槽位已满（6 个）且新驾驶员身份识别 | 提示用户“长期槽位已满，是否将最早未活跃槽转为临时槽或删除”，等待用户手动选择 | 用户手动释放槽位 |
| 临时槽位已满（1 个） | 新临时使用者自动降级为一次性记录槽，行程结束即清除 | — |
| 一次性槽位已满（1 个） | 覆盖当前一次性槽（该槽数据本身行程结束即清除） | — |
| 身份识别置信度低（< 0.6） | 不自动匹配，弹出中控屏确认对话框，等待用户手动选择 | 用户手动确认身份 |
| 5 秒内无身份确认 | 默认激活临时画像槽，行为数据标记为临时标签 | 用户可在行程中随时切换身份 |
| 紧急接管信号 | 立即冻结当前活跃槽写入，漏斗一全部锁定只读 | 险情解除且驾驶模式恢复人工驾驶 |
| 子画像槽数据校验失败 | 标记该槽为 DAMAGED，禁止读写，上报告警，提示用户是否删除重建 | 用户手动确认重建 |
| 临时槽 7 天到期时仍有未写入数据 | 等待当前写入批次完成（最长 2 秒），然后执行擦除 | 写入完成或 2 秒超时 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| MemoryBus | 读 | 驾驶员身份识别结果 | 只读 | 含身份 ID、识别方式、置信度 |
| MemoryBus | 读 | 中控屏手动选择信号 | 只读 | 用户手动选择的驾驶员 ID 与槽位类型 |
| MemoryBus | 读 | 驾驶模式信号 | 只读 | 由 ad-01 转发 |
| MemoryBus | 读 | 全局调度指令 | 只读 | 由 ad-01 下发 |
| MemoryBus | 写 | 子画像槽激活 / 冻结 / 创建指令 | 专属写入 | 向 ad-05、ad-06 下发 |
| MemoryBus | 写 | 临时槽清除触发信号 | 专属写入 | 向 ad-12 下发 |
| MemoryBus | 写 | 调度状态上报 | 周期性写入 | 每 1 秒向 ad-01 上报 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 长期槽位硬性上限 6 个，编译期强制，不可通过运行时配置绕过 |
| S-02 | 子画像槽之间绝对物理存储隔离，本模块仅做路由，不参与数据读写 |
| S-03 | 临时槽 7 天自动清除为硬约束，仅用户手动确认可提前清除 |
| S-04 | 一次性槽行程结束即清除，不保留任何数据 |
| S-05 | 漏斗一全部子画像槽数据编译期禁止以任何形式接入自动驾驶决策链路 |
| S-06 | 90 天未活跃仅提示用户，禁止系统自动删除长期槽 |
| S-07 | 紧急接管时立即冻结全部槽位写入，优先保障安全 |
| S-08 | 所有槽位创建、删除、冻结操作全量写入 ad-51 号变更日志 |
