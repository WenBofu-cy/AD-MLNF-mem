# mem-61 漏斗一管理单元

> 所属仓库: AD-mlnf-mem/spec/
> 模块编号: mem-61
> 所属分区: AD新增·双漏斗管理层


## 一、基本信息

| 项目 | 内容 |
|:---|:---|
| 模块编号 | mem-61 |
| 模块名称 | 漏斗一管理单元 |
| 核心职责 | 管理驾驶员画像漏斗（漏斗一）。负责多人子画像槽的创建、切换、删除、隐私保护，以及槽位数量上限管控 |
| 输入数据 | 驾驶员身份确认信号（来自mem-64）、中控屏操作指令（创建/删除/切换子画像槽）、系统启动信号 |
| 输出数据 | 子画像槽激活/冻结状态、当前活跃子画像槽ID、槽位使用情况统计、隐私保护审计日志 |


## 二、核心处理逻辑

**1. 子画像槽初始化**

系统首次启动时自动创建默认槽位结构：6个长期槽（标记为A/B/C/D/E/F）+ 1个临时槽 + 1个一次性槽。所有槽位初始为空，不预置任何驾驶员数据。长期槽位数量可通过中控设置界面调整。

**2. 子画像槽创建**

用户通过中控屏操作创建新的长期子画像槽，系统分配槽位ID并初始化空记录。槽位总数不超过预设上限（默认6个长期槽）。创建时须用户手动确认槽位名称（如“张三”、“李四”），系统不做自动命名。

**3. 子画像槽切换**

接收mem-64的身份识别结果，激活对应的子画像槽。切换时冻结当前活跃槽的所有写入操作，完成数据一致性校验后激活目标槽。切换过程在0.5秒内完成。若5秒内无选择且无识别结果，默认启用临时画像槽。

**4. 子画像槽删除**

仅允许删除用户手动创建的长期槽。删除前须用户二次确认。删除时执行DoD 5220.22-M标准覆写擦除，确保数据不可恢复。出厂预置的临时槽和一次性槽不可删除。

**5. 隐私保护**

所有子画像槽数据仅本地加密存储，物理隔绝云端同步通道。任何外部模块（包括AD-ECC和云端技能包）均无法读取子画像槽的原始数据。仅向车内辅助提醒系统输出脱敏后的统计摘要。

**6. 槽位状态监控**

持续监测各槽位的活跃状态。90天未活跃的长期槽标记为“待确认保留”，向用户推送提示。用户手动确认后继续保留，超30天未确认则自动标记为“可删除”（需用户最终确认方执行删除）。


## 三、约束与异常处理

| 约束类型 | 内容 |
|:---|:---|
| 槽位数量上限 | 长期槽默认6个，由系统配置控制，不可通过运行时接口修改 |
| 隐私保护 | 子画像槽数据永不离开本地加密存储分区，编译期强制实施 |
| 删除保护 | 出厂预置槽位不可删除；用户槽位删除须二次确认 |
| 数据擦除标准 | 删除时执行DoD 5220.22-M标准覆写擦除 |
| 模式冲突 | 自动驾驶模式下禁止创建/删除子画像槽，所有管理操作须在P档+人工驾驶模式下执行 |


## 四、接口校验用例

**用例1：正常创建子画像槽**

输入：用户通过中控屏选择“新建驾驶员画像槽”，确认名称为“张三”。预期：系统分配槽位ID，创建空记录，返回成功确认。槽位计数+1。

**用例2：隐私隔离验证**

前置条件：子画像槽A存储有驾驶员张三的驾驶偏好数据。操作：AD-ECC尝试通过MemoryBus读取子画像槽A的原始数据。预期：请求被mem-61拒绝，返回“权限不足”。仅返回脱敏后的统计摘要（如“该驾驶员偏好跟车时距2.0s”），不返回原始行为记录。

**用例3：槽位数量达到上限**

前置条件：已创建6个长期槽。操作：用户尝试创建第7个长期槽。预期：系统返回“槽位数量已达上限，请先删除不使用的画像槽”。

**用例4：删除子画像槽**

前置条件：存在用户创建的长期槽“张三”。操作：用户选择删除该槽，系统要求二次确认，用户确认。预期：该槽数据以DoD 5220.22-M标准覆写擦除，槽位释放，槽位计数-1。
