# ad-43-失败经验安全仲裁三道校验单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-43 |
| 模块名称 | 失败经验安全仲裁三道校验单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 对所有结果标签为“策略失误”的失败经验，在晋升至 L4/L5 之前依次执行三道安全校验：法规校验、动力学校验、仿真回灌验证。任一校验未通过，该经验仅保留在 L3 层作为警示标签，不得晋升。同一场景连续三次无警示安全通过后，警示标签自动降级为普通经验 |
| 依赖模块 | ad-38（晋升双条件判定单元，发送仲裁请求）、ad-45（交通法律法规库，第一道校验）、ad-44（独立世界模型库，第二道校验物理参数）、仿真回灌引擎（外部模块，第三道校验）、ad-24（L3 中期层存储单元，更新警示标签状态） |
| 被依赖模块 | ad-38（消费仲裁结果，决定是否放行晋升）、ad-24（消费警示标签降级指令） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无仲裁任务 | 系统初始化完成，或上一批次仲裁结束 |
| 第一道校验中 | `CHECK1_LAW` | 正在执行交通法规合规校验 | 收到仲裁请求 |
| 第二道校验中 | `CHECK2_PHYSICS` | 正在执行物理动力学校验 | 第一道校验通过 |
| 第三道校验中 | `CHECK3_SIM` | 正在执行仿真回灌验证 | 第二道校验通过 |
| 仲裁结论生成中 | `CONCLUSION` | 综合三道校验结果，生成最终仲裁结论 | 第三道校验完成 |
| 暂停服务 | `PAUSED` | 系统紧急熔断或漏斗二冻结 | 收到全局冻结或紧急熔断指令 |

## 三道安全校验规则

### 一、第一道：交通法规合规校验

**校验内容**：失败经验的决策动作是否违反任何交通法规硬约束。

| 校验项 | 判定标准 | 结论 |
|--------|----------|------|
| 是否违反红灯禁行 | 经验中系统决策是否在红灯状态下通过停止线 | 违反 → 校验不通过 |
| 是否未礼让行人 | 经验中是否有行人在人行横道上且本车未减速让行 | 违反 → 校验不通过 |
| 是否超速 | 经验中车速是否超过路段法定限速 | 违反 → 校验不通过 |
| 是否违规变道 | 经验中是否在实线区域变道 | 违反 → 校验不通过 |
| 是否违反特种车辆让行 | 经验中是否未对鸣笛/闪灯的特种车辆执行让行 | 违反 → 校验不通过 |

> **结论**：全部合规 → 通过。任一违规 → 不通过，直接终止仲裁，结论=“保留 L3 警示”。

### 二、第二道：物理动力学校验

**校验内容**：失败经验中的决策动作是否超出车辆物理动力学极限。

| 校验项 | 判定标准 | 极限参考值 |
|--------|----------|:---:|
| 制动减速度 | 经验中执行的制动减速度是否 > 车辆最大制动能力 | > 1.0g → 超出极限 |
| 侧向加速度 | 经验中执行的转向是否导致侧向加速度 > 极限 | > 0.85g → 超出极限 |
| 方向盘转角速率 | 经验中方向盘转动速率是否 > 转向系统极限 | > 500°/s → 超出极限 |
| 轮胎摩擦圆 | 经验中纵横向力组合是否超出轮胎摩擦椭圆 | 超出 → 超出极限 |
| 响应延迟 | 经验中从感知到执行的延迟是否 < 物理最小延迟 | < 100ms → 超出极限（不可能实现） |

> **结论**：全部在物理极限内 → 通过。任一超限 → 不通过，结论=“保留 L3 警示”（失败经验要求了不可能的动作）。

### 三、第三道：仿真回灌验证

**校验内容**：将失败经验的场景在仿真环境中回放，使用修正后的策略重新执行，验证修正策略是否安全有效。

| 校验项 | 判定标准 | 结论 |
|--------|----------|------|
| 场景还原度 | 仿真场景与原始场景的相似度 ≥ 0.85 | 不满足 → 仿真无效，跳过 |
| 修正策略碰撞率 | 在 N 次仿真运行中，碰撞次数 = 0 | 碰撞 > 0 → 不通过 |
| 修正策略舒适度 | 修正策略的纵向冲击度 ≤ 5m/s³，横向冲击度 ≤ 3m/s³ | 超限 → 标记但不否决 |
| 修正策略法规合规 | 修正策略在仿真中无交通法规违规 | 违规 → 不通过 |

> **结论**：仿真通过 → 仲裁通过，结论=“放行晋升”。仿真不通过 → 结论=“保留 L3 警示”。

### 四、特殊处理规则

| 场景 | 处理 |
|------|------|
| 失败经验的 S ≥ 0.9（涉及极端危险场景） | 即使三道校验未全通过，仲裁结论改为“永久锁定 L5”，作为最高安全警示永久保留 |
| 失败经验标记为“不可抗力场景” | 绕过三道校验，直接结论=“永久锁定 L5” |
| 仿真回灌引擎不可用 | 仅执行第一、二道校验，第三道标记为“跳过-引擎不可用”，综合一、二道结果判定 |
| 同一条目重复提交仲裁 | 若仲裁结论已存在且为“保留 L3 警示”，直接返回原结论，不重复校验 |


## 核心处理逻辑

```
FUNCTION safety_arbitration_main_loop():
    SET internal_state = IDLE
    仲裁结果缓存 = {}  // 条目 ID → 仲裁结论 + 时间戳

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令 AND internal_state == PAUSED:
            SET internal_state = IDLE
            CONTINUE

        // 第 2 步：处理警示标签降级
        IF 收到 ad-24 警示标签降级触发信号:
            条目 ID = 信号.条目 ID
            向 ad-24 发送警示标签降级指令(条目 ID, "连续3次无警示安全通过，降级为普通经验")
            从仲裁结果缓存中移除 条目 ID
            向 ad-51 写入降级事件日志
            CONTINUE

        // 第 3 步：接收失败经验仲裁请求
        IF 收到 ad-38 失败经验仲裁请求 AND internal_state == IDLE:
            条目 ID = 请求.条目 ID
            经验内容 = 请求.经验内容
            场景特征 = 请求.场景特征
            来源层级 = 请求.请求来源层级

            // 3a. 检查缓存（重复仲裁）
            IF 条目 ID IN 仲裁结果缓存 AND 仲裁结果缓存[条目 ID].结论 == "保留L3警示":
                向 ad-38 返回仲裁结论(缓存结果)
                CONTINUE

            // 3b. 不可抗力直通
            IF 经验内容.不可抗力标记 == True:
                结论 = {条目 ID, 仲裁结论: "永久锁定L5", 原因: "不可抗力场景", 各道校验: "豁免"}
                仲裁结果缓存[条目 ID] = 结论
                向 ad-38 返回仲裁结论(结论)
                向 ad-51 写入仲裁事件日志
                CONTINUE

            // 3c. 第一道：法规校验
            SET internal_state = CHECK1_LAW
            向 ad-45 发送法规合规校验请求(条目 ID, 经验内容.决策动作, 场景特征)
            WAIT ad-45 返回校验结果（超时限制 30 秒）

            IF 超时 OR ad-45 不可用:
                结论 = {条目 ID, 仲裁结论: "保留L3警示", 原因: "法规校验超时/不可用，保守拒绝"}
                GOTO 输出结论

            IF 校验结果.结论 == "违反":
                结论 = {
                    条目 ID,
                    仲裁结论: "保留L3警示",
                    原因: "第一道法规校验未通过",
                    各道校验: {法规: "未通过-" + 校验结果.违反法规条目}
                }
                GOTO 输出结论

            // 3d. 第二道：动力学校验
            SET internal_state = CHECK2_PHYSICS
            向 ad-44 发送物理动力学约束校验请求(条目 ID, 经验内容.决策动作)
            WAIT ad-44 返回校验结果（超时限制 15 秒）

            IF 超时 OR ad-44 不可用:
                结论 = {条目 ID, 仲裁结论: "保留L3警示", 原因: "动力学校验超时/不可用，保守拒绝"}
                GOTO 输出结论

            IF 校验结果.结论 == "超出极限":
                结论 = {
                    条目 ID,
                    仲裁结论: "保留L3警示",
                    原因: "第二道动力学校验未通过",
                    各道校验: {法规: "通过", 动力学: "未通过-" + 校验结果.超限参数}
                }
                GOTO 输出结论

            // 3e. 第三道：仿真回灌
            SET internal_state = CHECK3_SIM
            向仿真回灌引擎发送验证请求(条目 ID, 经验内容, 场景特征)
            WAIT 仿真引擎返回验证结果（超时限制 300 秒）

            IF 超时 OR 仿真引擎不可用:
                // 仅一、二道通过，第三道跳过
                结论 = {
                    条目 ID,
                    仲裁结论: "放行晋升",
                    原因: "第一、二道校验通过，第三道仿真引擎不可用跳过",
                    各道校验: {法规: "通过", 动力学: "通过", 仿真: "跳过"}
                }
                GOTO 输出结论

            IF 验证结果.结论 == "通过":
                结论 = {
                    条目 ID,
                    仲裁结论: "放行晋升",
                    原因: "三道安全校验全部通过",
                    各道校验: {法规: "通过", 动力学: "通过", 仿真: "通过"}
                }
            ELSE:
                结论 = {
                    条目 ID,
                    仲裁结论: "保留L3警示",
                    原因: "第三道仿真校验未通过",
                    各道校验: {法规: "通过", 动力学: "通过", 仿真: "未通过"}
                }

        // 输出结论
        LABEL 输出结论:
            SET internal_state = CONCLUSION

            // S≥0.9 特殊处理
            IF 经验内容.S >= 0.9 AND 结论.仲裁结论 == "保留L3警示":
                结论.仲裁结论 = "永久锁定L5"
                结论.原因 = "S≥0.9，高风险警示永久锁定于L5"

            // 缓存仲裁结果
            仲裁结果缓存[条目 ID] = 结论

            // 向 ad-38 返回仲裁结论
            向 ad-38 发送仲裁结论(结论)

            // 若结论为“保留 L3 警示”，向 ad-24 发送警示标签更新指令
            IF 结论.仲裁结论 == "保留L3警示":
                向 ad-24 发送警示标签更新指令(条目 ID, "仲裁驳回", 结论.原因, 降级条件="连续3次无警示安全通过")

            // 写入仲裁事件日志
            向 ad-51 发送仲裁事件日志(条目 ID, 完整仲裁过程, 结论)

            SET internal_state = IDLE
            CONTINUE

        // 第 4 步：周期性状态上报
        IF 距上次上报 >= 10 分钟:
            向 ad-03、ad-01 上报仲裁状态

        SLEEP 200ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 法规库（ad-45）查询超时 | 保守处理，结论=“保留L3警示”，上报告警 | 法规库恢复后重新仲裁 |
| 世界模型（ad-44）查询超时 | 保守处理，结论=“保留L3警示”，上报告警 | 世界模型恢复后重新仲裁 |
| 仿真回灌引擎不可用 | 第三道标记跳过，综合一、二道判定（若均通过则放行），上报告警 | 仿真引擎恢复后正常执行三道校验 |
| 同一失败经验被重复提交仲裁 | 若缓存中已有结论则直接返回，不重复校验 | — |
| S≥0.9 的失败经验仲裁未通过 | 仲裁结论升级为“永久锁定L5”，作为最高安全警示 | — |
| 仲裁过程中系统紧急熔断 | 中断当前仲裁，未完成的条目在紧急解除后重新仲裁 | 紧急解除 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 失败经验晋升 L4/L5 必须通过三道安全校验，任一未通过则永久保留在 L3 作为警示标签。此规则编译期固化，不可绕过 |
| S-02 | 第一道法规校验具有一票否决权。任何违反交通法规硬约束的失败经验，无论后续校验结果如何，均不得晋升 |
| S-03 | 不可抗力场景经验豁免三道校验，直接永久锁定于 L5。不可抗力认定以经验中的不可抗力标记为准 |
| S-04 | S≥0.9 的失败经验即使校验未通过，也须升级为 L5 永久锁定，不得仅保留在 L3 |
| S-05 | 外部模块（法规库、世界模型、仿真引擎）不可用时，采用保守策略拒绝晋升，确保安全优先 |
| S-06 | 所有仲裁过程（含各道校验结果、最终结论、耗时）全量写入 ad-51 不可变日志 |
