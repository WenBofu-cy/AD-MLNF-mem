# ad-14-场景判定与分槽路由单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-14 |
| 模块名称 | 场景判定与分槽路由单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 场景分槽管理 |
| 核心职责 | 依据独立世界模型库输出的道路属性与场景分类信号，判定当前驾驶场景类别，将经验写入请求精准路由至对应场景分槽。当场景无法精确判定时，降级路由至通用驾驶槽。当分槽数量达上限时，归并至相似度最高的已有分槽 |
| 依赖模块 | ad-44（独立世界模型库，提供道路属性与场景分类）、ad-03（漏斗二专属调度单元，接收路由调度指令）、ad-15 至 ad-19（五个场景分槽，接收经验写入） |
| 被依赖模块 | ad-03（漏斗二专属调度单元，消费路由完成回执）、ad-15 至 ad-19（消费经验写入指令） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常执行场景判定与路由 | 系统初始化完成，漏斗二已激活 |
| 世界模型查询中 | `QUERYING_WM` | 正在向世界模型查询当前道路属性 | 收到经验写入请求 |
| 场景判定计算中 | `JUDGING` | 正在根据世界模型返回数据进行场景类别匹配 | 世界模型返回结果 |
| 降级路由模式 | `FALLBACK` | 世界模型不可用或置信度低，降级路由至通用驾驶槽 | 世界模型查询超时/异常/置信度低 |
| 分槽归并模式 | `MERGING` | 分槽数量达上限，正在计算相似度并归并 | 分槽数量 ≥ Nmax_slot |
| 暂停服务 | `PAUSED` | 漏斗二冻结或非自动驾驶模式，暂停路由 | 驾驶模式切换或紧急接管 |
| 紧急只读 | `EMERGENCY_RO` | 紧急事件触发，暂停路由 | 收到 ad-01 紧急熔断指令 |

## 场景判定规则库

### 一、精确判定规则（按优先级从高到低）

| 优先级 | 场景类别 | 判定条件 | 目标分槽 |
|:---:|----------|----------|:---:|
| 1 | 特殊环境 | 天气 ∈ {暴雨/暴雪/大雾/沙尘暴} 或 路面类型 ∈ {积水/结冰/积雪} 或 施工区标记=True | ad-18 特殊环境槽 |
| 2 | 高速巡航 | 道路等级 ∈ {高速公路/城市快速路} 且 交通标识密度 ≥ 中 | ad-15 高速巡航槽 |
| 3 | 城区路口 | 道路等级 ∈ {城市主干道/次干道/支路} 且 路口密度 ≥ 1个/500m | ad-16 城区路口槽 |
| 4 | 泊车低速 | 车速 < 5km/h 持续 > 30秒 且 (停车场标记=True 或 狭窄空间标记=True) | ad-17 泊车低速槽 |
| 5 | 通用驾驶 | 以上均不满足，且路面类型 ∈ {沥青/水泥} | ad-19 通用驾驶槽 |
| 6 | 乡村道路 | 路面类型 ∈ {泥土/碎石/沙土} 或 道路等级=未分级 或 车道数=1 且 无标线 | ad-19 通用驾驶槽（乡村道路子类） |

### 二、降级与归并规则

| 场景 | 判定条件 | 处理方式 |
|------|----------|----------|
| 世界模型置信度 < 0.5 | 场景分类信号置信度不足 | 降级路由至 ad-19 通用驾驶槽，标记“低置信度” |
| 世界模型返回“未知道路” | 无法识别场景类别 | 降级路由至 ad-19 通用驾驶槽 |
| 目标分槽存储占用 > 90% | 该分槽容量告急 | 仍路由至该槽（由该槽内部触发加速遗忘），同时上报建议 |
| 分槽数量 ≥ Nmax_slot（默认 8）且新场景类别无法匹配 | 分槽上限已达 | 计算与已有分槽的相似度，归并至相似度最高的分槽 |


## 核心处理逻辑

```
FUNCTION scene_judgment_and_routing_main_loop():
    SET internal_state = NORMAL
    初始化世界模型查询失败计数器 wm_fail_count = 0

    WHILE 系统运行中:
        // 第 1 步：最高优先级——全局调度指令
        IF 收到 ad-03 全局调度指令:
            IF 指令 == 冻结 OR 锁定只读:
                SET internal_state = PAUSED or EMERGENCY_RO
                暂停所有路由处理
                CONTINUE
            ELSE IF 指令 == 激活:
                IF internal_state IN [PAUSED, EMERGENCY_RO]:
                    SET internal_state = NORMAL
                CONTINUE

        // 第 2 步：接收经验写入请求
        IF 收到 ad-03 经验写入请求:
            IF internal_state IN [EMERGENCY_RO, PAUSED]:
                暂存至临时队列（最长保留 5 秒）
                CONTINUE

            SET internal_state = QUERYING_WM
            query_start_time = NOW()

            // 2a. 向世界模型查询当前场景
            向 ad-44 发送场景查询请求（含当前 GPS 坐标、道路属性请求）
            WAIT ad-44 返回结果（超时限制 50ms）

            // 2b. 世界模型异常处理
            IF 超时 OR 返回异常:
                wm_fail_count++
                IF wm_fail_count >= 3:
                    上报告警 TO ad-01
                SET internal_state = FALLBACK
                目标分槽 = ad-19 通用驾驶槽
                判定置信度 = 0.3
                路由标记 = "世界模型降级"
                向 ad-03 发送降级路由告警
                GOTO 执行路由

            wm_fail_count = 0  // 重置失败计数

            // 2c. 场景判定
            SET internal_state = JUDGING
            场景类别 = 世界模型返回.场景分类信号
            置信度 = 世界模型返回.置信度

            IF 置信度 < 0.5:
                SET internal_state = FALLBACK
                目标分槽 = ad-19 通用驾驶槽
                判定置信度 = 置信度
                路由标记 = "低置信度降级"
                向 ad-03 发送降级路由告警
                GOTO 执行路由

            // 按判定规则库精确匹配
            目标分槽 = NULL
            IF 匹配“特殊环境”条件:
                目标分槽 = ad-18 特殊环境槽
            ELSE IF 匹配“高速巡航”条件:
                目标分槽 = ad-15 高速巡航槽
            ELSE IF 匹配“城区路口”条件:
                目标分槽 = ad-16 城区路口槽
            ELSE IF 匹配“泊车低速”条件:
                目标分槽 = ad-17 泊车低速槽
            ELSE IF 匹配“乡村道路”条件:
                目标分槽 = ad-19 通用驾驶槽
                路由标记 = "乡村道路子类"
            ELSE:
                目标分槽 = ad-19 通用驾驶槽
                路由标记 = "常规通用"

            判定置信度 = 置信度

            // 2d. 目标分槽容量检查
            IF 目标分槽.存储占用率 > 0.90:
                向 ad-03 上报容量告警（不改变路由）

            // 2e. 执行路由
            LABEL 执行路由:
            经验写入指令 = {
                目标分槽号: 目标分槽,
                经验条目: 经验写入请求.经验条目,
                写入优先级: 经验写入请求.优先级,
                路由标记: 路由标记,
                判定置信度: 判定置信度,
                判定时间戳: NOW()
            }
            向目标分槽发送经验写入指令
            向 ad-03 发送场景判定结果回执
            SET internal_state = NORMAL

        // 第 3 步：分槽数量监控与归并
        IF 分槽数量 >= Nmax_slot:
            SET internal_state = MERGING
            // 计算各分槽之间的场景相似度矩阵
            相似度矩阵 = 计算分槽间场景特征向量余弦相似度
            最佳归并对 = 相似度最高的两个分槽
            IF 最佳归并对.相似度 > 0.7:
                向 ad-03 发送分槽归并请求（源=相似度较低的槽，目标=相似度较高的槽）
            SET internal_state = NORMAL

        // 第 4 步：周期性状态上报
        IF 距上次上报 >= 5 秒:
            向 ad-03、ad-01 上报路由状态

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 世界模型查询超时（> 50ms） | 降级路由至通用驾驶槽，置信度 0.3，wm_fail_count++ | 世界模型恢复响应，连续 3 次成功后恢复 NORMAL |
| 世界模型连续 3 次查询失败 | 上报告警，后续请求不再查询世界模型，直接降级路由 | 每 30 秒重试一次，成功后恢复 |
| 世界模型置信度 < 0.5 | 降级路由至通用驾驶槽，保留低置信度标记 | 世界模型置信度恢复 |
| 目标分槽存储占用 > 90% | 仍路由至该槽，同步上报告警。由该分槽内部触发加速遗忘 | 存储占用回落至 < 70% |
| 经验写入请求中场景标签与判定结果不一致 | 以世界模型实时判定为准，忽略请求中的场景标签 | — |
| 分槽数量达上限且无相似度 > 0.7 的归并对 | 新经验写入路由至通用驾驶槽，附带“归并溢出”标记 | 手动调整 Nmax_slot 或删除闲置分槽 |
| 紧急熔断 | 立即暂停路由，状态 EMERGENCY_RO，暂存未处理请求（最长 5 秒） | 紧急事件解除 + 恢复指令 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| MemoryBus | 读 | 经验写入请求 | 只读 | 由 ad-03 下发 |
| MemoryBus | 读 | 世界模型道路属性与场景分类信号 | 只读 | 由 ad-44 返回 |
| MemoryBus | 读 | 全局调度指令 | 只读 | 由 ad-03 下发 |
| MemoryBus | 写 | 场景判定结果回执 | 专属写入 | 向 ad-03 返回 |
| MemoryBus | 写 | 经验写入指令 | 专属写入 | 向 ad-15 至 ad-19 下发 |
| MemoryBus | 写 | 降级路由告警 | 事件触发写入 | 向 ad-03、ad-01 上报 |
| MemoryBus | 写 | 分槽归并请求 | 事件触发写入 | 向 ad-03 发送 |
| MemoryBus | 写 | 路由状态上报 | 周期性写入 | 每 5 秒向 ad-03、ad-01 上报 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 场景判定以独立世界模型库实时输出为准，经验写入请求中携带的场景标签仅作参考，不可替代世界模型判定 |
| S-02 | 世界模型不可用时必须降级路由至通用驾驶槽，不可凭历史缓存猜测场景类别 |
| S-03 | 通用驾驶槽作为无条件兜底分槽，编译期保证其始终存在且不可被归并或删除 |
| S-04 | 乡村道路经验归入通用驾驶槽的子类标签，享有独立遗忘保护参数 |
| S-05 | 分槽归并前须校验归并后总条目数不超过单槽容量上限的 80% |
| S-06 | 紧急熔断时暂存队列最长保留 5 秒，超时则丢弃，优先保障安全 |
