# ad-41-最低复用次数校验单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-41 |
| 模块名称 | 最低复用次数校验单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 晋升与遗忘执行机制 |
| 核心职责 | 对遗忘候选清单中的每条经验进行最低复用次数校验。复用次数不足且 I 值低于阈值的条目通过校验，确认可遗忘；复用次数达到保护门槛的条目拒绝遗忘，从候选清单中移除。是遗忘判定流程的最后一道防线，防止尚有实战价值的经验被误删 |
| 依赖模块 | ad-40（遗忘阈值判定单元，提供遗忘候选清单）、ad-33（复用频次 C 值统计单元，提供准确复用次数） |
| 被依赖模块 | ad-42（冗余记忆删除与归档单元，接收最终通过校验的遗忘执行清单） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无校验任务，等待遗忘候选清单 | 系统初始化完成，或上次校验结束 |
| 校验执行中 | `VALIDATING` | 正在逐条目比对复用次数与保护阈值 | 收到遗忘候选清单 |
| 暂停服务 | `PAUSED` | 漏斗二冻结或紧急熔断 | 驾驶模式切换或紧急事件 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 遗忘候选清单 | Dict（层级 : List（条目 ID + 遗忘方式 + I 值 + 遗忘原因 + 遗忘优先级 + 来源分槽号 + 子类标记 + 结果分类标签）） | ad-40 遗忘阈值判定单元 | 每次遗忘判定完成后 | **高** |
| 条目复用次数字典 | Dict（条目 ID : {累计复用次数, 最近复用时间, C 值}） | ad-33 复用频次 C 值统计单元 | 随遗忘候选清单同步查询 | **高** |
| 各分槽最低复用保护阈值 | Dict（分槽号 : 最低复用次数保护门槛） | ad-35 三维权重系数配置单元 | 系统初始化及阈值变更时 | 普通 |
| 全局调度指令 | Enum（冻结/激活） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 通过校验的遗忘执行清单 | Dict（层级 : List（条目 ID + 遗忘方式 + I 值 + 复用次数 + 遗忘优先级 + 校验结论）） | ad-42 冗余记忆删除与归档单元 | 校验完成后 | **高** |
| 拒绝遗忘清单 | List（条目 ID + 拒绝原因 + 当前复用次数 + 保护阈值） | ad-40、来源层级存储单元 | 校验完成后 | **高** |
| 校验结果汇总 | Struct（校验总条目数 + 通过数 + 拒绝数 + 拒绝原因分布） | ad-40、ad-03、ad-51 | 每次校验完成后 | 普通 |


## 校验规则

### 一、通用校验公式

```
通过校验 = (复用次数 < C_min_protection)
拒绝遗忘 = (复用次数 >= C_min_protection)
```

### 二、各层级/分槽最低复用保护阈值

| 层级/分槽 | C_min_protection | 说明 |
|-----------|:---:|------|
| L1/L2 全部 | 3 | 临时层/近期层：复用 ≥ 3 次即保护 |
| L3 高速巡航槽 | 5 | 高速经验价值较高 |
| L3 城区路口槽 | 5 | 同上 |
| L3 泊车低速槽 | 2 | 低频场景，降低保护门槛 |
| L3 特殊环境槽 | 2 | 低频场景，降低保护门槛 |
| L3 通用驾驶槽-常规 | 4 | 标准保护门槛 |
| L3 通用驾驶槽-乡村 | 2 | 低频场景 |
| L4 全部（仅在容量红色告急时参与） | 1 | 复用 ≥ 1 次即保护 |

### 三、特殊豁免规则

| 条件 | 处理 |
|------|------|
| 条目.I 值 < 0.05（极低重要度） | 即使复用次数 ≥ 保护阈值，仍通过校验（可遗忘）。极低 I 值说明经验本身几乎无价值 |
| 条目.结果分类标签 = "策略失误" 且未通过安全仲裁 | 即使复用次数 ≥ 保护阈值，仍通过校验（可遗忘）。失败经验若无仲裁认可，不值得保护 |
| 条目在冷条目清单中（近 90 日未复用） | 保护阈值减半（如 3→1），加速淘汰长期闲置经验 |


## 核心处理逻辑

```
FUNCTION min_usage_count_validation_main_loop():
    SET internal_state = IDLE

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令 AND internal_state == PAUSED:
            SET internal_state = IDLE
            CONTINUE

        // 第 2 步：接收遗忘候选清单
        IF 收到 ad-40 遗忘候选清单 AND internal_state == IDLE:
            SET internal_state = VALIDATING

            遗忘候选清单 = ad-40 遗忘候选清单
            复用次数字典 = ad-33 条目复用次数字典
            分槽保护阈值库 = ad-35 各分槽最低复用保护阈值

            通过清单 = {L1: [], L2: [], L3: [], L4: []}
            拒绝清单 = []

            // 第 3 步：逐条目校验
            FOR EACH 层级 IN [L1, L2, L3, L4]:
                IF 遗忘候选清单[层级] == NULL:
                    CONTINUE

                FOR EACH 候选条目 IN 遗忘候选清单[层级]:
                    条目 ID = 候选条目.条目 ID
                    分槽号 = 候选条目.来源分槽号
                    I 值 = 候选条目.I 值
                    结果标签 = 候选条目.结果分类标签
                    遗忘方式 = 候选条目.遗忘方式

                    // 3a. 获取复用次数
                    复用数据 = 复用次数字典[条目 ID]
                    IF 复用数据 == NULL:
                        复用次数 = 0  // 从未复用
                    ELSE:
                        复用次数 = 复用数据.累计复用次数

                    // 3b. 特殊规则优先判定
                    IF I 值 < 0.05:
                        // 极低重要度，无视复用保护
                        候选条目.校验结论 = "通过-极低I值豁免复用保护"
                        通过清单[层级].append(候选条目)
                        CONTINUE

                    IF 结果标签 == "策略失误" AND 候选条目.安全仲裁状态 != "仲裁通过":
                        候选条目.校验结论 = "通过-失败经验无仲裁认可"
                        通过清单[层级].append(候选条目)
                        CONTINUE

                    // 3c. 获取保护阈值
                    保护阈值 = 分槽保护阈值库[分槽号]
                    IF 保护阈值 == NULL:
                        保护阈值 = 3  // 默认

                    // 3d. 冷条目阈值减半
                    IF 候选条目.在冷条目清单中:
                        保护阈值 = MAX(保护阈值 / 2, 1)

                    // 3e. 比对判定
                    IF 复用次数 < 保护阈值:
                        候选条目.校验结论 = "通过"
                        候选条目.复用次数 = 复用次数
                        通过清单[层级].append(候选条目)
                    ELSE:
                        拒绝清单.append({
                            条目 ID: 条目 ID,
                            拒绝原因: "复用次数达标",
                            当前复用次数: 复用次数,
                            保护阈值: 保护阈值,
                            来源层级: 层级
                        })

            // 第 4 步：输出校验结果
            IF 通过清单不为空（任一层级有条目）:
                向 ad-42 发送通过校验的遗忘执行清单

            IF 拒绝清单不为空:
                向 ad-40 发送拒绝遗忘清单
                向对应层级存储单元发送拒绝遗忘清单（条目保留在原层级）

            // 第 5 步：汇总上报
            校验结果汇总 = {
                校验总条目数: 各层级候选总数,
                通过数: 各层级通过总数,
                拒绝数: 拒绝清单.length,
                拒绝原因分布: 统计拒绝原因频次
            }
            向 ad-40、ad-03、ad-51 发送校验结果汇总

            SET internal_state = IDLE

        SLEEP 50ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 条目在复用次数字典中不存在 | 复用次数按 0 处理，正常参与校验 | — |
| 分槽保护阈值未配置 | 使用默认值 3，上报告警 | 阈值配置完成 |
| 某层级通过清单为空但拒绝清单也为空 | 正常情况（该层级无遗忘候选），不处理 | — |
| 同一条目出现在不同层级的遗忘候选清单中 | 按各自层级独立校验，若出现此情况则上报告警（数据一致性异常） | 数据修复 |

| 总线 | 操作 | 数据内容 | 权限 | 说明 |

| MemoryBus | 读 | 各分槽最低复用保护阈值 | 只读 | ad-35 下发 |

| MemoryBus | 写 | 校验结果汇总 | 专属写入 | ad-40、ad-03、ad-51 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 最低复用次数校验是遗忘流程的最后一道防线，本单元拒绝遗忘的条目在任何情况下不得被强制清除 |
| S-02 | 极低重要度（I < 0.05）豁免复用保护，防止无价值经验因偶发复用而长期占用存储 |
| S-03 | 失败经验若无安全仲裁认可，不享受复用保护。仅有经过验证的成功经验才值得用复用次数捍卫 |
| S-04 | 本单元仅执行校验判定，不直接操作存储介质。实际删除或归档由 ad-42 执行 |
| S-05 | 所有校验操作（含通过、拒绝）全量写入 ad-51 不可变日志 |
