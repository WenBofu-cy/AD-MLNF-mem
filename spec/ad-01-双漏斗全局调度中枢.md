# ad-01-双漏斗全局调度中枢 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-01 |
| 模块名称 | 总控漏斗 F₀ - 双漏斗全局调度中枢 |
| 所属分区 | 一、顶层总控中枢 |
| 核心职责 | 双漏斗记忆系统最高统筹单元。管控漏斗一与漏斗二的资源分配与模式切换，统一下发全局规则，对接 ECC 大脑经验查询请求 |
| 依赖模块 | ad-02（漏斗一调度状态）、ad-03（漏斗二调度状态）、ad-48（全局容量配额）、ad-51（变更日志） |
| 被依赖模块 | 全部 ad-02 至 ad-43 漏斗内核模块 + ad-44 至 ad-47 外挂模块（查询路由） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常双漏斗 | `NORMAL` | 双漏斗按驾驶模式各自激活，互不干扰 | 系统启动完成，模式信号明确 |
| 模式切换中 | `SWITCHING` | 人工↔自动驾驶模式切换，双漏斗临时冻结只读 | 驾驶模式信号变更 |
| 紧急只读 | `EMERGENCY_RO` | 紧急接管或安全事件触发，双漏斗强制只读，禁止一切写入 | 安全急停信号、碰撞检测信号 |
| 降级运行 | `DEGRADED` | 容量超限或子模块异常，限制新漏斗创建与晋升操作 | 容量使用率 > 90%、模块异常上报 |
| 维护只读 | `MAINTENANCE` | OTA 升级或手动维护状态，只读查询可用 | 系统维护指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 子漏斗状态快照 | Dict | ad-02 漏斗一调度单元、ad-03 漏斗二调度单元 | 周期性，每 1 秒 | 普通 |
| 驾驶模式信号 | Enum（人工驾驶 / 自动驾驶 / 紧急接管） | CerebellumBus（ECC→记忆） | 驾驶模式变更时 | **高** |
| ECC 经验查询请求 | Struct（含场景标签、查询条件、驾驶模式标记） | MemoryBus（ECC→记忆） | ECC 发起查询时 | **高** |
| 安全急停信号 | Enum（急停 / 碰撞 / 熔断） | MemoryBus（ECC→记忆） | 安全事件触发时 | **紧急** |
| 冷启动自检指令 | Signal | MemoryBus（ECC→记忆） | 系统上电时 | **高** |
| 容量配额使用率 | Float（0.0–1.0） | ad-48 全局容量配额管控单元 | 每 10 秒周期性上报 | 普通 |
| 模块异常上报 | List | 任意子模块 | 异常发生时即时上报 | **高** |
| 重要度权重配置更新 | Dict（α / β / γ） | ad-35 三维权重系数配置单元 | 用户或系统修改权重时 | 普通 |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 全局调度指令 | Enum（激活 / 冻结 / 锁定只读） | ad-02、ad-03、全部子漏斗 | 模式切换、安全事件时 | **紧急 / 高** |
| 模式切换信号 | Struct（切换类型 + 安全保障参数） | ad-02、ad-03 | 驾驶模式变更时 | **高** |
| 经验查询回执 | Struct（匹配经验列表 + 重要度排序） | MemoryBus（记忆→ECC） | 经验查询完成后 | 普通 |
| 全局容量配额分配 | Dict（各分区配额比例） | ad-48 全局容量配额管控单元 | 系统初始化、配额调整时 | 普通 |
| 双漏斗健康状态汇总 | Struct | MemoryBus（记忆→ECC） | 每 5 秒周期性上报 | 普通 |
| 遗忘阈值调整指令 | Dict（层级 : 新阈值） | ad-40 遗忘阈值判定单元 | 容量使用率 > 90% 时 | **高** |
| 全局紧急熔断指令 | Enum（暂停全部写入 / 恢复） | 全部漏斗内核模块（ad-03 至 ad-43） | 安全急停、碰撞事件时 | **紧急** |
| 漏斗合并或回收建议 | Enum | ad-02、ad-03 | 子漏斗数量接近 Nmax 时 | 普通 |


## 核心处理逻辑

```
FUNCTION F0_main_loop():
    WHILE 系统运行中:
        // 第 1 步：最高优先级——安全事件无条件优先处理
        IF 收到安全急停信号:
            SET internal_state = EMERGENCY_RO
            BROADCAST 全局紧急熔断指令 TO ALL 漏斗内核模块
            LOG 事件 TO ad-51 变更日志
            CONTINUE

        // 第 2 步：次高优先级——驾驶模式切换
        IF 收到驾驶模式变更信号:
            SET internal_state = SWITCHING
            激活目标漏斗，冻结源漏斗（超时限制 0.5 秒）
            IF 驾驶模式 == 紧急接管:
                双漏斗全部锁定只读
            WAIT 切换完成确认
            SET internal_state = NORMAL
            LOG 切换事件 TO ad-51 变更日志
            CONTINUE

        // 第 3 步：ECC 经验查询处理
        IF 收到 ECC 经验查询请求:
            IF internal_state IN [EMERGENCY_RO, SWITCHING]:
                返回只读缓存数据（若有）
            ELSE:
                根据驾驶模式标记路由到漏斗一（ad-02）或漏斗二（ad-03）
                汇总查询结果
                返回经验查询回执 TO MemoryBus

        // 第 4 步：周期性容量监控
        IF 容量配额使用率 > 0.90:
            SET internal_state = DEGRADED
            禁止新漏斗创建
            冻结晋升操作
            向 ad-40 号下发遗忘阈值收紧指令（I 阈值提升 20%）
            上报告警 TO MemoryBus

        // 第 5 步：子漏斗数量管控
        IF 子漏斗数量 >= Nmax * 0.95:
            向 ad-02、ad-03 下发漏斗合并建议

        // 第 6 步：周期性健康状态上报
        IF 距上次上报时间 >= 5 秒:
            汇总双漏斗运行状态
            通过 MemoryBus 上报 TO ECC

        // 第 7 步：子模块异常检测
        IF ad-02 或 ad-03 超时未上报状态:
            标记对应漏斗状态为 UNKNOWN
            使用最近有效快照继续运行
            上报告警 TO MemoryBus

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| ad-02 或 ad-03 超时未上报状态 | 标记对应漏斗状态为 UNKNOWN，使用最近一次有效快照继续运行，上报告警 | 连续 3 次正常上报后自动恢复 NORMAL |
| 容量使用率 > 90% | 自动收紧全局遗忘阈值（I 阈值提升 20%），冻结晋升与新建漏斗 | 容量回落至 < 70% 后逐步解除限制 |
| 收到非法或格式错误的指令 | 丢弃指令，记录安全日志到 ad-51 号，返回错误码 | — |
| 自身运行异常 | 触发降级模式：禁止新漏斗创建，冻结晋升操作，保留核心记忆只读 | 自检通过后手动或自动恢复 |
| 安全急停事件 | 最高优先级广播熔断指令，暂停全部非只读操作 | 安全事件解除且经人工确认 |
| 模式切换超时（> 0.5 秒） | 双漏斗维持只读状态，重试切换，上报告警 | 切换完成确认 |
| 紧急接管 | 双漏斗临时锁定只读，优先执行安全避险策略 | 险情解除且系统自检确认数据一致性 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| MemoryBus | 读 | ECC 经验查询请求 | 只读 | 含场景标签、查询条件、驾驶模式标记 |
| MemoryBus | 读 | 安全急停 / 碰撞 / 熔断信号 | 只读 | 最高优先级处理 |
| MemoryBus | 读 | 冷启动自检指令 | 只读 | 系统上电时接收 |
| MemoryBus | 写 | 经验查询回执 | 专属写入 | 含匹配经验列表与重要度排序 |
| MemoryBus | 写 | 双漏斗健康状态汇总 | 周期性写入 | 每 5 秒上报一次 |
| MemoryBus | 写 | 告警与异常上报 | 事件触发写入 | 异常发生时即时上报 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 模式切换须在 0.5 秒内完成，切换期间双漏斗强制只读，禁止任何写入操作 |
| S-02 | 紧急接管时双漏斗全部锁定只读，优先执行安全避险策略 |
| S-03 | 漏斗一子画像槽数据编译期禁止接入自动驾驶决策链路，本模块负责路由隔离 |
| S-04 | 降级运行期间禁止新漏斗创建与晋升操作，仅允许只读查询与安全相关写入 |
| S-05 | 安全急停信号为最高优先级，任何正在执行的非安全操作必须立即暂停 |
| S-06 | 所有规则变更与模式切换操作全量写入 ad-51 号变更日志，不可篡改 |
| S-07 | 固定知识底座不归本模块管辖，其查询请求直接路由至外挂模块区（ad-44 至 ad-47） |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-01-01 | 系统处于 NORMAL 状态，子漏斗数量已达 Nmax | 收到新场景事件 | 下发合并建议到 ad-02 与 ad-03，不创建新漏斗 |
| TC-01-02 | 容量使用率 = 92% | 周期性容量上报 | 下发遗忘阈值收紧指令到 ad-40，冻结晋升，上报告警 |
| TC-01-03 | 系统处于 NORMAL 状态 | MemoryBus 收到 ECC 安全急停信号 | 广播全局熔断指令，状态切换为 EMERGENCY_RO |
| TC-01-04 | NORMAL 状态，漏斗一处于激活状态 | 驾驶模式切换为自动驾驶 | 冻结漏斗一写入，激活漏斗二，0.5 秒内完成，状态恢复 NORMAL |
| TC-01-05 | NORMAL 状态 | ECC 经验查询请求（场景标签 = 高速巡航） | 路由到漏斗二，返回匹配经验列表与重要度排序 |
| TC-01-06 | NORMAL 状态，连续运行 5 秒 | — | 向 MemoryBus 上报双漏斗健康状态汇总 |
| TC-01-07 | NORMAL 状态 | 收到格式错误的指令包 | 丢弃，记录安全日志到 ad-51 号，返回错误码 |
| TC-01-08 | ad-02 号模块超时 3 秒未上报状态 | 周期性检查触发 | 标记漏斗一状态为 UNKNOWN，上报告警，使用最近快照继续 |
