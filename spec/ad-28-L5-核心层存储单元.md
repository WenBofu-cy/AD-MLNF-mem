# ad-28-L5-核心层存储单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-28 |
| 模块名称 | L5 核心层存储单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 五层记忆层级存储 |
| 核心职责 | 存储终身不可遗忘的驾驶安全底线、关键生存策略、不可抗力事件经验与人工锁定的核心规则。占漏斗二总存储容量的 0.5%，物理分区写保护。接收来自 L4 晋升的条目，或由安全显著性 S≥0.9 的事件直达写入。本层经验为系统最高智慧结晶，只读查询，写入须经多重校验，数据终身锁定不可删除 |
| 依赖模块 | ad-26（L4 长期层存储单元，接收晋升条目）、ad-29（L5 核心层安全规则硬锁定单元，执行写保护）、ad-30（L5 核心层防篡改与只读管控单元，管控读写权限）、ad-18（特殊环境槽，不可抗力事件直达入口）、ad-43（失败经验安全仲裁三道校验单元，特殊晋升路径） |
| 被依赖模块 | ad-29（接管存储分区写保护）、ad-30（提供只读查询服务）、ad-50（记忆导入导出与脱敏共享单元，特殊导出） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 存储已锁定，仅提供只读查询与安全写入 | 漏斗二激活，本层级初始化并锁定 |
| 写入允许窗口 | `WRITE_WINDOW` | 临时开放写入权限，用于安全事件直达或晋升写入 | 安全显著性 S≥0.9 触发或晋升操作 |
| 存储已满 | `FULL` | 0.5%容量已满，拒绝新写入（L5 不可扩容） | 容量使用率 ≥ 99% |
| 维护锁定 | `LOCKED` | 人工或系统维护，暂停一切读写 | 系统升级或人工安全巡检 |
| 降级保护 | `DEGRADED` | 存储介质异常但数据完整，只读服务正常 | 存储硬件部分故障，自动切换镜像 |

| 人工锁定指令 | Struct（条目 ID + 锁定理由 + 操作者权限令牌） | ad-01 总控漏斗 F₀（经 ad-30 校验） | 系统维护或特殊安全策略 | **高** |

| 查询结果数据 | Struct（匹配的经验条目列表 + 经验内容 + I 值 + 锁定状态） | ad-30（转发给请求方） | 只读查询命中时 | **高** |

| 存储异常告警 | Struct（异常类型 + 影响范围 + 当前处理） | ad-03、ad-01 | 存储介质故障或数据校验失败时 | **紧急** |


## 核心处理逻辑

```
FUNCTION l5_core_storage_main_loop():
    SET internal_state = NORMAL
    初始化 L5 存储分区（占漏斗二总容量 0.5%，约 2.5MB for 500MB 总容量）
    物理分区由 ad-29 硬件写保护锁定
    镜像分区 = 自动建立冗余镜像（RAID1 或等同机制）
    条目索引表（条目 ID → 存储地址 + 锁定时间 + I 值 + S 值 + 来源 + 锁定类型）
    最大容量 = 硬编码 0.5% 总漏斗容量，不可动态扩容

    WHILE 系统运行中:
        // 第 1 步：检查写保护与镜像健康
        IF 收到 ad-29 写保护状态报告:
            IF 写保护状态 != 已锁定:
                上报告警（L5 写保护异常）
                尝试重新锁定
            IF 镜像健康度 < 100%:
                触发镜像重建
                上报告警

        // 第 2 步：处理紧急安全事件直达写入（最高优先级）
        IF 收到安全事件直达写入 AND S >= 0.9:
            IF internal_state == FULL:
                返回写入拒绝通知（容量满）
                CONTINUE
            SET internal_state = WRITE_WINDOW
            ad-29 临时解除写保护（需提供安全令牌）
            新条目 = 直达写入条目
            新条目.锁定类型 = "自动安全锁定"
            新条目.终身锁定时间 = NOW()
            GOTO 执行写入

        // 第 3 步：处理 L4 晋升条目
        IF 收到 L4 晋升条目:
            IF internal_state == FULL:
                返回写入拒绝通知（容量满）
                CONTINUE
            // L5 晋升额外条件（已由 ad-38 判定，此处为二次校验）
            IF 晋升条目.I < 0.80 AND 晋升条目.不可抗力标记 != True:
                返回写入拒绝通知（I 值不满足）
                CONTINUE
            SET internal_state = WRITE_WINDOW
            ad-29 临时解除写保护
            新条目 = 晋升条目
            新条目.锁定类型 = IF 不可抗力标记 THEN "不可抗力锁定" ELSE "晋升锁定"
            新条目.终身锁定时间 = NOW()
            GOTO 执行写入

        // 第 4 步：处理人工锁定指令
        IF 收到人工锁定指令:
            IF 权限令牌验证失败:
                返回写入拒绝通知（权限不足）
                CONTINUE
            SET internal_state = WRITE_WINDOW
            ad-29 临时解除写保护
            新条目 = 从 L4 或其他层级提取（需提供完整经验内容）
            新条目.锁定类型 = "人工锁定"
            新条目.终身锁定时间 = NOW()
            GOTO 执行写入

        // 执行写入标签
        LABEL 执行写入:
            // 4a. 完整性校验
            IF 新条目.经验内容 == NULL OR 新条目.ID == NULL:
                ad-29 恢复写保护
                SET internal_state = NORMAL
                返回写入拒绝通知（条目不完整）
                CONTINUE

            // 4b. 检查剩余容量
            IF 剩余容量 < 新条目.大小:
                ad-29 恢复写保护
                SET internal_state = FULL
                返回写入拒绝通知（容量满）
                上报告警
                CONTINUE

            // 4c. 写入存储介质（物理只读区临时开放）
            存储地址 = 分配空间(新条目.大小)
            写入存储介质
            同步写入镜像分区
            更新条目索引表
            ad-29 恢复写保护

            // 4d. 返回成功回执
            向请求方发送存储完成回执（含终身锁定确认）
            向 ad-51 写入 L5 写入事件日志（不可变日志）
            SET internal_state = NORMAL
            CONTINUE

        // 第 5 步：处理只读查询
        IF 收到只读查询请求 AND internal_state != LOCKED:
            // 校验权限
            IF 请求来源模块 NOT IN [授权模块列表] OR 权限令牌无效:
                返回查询拒绝（权限不足）
                CONTINUE

            匹配条目 = 根据查询条件在索引表中检索
            IF 匹配条目为空:
                返回空结果
            ELSE:
                从存储介质读取经验内容
                向 ad-30 返回查询结果数据（由 ad-30 转发）

        // 第 6 步：定期状态上报
        IF 距上次上报 >= 5 分钟:
            向 ad-03、ad-01 上报 L5 状态快照

        SLEEP 500ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| L5 容量满（0.5% 配额用尽） | 拒绝新写入，上报告警。L5 不可通过清除释放空间，需手动扩展总漏斗配额（极罕见） | 总漏斗配额扩展 |
| 写入时镜像同步失败 | 主存储写入成功则返回成功，后台异步重建镜像，上报告警 | 镜像重建完成 |
| 存储介质出现坏块 | 自动切换至镜像分区，标记坏块，上报告警，尝试数据迁移 | 镜像完整且替代坏块 |
| 写保护解除失败（ad-29 异常） | 写入请求排队等待，最长等待 5 秒，超时则拒绝写入并告警 | ad-29 恢复 |
| 写入过程中系统掉电 | 恢复后通过镜像校验数据完整性，不完整条目回滚 | 系统重新初始化 |
| 晋升条目与 L5 已有经验高度重复 | 仍允许写入（L5 不执行归并，每条都是独立的不可遗忘资产） | — |
| 人工锁定指令尝试锁定已存在条目 | 返回拒绝（条目已锁定） | — |
| 只读查询请求来源为漏斗一 | 拒绝（编译期禁止漏斗一访问 L5） | — |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | L5 存储分区由 ad-29 硬件写保护锁定，任何运行时进程无法直接修改或删除。写入需临时解除保护并验证安全令牌 |
| S-02 | L5 条目终身不可删除。物理删除操作在编译期已移除，仅存在读取和写入接口 |
| S-03 | 安全显著性 S ≥ 0.9 的事件直达写入为最高优先级，无需满足 I 值和留存时间条件，直接锁定 |
| S-04 | 不可抗力场景经验自动标记为“不可抗力锁定”，享有永久保护，不可降级或归档 |
| S-05 | L5 容量硬编码为漏斗二总容量的 0.5%，不可通过运行时配置调整 |
| S-06 | L5 数据对外仅提供只读查询，写入仅接受来自 ad-26 的晋升、ad-18/ad-01 的安全直达、ad-01 的人工锁定 |
| S-07 | 所有写入、查询、拒绝事件全量写入 ad-51 不可变日志 |
| S-08 | 镜像分区与主存储实时同步，确保单点硬件故障不丢失核心经验 |
