# ad-03-漏斗二专属调度单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-03 |
| 模块名称 | 漏斗二专属调度单元 - 自成长经验漏斗管家 |
| 所属分区 | 一、顶层总控中枢 |
| 核心职责 | 漏斗二内部场景分槽的创建、激活与经验路由分发。依据世界模型输出的道路属性判定场景类别，将 ECC 大脑下发的经验写入请求路由至对应场景分槽 |
| 依赖模块 | ad-14（场景判定与分槽路由单元）、ad-15 至 ad-19（五个场景分槽）、ad-44（独立世界模型库）、ad-01（总控漏斗 F₀） |
| 被依赖模块 | ad-01（总控漏斗 F₀）、ad-20 至 ad-43（五层存储与晋升遗忘执行模块）、ad-37（重要度增量定时刷新单元） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 等待场景判定 | `WAITING_SCENE` | 等待世界模型返回当前道路属性与场景类别 | 收到经验写入请求但场景类别未确定 |
| 场景匹配中 | `MATCHING` | 正在将场景类别与已有分槽进行匹配 | 世界模型返回场景类别 |
| 分槽就绪 | `SLOT_READY` | 目标分槽已激活，经验写入路由畅通 | 目标分槽匹配成功 |
| 新槽创建中 | `CREATING_SLOT` | 当前场景类别未匹配到已有分槽，正在创建新场景分槽 | 场景类别未匹配到任何已有分槽 |
| 降级通用路由 | `FALLBACK_GENERAL` | 无法精确判定场景类别时，默认路由至通用驾驶槽 | 世界模型返回场景置信度低、未知道路 |
| 槽位冻结 | `FROZEN` | 漏斗二全部暂停写入，保留数据只读 | 驾驶模式切换为人工驾驶、紧急接管、安全急停 |
| 槽位维护 | `MAINTENANCE` | 正在执行分槽合并、遗忘策略调整等维护操作 | 容量超限、定期维护周期 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| ECC 经验写入请求 | Struct（含场景标签、驾驶经验条目、时间戳） | MemoryBus（ECC→记忆） | ECC 决策周期完成时 | **高** |
| 世界模型道路属性 | Struct（道路等级、车道数、路面类型、交通标识密度） | ad-44 独立世界模型库 | 每次经验写入请求时查询 | **高** |
| 世界模型场景分类信号 | Enum（高速巡航 / 城区路口 / 泊车低速 / 特殊环境 / 通用驾驶 / 乡村道路） | ad-44 独立世界模型库 | 场景类别判定完成时 | **高** |
| 驾驶模式信号 | Enum（人工驾驶 / 自动驾驶 / 紧急接管） | ad-01 总控漏斗 F₀ | 驾驶模式变更时 | **高** |
| 全局调度指令 | Enum（激活 / 冻结 / 锁定只读 / 维护） | ad-01 总控漏斗 F₀ | 模式切换、安全事件时 | **紧急 / 高** |
| 场景分槽状态快照 | Dict（各槽状态 + 存储占用 + 遗忘策略参数） | ad-15 至 ad-19 五个场景分槽 | 周期性，每 1 秒 | 普通 |
| 容量配额使用率 | Float（0.0–1.0） | ad-48 全局容量配额管控单元 | 每 10 秒周期性上报 | 普通 |
| 分槽合并建议 | Struct（源槽号 + 目标槽号 + 合并理由） | ad-01 总控漏斗 F₀ | 子漏斗数量接近 Nmax 时 | 普通 |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 目标场景分槽激活指令 | Struct（槽号 + 激活模式） | ad-15 至 ad-19 对应分槽 | 场景匹配或新槽创建完成时 | **高** |
| 新场景分槽创建指令 | Struct（槽号 + 场景类别 + 初始遗忘策略参数） | ad-14 场景判定与分槽路由单元 | 未匹配到已有分槽时 | **高** |
| 经验条目路由分发信号 | Struct（目标槽号 + 经验条目 ID + 写入优先级） | ad-14 场景判定与分槽路由单元 | 经验写入请求匹配到目标分槽时 | **高** |
| 降级通用路由信号 | Struct（目标槽 = 通用驾驶槽 + 经验条目 + 降级标记） | ad-19 通用驾驶槽 | 场景类别无法精确判定时 | 普通 |
| 分槽冻结指令 | Enum（冻结 / 解冻） | ad-15 至 ad-19 全部场景分槽 | 驾驶模式切换为非自动驾驶时 | **高** |
| 分槽级遗忘策略参数 | Dict（槽号 : 遗忘阈值、晋升阈值调整） | ad-15 至 ad-19 对应分槽 | 新槽创建或策略调整时 | 普通 |
| 调度状态上报 | Struct（当前活跃分槽 + 各槽存储占用统计） | ad-01 总控漏斗 F₀ | 周期性，每 1 秒 | 普通 |


## 核心处理逻辑

```
FUNCTION F2_dispatcher_main_loop():
    WHILE 系统运行中:
        // 第 1 步：最高优先级——全局调度指令
        IF 收到 ad-01 下发的全局调度指令:
            IF 指令 == 冻结 OR 锁定只读:
                SET internal_state = FROZEN
                冻结全部场景分槽写入
            ELSE IF 指令 == 维护:
                SET internal_state = MAINTENANCE
                暂停新经验写入，保留只读查询
            ELSE IF 指令 == 激活:
                IF internal_state == FROZEN:
                    解除冻结，恢复写入
                    SET internal_state = SLOT_READY
            CONTINUE

        // 第 2 步：驾驶模式监听
        IF 驾驶模式 != 自动驾驶:
            IF internal_state NOT IN [FROZEN]:
                冻结全部场景分槽写入
                SET internal_state = FROZEN
            CONTINUE  // 漏斗二仅自动驾驶模式运行

        // 第 3 步：经验写入请求路由
        IF 收到 ECC 经验写入请求:
            // 3a. 查询世界模型获取场景类别
            SET internal_state = WAITING_SCENE
            向 ad-44 查询当前道路属性与场景分类
            WAIT 世界模型返回结果

            // 3b. 场景判定与分槽匹配
            IF 场景分类置信度 < 0.5:
                // 无法精确判定，降级路由至通用驾驶槽
                SET internal_state = FALLBACK_GENERAL
                向 ad-19 通用驾驶槽下发经验写入指令（含降级标记）
                LOG 降级路由事件
            ELSE:
                SET internal_state = MATCHING
                在已有分槽中检索匹配的场景类别

                IF 匹配成功:
                    向对应分槽下发经验写入指令
                    SET internal_state = SLOT_READY
                ELSE IF 分槽数量 < Nmax_slot:
                    // 创建新场景分槽
                    SET internal_state = CREATING_SLOT
                    向 ad-14 发送新槽创建指令（含场景类别 + 初始遗忘策略）
                    WAIT 创建完成
                    激活新分槽
                    SET internal_state = SLOT_READY
                ELSE:
                    // 分槽数量达上限，归并至最相似分槽
                    向相似度最高的分槽下发经验写入指令（含归并标记）
                    SET internal_state = SLOT_READY

        // 第 4 步：分槽级遗忘策略动态调整
        IF 收到 ad-01 下发的遗忘阈值调整指令:
            更新对应分槽的遗忘策略参数
            // 各分槽独立配置：
            //   高速巡航槽：复用频次权重上调 20%
            //   城区路口槽：安全显著性权重上调 20%
            //   泊车低速槽：最低重要度阈值下调 50%
            //   特殊环境槽：所有晋升阈值下调 30%
            //   通用驾驶槽：乡村道路子类遗忘保护

        // 第 5 步：定期分槽健康检查
        IF 距上次健康检查 >= 60 秒:
            FOR EACH 场景分槽:
                IF 分槽存储占用 > 该槽配额 90%:
                    触发该槽的加速遗忘机制
                IF 分槽连续 7 天无新经验写入:
                    标记为 LOW_ACTIVITY，不删除，降低轮询优先级

        // 第 6 步：周期性状态上报
        IF 距上次上报 >= 1 秒:
            汇总当前活跃分槽信息与各槽存储占用统计
            上报 TO ad-01

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 世界模型返回场景分类置信度 < 0.5 | 经验降级路由至通用驾驶槽，标记为降级经验标签 | 世界模型置信度恢复至 ≥ 0.5 |
| 场景分槽数量已达 Nmax_slot | 归并至相似度最高的已有分槽，经验条目附带归并标记 | 手动调整 Nmax_slot 或分槽合并回收 |
| 目标分槽存储占用 > 90% | 触发该槽的加速遗忘机制，暂停该槽新经验写入，上报告警 | 存储占用回落至 < 70% |
| 分槽创建超时（> 3 秒） | 回退至降级通用路由，经验写入通用驾驶槽，上报告警 | 系统恢复后手动或自动重试创建 |
| 收到场景标签与经验内容不匹配的写入请求 | 以世界模型实时判定为准，忽略请求中的场景标签，重新判定 | — |
| 紧急接管或安全急停 | 立即冻结全部场景分槽写入，漏斗二全部锁定只读 | 险情解除且驾驶模式恢复自动驾驶 |
| 驾驶模式从自动驾驶切换为人工驾驶 | 冻结全部写入，漏斗二进入休眠冻结状态 | 驾驶模式恢复自动驾驶 |
| 分槽连续 7 天无新经验 | 标记为 LOW_ACTIVITY，降低轮询优先级，不删除不回收 | 有新经验写入时自动恢复 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| MemoryBus | 读 | ECC 经验写入请求 | 只读 | 含场景标签、驾驶经验条目、时间戳 |
| MemoryBus | 读 | 世界模型道路属性与场景分类信号 | 只读 | 由 ad-44 提供 |
| MemoryBus | 读 | 驾驶模式信号 | 只读 | 由 ad-01 转发 |
| MemoryBus | 读 | 全局调度指令 | 只读 | 由 ad-01 下发 |
| MemoryBus | 读 | 分槽合并建议 | 只读 | 由 ad-01 下发 |
| MemoryBus | 写 | 场景分槽激活 / 冻结 / 创建指令 | 专属写入 | 向 ad-14 至 ad-19 下发 |
| MemoryBus | 写 | 经验条目路由分发信号 | 专属写入 | 向 ad-14 下发 |
| MemoryBus | 写 | 分槽级遗忘策略参数 | 专属写入 | 向各场景分槽下发 |
| MemoryBus | 写 | 调度状态上报 | 周期性写入 | 每 1 秒向 ad-01 上报 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 漏斗二仅在自动驾驶模式下运行，人工驾驶模式时全部冻结只读 |
| S-02 | 场景判定以世界模型实时输出为准，不信任经验请求中的场景标签 |
| S-03 | 新场景分槽创建须校验分槽总数上限，编译期硬编码 Nmax_slot |
| S-04 | 紧急接管或安全急停时，漏斗二全部锁定只读，优先保障安全 |
| S-05 | 乡村道路经验归入通用驾驶槽子类，享独立遗忘保护参数 |
| S-06 | 各分槽遗忘策略独立配置，不得跨槽混用 |
| S-07 | 所有分槽创建、冻结、遗忘策略调整操作全量写入 ad-51 号变更日志 |
| S-08 | 漏斗二与漏斗一物理存储分区隔离，编译期强制，本模块负责路由隔离 |
