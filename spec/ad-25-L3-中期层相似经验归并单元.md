# ad-25-L3-中期层相似经验归并单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-25 |
| 模块名称 | L3 中期层相似经验归并单元 |
| 所属分区 | 三、漏斗二：自动驾驶自成长漏斗 / 五层记忆层级存储 |
| 核心职责 | 检测并合并 L3 中期层中场景特征高度相似的重复经验条目。通过场景特征向量余弦相似度计算，识别冗余经验，合并复用计数，保留重要度更高的条目，释放存储空间。归并操作不可逆，须经严格校验 |
| 依赖模块 | ad-24（L3 中期层存储单元，提供 L3 条目索引与经验数据）、ad-36（综合重要度 I 值聚合计算单元，提供条目当前 I 值）、ad-35（三维权重系数配置单元，提供各分槽归并相似度阈值） |
| 被依赖模块 | ad-24（消费归并执行指令，更新存储）、ad-51（记录归并变更日志） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无归并任务，等待触发 | 系统初始化完成，漏斗二激活 |
| 特征提取中 | `EXTRACTING` | 正在从 L3 存储中提取条目的场景特征向量 | 收到归并请求或周期性触发 |
| 相似度计算中 | `CALCULATING` | 正在计算条目对之间的余弦相似度 | 特征提取完成 |
| 归并方案生成中 | `PLANNING` | 正在根据相似度结果生成归并建议 | 相似度计算完成 |
| 归并执行中 | `EXECUTING` | 正在执行归并操作 | 归并方案经校验通过后 |
| 暂停服务 | `PAUSED` | L3 处于维护或冻结状态，暂停归并 | 驾驶模式切换、L3 维护中 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 归并触发请求 | Struct（触发类型：周期/容量告急/手动 + 目标分槽号 + 归并范围） | ad-24 L3 中期层存储单元 | L3 存储占用 > 80% 或距上次归并 ≥ 7 日 | **高** |
| L3 条目特征快照 | List（条目 ID + 场景特征向量 + 当前 I 值 + 来源分槽号 + 子类标记 + 结果分类标签 + 复用计数） | ad-24 L3 中期层存储单元 | 归并触发后请求 | **高** |
| 分槽归并相似度阈值 | Dict（分槽号 : 最低归并相似度） | ad-35 三维权重系数配置单元 | 系统初始化及阈值变更时 | **高** |
| 全局调度指令 | Enum（冻结/激活） | ad-03 漏斗二专属调度单元 | 模式切换时 | **高** |

| 异常归并告警 | Struct（归并对 ID + 异常类型 + 详情） | ad-03、ad-01 | 归并校验异常时 | **高** |


## 归并判定规则

### 一、归并相似度阈值

| 分槽 | 默认相似度阈值 | 说明 |
|------|:---:|------|
| 高速巡航槽 (ad-15) | 0.80 | 高速场景标准化程度高，阈值适中 |
| 城区路口槽 (ad-16) | 0.75 | 城区场景复杂多变，阈值略低以促进归并 |
| 泊车低速槽 (ad-17) | 0.75 | 泊车场景重复性高，阈值放宽 |
| 特殊环境槽 (ad-18) | 0.85 | 特殊环境经验宝贵，阈值严格防止误归并 |
| 通用驾驶槽 (ad-19) | 0.80 | 标准阈值 |

### 二、归并禁止规则（无条件跳过）

| 规则编号 | 条件 | 说明 |
|:---:|------|------|
| R-01 | 两条经验的结果分类标签不同（如“成功优化”与“策略失误”） | 不同结果类型的经验不可归并 |
| R-02 | 两条经验属于不同分槽 | 不同场景分槽的经验不可跨槽归并 |
| R-03 | 两条经验属于不同子类（如“乡村道路”与“常规通用”） | 不同子类的经验不可归并 |
| R-04 | 任一条经验的结果分类标签为“不可抗力场景” | 不可抗力经验不可归并，须永久独立保留 |
| R-05 | 任一条经验当前正被 ECC 查询锁定 | 被活跃使用的经验不可归并 |
| R-06 | 两条经验的“核心风险目标类别”不同（如行人 vs 机动车） | 不同风险类型的经验不可归并 |

### 三、归并策略

| 策略项 | 规则 |
|--------|------|
| 保留条目 | I 值更高的条目作为保留主体 |
| I 值合并 | 取两条经验 I 值的加权平均值：I_new = (I_a × 复用计数_a + I_b × 复用计数_b) / (复用计数_a + 复用计数_b) |
| 复用计数合并 | 保留条目的复用计数 = 原计数_a + 原计数_b |
| 经验内容 | 保留条目的经验内容不变，合并条目被删除 |
| 删除安全 | 合并条目删除前须校验保留条目完整性，删除后不可恢复 |


## 核心处理逻辑

```
FUNCTION l3_similarity_merge_main_loop():
    SET internal_state = IDLE
    归并间隔 = 7 日（默认）
    容量触发阈值 = 0.80

    WHILE 系统运行中:
        // 第 1 步：全局调度
        IF 收到 ad-03 冻结指令 OR L3 处于维护状态:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF 激活指令 AND internal_state == PAUSED:
            SET internal_state = IDLE
            CONTINUE

        // 第 2 步：检查归并触发条件
        归并触发 = False
        IF 收到 ad-24 归并触发请求:
            归并触发 = True
            触发类型 = 请求.触发类型
            目标分槽 = 请求.目标分槽号（若指定）
        ELSE IF internal_state == IDLE:
            // 主动周期性检查
            IF 距上次归并 >= 归并间隔:
                归并触发 = True
                触发类型 = "周期"
            ELSE IF ad-24 L3 占用率 > 容量触发阈值:
                归并触发 = True
                触发类型 = "容量告急"

        IF NOT 归并触发:
            SLEEP 60 秒
            CONTINUE

        // 第 3 步：提取 L3 条目特征
        SET internal_state = EXTRACTING
        向 ad-24 请求 L3 条目特征快照（可按目标分槽筛选）
        WAIT ad-24 返回特征快照

        IF 特征快照为空 OR 条目数 < 2:
            SET internal_state = IDLE
            CONTINUE

        // 第 4 步：计算相似度
        SET internal_state = CALCULATING
        归并对候选列表 = []

        // 按分槽分组，组内两两比较
        FOR EACH 分槽组 IN 按分槽号分组(特征快照):
            IF 分槽组.条目数 < 2:
                CONTINUE

            阈值 = 分槽归并相似度阈值[分槽组.分槽号]

            FOR i = 0 TO 分槽组.条目数 - 2:
                FOR j = i + 1 TO 分槽组.条目数 - 1:
                    条目 A = 分槽组[i]
                    条目 B = 分槽组[j]

                    // 4a. 归并禁止规则检查
                    IF 违反任一禁止规则(条目 A, 条目 B):
                        CONTINUE

                    // 4b. 计算余弦相似度
                    相似度 = 余弦相似度(条目 A.场景特征向量, 条目 B.场景特征向量)

                    // 4c. 相似度阈值判定
                    IF 相似度 >= 阈值:
                        归并对候选列表.append({
                            条目 A: 条目 A.ID,
                            条目 B: 条目 B.ID,
                            相似度: 相似度,
                            保留条目: IF 条目 A.I >= 条目 B.I THEN 条目 A ELSE 条目 B,
                            合并条目: IF 条目 A.I >= 条目 B.I THEN 条目 B ELSE 条目 A
                        })

        // 第 5 步：生成归并方案
        SET internal_state = PLANNING
        IF 归并对候选列表为空:
            SET internal_state = IDLE
            CONTINUE

        // 按相似度降序排列
        按相似度降序排列 归并对候选列表

        // 消解冲突：同一条目出现在多个归并对中，优先保留相似度最高的归并对
        已分配条目集合 = {}
        最终归并方案 = []
        FOR EACH 候选 IN 归并对候选列表:
            IF 候选.条目 A IN 已分配条目集合 OR 候选.条目 B IN 已分配条目集合:
                CONTINUE  // 该条目已被更高相似度的归并对占用

            // 计算归并后属性
            保留 = 候选.保留条目
            合并 = 候选.合并条目
            合并后 I 值 = (保留.I × 保留.复用计数 + 合并.I × 合并.复用计数) / (保留.复用计数 + 合并.复用计数)
            合并后复用计数 = 保留.复用计数 + 合并.复用计数

            最终归并方案.append({
                保留条目 ID: 保留.ID,
                合并条目 ID: 合并.ID,
                相似度: 候选.相似度,
                归并后 I 值: 合并后 I 值,
                归并后复用计数: 合并后复用计数,
                释放空间: 合并.经验内容.大小
            })

            已分配条目集合.ADD(保留.ID)
            已分配条目集合.ADD(合并.ID)

        // 第 6 步：执行归并
        IF 最终归并方案为空:
            SET internal_state = IDLE
            CONTINUE

        SET internal_state = EXECUTING
        成功数 = 0
        失败数 = 0
        总释放空间 = 0

        FOR EACH 归并项 IN 最终归并方案:
            // 6a. 执行前二次校验（防并发变更）
            当前保留条目 = 查询 ad-24 获取条目最新状态(归并项.保留条目 ID)
            当前合并条目 = 查询 ad-24 获取条目最新状态(归并项.合并条目 ID)
            IF 当前保留条目 == NULL OR 当前合并条目 == NULL:
                // 条目可能已被其他操作删除
                失败数++
                CONTINUE
            IF 违反任一禁止规则(当前保留条目, 当前合并条目):
                失败数++
                上报告警（并发冲突）
                CONTINUE

            // 6b. 更新保留条目
            更新保留条目.I 值 = 归并项.归并后 I 值
            更新保留条目.复用计数 = 归并项.归并后复用计数
            向 ad-24 发送更新指令

            // 6c. 删除合并条目
            向 ad-24 发送删除指令（合并条目 ID）
            向 ad-42 发送安全删除执行（合并条目 ID）

            // 6d. 记录归并日志
            向 ad-51 写入归并事件日志（含归并对详情 + 时间戳）

            成功数++
            总释放空间 += 归并项.释放空间

        // 第 7 步：输出归并执行结果
        归并结果 = {
            归并对总数: 最终归并方案.length,
            成功归并数: 成功数,
            失败归并数: 失败数,
            释放空间量: 总释放空间,
            失败详情: 各失败条目原因
        }
        向 ad-24 发送归并执行结果
        向 ad-51 写入归并汇总日志

        // 第 8 步：状态上报
        向 ad-03、ad-01 上报归并状态（含本轮统计）

        SET internal_state = IDLE
        SLEEP 10 秒
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 归并过程中条目被其他操作修改或删除 | 二次校验时发现不一致则跳过该归并对，上报告警 | — |
| 归并执行到一半时系统崩溃 | 恢复后 L3 回滚到归并前快照（由 ad-24 备份机制保障） | 系统恢复后重新触发归并 |
| 某分槽归并阈值被设为 1.0（禁止归并） | 该分槽本轮跳过，不进行归并 | — |
| 单次归并对数量 > 100 | 分批执行，每批 50 对，避免长时占用 | 批次间释放资源 |
| 归并后释放空间 < 预期 | 上报告警（可能存在碎片），触发碎片整理 | 碎片整理完成 |
| 冻结状态收到归并请求 | 暂不处理，状态 PAUSED | 激活指令 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 归并操作不可逆。合并条目删除后不可恢复，执行前须完成二次校验 |
| S-02 | 不可抗力场景经验（结果标签=“不可抗力”）绝对不可参与归并，须永久独立保留 |
| S-03 | 失败经验（策略失误）可与同类型失败经验归并，但不可与成功经验归并 |
| S-04 | 跨分槽归并绝对禁止。分槽是经验的天然隔离边界 |
| S-05 | 跨子类归并绝对禁止。乡村道路经验不可与常规通用经验归并 |
| S-06 | 归并相似度阈值不可低于编译期硬编码下限（0.60），防止过度归并导致经验失真 |
| S-07 | 所有归并操作（含成功、失败、跳过）全量写入 ad-51 不可变日志 |
