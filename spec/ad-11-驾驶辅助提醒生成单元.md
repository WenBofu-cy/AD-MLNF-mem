# ad-11-驾驶辅助提醒生成单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-11 |
| 模块名称 | 驾驶辅助提醒生成单元 |
| 所属分区 | 二、漏斗一：驾驶员画像漏斗 |
| 核心职责 | 依据行为累积统计单元输出的统计周期报表，结合当前场景特征，生成车内驾驶辅助语音提示与仪表显示内容。提示内容覆盖陋习纠正、优良鼓励、应急总结三种类型。仅输出至车内人机交互界面，编译期禁止接入自动驾驶决策链路 |
| 依赖模块 | ad-10（行为累积统计单元，提供统计周期报表）、ad-08（上下文场景标记单元，可选查询当前场景特征）、ad-02（漏斗一专属调度单元，提供当前活跃槽号与用户偏好设置） |
| 被依赖模块 | 无（终端输出模块，不向漏斗一内部其他模块输出） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收统计报表并生成提醒 | 系统初始化完成，活跃槽已激活 |
| 提醒冷却中 | `COOLDOWN` | 刚发送过同类型提醒，正在冷却期 | 上一条同类型提醒发送后未满冷却时长 |
| 夜间静默 | `NIGHT_SILENT` | 夜间时段，仅保留仪表显示，暂停语音提示 | 时段 = 夜间（22:00–07:00） |
| 用户暂停提醒 | `USER_PAUSED` | 用户通过中控屏暂时关闭了语音提醒 | 用户在中控屏操作 |
| 暂停服务 | `PAUSED` | 漏斗一冻结或驾驶模式切换，暂停全部提醒 | 驾驶模式切换为非人工驾驶 |
| 紧急只读 | `EMERGENCY_RO` | 紧急事件触发，暂停全部提醒 | 收到 ad-01 紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 统计周期报表 | Struct（含各行为维度统计、标签汇总、陋习排行榜、改善趋势） | ad-10 行为累积统计单元 | 每次统计更新后 | **高** |
| 当前活跃槽号 | Int | ad-02 漏斗一专属调度单元 | 槽位激活/切换时 | **高** |
| 用户提醒偏好设置 | Struct（语音开关、提醒频率、陋习提醒类型开关、夜间静默开关） | ad-02 漏斗一专属调度单元 | 用户修改设置时 | **高** |
| 驾驶模式信号 | Enum（人工驾驶/自动驾驶/紧急接管） | ad-02 漏斗一专属调度单元 | 驾驶模式变更时 | **高** |
| 紧急熔断指令 | Enum（全局熔断） | ad-01 总控漏斗 F₀ | 安全急停或碰撞事件时 | **紧急** |
| 系统时钟信号 | UTC 时间戳 | 系统时钟 | 周期性，每 10 秒 | 普通 |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 车内语音提示指令 | Struct（提示内容文本 + 语音优先级 + 语音类型） | 车内语音播报系统（硬件） | 满足提醒触发条件时 | 普通 |
| 仪表显示内容 | Struct（显示文本 + 图标类型 + 显示时长 + 颜色） | 车内仪表/中控屏（硬件） | 每次统计报表更新后 | 普通 |
| 提醒发送日志 | Struct（槽号 + 提醒类型 + 触发统计值 + 发送时间戳） | ad-51 记忆变更日志追溯单元 | 每次发送提醒后 | 普通 |
| 提醒状态上报 | Struct（当前状态 + 近期发送提醒次数 + 用户关闭提醒次数） | ad-01 总控漏斗 F₀ | 每 60 秒周期性上报 | 普通 |

## 核心处理逻辑

```
FUNCTION driving_assist_reminder_main_loop():
    SET internal_state = NORMAL
    初始化冷却计时器字典（每个提醒类型一个计时器）
    初始化夜间静默时段 = [22:00, 07:00]

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = EMERGENCY_RO
            暂停全部提醒
            清除待发送提醒队列
            CONTINUE

        // 第 2 步：驾驶模式检查
        IF 驾驶模式 != 人工驾驶:
            SET internal_state = PAUSED
            CONTINUE

        // 第 3 步：夜间静默检查
        IF 用户设置.夜间静默开关 == True AND 当前时间 IN 夜间静默时段:
            SET internal_state = NIGHT_SILENT
        ELSE IF internal_state == NIGHT_SILENT AND 当前时间 NOT IN 夜间静默时段:
            SET internal_state = NORMAL

        // 第 4 步：用户暂停检查
        IF 用户设置.语音开关 == False:
            SET internal_state = USER_PAUSED
        ELSE IF internal_state == USER_PAUSED AND 用户设置.语音开关 == True:
            SET internal_state = NORMAL

        // 第 5 步：接收统计周期报表
        IF 收到 ad-10 统计周期报表:
            IF internal_state IN [EMERGENCY_RO, PAUSED]:
                丢弃报表
                CONTINUE

            // 5a. 仪表显示始终更新（即使静默或用户暂停也更新仪表）
            生成仪表显示内容:
                综合优良率数值 + 趋势箭头
                陋习排行榜 Top 3
                改善趋势说明
            向仪表/中控屏发送显示指令（显示时长 = 15 秒）

            // 5b. 语音提示（须通过状态检查）
            IF internal_state IN [NIGHT_SILENT, USER_PAUSED, COOLDOWN]:
                SKIP 语音提示
            ELSE:
                // 按优先级从高到低检查触发条件
                语音候选列表 = []

                // 陋习提醒（按严重程度排序）
                FOR EACH 陋习行为 IN [未礼让行人, 未打灯变道, 急刹车, 跟车过近, 转弯过快, 急加速, 刹停点头]:
                    IF 报表.按行为维度[陋习行为].近30日.陋习次数 >= 触发规则库[陋习行为].触发阈值:
                        IF 冷却计时器[陋习行为] == 0:
                            语音候选列表.append({
                                类型: "陋习纠正",
                                行为: 陋习行为,
                                模板: 触发规则库[陋习行为].语音模板,
                                统计值: 报表.按行为维度[陋习行为].近30日.陋习次数,
                                优先级: 根据陋习危险程度（未礼让行人最高）
                            })

                // 应急总结提醒
                IF 报表.按判定标签汇总.近7日.应急总数 >= 3:
                    IF 冷却计时器["应急总结"] == 0:
                        语音候选列表.append({
                            类型: "应急总结",
                            模板: 触发规则库["应急总结"].语音模板,
                            统计值: 报表.按判定标签汇总.近7日.应急总数,
                            优先级: 中
                        })

                // 优良鼓励提醒
                改善趋势 = 报表.改善趋势
                IF 改善趋势.近7日综合优良率 - 改善趋势.近30日综合优良率 > 0.1:
                    IF 冷却计时器["优良率提升"] == 0:
                        语音候选列表.append({
                            类型: "优良鼓励",
                            模板: 触发规则库["优良率提升"].语音模板,
                            统计值: 改善趋势差值,
                            优先级: 低
                        })
                IF 报表.按判定标签汇总.近30日.综合优良率 >= 0.9:
                    IF 冷却计时器["优良率优秀"] == 0:
                        语音候选列表.append({
                            类型: "优良鼓励",
                            模板: 触发规则库["优良率优秀"].语音模板,
                            统计值: 优良率,
                            优先级: 低
                        })

                // 选优先级最高的 1 条发送（每次只发 1 条，避免信息轰炸）
                IF 语音候选列表不为空:
                    按优先级排序
                    最佳候选 = 语音候选列表[0]
                    语音文本 = 填充模板(最佳候选.模板, 最佳候选.统计值)
                    向车内语音播报系统发送语音提示指令
                    重置 冷却计时器[最佳候选.行为/类型] = 触发规则库[最佳候选].冷却时间
                    向 ad-51 写入提醒发送日志

        // 第 6 步：冷却计时器递减（每 10 秒检查一次）
        IF 距上次冷却递减 >= 10 秒:
            FOR EACH 冷却计时器:
                IF 冷却计时器[key] > 0:
                    冷却计时器[key] -= 10 秒

        // 第 7 步：周期性状态上报
        IF 距上次上报 >= 60 秒:
            向 ad-01 上报提醒状态

        SLEEP 5 秒  // 提醒生成不需要高频轮询
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 用户关闭语音提醒 | 继续更新仪表显示，暂停语音输出，状态 USER_PAUSED | 用户重新开启语音提醒 |
| 夜间静默时段 | 继续更新仪表显示，暂停语音输出，状态 NIGHT_SILENT | 进入日间时段（07:00–22:00） |
| 驾驶模式切换为非人工驾驶 | 暂停全部提醒，状态 PAUSED | 恢复人工驾驶 |
| 紧急熔断 | 暂停全部提醒，清空待发送队列，状态 EMERGENCY_RO | 紧急事件解除 + 恢复指令 |
| 语音播报系统硬件故障 | 仅输出仪表显示，上报告警，不阻塞统计报表处理 | 硬件恢复 |
| 仪表/中控屏硬件故障 | 继续语音提示，上报告警 | 硬件恢复 |
| 用户连续 3 次手动关闭同类型提醒 | 将该类型提醒冷却时间延长至 2 倍，避免骚扰用户 | 用户重置提醒偏好或 7 天后自动恢复 |
| 统计报表中槽号与当前活跃槽号不一致 | 丢弃报表（可能是旧槽的延迟报表） | — |
| 提醒发送日志写入失败 | 缓存至本地临时队列（最长保留 5 分钟），重试写入 | 日志模块恢复 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 本模块输出仅限车内人机交互界面（语音播报 + 仪表/中控屏显示），编译期禁止向自动驾驶决策链路、ECC 大脑或任何外部系统发送数据 |
| S-02 | 提醒内容不得包含任何驾驶员身份明文信息（姓名、ID），仅以通用称谓或行为数据呈现 |
| S-03 | 陋习提醒采用鼓励式措辞，禁止使用羞辱、恐吓、贬低性语言。语音模板须经合规审查 |
| S-04 | 未礼让行人提醒为法规强制项，触发阈值 = 1 次即提醒，不可被用户关闭 |
| S-05 | 用户可通过中控屏关闭除未礼让行人外的任意类型语音提醒，关闭后冷却计时器不受影响 |
| S-06 | 夜间静默时段（默认 22:00–07:00）仅保留仪表显示，语音暂停，用户可自行调整时段 |
| S-07 | 每次提醒发送均全量记录于 ad-51 变更日志（含槽号、提醒类型、触发统计值、时间戳） |
| S-08 | 应急操作后 10 分钟内不主动语音打扰，仅仪表显示，避免增加驾驶员紧张情绪 |
