# ad-09-行为判定标签单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-09 |
| 模块名称 | 行为判定标签单元 |
| 所属分区 | 二、漏斗一：驾驶员画像漏斗 |
| 核心职责 | 将每条带场景标签的驾驶行为观测条目与交通法规库基准进行比对，自动判定并标记为「优良习惯」「常态陋习」或「应急特殊操作」。应急标记为 True 的行为自动归入应急特殊操作，不纳入陋习统计 |
| 依赖模块 | ad-08（上下文场景标记单元，提供带场景标签的行为条目）、ad-45（交通法律法规库，提供法规基准与判定阈值）、ad-02（漏斗一专属调度单元，提供当前活跃槽号） |
| 被依赖模块 | ad-10（行为累积统计单元，消费带判定标签的行为条目） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 正常接收行为条目并执行判定 | 系统初始化完成，法规库可用 |
| 法规查询中 | `QUERYING` | 正在向交通法规库查询当前场景的判定基准 | 收到行为条目后触发 |
| 规则匹配中 | `MATCHING` | 正在将驾驶员行为与法规基准进行对比匹配 | 法规库返回结果后 |
| 降级保守判定 | `FALLBACK` | 法规库暂时不可用，使用内置保守规则判定 | 法规库超时或返回异常 |
| 暂停判定 | `PAUSED` | 漏斗一冻结或驾驶模式切换，暂停处理 | 驾驶模式切换为非人工驾驶 |
| 紧急只读 | `EMERGENCY_RO` | 紧急事件触发，暂停全部判定 | 收到 ad-01 紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 带场景标签的行为条目 | Struct（含原始观测 + 场景标签 + 应急标记 + 标签置信度） | ad-08 上下文场景标记单元 | 每次场景标记完成后 | **高** |
| 交通法规判定基准 | Struct（行为类型 : 合规阈值 + 违规判定标准） | ad-45 交通法律法规库 | 每次法规查询请求后 | **高** |
| 驾驶模式信号 | Enum（人工驾驶/自动驾驶/紧急接管） | ad-02 漏斗一专属调度单元 | 驾驶模式变更时 | **高** |
| 当前活跃槽号 | Int | ad-02 漏斗一专属调度单元 | 槽位激活/切换时 | **高** |
| 紧急熔断指令 | Enum（全局熔断） | ad-01 总控漏斗 F₀ | 安全急停或碰撞事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 带判定标签的行为条目 | Struct（原条目 + 判定标签 + 判定依据 + 判定置信度） | ad-10 行为累积统计单元 | 判定完成后 | **高** |
| 法规库查询异常告警 | Struct（异常类型 + 持续时长） | ad-01 总控漏斗 F₀ | 法规库超时或异常超过 3 次 | 普通 |
| 判定状态上报 | Struct（当前状态 + 近期各标签统计） | ad-01 总控漏斗 F₀ | 每 5 秒周期性上报 | 普通 |

**判定标签详表：**

| 标签 | 标识 | 含义 | 对统计的影响 |
|------|------|------|-------------|
| 优良习惯 | `GOOD` | 该行为完全符合或优于交通法规基准 | 正向统计，用于驾驶辅助表扬提示 |
| 常态陋习 | `BAD` | 该行为未达到交通法规基准，且非应急场景 | 负向统计，用于驾驶辅助提醒与纠正 |
| 应急特殊操作 | `EMERGENCY_OP` | 该行为发生在紧急/特殊场景下，不适用常规判定 | **不纳入陋习统计**，单独记录供复盘 |


## 判定规则库（内置）

| 行为类型 | 合规基准（优良习惯阈值） | 违规判定（常态陋习阈值） | 特殊说明 |
|----------|------------------------|------------------------|----------|
| 变道 | 转向灯提前 ≥ 3 秒开启 + 方向盘转角速率 ≤ 200°/s | 未开转向灯变道、转向灯开启 < 1 秒即变道 | 应急场景下未打灯变道标记为应急特殊操作 |
| 跟车 | 跟车时距 ≥ 2.0s（默认风格） | 跟车时距 < 1.5s（默认风格） | 风格设定为“高效通行”时阈值调整为 ≥ 1.8s / < 1.5s |
| 制动 | 制动减速度 ≤ 3m/s²，刹停无点头 | 制动减速度 > 5m/s²（非紧急）、刹停点头明显 | TTC < 2s 时的紧急制动标记为应急特殊操作 |
| 加速 | 纵向冲击度 ≤ 3m/s³ | 纵向冲击度 > 5m/s³（急加速） | 非铺装路面阈值全面收紧 40% |
| 转弯 | 转弯车速 ≤ 限速 × 0.7，方向盘转角平顺 | 转弯车速 > 限速 × 0.9，方向盘转角突变 | 夜间/恶劣天气阈值收紧 30% |
| 路口通行 | 完全停止后再起步（红灯/停止线） | 未完全停止（滑行通过）、抢黄灯 | 黄灯通过视具体情况：已进入路口则标记优良，加速抢行标记陋习 |
| 让行 | 主动减速/停车礼让行人 | 未礼让人行横道行人、抢行非机动车 | 此条为硬约束，任何场景下不礼让均标记陋习 |
| 停车 | 平稳刹停，距前车 ≥ 2m | 刹停点头、距前车 < 1m | — |
| 起步 | 平缓起步，纵向冲击度 ≤ 2m/s³ | 猛起步（纵向冲击度 > 5m/s³） | — |


## 核心处理逻辑

```
FUNCTION behavior_judgment_main_loop():
    SET internal_state = NORMAL

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断
        IF 收到紧急熔断指令:
            SET internal_state = EMERGENCY_RO
            暂停全部判定处理
            丢弃未完成的判定任务
            CONTINUE

        // 第 2 步：驾驶模式检查
        IF 驾驶模式 != 人工驾驶:
            SET internal_state = PAUSED
            CONTINUE

        // 第 3 步：接收带场景标签的行为条目
        IF 收到 ad-08 带场景标签的行为条目:
            IF internal_state == EMERGENCY_RO:
                丢弃该条目
                CONTINUE

            // 3a. 优先判定：应急标记
            IF 行为条目.应急标记 == True:
                判定标签 = "应急特殊操作"
                判定依据 = "应急场景：" + 行为条目.场景标签
                判定置信度 = 行为条目.标签置信度
                GOTO 打包输出

            // 3b. 查询交通法规库获取当前场景判定基准
            SET internal_state = QUERYING
            向 ad-45 发送法规查询请求（含行为类型 + 场景标签 + 道路等级）
            WAIT ad-45 返回结果（超时限制 50ms）

            IF 超时 OR 返回异常:
                query_fail_count++
                IF query_fail_count >= 3:
                    上报告警 TO ad-01
                SET internal_state = FALLBACK
                使用内置保守规则判定
                GOTO 内置规则判定

            query_fail_count = 0

            // 3c. 规则匹配判定
            SET internal_state = MATCHING
            法规基准 = ad-45 返回结果
            判定置信度 = 行为条目.标签置信度

            // 根据行为类型逐项比对
            CASE 行为类型 OF:
                "变道":
                    IF 转向灯提前时间 >= 法规基准.变道.转向灯提前阈值 AND
                       方向盘转角速率 <= 法规基准.变道.转角速率上限:
                        判定标签 = "优良习惯"
                    ELSE:
                        判定标签 = "常态陋习"
                        判定依据 = 具体未达标项

                "跟车":
                    风格设定 = 查询当前用户驾驶风格（平顺舒适/标准通勤/高效通行）
                    合规阈值 = 根据风格获取对应跟车时距阈值
                    IF 跟车时距 >= 合规阈值:
                        判定标签 = "优良习惯"
                    ELSE:
                        判定标签 = "常态陋习"
                        判定依据 = "跟车时距 " + 跟车时距 + " < 阈值 " + 合规阈值

                "制动":
                    IF 制动减速度 <= 3.0:
                        判定标签 = "优良习惯"
                    ELSE IF 制动减速度 > 5.0:
                        判定标签 = "常态陋习"
                        判定依据 = "制动过猛：" + 制动减速度 + "m/s²"
                    ELSE:
                        // 3.0 ~ 5.0 区间，结合场景判断
                        IF 场景标签 IN ["恶劣天气", "夜间无灯", "非铺装道路"]:
                            判定标签 = "常态陋习"（特殊场景应更谨慎）
                        ELSE:
                            判定标签 = "优良习惯"（可接受范围内）

                "加速":
                    IF 纵向冲击度 <= 3.0:
                        判定标签 = "优良习惯"
                    ELSE IF 纵向冲击度 > 5.0:
                        判定标签 = "常态陋习"
                    ELSE:
                        判定标签 = "优良习惯"

                "转弯":
                    IF 转弯车速 <= 限速 × 0.7 AND 方向盘转角速率 <= 200:
                        判定标签 = "优良习惯"
                    ELSE IF 转弯车速 > 限速 × 0.9:
                        判定标签 = "常态陋习"
                    ELSE:
                        判定标签 = "优良习惯"

                "路口通行":
                    IF 车速 == 0 AND 档位可能切换:
                        判定标签 = "优良习惯"  // 完全停止
                    ELSE IF 车速 > 0 AND 信号灯 == 红灯:
                        判定标签 = "常态陋习"
                        判定依据 = "未完全停止，滑行通过红灯"
                    ELSE:
                        判定标签 = "优良习惯"

                "让行":
                    IF 行人横道检测到行人 AND 本车未减速/未停车:
                        判定标签 = "常态陋习"  // 硬约束，不礼让即陋习
                        判定依据 = "未礼让人行横道行人"
                    ELSE:
                        判定标签 = "优良习惯"

                "停车":
                    IF 制动末段平顺（纵向冲击度 < 2.0） AND 距前车 >= 2m:
                        判定标签 = "优良习惯"
                    ELSE:
                        判定标签 = "常态陋习"

                "起步":
                    IF 纵向冲击度 <= 2.0:
                        判定标签 = "优良习惯"
                    ELSE:
                        判定标签 = "常态陋习"

                ELSE:
                    // 未定义行为类型
                    判定标签 = "优良习惯"（默认放行）
                    判定置信度 = 0.5

            SET internal_state = NORMAL
            GOTO 打包输出

        // 内置规则判定（法规库不可用时）
        LABEL 内置规则判定:
            // 使用内置判定规则库（见上方“判定规则库”表格）
            // 规则匹配逻辑同上，但阈值使用内置默认值
            // 判定置信度 = MIN(行为条目.标签置信度, 0.6)
            GOTO 打包输出

        // 打包输出
        LABEL 打包输出:
            judged_entry = {
                原始条目 = 行为条目,
                判定标签 = 判定标签,
                判定依据 = 判定依据,
                判定置信度 = 判定置信度,
                判定时间戳 = NOW()
            }
            向 ad-10 发送 judged_entry

        // 第 4 步：周期性状态上报
        IF 距上次上报 >= 5 秒:
            向 ad-01 上报判定状态（含近期各标签数量统计、法规库查询成功率）

        SLEEP 10ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 应急标记为 True | 自动标记为“应急特殊操作”，跳过法规比对，不纳入陋习统计 | — |
| 法规库查询超时（> 50ms） | 降级使用内置保守规则，判定置信度上限 0.6，继续处理 | 法规库恢复响应，连续 3 次成功后恢复 NORMAL |
| 法规库连续 3 次查询失败 | 上报告警 TO ad-01，后续使用内置规则，不再尝试查询 | 每 30 秒重试一次，成功后恢复 |
| 行为类型不在内置规则库中 | 默认标记为“优良习惯”，置信度 0.5 | 规则库扩展后覆盖 |
| 场景标签置信度 < 0.5 | 采用保守判定：阈值全面收紧 30%，宁可标陋习不标优良 | 标签置信度恢复 |
| 场景标签为“未知道路” | 采用保守判定，阈值收紧 30% | 世界模型恢复正常 |
| 驾驶模式切换为非人工驾驶 | 暂停处理，状态 PAUSED | 恢复人工驾驶 |
| 紧急熔断 | 停止处理，丢弃未完成条目 | 紧急事件解除 + 恢复指令 |
| 让行行为在任何场景下未礼让行人 | 一律标记为“常态陋习”，无例外 | — |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 漏斗一数据编译期禁止接入自动驾驶决策链路。本模块输出的判定结果仅流向 ad-10，不向 ECC 大脑传输 |
| S-02 | 应急标记为 True 的行为条目，必须无条件标记为“应急特殊操作”，跳过法规比对，不纳入陋习统计。此规则不可被任何配置覆盖 |
| S-03 | 让行行人在任何场景下均为硬约束，未礼让即标记“常态陋习”，无例外 |
| S-04 | 法规库不可用时，必须降级使用内置保守规则，不可凭空猜测或使用历史缓存判定基准 |
| S-05 | 判定置信度 < 0.5 时，ad-10 统计应采用保守加权（陋习权重上调 20%） |
| S-06 | 所有判定结果（含优良、陋习、应急）全量写入 ad-51 变更日志 |
| S-07 | 判定规则库（内置）的更新须经 OTA 升级流程，不可运行时动态修改 |
