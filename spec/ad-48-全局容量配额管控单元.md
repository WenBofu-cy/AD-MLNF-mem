# ad-48-全局容量配额管控单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-48 |
| 模块名称 | 全局容量配额管控单元 |
| 所属分区 | 五、存储与系统运维 |
| 核心职责 | 监控双漏斗记忆系统的总存储占用量，执行 500MB 终身容量硬约束。当总使用率超过安全阈值时，向遗忘阈值判定单元发出收紧指令，并在必要时直接触发低重要度条目的批量清除。保护 L4/L5 及不可抗力标记条目不被强制清除 |
| 依赖模块 | ad-20/22/24/26（各层级存储单元，上报容量使用数据）、ad-40（遗忘阈值判定单元，接收阈值收紧指令）、ad-42（冗余记忆删除与归档单元，接收批量清除指令）、ad-01（总控漏斗 F₀，接收容量告警） |
| 被依赖模块 | ad-40（消费容量告急信号以动态调节遗忘阈值）、ad-42（消费批量清除指令）、ad-01（接收容量状态汇报） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 正常运行 | `NORMAL` | 存储使用率在安全范围内，正常监控 | 系统初始化完成，使用率 < 85% |
| 黄色告警 | `YELLOW_ALERT` | 使用率在 85%–90% 之间，已通知收紧遗忘阈值 | 使用率超过 85% |
| 橙色告警 | `ORANGE_ALERT` | 使用率在 90%–95% 之间，L3 遗忘方式升级为直接删除 | 使用率超过 90% |
| 红色告警 | `RED_ALERT` | 使用率 > 95%，触发 L4 有限扫描与紧急批量清除 | 使用率超过 95% |
| 暂停服务 | `PAUSED` | 系统紧急熔断，暂停容量监控与清除 | 收到 ad-01 紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 各层级存储占用快照 | Dict（层级 : {当前使用容量, 条目数, 最大配额}） | ad-20/22/24/26 各层级存储单元 | 周期性，每 10 秒 | 普通 |
| 漏斗一存储占用快照 | Struct（各子画像槽使用容量 + 总漏斗一使用量） | ad-02 漏斗一专属调度单元 | 周期性，每 30 秒 | 普通 |
| 全局总容量上限 | Int（500MB） | 编译期常量 + OTA 可配置 | 系统初始化 | — |
| 清除完成确认 | Struct（清除条目数 + 释放空间量） | ad-42 冗余记忆删除与归档单元 | 每次批量清除完成后 | **高** |
| 紧急熔断指令 | Enum（全局熔断） | ad-01 总控漏斗 F₀ | 安全急停或碰撞事件时 | **紧急** |


## 输出数据

| 输出项 | 数据类型 | 目标模块 | 输出条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 容量告急信号 | Struct（当前使用率 + 告急等级 + 动态阈值调整系数 + 时间戳） | ad-40 遗忘阈值判定单元 | 使用率超过各级阈值时 | **高** |
| 批量清除指令 | Struct（清除范围 + 清除条目数目标 + 清除优先级规则 + 保护名单） | ad-42 冗余记忆删除与归档单元 | 红色告警且自动清除触发时 | **高** |
| 容量状态汇报 | Struct（总使用率 + 漏斗一/二各自使用量 + 各层级占比 + 告急等级 + 预估剩余天数） | ad-01 总控漏斗 F₀、ad-03 | 每 30 秒周期性上报 | 普通 |
| 配额调整建议 | Struct（建议调整的层级配额比例 + 调整原因） | ad-03、ad-01 | 某层级长期占用过高或过低时 | 普通 |


## 核心处理逻辑

```
FUNCTION global_quota_control_main_loop():
    SET internal_state = NORMAL
    总容量上限 = 500MB
    各层级配额比例 = {L1: 60%, L2: 25%, L3: 10%, L4: 4.5%, L5: 0.5%}
    各层级当前使用量 = {}
    漏斗一当前使用量 = 0
    last_report_time = 0

    WHILE 系统运行中:
        // 第 1 步：紧急熔断优先
        IF 收到紧急熔断指令:
            SET internal_state = PAUSED
            CONTINUE
        ELSE IF internal_state == PAUSED AND 紧急解除:
            SET internal_state = NORMAL

        // 第 2 步：接收各层级存储占用快照并计算
        IF 收到各层级存储占用快照:
            更新 各层级当前使用量
            漏斗二总使用量 = SUM(各层级当前使用量)
            总使用量 = 漏斗二总使用量 + 漏斗一当前使用量
            总使用率 = 总使用量 / 总容量上限

        // 第 3 步：接收漏斗一存储快照
        IF 收到漏斗一存储占用快照:
            漏斗一当前使用量 = 快照.总漏斗一使用量

        // 第 4 步：告急等级判定与动作
        新告急等级 = NORMAL
        阈值调整系数 = 1.0

        IF 总使用率 >= 0.95:
            新告急等级 = RED_ALERT
            阈值调整系数 = 2.0
        ELSE IF 总使用率 >= 0.90:
            新告急等级 = ORANGE_ALERT
            阈值调整系数 = 1.5
        ELSE IF 总使用率 >= 0.85:
            新告急等级 = YELLOW_ALERT
            阈值调整系数 = 1.2

        // 告急等级变化时立即通知 ad-40
        IF 新告急等级 != internal_state OR 阈值调整系数变化:
            SET internal_state = 新告急等级
            向 ad-40 发送容量告急信号(总使用率, 新告急等级, 阈值调整系数)

        // 第 5 步：红色告警时执行紧急批量清除
        IF internal_state == RED_ALERT AND 距上次清除 >= 60 秒:
            清除目标量 = 总使用量 - (总容量上限 × 0.85)  // 释放至 85% 以下

            // 构建清除指令：优先清理 L1 最低 I 值条目，L2 次之
            清除指令 = {
                清除范围: [L1, L2],
                清除条目数目标: 根据清除目标量估算,
                清除优先级规则: "按 I 值升序，优先清除非安全条目",
                保护名单: [L4全部, L5全部, 不可抗力标记条目, S≥0.7条目]
            }
            向 ad-42 发送批量清除指令(清除指令)

        // 第 6 步：配额调整建议
        IF 距上次配额检查 >= 1 小时:
            FOR EACH 层级 IN [L1, L2, L3, L4]:
                IF 各层级当前使用量[层级] > 各层级配额比例[层级] × 总容量上限 × 1.2:
                    向 ad-03、ad-01 发送配额调整建议(层级, 当前占用, 建议提高配额)
                ELSE IF 各层级当前使用量[层级] < 各层级配额比例[层级] × 总容量上限 × 0.5:
                    向 ad-03、ad-01 发送配额调整建议(层级, 当前占用, 建议回收配额)

        // 第 7 步：周期性状态汇报
        IF NOW() - last_report_time >= 30 秒:
            last_report_time = NOW()
            预估剩余天数 = IF 总使用率增长速率 > 0 THEN (总容量上限 × 0.85 - 总使用量) / 增长速率 ELSE ">30日"
            向 ad-01、ad-03 发送容量状态汇报(总使用率, 漏斗一/二使用量, 各层级占比, 当前告急等级, 预估剩余天数)

        SLEEP 1 秒
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 某层级存储单元超时未上报占用快照 | 使用上一次有效快照值，标记该层级为“数据可能过时”，上报告警 | 该层级恢复上报后清除标记 |
| 紧急清除释放空间不足 | 扩大清除范围至 L3 的低 I 值条目（不含不可抗力），继续上报红色告警 | 手动扩容或 OTA 提高总容量上限 |
| 总容量上限被 OTA 修改 | 校验新上限 ≥ 100MB 且为 10MB 整数倍，校验通过后更新总容量上限，记录变更日志 | — |
| 漏斗一使用量异常增长（短时间 > 10MB） | 上报告警，提示可能数据采集异常或子画像槽泄漏 | 人工确认后手动清理 |
| 紧急熔断期间容量告急 | 暂停清除操作，保持当前告急等级，紧急解除后立即执行清除 | 紧急解除 |


## 总线契约

| 总线 | 操作 | 数据内容 | 权限 | 说明 |
|------|------|----------|------|------|
| MemoryBus | 读 | 各层级存储占用快照 | 只读 | ad-20/22/24/26 每 10 秒推送 |
| MemoryBus | 读 | 漏斗一存储占用快照 | 只读 | ad-02 每 30 秒推送 |
| MemoryBus | 读 | 清除完成确认 | 只读 | ad-42 返回 |
| MemoryBus | 读 | 紧急熔断指令 | 只读 | ad-01 下发 |
| MemoryBus | 写 | 容量告急信号 | 专属写入 | ad-40 |
| MemoryBus | 写 | 批量清除指令 | 专属写入 | ad-42 |
| MemoryBus | 写 | 容量状态汇报 | 周期性写入 | ad-01、ad-03 |
| MemoryBus | 写 | 配额调整建议 | 周期性写入 | ad-03、ad-01 |


## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 总容量上限 500MB 为编译期默认值，OTA 可调整范围 100MB–1GB，调整时须校验下限与整数倍约束，且不可低于当前已使用量 |
| S-02 | L4/L5 层级条目在任何告急等级下均不被纳入批量清除范围，L5 条目终身不可清除 |
| S-03 | 不可抗力标记条目与 S≥0.7 的安全高价值条目在任何批量清除指令中均享有一级保护，不得被强制删除 |
| S-04 | 容量告急时仅能通过收紧遗忘阈值或触发低重要度清除来释放空间，不得绕过 ad-40/ad-42 直接操作存储 |
| S-05 | 所有容量告急、批量清除、配额调整操作全量写入 ad-51 不可变日志 |
