# ad-12-临时画像槽自动清除单元 接口规格

---

## 基本信息

| 项 | 内容 |
|----|------|
| 模块编号 | ad-12 |
| 模块名称 | 临时画像槽自动清除单元 |
| 所属分区 | 二、漏斗一：驾驶员画像漏斗 |
| 核心职责 | 负责临时画像槽（7 天到期）与一次性记录槽（行程结束）的自动安全擦除。执行 DoD 5220.22-M 标准单次覆写擦除，确保被清除数据不可恢复。擦除前校验权限，擦除后校验完整性 |
| 依赖模块 | ad-02（漏斗一专属调度单元，下发清除触发信号）、ad-06（子画像槽数据隔离管控单元，校验擦除权限） |
| 被依赖模块 | 无（擦除操作的最终执行者，不向漏斗一内部其他模块输出业务数据） |


## 内部状态定义

| 状态 | 标识 | 含义 | 触发条件 |
|------|------|------|----------|
| 空闲等待 | `IDLE` | 无擦除任务，等待指令 | 系统初始化完成、上一次擦除任务结束 |
| 权限校验中 | `AUTH_CHECK` | 正在向 ad-06 申请擦除权限 | 收到 ad-02 清除触发信号 |
| 擦除执行中 | `ERASING` | 正在对目标槽存储分区执行覆写擦除 | 权限校验通过 |
| 擦除校验中 | `VERIFYING` | 擦除完成后，正在校验数据是否彻底不可恢复 | 覆写擦除完成 |
| 擦除完成 | `DONE` | 擦除成功，存储分区已释放 | 校验通过 |
| 擦除失败 | `FAILED` | 权限被拒、擦除过程异常或校验未通过 | 任一环节失败 |
| 暂停等待 | `PAUSED` | 目标槽仍有未完成的写入操作，等待写入完成 | 擦除时检测到活跃写入 |
| 紧急中断 | `ABORTED` | 紧急熔断指令中断擦除操作 | 收到 ad-01 紧急熔断指令 |


## 输入数据

| 输入项 | 数据类型 | 来源模块 | 触发条件 | 优先级 |
|--------|----------|----------|----------|:---:|
| 临时槽清除触发信号 | Struct（目标槽号 + 槽位类型 + 触发原因 + 创建时间戳） | ad-02 漏斗一专属调度单元 | 临时槽 7 天到期 | **高** |
| 一次性槽清除触发信号 | Struct（目标槽号 + 槽位类型 + 触发原因 + 行程结束时间戳） | ad-02 漏斗一专属调度单元 | 一次性行程结束（车辆熄火 + P 档确认） | **高** |
| 擦除放行/拦截信号 | Struct（请求 ID + 放行确认 / 拦截原因码） | ad-06 子画像槽数据隔离管控单元 | 每次权限校验后返回 | **高** |
| 紧急熔断指令 | Enum（全局熔断） | ad-01 总控漏斗 F₀ | 安全急停或碰撞事件时 | **紧急** |
| 目标槽活跃写入状态 | Struct（槽号 + 是否有未完成写入 + 未完成写入数量） | ad-06 子画像槽数据隔离管控单元 | 擦除前查询 | **高** |

## 核心处理逻辑

```
FUNCTION temporary_slot_cleanup_main_loop():
    SET internal_state = IDLE

    WHILE 系统运行中:
        // 第 1 步：最高优先级——紧急熔断
        IF 收到紧急熔断指令:
            IF internal_state IN [AUTH_CHECK, ERASING, VERIFYING]:
                // 正在擦除中，立即中断
                SET internal_state = ABORTED
                停止擦除操作
                标记目标槽为 PARTIALLY_ERASED
                上报告警 TO ad-01
                LOG 中断事件 TO ad-51
            CONTINUE

        // 第 2 步：等待擦除指令
        IF 收到 ad-02 清除触发信号:
            目标槽号 = 信号.目标槽号
            槽位类型 = 信号.槽位类型
            触发原因 = 信号.触发原因

            // 2a. 校验槽位类型
            IF 槽位类型 NOT IN [临时, 一次性]:
                返回失败（槽位类型不匹配，长期槽不可自动清除）
                CONTINUE

            // 2b. 校验触发原因合法性
            IF 槽位类型 == 临时 AND 触发原因 != "7天到期":
                IF 触发原因 == "用户手动确认删除":
                    继续执行（用户主动删除）
                ELSE:
                    返回失败（触发原因不合法）
                    CONTINUE
            ELSE IF 槽位类型 == 一次性 AND 触发原因 != "行程结束":
                返回失败
                CONTINUE

            // 第 3 步：检查目标槽活跃写入状态
            SET internal_state = AUTH_CHECK
            向 ad-06 查询目标槽是否有未完成写入

            IF 目标槽有未完成写入:
                SET internal_state = PAUSED
                等待写入完成（最长等待 2 秒）
                IF 2 秒超时:
                    强制结束写入（写入不完整的数据由 ad-06 丢弃）
                SET internal_state = AUTH_CHECK

            // 第 4 步：申请擦除权限
            向 ad-06 发送擦除请求（目标槽号 + 擦除范围 = 全部分区）
            WAIT ad-06 返回放行/拦截信号

            IF 拦截:
                SET internal_state = FAILED
                失败原因 = "擦除权限被拒：" + 拦截原因码
                GOTO 失败处理

            // 第 5 步：执行安全擦除
            SET internal_state = ERASING

            // 5a. 确定擦除标准
            IF 槽位类型 == 临时:
                擦除标准 = DoD 5220.22-M 单次覆写
                覆写模式 = 0x00  // 全零覆写
            ELSE IF 槽位类型 == 一次性:
                擦除标准 = DoD 5220.22-M 单次覆写
                覆写模式 = 0xFF  // 全一覆写后再全零覆写（双次，更安全）

            // 5b. 执行覆写
            获取目标槽存储分区基址 + 大小
            FOR offset = 0 TO 分区大小 STEP 覆写块大小（默认 4KB）:
                写入覆写模式到 基址 + offset
                IF 写入失败:
                    SET internal_state = FAILED
                    失败原因 = "擦除写入失败 at offset " + offset
                    GOTO 失败处理

            // 5c. 刷新存储控制器缓存，确保覆写落地到物理介质
            FLUSH 存储控制器缓存

            // 第 6 步：擦除后校验
            SET internal_state = VERIFYING
            校验通过 = True
            // 随机抽样校验（抽样率 = 10%，均匀分布）
            FOR 抽样点 IN 随机抽样(0 TO 分区大小, 抽样率 10%):
                读取 抽样点 处的数据
                IF 读取值 != 覆写模式:
                    校验通过 = False
                    BREAK

            IF NOT 校验通过:
                SET internal_state = FAILED
                失败原因 = "擦除后校验失败，数据可能残留"
                GOTO 失败处理

            // 第 7 步：释放存储分区
            向 ad-48 发送存储分区释放通知
            向 ad-02 发送擦除完成回执
            向 ad-51 写入擦除事件日志
            SET internal_state = DONE
            SET internal_state = IDLE
            CONTINUE

        // 失败处理
        LABEL 失败处理:
            向 ad-02 发送擦除失败通知（含槽号 + 失败原因码）
            向 ad-51 写入擦除失败事件日志
            向 ad-01 上报告警
            SET internal_state = IDLE

        SLEEP 100ms
```


## 约束与异常处理

| 场景 | 处理方式 | 恢复条件 |
|------|----------|----------|
| 擦除权限被 ad-06 拦截 | 返回失败，原因=“权限被拒”，上报告警。不强行擦除 | 拦截原因消除后重新触发 |
| 目标槽仍有未完成写入（2 秒内） | 等待写入完成，状态 PAUSED | 写入完成 |
| 目标槽仍有未完成写入（超 2 秒） | 强制结束写入，继续擦除。不完整数据已由 ad-06 丢弃 | — |
| 擦除过程中存储硬件写入失败 | 返回失败，标记目标槽为 PARTIALLY_ERASED，上报告警，禁止该分区被重新分配 | 存储硬件修复后手动重建该分区 |
| 擦除后校验发现数据残留 | 返回失败，标记目标槽为 PARTIALLY_ERASED，上报告警，禁止该分区被重新分配 | 手动触发深度擦除（多次覆写） |
| 紧急熔断中断擦除 | 立即停止，标记目标槽为 PARTIALLY_ERASED，上报告警 | 紧急事件解除后，重新从 AUTH_CHECK 开始完整擦除流程 |
| 一次性槽覆盖旧槽时旧槽仍存在 | 先完成旧槽擦除（快速擦除模式），再创建新槽。旧槽擦除失败则新槽创建也失败 | 旧槽擦除成功 |
| 系统掉电后残留未擦除的临时槽 | 上电自检时扫描所有临时/一次性槽，超期或行程已结束的自动触发擦除 | 上电自检完成 |

| MemoryBus | 写 | 擦除完成回执 | 专属写入 | 向 ad-02 返回 |

## 安全边界

| 规则编号 | 内容 |
|:---:|------|
| S-01 | 长期槽（槽位类型 = 长期）禁止本模块自动清除。长期槽删除须用户手动确认 + ad-02 单独下发删除指令 |
| S-02 | 擦除标准硬编码：临时槽 DoD 5220.22-M 单次全零覆写，一次性槽 DoD 5220.22-M 双次覆写（全一 + 全零），编译期固定不可降级 |
| S-03 | 擦除后必须执行数据残留校验（随机抽样率 ≥ 10%），校验未通过的存储分区禁止重新分配 |
| S-04 | 擦除操作执行前须经 ad-06 权限校验，校验未通过则终止，不可强行擦除 |
| S-05 | 紧急熔断中断擦除时，目标槽标记为 PARTIALLY_ERASED，紧急解除后必须重新执行完整擦除，不可跳过 |
| S-06 | 系统上电自检时须扫描所有临时/一次性槽，超期或行程已结束的自动触发擦除 |
| S-07 | 所有擦除操作（含成功、失败、中断）全量写入 ad-51 不可变日志，含槽号、擦除方式、时间戳、校验结果 |


## 接口校验用例

| 用例编号 | 前置条件 | 输入 | 预期输出 |
|----------|----------|------|----------|
| TC-12-01 | IDLE 状态，临时槽 A 满 7 天，无活跃写入 | ad-02 清除触发信号（槽 A，临时，7天到期） | 权限通过→单次全零覆写→校验通过→释放分区→返回完成回执 |
| TC-12-02 | IDLE 状态，一次性槽 B 行程结束，无活跃写入 | ad-02 清除触发信号（槽 B，一次性，行程结束） | 权限通过→双次覆写（全一+全零）→校验通过→返回完成回执 |
| TC-12-03 | 目标槽有未完成写入（1 秒内完成） | 同 TC-12-01，但 ad-06 返回有活跃写入 | 等待 1 秒→写入完成→继续正常擦除流程 |
| TC-12-04 | 目标槽有未完成写入（超 2 秒） | 同 TC-12-01，但写入持续未完成 | 2 秒超时→强制结束写入→继续擦除 |
| TC-12-05 | 擦除权限被 ad-06 拦截 | 清除触发信号，ad-06 返回拦截（槽状态异常） | 返回失败（权限被拒），上报告警 |
| TC-12-06 | 擦除后校验发现数据残留 | 覆写完成，随机抽样发现 1 个字节未覆盖 | 返回失败（校验失败），标记 PARTIALLY_ERASED，上报告警，禁止分区重新分配 |
| TC-12-07 | 擦除进行中收到紧急熔断 | 擦除 50% 完成时收到紧急熔断 | 立即中断，标记 PARTIALLY_ERASED，上报告警 |
| TC-12-08 | 紧急熔断解除后重新擦除 | 目标槽标记 PARTIALLY_ERASED，紧急解除 | 从 AUTH_CHECK 重新开始完整擦除流程 |
| TC-12-09 | 系统上电自检扫描 | 检测到临时槽 C 已到期 8 天未擦除 | 自动触发擦除流程 |
| TC-12-10 | 长期槽（槽 D）收到清除信号 | ad-02 清除触发信号（槽 D，长期，用户确认删除） | 返回失败（槽位类型不匹配，长期槽不可自动清除） |
